# Question

A Product Owner describes a feature to the team, but when implemented, the result doesn't match what the PO envisioned. The team claims they built what was requested. How can future misunderstandings be prevented?

## Options

- **A**: Have the PO write more detailed requirements
- **B**: Have developers ask more questions
- **C**: Incorporate techniques like Three Amigos sessions, BDD scenarios, or rapid prototyping to validate shared understanding before implementation begins
- **D**: Accept that some misunderstanding is inevitable

## Answer

C

## Explanation

The correct answer is C. "Build what was requested" vs. "what was envisioned" gaps indicate failure to validate understanding before building. Three Amigos (developer, tester, PO) discussions surface different interpretations. BDD scenarios make expected behaviour explicit and testable. Rapid prototyping provides visual validation. These techniques catch misunderstanding when correction is cheap.

Option A more text doesn't ensure better understanding.

Option B helps but isn't structured enough for complex requirements.

Option D accepts preventable waste.
