# Question

A complex integration project involves multiple systems owned by different teams, each with their own technical vocabulary. Integration discussions are hampered by terminology confusion. How should the PM address this?

## Options

- **A**: Use only one team's terminology
- **B**: Avoid technical discussions and focus on requirements only
- **C**: Create a shared glossary through collaborative workshops, establish naming conventions for integration points, and validate understanding with concrete examples
- **D**: Hire a translator to attend all meetings

## Answer

C

## Explanation

The correct answer is C. Cross-system integration requires shared vocabulary. Collaborative glossary development builds mutual understanding and surfaces where the same word means different things. Naming conventions for integration points create unambiguous references. Concrete examples validate that abstract terms are understood the same way by all parties.

Option A privileges one team and may lose important distinctions.

Option B avoids rather than solves the problem.

Option D doesn't build team capability.
