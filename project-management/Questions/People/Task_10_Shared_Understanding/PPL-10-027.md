# Question

A project team has inherited legacy systems that few people understand. The original developers have left the organisation. Critical knowledge is being lost. How should the PM build team understanding of these systems?

## Options

- **A**: Avoid touching the legacy systems
- **B**: Rewrite everything from scratch
- **C**: Invest in knowledge archaeology—code walkthroughs, documentation sprints, reverse-engineering sessions—and capture learning for future reference
- **D**: Hire the original developers as consultants indefinitely

## Answer

C

## Explanation

The correct answer is C. Legacy system understanding must be deliberately rebuilt when original knowledge holders are gone. Code walkthroughs build collective understanding. Documentation sprints capture learning systematically. Reverse-engineering sessions uncover implicit behaviour. This investment pays off in reduced risk and faster future changes.

Option A doesn't solve the fundamental problem.

Option B is costly, risky, and may not be justified.

Option D creates ongoing dependency and may not be possible.
