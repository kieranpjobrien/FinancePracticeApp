# Question

A team practices pair programming and claims it builds shared understanding of the codebase. However, after six months, knowledge is still siloed with certain developers owning specific areas. What might be missing?

## Options

- **A**: Pair programming doesn't work
- **B**: Pairs aren't rotating enough to spread knowledge across the full codebase
- **C**: Knowledge silos are inevitable
- **D**: The team needs more documentation

## Answer

B

## Explanation

The correct answer is B. Pair programming builds shared understanding between paired individuals, but if pairs don't rotate systematically, knowledge spreads only within limited groups. Intentional rotation ensures everyone pairs with everyone else across all codebase areas over time. This requires deliberate pairing policies, not just ad-hoc pairing.

Option A throws out a valuable practice due to incomplete implementation.

Option C accepts a solvable problem.

Option D doesn't address the practice gap.
