# Question

Stakeholders agreed to a project scope document, but during implementation they frequently express surprise that certain features "aren't included." How should the PM prevent this misalignment?

## Options

- **A**: Add more detail to scope documents
- **B**: Have stakeholders sign-off more formally
- **C**: Use visual methods (prototypes, mockups, walk-throughs) to validate understanding of what's in and out of scope before finalising, not just after
- **D**: Reduce scope to minimise surprises

## Answer

C

## Explanation

The correct answer is C. Written scope documents often fail to create shared understanding because people interpret text differently. Visual methods make scope concrete and testable—stakeholders can see what they're getting and what they're not. Validating understanding before commitment catches misalignment when it's easier to address.

Option A more text doesn't guarantee better understanding.

Option B formal sign-off doesn't mean actual understanding.

Option D doesn't solve the understanding problem.
