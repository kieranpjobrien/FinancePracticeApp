# Question

A Scrum team's Definition of Done is unclear, with different team members applying different quality standards. What should the Scrum Master do?

## Options

- **A**: Define the Definition of Done and impose it on the team
- **B**: Let each developer maintain their own standards
- **C**: Facilitate a team session to collaboratively create a clear, shared Definition of Done that everyone understands and commits to
- **D**: Remove quality gates to simplify the process

## Answer

C

## Explanation

The correct answer is C. The Definition of Done must be a shared understanding, not individual interpretation. Collaborative creation builds buy-in and surfaces different perspectives on quality. The session should result in explicit, testable criteria everyone understands. This becomes a living agreement the team can refine over time.

Option A doesn't create genuine shared understanding.

Option B guarantees inconsistent quality.

Option D sacrifices quality for simplicity.
