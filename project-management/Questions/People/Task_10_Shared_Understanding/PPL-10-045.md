# Question

A team uses a wiki for documentation but information is scattered, outdated, and difficult to navigate. Team members resort to asking colleagues rather than checking documentation. How can the PM improve shared understanding through better documentation practices?

## Options

- **A**: Force team members to read the wiki instead of asking questions
- **B**: Abandon the wiki and rely only on verbal knowledge transfer
- **C**: Conduct a documentation audit, establish structure and ownership, implement regular review cycles, and make documentation part of the team's definition of done
- **D**: Hire a technical writer to maintain all documentation

## Answer

C

## Explanation

The correct answer is C. Documentation supports but doesn't replace shared understanding. Audit reveals what exists and what's missing. Structure makes information findable. Ownership ensures accuracy. Regular reviews keep content current. Making documentation part of "done" ensures it's maintained as work is completed, not as afterthought.

Option A doesn't address why the wiki isn't useful.

Option B loses the value of persistent reference material.

Option D creates single point of failure and may not have technical depth.
