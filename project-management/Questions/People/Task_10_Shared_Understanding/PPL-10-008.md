# Question

During a requirements workshop, the PM notices that stakeholders are using the same term to mean different things. This is causing confusion in discussions. What should the PM do?

## Options

- **A**: Continue the workshop and clarify later
- **B**: Stop and create a shared glossary, ensuring everyone agrees on terminology before proceeding
- **C**: Assume everyone understands despite using different terms
- **D**: Choose one definition and tell others to adopt it

## Answer

B

## Explanation

The correct answer is B. Terminology confusion undermines all subsequent discussion. Taking time to create shared definitions is an investment that pays off in clearer requirements. Get explicit agreement on key terms before proceeding. This is a foundational element of shared understanding.

Option A allows confusion to compound.

Option C ignores an evident problem.

Option D imposes without building agreement.
