# Question

During a sprint planning meeting, two senior developers strongly disagree about the technical approach for a critical feature. One advocates for a microservices architecture while the other insists on a monolithic design. The debate becomes heated, and other team members are becoming uncomfortable. As the project manager, what should you do FIRST?

## Options

- **A**: Make the technical decision yourself to end the conflict quickly
- **B**: Acknowledge the conflict and facilitate a discussion focused on project requirements and constraints
- **C**: Escalate the issue to the technical lead or architect immediately
- **D**: End the meeting and schedule separate one-on-one sessions with each developer

## Answer

B

## Explanation

The correct answer is B. As a servant leader, the project manager should facilitate constructive dialogue by acknowledging the conflict and redirecting the discussion toward objective criteria (project requirements, constraints, timeline). This demonstrates the collaborating/problem-solving conflict resolution approach, which seeks a win-win solution.

Option A is incorrect because the project manager should not make unilateral technical decisions, especially when qualified team members have valid perspectives. This uses forcing/directing style inappropriately.

Option C is premature - escalation should occur only after attempting to resolve at the team level. The PM should first try to facilitate resolution.

Option D (avoiding/withdrawing) delays resolution and may signal that conflict is not acceptable, discouraging healthy technical debates in the future.
