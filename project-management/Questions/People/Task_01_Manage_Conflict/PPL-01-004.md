# Question

In a hybrid project, the QA lead and development lead have an ongoing disagreement about testing standards. The QA lead wants comprehensive test coverage before any release, while the development lead argues this slows delivery unacceptably. This has caused delays in two consecutive sprints. What should the project manager do?

## Options

- **A**: Set a firm testing standard that both must follow to eliminate the disagreement
- **B**: Facilitate a working session where both leads define quality criteria tied to the Definition of Done that balances thoroughness with delivery speed
- **C**: Let the product owner decide the appropriate testing level for each release
- **D**: Separate the teams so they can work independently with their own standards

## Answer

B

## Explanation

The correct answer is B. This uses the collaborating/problem-solving approach by bringing both parties together to find a solution that addresses both concerns - quality AND delivery speed. By focusing on the Definition of Done (DoD), the discussion becomes objective and tied to project outcomes rather than personal positions. This creates shared ownership of the solution.

Option A (forcing) imposes a solution without buy-in, which likely won't resolve the underlying tension and may create resentment. The PM may also lack the technical expertise to set appropriate standards.

Option C delegates the decision but doesn't resolve the conflict between the leads. The product owner may not have the technical depth, and the leads still need to work together.

Option D (avoiding through separation) doesn't address the root issue and creates silos. QA and development must collaborate effectively for successful delivery - avoiding interaction isn't sustainable.
