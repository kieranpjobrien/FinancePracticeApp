# Question

In a hybrid project, the QA lead and development lead disagree about testing standards. The QA lead wants comprehensive coverage before any release; the development lead says this slows delivery unacceptably. Sprints have slipped twice in a row. What should the project manager do?

## Options

- **A**: Bring both leads together to agree quality criteria tied to the Definition of Done that balance thoroughness and delivery speed.
- **B**: Set a single testing standard yourself so the disagreement stops immediately.
- **C**: Ask the product owner to decide the right testing level for each release going forward.
- **D**: Separate QA and development streams so each can work to its own standards.

## Answer

A

## Explanation

(A) is collaborating: the parties define shared, objective quality criteria via the Definition of Done, which PMI treats as the authoritative artefact for "done". (B) forces a solution the PM may lack technical authority to set and produces no buy-in. (C) offloads a technical/process decision to the product owner and leaves the two leads still misaligned. (D) avoids the conflict by creating silos, which is unsustainable when QA and dev must collaborate.
