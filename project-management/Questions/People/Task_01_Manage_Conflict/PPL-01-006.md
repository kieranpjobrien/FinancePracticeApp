# Question

A distributed agile team has members in three time zones. During retrospectives, the team members in Asia consistently remain silent while North American members dominate discussions. When asked privately, Asian team members express frustration that their ideas are dismissed or interrupted. How should the Scrum Master address this conflict?

## Options

- **A**: Rotate meeting times so each region occasionally has convenient hours
- **B**: Implement structured facilitation techniques such as round-robin sharing, written input before meetings, and explicit pauses for all voices
- **C**: Create separate retrospectives for each region to ensure everyone can participate
- **D**: Coach the North American team members on cultural sensitivity in a private session

## Answer

B

## Explanation

The correct answer is B. This addresses the root cause - the meeting structure and facilitation approach that inadvertently favors certain communication styles. Techniques like round-robin sharing ensure everyone speaks, written input allows time for reflection (important in some cultures), and explicit pauses create space for those who may not interrupt. This is a systemic solution that improves inclusion.

Option A addresses timing but not the communication dynamic. Even at convenient times, the same dominance patterns would likely continue.

Option C creates silos and prevents cross-cultural team building. Retrospectives should involve the whole team to build shared understanding and address team-wide issues.

Option D singles out one group and may create defensiveness. While cultural awareness is valuable, the solution should be structural changes that benefit everyone, not labeling some members as problems.
