# Question

When should a project team revisit its ground rules?

## Options

- **A**: Only at the end of the project as part of lessons learned.
- **B**: Only when a serious conflict has surfaced.
- **C**: Periodically, and whenever the team or context changes.
- **D**: Never; consistency is the value of having ground rules.

## Answer

C

## Explanation

Ground rules are living agreements. (C) reflects PMI's view that retrospectives, new joiners, distributed-work changes, and phase transitions all warrant a check on whether the rules still serve the team. (A) misses opportunities throughout the project. (B) is reactive and lets dysfunction build. (D) treats rules as immutable, which guarantees they will eventually drift from how the team actually works.
