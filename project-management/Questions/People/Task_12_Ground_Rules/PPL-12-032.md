# Question

A Scrum team's Definition of Done requires code review before completion. During a sprint, a developer completes a story without code review because the reviewer was unavailable and the sprint was ending. What should the Scrum Master do?

## Options

- **A**: Accept the story since the code works
- **B**: Mark the story as incomplete and carry it to the next sprint
- **C**: Change the Definition of Done to be more flexible
- **D**: Reprimand the developer for violating the rule

## Answer

B

## Explanation

The correct answer is B. The Definition of Done is a ground rule for quality. If code review is part of Done, then work without review isn't done, regardless of time pressure. Carrying it forward reinforces that the rule matters and protects quality.

Option A undermines the Definition of Done and the discipline it provides.

Option C changes rules for convenience rather than addressing the real issue (why was reviewer unavailable?).

Option D focuses on blame rather than upholding the standard.
