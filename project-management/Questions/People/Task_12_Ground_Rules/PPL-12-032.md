# Question

A Scrum team's Definition of Done requires peer code review before a story is considered complete. At the end of the sprint, a developer has merged a story without review because the reviewer was on leave. What should the Scrum Master do?

## Options

- **A**: Accept the story since the code is working in the build.
- **B**: Mark the story as not done and roll it into the next sprint.
- **C**: Update the Definition of Done to permit deferred reviews when reviewers are unavailable.
- **D**: Counsel the developer privately and accept the story this once.

## Answer

B

## Explanation

The Definition of Done is a team agreement that protects quality; honouring it is what makes it a meaningful standard. (B) keeps the rule intact and creates the right pressure to address the underlying problem (single point of failure on review). (A) erodes the standard. (C) weakens the rule for convenience instead of solving why a single absence broke the workflow. (D) treats the breach as a personal issue, not a process issue.
