# Question

A vendor proposes a fixed-price contract for a project with unclear requirements. The PM is concerned about change order costs if requirements evolve. What contract structure should the PM negotiate?

## Options

- **A**: Accept fixed-price to cap costs
- **B**: Insist on time-and-materials
- **C**: Negotiate a hybrid approach such as time-and-materials for discovery/design phases and fixed-price for well-defined implementation, with clear change control mechanisms
- **D**: Delay the project until requirements are fully defined

## Answer

C

## Explanation

The correct answer is C. Contract structure should match requirement certainty. Time-and-materials suits exploratory phases where scope is unclear. Fixed-price suits well-defined work. Hybrid contracts can include not-to-exceed caps for T&M phases, clear criteria for transitioning to fixed-price, and transparent change control. This balances risk appropriately across the project lifecycle.

Option A creates change order risk if requirements evolve.

Option B removes all price certainty.

Option D may delay indefinitely since some requirements emerge during development.
