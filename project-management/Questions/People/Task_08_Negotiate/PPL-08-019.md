# Question

A PM is negotiating scope with a stakeholder who wants additional features included without schedule or budget changes. The stakeholder has significant organisational influence. How should the PM handle this?

## Options

- **A**: Accept all requests to maintain the relationship
- **B**: Refuse all additional scope
- **C**: Acknowledge the value of the requests, explain trade-offs using data, offer options (what could be deferred to add this? what additional resources would enable it?), and seek agreement on prioritisation
- **D**: Agree now and plan to cut the features later

## Answer

C

## Explanation

The correct answer is C. Scope negotiation requires balancing stakeholder relationships with project constraints. Acknowledging value shows you understand their needs. Data (schedule impact, resource needs) makes trade-offs objective. Options empower the stakeholder to make informed choices. This maintains the relationship while protecting project viability.

Option A leads to project failure through scope creep.

Option B may damage an important relationship unnecessarily.

Option D destroys trust when features are later cut.
