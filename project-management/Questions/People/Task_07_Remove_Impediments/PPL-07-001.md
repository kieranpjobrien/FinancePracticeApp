# Question

During a daily standup, a developer reports being blocked for two days waiting for access credentials from IT security. Previous requests through the normal ticketing system haven't been prioritized. What should the Scrum Master do?

## Options

- **A**: Tell the developer to continue waiting since proper process must be followed
- **B**: Personally contact IT security to explain the project impact and expedite the request
- **C**: Reassign the developer to different work while waiting
- **D**: Escalate to the project sponsor to pressure IT security

## Answer

B

## Explanation

The correct answer is B. The Scrum Master's responsibility includes removing impediments. Two days blocked is significant, and normal channels aren't working. Personally intervening to explain project impact and build relationships with IT security addresses both the immediate blocker and potentially improves future cooperation. This is servant leadership in action.

Option A accepts the impediment without action. Following process doesn't mean accepting blocked team members. The SM should work the system to unblock the team.

Option C works around the symptom but doesn't remove the impediment. The credentials are still needed, and the pattern may repeat. Also, the original work may be higher priority.

Option D escalates before the SM has tried to resolve it directly. Escalation should come after direct engagement fails, not as a first response.
