# Question

A team's impediment board shows 15 open items, some dating back several sprints. Many items show no recent activity. What should the Scrum Master do?

## Options

- **A**: Continue adding new impediments without addressing old ones
- **B**: Delete all old impediments to clean up the board
- **C**: Review each impediment: close those no longer relevant, prioritise remaining ones by impact, assign owners and next actions, and establish regular review cadence
- **D**: Stop using the impediment board

## Answer

C

## Explanation

The correct answer is C. A stale impediment board loses value. Regular hygiene ensures accuracy. Some old items may have been resolved informally or become irrelevant. Remaining items need prioritisation and ownership. Establishing cadence (weekly review?) prevents future staleness. The board should reflect current reality and drive action.

Option A makes the board increasingly useless.

Option B loses valid items along with resolved ones.

Option D removes a valuable visibility tool.
