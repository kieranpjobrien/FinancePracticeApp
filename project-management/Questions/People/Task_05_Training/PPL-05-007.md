# Question

A project delivers a new system to operations staff. User acceptance testing passes, but post-launch support tickets are extremely high because users don't understand how to use the system effectively. What went wrong and what should have been done?

## Options

- **A**: The system was poorly designed; it should be redesigned
- **B**: Training was insufficient; comprehensive user training should have been part of the project scope
- **C**: Users are resistant to change; nothing could have prevented this
- **D**: Support tickets are normal; no action is needed

## Answer

B

## Explanation

The correct answer is B. UAT passing but high support tickets indicates users can operate the system but don't know how to use it effectively for their work. Training should have been planned as part of the project, teaching not just mechanics but workflows and best practices. This is a common oversight - treating training as separate from delivery.

Option A (bad design) doesn't fit the evidence - UAT passed, indicating the system works.

Option C (blame users) avoids accountability for inadequate change management.

Option D (accept tickets) doesn't address the root cause or the user frustration.
