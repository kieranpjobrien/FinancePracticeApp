# Question

A new ERP module is six weeks from go-live. The 200 end users have only seen brief demos at steering committees and were not invited to UAT. The change manager raises that benefits realisation depends on user adoption. What should the project manager do FIRST?

## Options

- **A**: Issue the user manuals and quick-reference guides as soon as they are finalised so users can self-serve.
- **B**: Recommend delaying go-live until every user has completed instructor-led training.
- **C**: Work with the change manager to design a role-based training plan and arrange UAT exposure for a representative user group before cutover.
- **D**: Stand up an extended help desk for the first 30 days post go-live to handle user questions as they arise.

## Answer

C

## Explanation

PMI links training to benefits realisation and stakeholder engagement, and the gap here is that users have had no hands-on exposure. (C) addresses both: role-based training matches learning to actual usage, and UAT involvement builds familiarity and surfaces issues before cutover. (A) assumes documentation alone drives adoption. (B) over-corrects with a delay before lighter-touch options have been tried. (D) is a useful complement post go-live but does not replace pre-cutover preparation.
