# Question

Which of the following BEST describes a team's Definition of Done (DoD)?

## Options

- **A**: A list of features to be completed in the current sprint
- **B**: A shared understanding of what criteria must be met for work to be considered complete
- **C**: The project acceptance criteria defined by the customer
- **D**: A checklist used only by QA to verify deliverables

## Answer

B

## Explanation

The correct answer is B. The Definition of Done is a shared team agreement on the criteria that every product increment must meet to be considered complete. It typically includes coding standards, testing requirements, documentation, and any other activities needed before work can be released. It ensures consistent quality and shared understanding of "done."

Option A describes the sprint backlog, not the Definition of Done. The sprint backlog is what will be built; the DoD is the quality standard for how it's built.

Option C describes acceptance criteria, which are specific to individual user stories. The DoD is broader - it applies to all work items and defines the minimum quality bar.

Option D is too narrow. The DoD belongs to the entire team, not just QA. All team members contribute to and are accountable for meeting the DoD.
