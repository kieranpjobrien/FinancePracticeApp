# Question

The team uses a mix of senior and junior developers. Code review feedback from seniors to juniors has become harsh and demoralising, though technically accurate. Junior developers are becoming reluctant to submit code. How should the PM address this?

## Options

- **A**: Tell juniors to develop thicker skin
- **B**: Stop code reviews to avoid conflict
- **C**: Facilitate discussion about constructive feedback norms, provide coaching on effective code review practices, and monitor review interactions for improvement
- **D**: Have seniors review each other and exclude juniors from the process

## Answer

C

## Explanation

The correct answer is C. Code review is valuable but must be psychologically safe. Facilitate discussion: technical accuracy matters, but so does delivery. Coach on constructive feedback: focus on code not coder, explain rationale, acknowledge good aspects, suggest improvements constructively. Model and monitor improved behaviour. This maintains the benefits of review while protecting learning culture.

Option A accepts harmful communication and will drive away junior talent.

Option B loses valuable quality and learning opportunities.

Option D excludes juniors from a key learning activity.
