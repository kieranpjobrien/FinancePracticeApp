# Question

What does a sprint burndown chart primarily show?

## Options

- **A**: Each team member's contribution to the sprint by story points completed.
- **B**: A comparison of this sprint's velocity to previous sprints in the release.
- **C**: Remaining work in the sprint plotted against time so the team can see if it is on track.
- **D**: Cumulative defects discovered during the sprint by severity.

## Answer

C

## Explanation

A burndown chart plots remaining work over time so the team can see at a glance whether it is converging on the sprint goal and adjust if not. (A) describes individual tracking, which is not the purpose of a sprint burndown and is contrary to team-level metrics. (B) describes a velocity chart across sprints, not a within-sprint burndown. (D) describes a defect trend chart, which is a quality artefact, not a progress chart.
