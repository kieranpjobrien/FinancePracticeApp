# Question

A project team is struggling to complete work within sprints. Analysis reveals that requirements are frequently clarified or changed during the sprint after work has started. What improvement should the Scrum Master prioritize?

## Options

- **A**: Extend sprint duration to accommodate requirement changes
- **B**: Improve backlog refinement to ensure stories are well-defined before sprint planning
- **C**: Add a formal change control process for mid-sprint requirement changes
- **D**: Increase the team's capacity by adding more developers

## Answer

B

## Explanation

The correct answer is B. The root cause is inadequate refinement - stories entering sprints without sufficient clarity. Improving backlog refinement ensures requirements are well-understood, questions are answered, and acceptance criteria are clear BEFORE work begins. This addresses the cause rather than symptoms.

Option A (longer sprints) gives more time to handle changes but doesn't prevent them. It reduces agility without solving the preparation problem.

Option C (change control) adds bureaucracy that slows the team but doesn't address why changes are needed. If requirements aren't clear upfront, formal controls just document the problem.

Option D (more developers) increases capacity but doesn't improve throughput if the bottleneck is requirement clarity. Adding people to a team with process problems often makes things worse (Brooks' Law concept).
