# Question

What is the purpose of a Definition of Done (DoD) in agile?

## Options

- **A**: To document what was done at the end of the project
- **B**: To establish shared understanding of what "complete" means for each work item, ensuring quality and consistency
- **C**: To define when the project will be finished
- **D**: To list tasks assigned to each team member

## Answer

B

## Explanation

The correct answer is B. The Definition of Done establishes team-wide criteria that must be met for work to be considered complete. It typically includes coding standards, testing requirements, documentation, and review processes. This ensures quality, consistency, and shared understanding of "done" - preventing incomplete work from being called finished.

Option A confuses DoD with project closure documentation.

Option C relates to project completion criteria, not work item completion.

Option D describes task assignment, not completion criteria.
