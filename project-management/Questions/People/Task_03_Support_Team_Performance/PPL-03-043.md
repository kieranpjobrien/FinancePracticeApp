# Question

What is the purpose of Work in Progress (WIP) limits in Kanban?

## Options

- **A**: To limit how many hours team members work
- **B**: To constrain the amount of work being actively worked on, reducing context switching and improving flow
- **C**: To limit the total number of items in the backlog
- **D**: To set maximum project budget

## Answer

B

## Explanation

The correct answer is B. WIP limits constrain how much work can be in progress simultaneously. This reduces context switching, improves focus, highlights bottlenecks, and typically increases throughput. When a WIP limit is reached, the team must finish current work before starting new work, promoting completion over starting.

Option A confuses WIP with working hours.

Option C relates to backlog management, not active work limits.

Option D relates to financial constraints, not work flow.
