# Question

Performance metrics show the team is completing many story points, but customer satisfaction scores are declining. Users report features don't meet their needs despite matching specifications. What should the PM do?

## Options

- **A**: Celebrate the velocity and dismiss user concerns as out of scope
- **B**: Investigate the disconnect - involve users earlier in the process, improve feedback loops, and adjust how success is measured beyond output metrics
- **C**: Increase velocity targets to ship more features faster
- **D**: Blame the product owner for poor requirements

## Answer

B

## Explanation

The correct answer is B. Output (story points completed) doesn't equal outcomes (user value). Investigate the gap: are specifications capturing actual needs? Is user feedback obtained too late? Adjust the process: involve users in refinement, shorten feedback cycles, consider usability testing before completion. Add outcome metrics alongside output metrics.

Option A optimises for the wrong thing and ignores real problems.

Option C compounds the problem by shipping more misaligned features faster.

Option D assigns blame rather than improving the system.
