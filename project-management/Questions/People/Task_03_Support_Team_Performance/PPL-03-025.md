# Question

A team's velocity has dropped by roughly 25% across the last three sprints. The team is fully staffed, engagement seems healthy, and no one has left. What should the project manager do FIRST?

## Options

- **A**: Set a velocity target for next sprint that returns to the original baseline so the team has a clear goal.
- **B**: Bring an additional developer onto the team for the next two sprints to recover throughput.
- **C**: Use the next retrospective to examine what has changed in work type, dependencies, or interruptions.
- **D**: Report the trend to the sponsor and recommend rebaselining the release plan.

## Answer

C

## Explanation

A velocity drop is a symptom, and the PM's first move is to understand the cause with the team rather than react to the number. (C) does that through the existing inspect-and-adapt mechanism. (A) treats velocity as a target to be hit, which encourages estimation inflation rather than honest planning. (B) adds capacity before the cause is known and risks Brooks's Law. (D) escalates and rebaselines without first determining whether the change is a noise blip or a structural issue the team can address.
