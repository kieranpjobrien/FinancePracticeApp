# Question

A senior stakeholder has excluded end users from project discussions, saying they "just need to be told what to do at go-live". The PM believes user input would materially improve outcomes. What should the PM do?

## Options

- **A**: Make the case to the senior stakeholder using outcomes she cares about — fewer change requests, faster adoption — and propose specific lightweight ways to involve users.
- **B**: Defer to the senior stakeholder's preference; she has the authority to set engagement.
- **C**: Engage end users informally without informing the senior stakeholder.
- **D**: Wait until adoption issues emerge during rollout, then point to lack of user input.

## Answer

A

## Explanation

PMI advocates for appropriate stakeholder inclusion using arguments framed in the deciding stakeholder's value system, while respecting her authority. (B) accepts a real project risk without contesting it. (C) goes around the senior stakeholder, which is a trust and governance breach. (D) deliberately allows preventable damage and is an ethics issue.
