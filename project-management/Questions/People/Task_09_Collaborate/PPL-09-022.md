# Question

A stakeholder group (end users) has been excluded from project discussions by a senior stakeholder who believes "they just need to be told what to do." The PM believes user input is valuable. How should the PM handle this?

## Options

- **A**: Defer to the senior stakeholder's preference
- **B**: Secretly engage end users without the senior stakeholder's knowledge
- **C**: Make the case for user involvement based on project benefits (better requirements, smoother adoption), propose specific ways to include users efficiently, and seek agreement on an engagement approach
- **D**: Wait until problems emerge from lack of user input

## Answer

C

## Explanation

The correct answer is C. Advocating for appropriate stakeholder inclusion is part of the PM role. Framing user involvement in terms the senior stakeholder values (fewer change requests, better adoption, reduced training costs) builds the case. Proposing efficient engagement methods addresses time concerns. Seeking agreement respects their position while advancing project interests.

Option A may compromise project success.

Option B undermines trust and creates governance issues.

Option D allows preventable problems to occur.
