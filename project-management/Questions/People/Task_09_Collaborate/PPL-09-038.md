# Question

Which stakeholder collaboration technique involves regularly showing working product increments to stakeholders for feedback?

## Options

- **A**: Stakeholder mapping
- **B**: Sprint review or demo
- **C**: RACI matrix
- **D**: Issue log

## Answer

B

## Explanation

The correct answer is B. Sprint reviews (or demos) are collaborative sessions where the team shows working product increments to stakeholders. This enables feedback on actual functionality rather than documentation, ensures stakeholders see progress, and provides opportunities for course correction. Regular demos are a core agile collaboration practice.

Option A is an analysis technique for understanding stakeholder relationships.

Option C is a responsibility assignment tool.

Option D is for tracking problems.
