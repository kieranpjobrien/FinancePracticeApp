# Question

A distributed agile team is experiencing friction because different team members interpret "done" differently. User stories are being marked complete but later require rework. What is the BEST approach?

## Options

- **A**: Create more detailed user story acceptance criteria
- **B**: Have the product owner personally review every completed item
- **C**: Collaboratively develop a Definition of Done and display it visually in the virtual workspace
- **D**: Implement peer code reviews for all work

## Answer

C

## Explanation

The correct answer is C. A Definition of Done is a shared team agreement that makes "done" explicit and consistent. Collaborative development ensures buy-in and shared understanding. Displaying it visually in the virtual workspace (shared board, wiki, team channel) keeps it visible and reinforces the standard.

Option A helps for individual stories but doesn't create a consistent standard across all work.

Option B creates a bottleneck and doesn't scale; it also removes team ownership of quality.

Option D addresses code quality but not the broader definition of done for all types of work.
