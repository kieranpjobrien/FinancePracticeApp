# Question

A project is implementing a supply chain optimization system. Global shipping disruptions have caused the organization to fundamentally rethink its supply chain strategy - moving from global sourcing to regional suppliers. This changes many assumptions in the project's design. What should the project manager do?

## Options

- **A**: Complete the current design and add regional capabilities in a future phase
- **B**: Halt the project until supply chain strategy stabilizes
- **C**: Work with stakeholders to understand the new supply chain strategy, assess the impact on project requirements and design, and propose how to adapt the project to support the new direction
- **D**: Reduce project scope to features that work for both global and regional models

## Answer

C

## Explanation

The correct answer is C. When the business strategy driving project requirements changes fundamentally, the project must adapt. The PM should understand the new strategy, assess how it changes requirements, and work with stakeholders to determine the appropriate project response - which might mean significant replanning.

Option A delivers a system designed for an outdated strategy.

Option B pauses indefinitely when the project might be able to support the new strategy with modifications.

Option D compromises the solution without understanding if a coherent dual-purpose design is even possible or desirable.
