# Question

A project is implementing a new warehouse system when the organization announces an acquisition of another company with its own warehouse operations. The acquired company uses different systems and processes. The project was designed for current operations only. What should the project manager do?

## Options

- **A**: Complete the project as planned for current operations
- **B**: Pause the project until the acquisition integration strategy is defined
- **C**: Engage with integration planning, understand how the acquisition affects system requirements, and collaborate with stakeholders to determine whether to continue, expand scope, or pause
- **D**: Expand scope to include the acquired company's warehouses

## Answer

C

## Explanation

The correct answer is C. An acquisition fundamentally changes the context. The PM should engage with integration planning to understand: Will systems be consolidated? Which system will be the target? What's the timeline? This information enables informed decisions about how the project should proceed.

Option A may deliver a system that becomes obsolete when systems are consolidated.

Option B pauses without understanding whether the project might contribute to integration.

Option D expands scope without understanding the integration strategy - the acquired system might become the standard instead.
