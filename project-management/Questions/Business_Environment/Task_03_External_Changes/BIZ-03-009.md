# Question

A software project has been using a third-party API as a core component. The API provider announces they will discontinue the service in 18 months. The project is scheduled to complete in 6 months. What should the project manager do?

## Options

- **A**: Complete the project as planned since the API will still work for the project duration
- **B**: Assess the long-term impact, identify alternatives, and work with stakeholders to decide whether to proceed with the original plan or pivot to a sustainable solution
- **C**: Immediately switch to a different API
- **D**: Negotiate with the API provider to extend the discontinuation date

## Answer

B

## Explanation

The correct answer is B. While the project timeline fits before discontinuation, the PM should consider the full picture. Will the delivered product be maintainable? What's the cost of migrating later versus now? Are there better alternatives? Stakeholders need this analysis to make an informed decision about long-term value.

Option A delivers a product with a known short lifespan without stakeholder awareness.

Option C makes a significant decision without proper analysis or stakeholder input.

Option D may be worth trying but is unlikely to succeed and doesn't address the fundamental dependency risk.
