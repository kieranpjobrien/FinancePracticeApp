# Question

Midway through a project, the sponsor asks the project manager to add a feature that wasn't in the original scope. The sponsor believes it will significantly increase project value. What should the project manager do?

## Options

- **A**: Add the feature since the sponsor requested it
- **B**: Refuse to add the feature since it wasn't in the original scope
- **C**: Evaluate the feature's value proposition, assess impact on schedule, budget, and risk, and follow the change control process
- **D**: Add the feature but document that it was a sponsor request

## Answer

C

## Explanation

The correct answer is C. New features should be evaluated through change control. The PM should assess: Does the feature deliver value that justifies the impact? What are the schedule, budget, and risk implications? This analysis enables an informed decision by appropriate stakeholders.

Option A adds scope without proper analysis or change control.

Option B rigidly rejects a potentially valuable change without evaluation.

Option D adds scope without proper approval, using documentation as cover rather than proper process.
