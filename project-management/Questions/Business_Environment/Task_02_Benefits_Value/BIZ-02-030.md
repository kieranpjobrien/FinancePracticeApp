# Question

A project team using Scrum has completed five sprints. The product owner wants to measure whether the releases are delivering value. What should the project manager recommend?

## Options

- **A**: Wait until the project is complete to measure value
- **B**: Track velocity as the measure of value delivery
- **C**: Establish value metrics tied to user outcomes and business results, and measure these against each release to assess incremental value delivery
- **D**: Use story points completed as the value measure

## Answer

C

## Explanation

The correct answer is C. Value should be measured in terms of user outcomes and business results, not development metrics. For each release, track relevant value indicators - user adoption, task completion rates, customer satisfaction, revenue impact - that demonstrate whether the delivered features are providing real value.

Option A delays feedback, missing the agile principle of early and continuous value delivery.

Option B measures development throughput, not business value. High velocity can deliver low value.

Option D, like velocity, measures output volume rather than value delivered.
