# Question

A project is using earned value management. The project is ahead of schedule (SPI = 1.15) and under budget (CPI = 1.10). However, user acceptance testing reveals that users are dissatisfied with the solution's usability. How should the project manager interpret this situation?

## Options

- **A**: The project is successful based on EVM metrics
- **B**: Schedule and cost performance are positive, but value delivery is at risk if users won't adopt the solution; additional work may be needed
- **C**: User satisfaction is not a project metric
- **D**: Complete the project as planned since EVM shows success

## Answer

B

## Explanation

The correct answer is B. EVM measures schedule and cost performance but not value delivery. A project that delivers on time and under budget but produces an unusable solution has failed its fundamental purpose. The PM should recognize that favorable EVM metrics don't guarantee success if benefits won't be realized.

Option A equates EVM performance with project success, ignoring the purpose of delivering value.

Option C dismisses an important success factor that affects benefits realization.

Option D prioritizes completing the plan over delivering value.
