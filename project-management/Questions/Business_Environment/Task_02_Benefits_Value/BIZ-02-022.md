# Question

The sponsor is uncomfortable that the planned MVP release will not deliver enough business value to justify the investment. What should the project manager do?

## Options

- **A**: Expand the MVP to include more features so it has more visible value.
- **B**: Explain that low initial value is a characteristic of MVPs and ask the sponsor to be patient.
- **C**: Abandon the MVP approach and plan a traditional full-feature launch.
- **D**: Work with the sponsor to ensure the MVP contains genuine core value and share the roadmap of follow-on increments.

## Answer

D

## Explanation

An MVP is not a weak first release — it should deliver real early value and prove the direction (D). (A) defeats the point by over-scoping. (B) confuses "minimum" with "token". (C) discards the learning and risk-reduction benefits of incremental delivery.
