# Question

A project team is implementing a minimum viable product (MVP) approach. The sponsor is concerned that the MVP won't deliver enough value to justify the investment. What should the project manager do?

## Options

- **A**: Add more features to the MVP to increase value
- **B**: Explain that MVP value is limited by definition
- **C**: Work with the sponsor to ensure the MVP includes features that deliver core value, and articulate how subsequent iterations will build on that value
- **D**: Replace the MVP approach with a traditional full-feature release

## Answer

C

## Explanation

The correct answer is C. The MVP should deliver meaningful value - it's not an excuse for a poor initial release. The PM should work with the sponsor to ensure the MVP scope includes features that demonstrate value and address core user needs, with a clear roadmap for how subsequent releases will enhance value.

Option A defeats the MVP purpose if it becomes feature-bloated.

Option B accepts limited value without working to maximize MVP value.

Option D abandons an approach that offers faster feedback and risk reduction without exploring how to make it work.
