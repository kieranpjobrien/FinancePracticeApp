# Question

A project is delivering incremental releases. After the third release, users report they're not seeing the efficiency gains promised. The features work correctly but the workflow is cumbersome. What should the project manager do?

## Options

- **A**: Continue delivering planned features and address efficiency in a future release
- **B**: Investigate the workflow issues, gather user feedback, and adjust upcoming releases to improve usability and efficiency
- **C**: Blame users for not adapting to the new system
- **D**: Add more training to help users understand the system

## Answer

B

## Explanation

The correct answer is B. If features work but don't deliver value due to poor workflow, the PM should investigate and adjust. Agile and incremental approaches should use feedback to improve subsequent releases. Continuing to deliver features that don't provide value wastes resources.

Option A ignores current feedback and continues building potentially wrong features.

Option C blames users rather than addressing legitimate feedback about system design.

Option D assumes training will fix a design problem. If the workflow is cumbersome, training won't make it efficient.
