# Question

A project manager is reviewing benefits realization six months after project closure. The project achieved 120% of projected benefits. Upon investigation, the PM discovers this is because the original baseline was incorrectly measured (it was artificially low). What should the PM report?

## Options

- **A**: Report 120% achievement since that's what the data shows
- **B**: Report the baseline error, recalculate benefits using correct baseline data, and document lessons learned about baseline establishment
- **C**: Keep quiet about the baseline error to avoid undermining confidence in benefits reporting
- **D**: Attribute the over-achievement to excellent project execution

## Answer

B

## Explanation

The correct answer is B. Integrity in benefits reporting matters more than impressive numbers. If the baseline was wrong, the benefits figure is meaningless. The PM should correct the record, provide accurate benefits assessment, and ensure future projects establish baselines correctly. This builds trust in the benefits management process.

Option A reports misleading data that overstates actual improvement.

Option C prioritizes optics over accuracy, undermining the integrity of benefits reporting.

Option D misattributes measurement error to performance, providing false lessons learned.
