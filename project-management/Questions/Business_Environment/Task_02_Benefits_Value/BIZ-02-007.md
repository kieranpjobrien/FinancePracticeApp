# Question

What is the difference between project outputs and project benefits?

## Options

- **A**: There is no difference; they are the same thing
- **B**: Outputs are what the project delivers (products, systems, documents); benefits are the positive outcomes the organization gains from using those outputs
- **C**: Outputs are intangible; benefits are tangible
- **D**: Outputs are measured after project closure; benefits are measured during the project

## Answer

B

## Explanation

The correct answer is B. Outputs are the direct deliverables of the project (a new system, a building, a process document). Benefits are the value the organization realizes from using those outputs (increased efficiency, cost savings, improved customer satisfaction). A project can deliver outputs successfully but fail to achieve benefits if adoption is poor or the outputs don't meet actual needs.

Option A conflates delivery with value realization, which are distinct concepts.

Option C reverses the typical characterization - outputs are often tangible; benefits can be tangible or intangible.

Option D reverses the timing - outputs are delivered during the project; benefits often materialize after project closure.
