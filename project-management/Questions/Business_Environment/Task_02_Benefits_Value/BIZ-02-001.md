# Question

A project to implement an automated system was justified by an expected 20% reduction in processing time. The project is complete and deployed, but actual measurement shows only 12% improvement. What should the project manager recommend?

## Options

- **A**: Close the project since the deliverable was completed as specified
- **B**: Report the benefits shortfall, analyze root causes, and recommend follow-on actions to close the gap
- **C**: Redefine the success criteria to match actual results
- **D**: Wait for additional time before measuring benefits

## Answer

B

## Explanation

The correct answer is B. Projects exist to deliver business value, not just deliverables. A significant gap between expected and actual benefits deserves analysis - perhaps user adoption is low, training was insufficient, or the original estimate was flawed. Understanding why enables corrective action. Transparent reporting maintains credibility.

Option A treats project completion as success regardless of value delivered. The project was justified by benefits, and those should be tracked.

Option C manipulates success criteria to hide the shortfall, which is dishonest and prevents learning.

Option D may be appropriate if benefits take time to materialize, but shouldn't be used to avoid facing disappointing initial results.
