# Question

A project delivered all its features, but the expected benefits won't materialize because a dependent project was cancelled. The delivered system requires data from the cancelled project to function as intended. What should the project manager do?

## Options

- **A**: Close the project as successful since all features were delivered
- **B**: Document the dependency failure, assess options for the delivered system, and escalate to governance for decisions on next steps
- **C**: Modify the delivered system to work without the cancelled project's data
- **D**: Blame the other project for the failure

## Answer

B

## Explanation

The correct answer is B. The project delivered outputs but cannot achieve benefits due to external dependency failure. This situation needs escalation to governance to decide: Can the delivered system be repurposed? Should investment be made to create alternative data sources? Should the system be decommissioned? This is an organizational decision.

Option A ignores that the project cannot deliver value, regardless of feature completion.

Option C may be appropriate but requires analysis and approval - it's a significant scope change post-closure.

Option D assigns blame rather than solving the problem.
