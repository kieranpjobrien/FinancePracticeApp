# Question

Your project delivered its full scope but the expected benefits cannot materialise because a sister project it depended on was cancelled. What should the project manager do?

## Options

- **A**: Close the project as successful because all features were delivered as agreed.
- **B**: Quietly repurpose the delivered system so it can run without the cancelled project's data.
- **C**: Record the dependency failure, lay out options for the delivered system, and escalate to governance for a decision.
- **D**: Document the other project's cancellation as the root cause and close out.

## Answer

C

## Explanation

The system is orphaned — a governance decision is needed on repurpose, decommission, or invest (C). (A) closes on outputs when outcomes are impossible. (B) is a major scope change without approval. (D) assigns blame without making a forward decision.
