# Question

A project delivered a new CRM system on time and budget. Three months post-launch, sales have not increased as projected in the business case. Investigation reveals sales staff are using the old system because the new one requires more data entry. What should be done?

## Options

- **A**: Force sales staff to use the new system by disabling the old one
- **B**: Declare the project unsuccessful and recommend decommissioning the new system
- **C**: Analyze the adoption barriers, consider system modifications to reduce friction, develop a change management plan, and work toward realizing the projected benefits
- **D**: Update the business case projections to reflect actual sales results

## Answer

C

## Explanation

The correct answer is C. Low adoption is preventing benefit realization. The solution involves understanding why users prefer the old system (more data entry is a symptom), addressing root causes (perhaps streamlining workflows, reducing required fields, or demonstrating value to users), and actively managing the change. Technical delivery without adoption delivers no value.

Option A uses force rather than addressing legitimate user concerns. Forced adoption typically leads to workarounds and data quality issues.

Option B gives up on the investment without attempting to improve the situation.

Option D revises expectations downward rather than working to improve actual results.
