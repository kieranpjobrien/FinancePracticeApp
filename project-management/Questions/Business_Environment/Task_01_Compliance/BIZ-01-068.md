# Question

A privacy impact assessment completed late in execution reveals that the design collects more personal data than the stated purpose requires. The build is 80% complete. Which response best reflects privacy-by-design?

## Options

- **A**: Document the over-collection as a post-launch remediation item in the benefits realisation plan.
- **B**: Redesign the affected data flows now to collect only what the stated purpose requires before launch.
- **C**: Update the public privacy notice to describe the broader data use and publish it before go-live.
- **D**: Encrypt the additional data at rest so the residual privacy risk is effectively mitigated at launch.

## Answer

B

## Explanation

Privacy-by-design requires data minimisation at the source; the fix is redesign, not disclosure or encryption (B). (A) ships a known non-compliant design. (C) attempts to "consent-wash" over-collection with notice. (D) protects data that should never have been collected in the first place.
