# Question

An agile project team wants to skip security compliance testing for a sprint to meet a deadline. They argue the features are low-risk and testing can be done later. What should the project manager do?

## Options

- **A**: Allow the skip since the team understands the features best
- **B**: Explain that compliance requirements are non-negotiable and work with the team to adjust scope or timeline to include testing
- **C**: Escalate to the sponsor to override compliance requirements
- **D**: Document the team's decision and proceed without testing

## Answer

B

## Explanation

The correct answer is B. Security compliance testing exists to protect the organization and users. The PM should maintain compliance requirements while being flexible on how to achieve them - perhaps fewer features this sprint, or extended timeline, but not skipped security testing.

Option A delegates compliance decisions to the team inappropriately. Teams prioritize delivery; PMs must balance delivery with organizational constraints.

Option C assumes sponsors can override compliance. Security requirements typically aren't optional regardless of who approves.

Option D creates accountability documentation but still accepts the compliance risk, which doesn't solve the problem.
