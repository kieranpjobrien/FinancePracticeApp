# Question

In a sprint review for a HIPAA-regulated patient portal, the team demonstrates a feature that stores protected health information without the required field-level encryption. Release is scheduled for the end of the sprint. What should the project manager do?

## Options

- **A**: Hold the feature back from release and add the encryption work to the next sprint backlog.
- **B**: Release the feature and request a HIPAA waiver from the compliance team.
- **C**: Release as-is since the functional acceptance criteria were met.
- **D**: Document the gap and plan remediation for the next quarterly release.

## Answer

A

## Explanation

HIPAA encryption requirements are not optional and a release with non-compliant PHI handling exposes the organisation to mandatory penalties. (A) blocks the release and addresses the gap immediately. (B) misunderstands HIPAA — there is no waiver mechanism. (C) prioritises functional completion over a legal requirement. (D) accepts the violation as known debt rather than fixing it.
