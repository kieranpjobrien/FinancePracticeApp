# Question

During project execution, the compliance officer raises concerns that the project's documentation doesn't meet audit trail requirements. The project is already 60% complete. What should the project manager do?

## Options

- **A**: Continue as planned and improve documentation on future projects
- **B**: Assess the documentation gap, determine what can be reconstructed for completed work, and improve practices for remaining work
- **C**: Challenge the compliance officer's interpretation of requirements
- **D**: Add a documentation task to the project backlog for the final sprint

## Answer

B

## Explanation

The correct answer is B. Audit trail requirements exist for accountability and traceability. The PM should understand the gap, determine what historical documentation can be reconstructed (meeting notes, decision logs, change records), and ensure remaining work meets requirements. 60% complete means 40% can definitely be done right.

Option A accepts a compliance gap that could cause audit failures or regulatory issues.

Option C is adversarial and may be inappropriate. Compliance officers typically have legitimate authority over compliance interpretations.

Option D defers the fix until late, when reconstruction of decisions and changes becomes harder. Address documentation immediately.
