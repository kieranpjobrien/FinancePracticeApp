# Question

A feature released in the last sprint passed all functional tests but fails an accessibility audit against WCAG 2.1 AA, which is required by the customer's procurement contract. What should the project manager do?

## Options

- **A**: Log the gap as technical debt and schedule remediation in a future quarter.
- **B**: Add the fix to the next sprint and update the definition of done to include accessibility checks.
- **C**: Continue delivery and wait for the customer to flag the issue formally.
- **D**: Roll the feature back from production until full WCAG compliance can be retested.

## Answer

B

## Explanation

The PM must address both the immediate gap and the process gap that allowed it through. (B) fixes the feature and prevents recurrence by changing the DoD. (A) defers a contractual obligation, exposing the organisation. (C) is reactive and damages credibility with the customer. (D) is disproportionate; rollback is appropriate only if the feature creates active harm or contractual breach is imminent.
