# Question

A project team is using an agile approach. The compliance department insists on detailed upfront documentation that seems to conflict with agile principles. What should the project manager do?

## Options

- **A**: Explain to compliance that agile doesn't require upfront documentation
- **B**: Work with compliance to understand their requirements and find approaches that satisfy compliance needs while maintaining agile practices
- **C**: Create the documentation compliance wants, even if the team doesn't use it
- **D**: Escalate to leadership to override compliance requirements

## Answer

B

## Explanation

The correct answer is B. Apparent conflicts between agile and compliance often have solutions when both sides understand each other's needs. Perhaps compliance needs traceability (achievable through user story tracking), audit trails (achievable through tool configurations), or approval evidence (achievable through digital sign-offs). Collaboration finds workable solutions.

Option A dismisses legitimate compliance needs by citing methodology. Compliance requirements typically supersede methodology preferences.

Option C creates documentation theater - artifacts that satisfy audits but add no value. This is waste that should be avoided if possible.

Option D treats compliance as an obstacle to overcome rather than a partner to work with.
