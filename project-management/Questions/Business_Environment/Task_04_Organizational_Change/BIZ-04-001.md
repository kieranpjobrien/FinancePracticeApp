# Question

A project is implementing a new system that will significantly change how a department works. The technical implementation is on track, but there are signs of user resistance: low attendance at training, negative comments in the hallways, and minimal engagement with pilot users. What should the project manager prioritize?

## Options

- **A**: Focus on completing the technical implementation since that's the project's core deliverable
- **B**: Mandate attendance at training and system usage
- **C**: Proactively address change resistance through communication, involvement, addressing concerns, and working with change sponsors
- **D**: Delay the rollout until users are more receptive

## Answer

C

## Explanation

The correct answer is C. Technical success without user adoption delivers no value. The symptoms indicate change resistance that must be addressed through change management: communicating the "why," involving resistors in shaping the implementation, addressing legitimate concerns, and working with organizational leaders (change sponsors) to support adoption.

Option A achieves technical completion but fails business value. A system nobody uses is a failed project regardless of technical quality.

Option B uses force, which may achieve compliance but not commitment. Forced users often resist passively and adoption remains low.

Option D delays without addressing root causes. Unless you work on the resistance, delay just postpones the problem.
