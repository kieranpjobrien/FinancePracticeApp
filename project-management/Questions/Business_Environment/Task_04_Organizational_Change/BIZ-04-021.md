# Question

A project team is deploying a new system in phases. The first phase went live with significant user complaints about usability. The project is about to begin phase two deployment. What should the project manager do?

## Options

- **A**: Proceed with phase two since it was planned
- **B**: Delay phase two until all phase one complaints are resolved
- **C**: Address critical usability issues before phase two, communicate actions taken to phase one users, and apply lessons learned to improve phase two deployment
- **D**: Lower expectations for phase two by warning users about limitations

## Answer

C

## Explanation

The correct answer is C. Phase one problems create risk for phase two adoption if not addressed. The PM should fix critical issues, demonstrate responsiveness to feedback (which builds trust), and improve the phase two approach based on lessons learned. This shows users their feedback matters.

Option A ignores feedback and risks repeating problems, damaging credibility further.

Option B may be too extreme if issues can be addressed in parallel with phase two preparation.

Option D manages expectations downward rather than fixing problems.
