# Question

An agile project is delivering frequent releases. Users complain about "constant change" and express preference for less frequent, larger releases. What should the project manager do?

## Options

- **A**: Continue frequent releases since that's the agile approach
- **B**: Switch to infrequent large releases
- **C**: Investigate the source of frustration - is it release frequency, quality issues, inadequate communication, or insufficient time to adopt each change - and address root causes
- **D**: Add more training between releases

## Answer

C

## Explanation

The correct answer is C. Complaints about "constant change" may have various root causes. Are releases causing disruption due to quality issues? Is communication about changes inadequate? Do users need more time to learn before the next change? Understanding the specific frustration enables targeted response without abandoning agile benefits.

Option A ignores legitimate user feedback.

Option B may sacrifice agile benefits without addressing root causes.

Option D assumes knowledge is the issue without understanding the actual problem.
