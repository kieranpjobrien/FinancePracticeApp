# Question

A project has completed technical deployment, but six weeks later, usage metrics show that only 30% of the target users are actively using the new system. What should the project manager recommend?

## Options

- **A**: Close the project since the technical deployment was successful
- **B**: Mandate system usage and disable alternative workarounds
- **C**: Analyze the adoption barriers, engage with non-users to understand their concerns, and develop targeted interventions to improve adoption
- **D**: Provide additional training to all users

## Answer

C

## Explanation

The correct answer is C. Low adoption indicates failed change management. The PM should understand why users aren't adopting - is it awareness, ability, or willingness? Are there legitimate usability issues? Understanding specific barriers enables targeted intervention rather than generic responses.

Option A accepts technical success while ignoring value delivery failure.

Option B uses force, which typically drives workarounds and resentment rather than genuine adoption.

Option D assumes knowledge is the barrier when it might be usability, motivation, or something else.
