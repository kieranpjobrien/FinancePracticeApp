# Question

An external event (natural disaster, pandemic, political upheaval) outside project control has created significant project risk. How should this type of risk be managed?

## Options

- **A**: It can't be managed since it's external
- **B**: Focus on response planning and resilience since the event itself can't be prevented - business continuity plans, alternative arrangements, and scenario planning
- **C**: Ignore external risks
- **D**: Stop the project until external conditions stabilise

## Answer

B

## Explanation

The correct answer is B. External risks can't be prevented but can be prepared for. Response planning: what will you do if it happens? Resilience: distributed teams, backup systems, alternative suppliers. Scenario planning: how do different scenarios affect the project? Monitoring: watch for early warning signs. This builds ability to respond even if you can't prevent.

Option A assumes external means unmanageable, which isn't true.

Option C ignores significant risks.

Option D may not be practical and doesn't improve preparedness.
