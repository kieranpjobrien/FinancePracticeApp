# Question

The project is 80% complete and has used only 30% of contingency reserve. Some identified risks can now be closed as their trigger windows have passed. What should be done with the unused contingency?

## Options

- **A**: Spend it on scope additions
- **B**: Review remaining risks, release contingency no longer needed to the appropriate parties, and retain contingency proportional to remaining risk exposure
- **C**: Keep all contingency until project end regardless
- **D**: Return all unused contingency immediately

## Answer

B

## Explanation

The correct answer is B. Contingency should be managed throughout the project. Review: which risks are still active? Close risks whose trigger windows have passed. Release: contingency no longer needed can be released (either to budget, other projects, or retained as profit depending on contracts). Retain: keep appropriate contingency for remaining risks.

Option A uses risk funds for scope, not risk.

Option C over-retains when some can be released.

Option D under-retains when risks remain.
