# Question

A project manager is distinguishing between risks and issues during a status review. Which statement best describes the difference?

## Options

- **A**: A risk is an uncertain future event; an issue is a condition that has already occurred.
- **B**: A risk is a negative event; an issue is any situation requiring management attention.
- **C**: A risk is a long-term concern; an issue is a short-term problem that resolves itself.
- **D**: A risk is owned by the sponsor; an issue is owned by the project manager.

## Answer

A

## Explanation

Per PMI, a risk is an uncertain event or condition that, if it occurs, has an effect on at least one project objective; an issue is a point or matter in question or dispute that is currently affecting the project. (A) captures the core distinction of uncertainty versus realised condition. (B) conflates risk with negative outcomes only, ignoring that risks can be positive (opportunities). (C) invents a time-horizon distinction that isn't in PMI guidance. (D) confuses ownership rules with the definitional difference.
