# Question

A project has identified cybersecurity as a significant risk area, but no one on the team has security expertise. How should the PM address this risk area?

## Options

- **A**: Ignore security risks since the team can't assess them
- **B**: Engage security experts (internal or external) to help identify, assess, and plan responses for security risks
- **C**: List generic security risks from the internet
- **D**: Assume the IT department will handle security

## Answer

B

## Explanation

The correct answer is B. When the team lacks expertise in a risk area, bring in specialists. Engage experts: security specialists can identify risks the team would miss, assess them accurately, and recommend appropriate responses. This might be internal security team, external consultants, or security-focused vendors.

Option A ignores a critical risk area.

Option C may miss project-specific risks and not appropriately assess them.

Option D assumes without confirming security is someone else's responsibility.
