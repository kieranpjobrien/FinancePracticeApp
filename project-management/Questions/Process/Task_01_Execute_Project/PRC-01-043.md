# Question

The project delivered an MVP that users love, and there's pressure to immediately scale it across the organisation. However, the technical architecture was designed for MVP scale, not enterprise scale. What should the PM advise?

## Options

- **A**: Scale immediately since users want it
- **B**: Recommend a planned scaling approach that addresses architectural limitations before broad rollout, balancing user demand with sustainable growth
- **C**: Refuse all scaling until the architecture is perfect
- **D**: Ignore the architectural limitations

## Answer

B

## Explanation

The correct answer is B. MVP success is great, but scaling prematurely on an MVP architecture can create major problems. Plan a scaling approach: what architectural changes are needed? Can scaling be phased (critical users first)? Balance user demand (it's a good problem to have) with sustainable growth. Communicate timing honestly while addressing legitimate architecture concerns.

Option A risks serious performance or stability issues at scale.

Option C delays value delivery when a phased approach may work.

Option D creates technical debt that will be costly to address later.
