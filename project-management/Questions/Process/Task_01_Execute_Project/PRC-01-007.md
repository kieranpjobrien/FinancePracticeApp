# Question

What is the PRIMARY benefit of delivering project value incrementally rather than all at once at project end?

## Options

- **A**: It reduces overall project risk
- **B**: It provides earlier return on investment and allows course correction based on feedback
- **C**: It makes project planning simpler
- **D**: It requires fewer resources

## Answer

B

## Explanation

The correct answer is B. Incremental delivery provides value sooner (earlier ROI) and enables feedback loops. Stakeholders can use early deliverables and provide feedback that informs later work, reducing the risk of building the wrong thing.

Option A (reduces risk) is partially true but is a secondary benefit. Incremental delivery does reduce risk through feedback, but the primary benefit is earlier value realisation.

Option C (simpler planning) is incorrect. Incremental delivery often requires more planning sophistication (integration points, release management, etc.).

Option D (fewer resources) isn't accurate. Incremental delivery doesn't inherently require fewer resources - it's about when and how value is delivered.
