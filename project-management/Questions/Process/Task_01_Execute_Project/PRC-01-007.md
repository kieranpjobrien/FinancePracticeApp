# Question

Compared with delivering all scope in a single end-of-project release, what is the PRIMARY benefit of incremental delivery?

## Options

- **A**: It reduces total project cost by eliminating integration work.
- **B**: It enables earlier realisation of business value and feedback that informs later increments.
- **C**: It simplifies the work breakdown structure and project schedule.
- **D**: It removes the need for a formal change control process.

## Answer

B

## Explanation

Incremental delivery's defining benefit is putting working capability in users' hands sooner so the organisation captures value earlier and learns. (A) is wrong — incremental delivery typically increases integration overhead, not eliminates it. (C) is incorrect; incremental work often requires more sophisticated planning, not less. (D) is false in any methodology — change control still applies, though it may be lighter weight in agile contexts.
