# Question

A project to develop a new product is competing with a known competitor release in six months. The full feature set would take twelve months to develop. What approach should the PM recommend?

## Options

- **A**: Accept losing to the competitor
- **B**: Work double time to deliver in six months
- **C**: Define an MVP that can be delivered in six months with essential competitive features, then iterate to add more features post-launch
- **D**: Cancel the project since it can't beat the competitor

## Answer

C

## Explanation

The correct answer is C. When time-to-market is critical, define an MVP that delivers essential functionality within the window. This gets to market competitively with a viable product. Post-launch iterations add remaining features while gathering market feedback. Being in market with something beats being out of market with everything planned.

Option A concedes without exploring options.

Option B is likely unsustainable and risks quality.

Option D gives up when a viable path exists.
