# Question

During sprint execution, the team identifies that a user story is much larger than estimated and cannot be completed in the sprint. What should happen?

## Options

- **A**: Force the team to work overtime to complete it
- **B**: Split the story into smaller pieces, complete what can be done this sprint, and carry the remainder to the backlog for prioritisation
- **C**: Mark it as complete anyway
- **D**: Cancel the sprint

## Answer

B

## Explanation

The correct answer is B. When a story is too large, split it. Complete the portion that can be done this sprint (if it delivers standalone value). Return the remainder to the backlog for the product owner to prioritise. This maintains integrity of "done" while still delivering value.

Option A is unsustainable and affects quality.

Option C violates the definition of done and creates technical debt.

Option D is excessive for a single story problem.
