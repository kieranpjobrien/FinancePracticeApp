# Question

Technical debt has accumulated to the point where new feature development is significantly slowed. The product owner only wants to prioritise visible features. How should the PM address this?

## Options

- **A**: Let technical debt continue to accumulate
- **B**: Quantify the impact of technical debt on velocity and value delivery, and work with the PO to allocate capacity for debt reduction as an investment in future delivery speed
- **C**: Have the team secretly work on technical debt
- **D**: Stop all feature development to pay down debt

## Answer

B

## Explanation

The correct answer is B. Technical debt is a business decision. Quantify its impact: how much does it slow new development? What's the cost of continuing vs addressing it? Present this to the PO so they can make informed prioritisation decisions. Typically, allocating consistent capacity (e.g., 20%) for debt reduction balances immediate features with sustainable velocity.

Option A compounds the problem until development grinds to a halt.

Option C lacks transparency and doesn't involve the PO in prioritisation.

Option D stops value delivery entirely when balance is possible.
