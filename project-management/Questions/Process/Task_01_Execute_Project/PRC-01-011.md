# Question

What is the defining principle of value-driven delivery?

## Options

- **A**: Backlog items are sequenced by who requested them, so the loudest stakeholders are served first.
- **B**: Work is sequenced by ease of completion, so the team finishes more items per iteration.
- **C**: Work is sequenced by business value and risk reduction, so the highest-return items are delivered first.
- **D**: Work is sequenced strictly by dependency order to keep the technical architecture clean.

## Answer

C

## Explanation

Value-driven delivery sequences work by business value (and risk burndown), so that if the project is cut short the most important capabilities have already shipped. (A) optimises for politics, not value. (B) optimises for throughput of low-value work — high count, low return. (D) is a real consideration but a constraint on sequencing, not the driving principle; pure dependency order often pushes the highest-value features to the end.
