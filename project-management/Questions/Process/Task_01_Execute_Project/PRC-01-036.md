# Question

The team has been consistently delivering features, but velocity varies significantly sprint to sprint (30 to 50 points). Stakeholders are frustrated with unpredictable delivery. How should the PM address this?

## Options

- **A**: Force the team to commit to exactly 40 points every sprint
- **B**: Analyse velocity variation causes, improve estimation practices, stabilise team capacity, and set realistic ranges rather than point estimates for forecasting
- **C**: Stop measuring velocity
- **D**: Accept that prediction is impossible

## Answer

B

## Explanation

The correct answer is B. Velocity variation has causes that can be investigated. Common causes: inconsistent estimation, varying work complexity, team disruptions, unplanned work. Address root causes. Improve estimation through refinement practices. Consider forecasting in ranges that acknowledge uncertainty. Stable teams with consistent practices typically achieve more predictable velocity over time.

Option A ignores underlying causes and creates false precision.

Option C loses valuable planning information.

Option D gives up on improvement when causes can be addressed.
