# Question

What does "working software over comprehensive documentation" mean in the Agile Manifesto?

## Options

- **A**: Never write any documentation
- **B**: Prioritise delivering functional software as the primary measure of progress, while still producing necessary documentation
- **C**: Working software is the only deliverable
- **D**: Documentation is forbidden in agile

## Answer

B

## Explanation

The correct answer is B. The Agile Manifesto values working software more highly than comprehensive documentation, but doesn't eliminate documentation. Functional software that delivers value is the primary measure of progress. Documentation is still produced where needed (user guides, API docs, regulatory requirements), but isn't created for its own sake.

Options A, C, and D misinterpret the manifesto as eliminating documentation entirely, which is not the intent.
