# Question

In Scrum, what is the primary measure of progress?

## Options

- **A**: The percentage of story points completed against the release plan.
- **B**: The number of sprint ceremonies the team holds without skipping any.
- **C**: Working software that meets the definition of done.
- **D**: The team's velocity trend over the last three sprints.

## Answer

C

## Explanation

The Agile Manifesto and the Scrum Guide both name working software as the primary measure of progress — anything else is a proxy that can mislead. (C) reflects this directly. (A) measures planned consumption, which can be inflated by changing estimates. (B) measures process compliance, not outcome. (D) is a useful planning input but a leading indicator of capacity, not a measure of delivered value.
