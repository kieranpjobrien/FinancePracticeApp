# Question

A project team is consistently delivering features on time, but feedback from users indicates the features aren't being used because they don't address actual pain points. The product owner believes they know what users need. What should the project manager recommend?

## Options

- **A**: Trust the product owner's expertise and continue as planned
- **B**: Recommend implementing direct user research and validation loops to ensure delivered features actually solve user problems
- **C**: Pause feature development until user needs are better understood
- **D**: Add more features to increase the chances of hitting user needs

## Answer

B

## Explanation

The correct answer is B. Delivering features users don't use generates no value - it's waste. The solution is validating with actual users through research, prototyping, early releases, and usage analytics. This creates feedback loops that correct course without stopping work entirely.

Option A trusts assumptions that evidence suggests are wrong.

Option C stops all progress when the issue is direction, not pace. Validation can happen in parallel with development.

Option D increases output while maintaining the same misalignment - more waste, not more value.
