# Question

A SaaS integration project depends on a partner API that was promised ready two weeks ago but is still unstable. The partner's PM provides weekly updates with no commitment date. The team's integration work is blocked. What should the PM do FIRST?

## Options

- **A**: Instruct the team to mock the partner API so integration and downstream testing can proceed against a simulated endpoint.
- **B**: Escalate the missed commitment to both sponsors and ask them to jointly agree a revised integration readiness date.
- **C**: Reassign the blocked team to unrelated backlog items until the partner confirms the API is fully production-ready.
- **D**: Request a working session with the partner's engineering lead to understand actual blockers and agree a concrete stabilisation plan.

## Answer

D

## Explanation

PMI expects the PM to pursue root cause and work the problem at the level where it can be solved before escalating or working around it. (D) engages engineering reality. (A) mocking hides the dependency risk and can mask integration defects until late. (B) escalating without engineering facts in hand will generate heat, not a plan. (C) reassigning is sensible tactically but doesn't address the unresolved external dependency.
