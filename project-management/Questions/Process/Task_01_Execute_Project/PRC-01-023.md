# Question

The team is consistently delivering completed features, but user adoption is low because the features don't match actual user needs. Users weren't adequately involved during development. How should the PM address this going forward?

## Options

- **A**: Continue delivering features as planned since that's what was agreed
- **B**: Increase user involvement through regular demos, feedback sessions, and usability testing to ensure delivered features actually meet user needs
- **C**: Stop development until users are satisfied
- **D**: Build whatever users request without analysis

## Answer

B

## Explanation

The correct answer is B. Delivery without user validation creates waste. Increase user involvement: regular demos provide feedback opportunities, usability testing validates assumptions, direct user input shapes development. Value isn't delivered until users can actually use and benefit from features.

Option A continues delivering low-value features.

Option C is too drastic when adjustment can happen iteratively.

Option D may lead to scope creep without prioritisation.
