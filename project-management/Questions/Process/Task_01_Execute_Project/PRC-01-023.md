# Question

A team has shipped four releases over a year. Adoption telemetry shows users opening features briefly and then abandoning them. End users were not involved during requirement gathering or in any sprint review. What should the PM put in place going forward?

## Options

- **A**: Schedule monthly steering committee reviews where business sponsors validate features on behalf of users.
- **B**: Embed regular user sessions — sprint reviews with real users, prototype testing, and adoption telemetry reviews — into the cadence.
- **C**: Pause development and run a four-week discovery phase with users before resuming the roadmap.
- **D**: Add a UX writer to the team to improve in-product copy and reduce abandonment.

## Answer

B

## Explanation

The root cause is the missing user feedback loop, not a one-time discovery gap or a copy problem. (B) institutionalises the loop so each release is informed by real users — the structural fix. (A) substitutes sponsors for users, which is exactly the gap that produced the problem. (C) over-corrects by stopping delivery when continuous validation can run alongside it. (D) treats a symptom (copy) when the issue is feature-market fit.
