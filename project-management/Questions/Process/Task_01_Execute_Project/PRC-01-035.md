# Question

The project is delivering features on schedule, but the business hasn't prepared for adoption - change management, training, and process updates are lagging. Even delivered features aren't being used. What should the PM do?

## Options

- **A**: Continue delivering features as that's the project scope
- **B**: Raise the adoption gap as a risk to value realisation, coordinate with change management and training teams, and adjust delivery pace if needed to allow adoption to catch up
- **C**: Stop development until adoption catches up
- **D**: Add adoption responsibilities to the development team

## Answer

B

## Explanation

The correct answer is B. Features without adoption don't deliver value. Raise the issue: value realisation requires adoption, which requires change management and training. Coordinate across workstreams to align delivery with adoption readiness. Consider pacing delivery to allow adoption activities to keep up. Value isn't delivered until users are successfully using the features.

Option A delivers features that don't translate to business value.

Option C may be excessive when coordination can solve the problem.

Option D mixes responsibilities that may require different skills.
