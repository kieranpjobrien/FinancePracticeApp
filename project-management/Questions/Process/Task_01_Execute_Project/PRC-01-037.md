# Question

What is continuous integration (CI) in software delivery?

## Options

- **A**: Frequent merging of developer changes into a shared branch with automated builds and tests to surface integration problems early.
- **B**: A practice of integrating end users into the development team to gather requirements continuously.
- **C**: Maintaining a single, always-on production environment that receives every change instantly.
- **D**: A documentation method that keeps requirements, design, and test artefacts in a single repository.

## Answer

A

## Explanation

CI is specifically about merging code frequently and verifying with automated build and test on every merge. (A) is the standard definition. (B) describes user research or embedded users, not CI. (C) describes continuous deployment, which is downstream of CI. (D) describes documentation hygiene, unrelated to CI.
