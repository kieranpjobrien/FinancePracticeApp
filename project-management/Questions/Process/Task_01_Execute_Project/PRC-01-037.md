# Question

What is "continuous integration" in software development?

## Options

- **A**: Integrating the team socially through events
- **B**: Frequently merging code changes into a shared repository with automated builds and tests to detect problems early
- **C**: Continuously adding new team members
- **D**: Integrating multiple projects into one

## Answer

B

## Explanation

The correct answer is B. Continuous integration involves frequently merging developer code changes into a shared repository, typically multiple times per day. Each merge triggers automated builds and tests. This practice detects integration problems early when they're easier to fix, rather than discovering them late in development.

Options A, C, and D misinterpret the "integration" concept.
