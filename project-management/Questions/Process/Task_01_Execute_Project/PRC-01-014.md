# Question

Which statement best describes a Minimum Viable Product (MVP)?

## Options

- **A**: A low-cost, low-quality version of the product built to test if customers will pay anything.
- **B**: The smallest set of features that delivers real value to early users and produces validated learning.
- **C**: A product that meets only the legal and regulatory minimums needed to ship.
- **D**: The first internal release built to test the team's tooling and deployment pipeline.

## Answer

B

## Explanation

An MVP is defined by learning and value, not cost or quality. (B) captures both elements — minimum scope, real value, and learning that informs the next increment. (A) confuses "minimum" with "cheap and shoddy"; an MVP must still meet quality standards or it produces misleading feedback. (C) describes regulatory minimums, a different concept. (D) describes an internal release or smoke test, not an MVP.
