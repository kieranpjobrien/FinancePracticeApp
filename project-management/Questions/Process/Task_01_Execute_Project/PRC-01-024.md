# Question

A waterfall project has reached the testing phase. Multiple defects are found that trace back to requirements ambiguity. Fixing them properly would require significant rework. The deadline is approaching. What should the PM do?

## Options

- **A**: Ship with known defects and fix them later
- **B**: Assess defect severity, fix critical issues, defer non-critical ones with stakeholder agreement, and implement lessons learned for requirements clarity
- **C**: Extend the deadline without consultation
- **D**: Blame the requirements team

## Answer

B

## Explanation

The correct answer is B. Triage defects by severity and business impact. Critical defects affecting core functionality must be fixed. Non-critical defects can be deferred with stakeholder agreement (documented). Learn from the situation: improve requirements processes for future phases/projects. This balances quality with deadline reality.

Option A may ship an unusable or risky product.

Option C changes commitments without stakeholder involvement.

Option D doesn't solve the problem or deliver value.
