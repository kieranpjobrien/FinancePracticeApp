# Question

What is the primary purpose of timeboxing in agile delivery?

## Options

- **A**: To create fixed-duration intervals that force prioritisation and protect a regular cadence of delivery.
- **B**: To set a maximum number of hours team members are allowed to work on a project each week.
- **C**: To allocate buffer time at the end of every sprint to absorb estimation errors.
- **D**: To establish a firm deadline beyond which scope can no longer be added.

## Answer

A

## Explanation

A timebox fixes time so scope must flex inside it — the cadence and the forced prioritisation are the point. (A) captures both. (B) describes work-hour limits, a different concept. (C) reframes timeboxing as a buffer mechanism — but the timebox doesn't end early if work finishes, and adding buffer changes the timebox's purpose. (D) describes a scope-freeze, which is related to but distinct from timeboxing.
