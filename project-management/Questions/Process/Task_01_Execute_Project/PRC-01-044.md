# Question

The team uses a Kanban approach with WIP limits. A critical issue arises that must be addressed immediately, but all WIP slots are occupied. What should happen?

## Options

- **A**: Ignore WIP limits for critical issues
- **B**: Expedite the critical issue through a dedicated swim lane for emergencies, while ensuring the team finishes current work before starting new non-critical items
- **C**: Wait until a slot opens naturally
- **D**: Abandon Kanban for this project

## Answer

B

## Explanation

The correct answer is B. Kanban systems typically include an expedite lane for genuine emergencies that bypasses normal WIP limits. Use it for the critical issue. Meanwhile, maintain discipline on other work - finish current items before starting new ones. If emergencies are frequent, examine why - they may indicate systemic issues needing attention.

Option A removes a valuable constraint without appropriate governance.

Option C may be too slow for a genuine critical issue.

Option D abandons valuable practices for a manageable situation.
