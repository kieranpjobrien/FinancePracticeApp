# Question

A project is running behind schedule, and the team identifies that some planned features are "nice to have" while others are essential for the business case. What should the PM do?

## Options

- **A**: Work overtime to deliver everything as planned
- **B**: Negotiate with stakeholders to descope or defer lower-value features while protecting essential functionality
- **C**: Cut all features equally to reduce scope
- **D**: Extend the deadline without discussing with stakeholders

## Answer

B

## Explanation

The correct answer is B. When facing schedule pressure, the PM should facilitate value-based decisions. Work with stakeholders to identify must-have vs nice-to-have features. Protecting essential functionality while descoping lower-value items ensures the project delivers its core value proposition even under constraints.

Option A may not be sustainable and doesn't address the fundamental scope/time trade-off.

Option C treats all features equally when they have different business values.

Option D changes commitments without stakeholder involvement.
