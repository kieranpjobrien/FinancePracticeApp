# Question

A project is implementing a regulatory requirement with a hard compliance deadline. The team is using agile methods but must meet the fixed scope by the fixed date. How should the PM approach this situation?

## Options

- **A**: Abandon agile for waterfall since the deadline is fixed
- **B**: Use agile practices within the constraint - prioritise ruthlessly so essential compliance is delivered first, have contingency if full scope is at risk, and maintain transparency about progress
- **C**: Ignore the regulatory deadline
- **D**: Commit to everything and hope for the best

## Answer

B

## Explanation

The correct answer is B. Fixed deadlines don't preclude agile practices. Prioritise: ensure must-have compliance features are delivered first. Contingency: if the full scope is at risk, what's the minimum for compliance? Transparency: keep stakeholders informed about progress so decisions can be made early. Agile's frequent delivery and feedback help identify risks sooner than waterfall.

Option A throws out valuable practices unnecessarily.

Option C ignores legal requirements.

Option D doesn't actively manage the risk.
