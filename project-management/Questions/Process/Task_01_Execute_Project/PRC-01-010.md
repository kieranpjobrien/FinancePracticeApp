# Question

In agile methodology, what does "working software over comprehensive documentation" mean?

## Options

- **A**: Documentation should never be created
- **B**: Functional deliverables that provide value are prioritised over extensive documentation, though necessary documentation is still produced
- **C**: Software developers should not write any comments in code
- **D**: Only user manuals are needed, not technical documentation

## Answer

B

## Explanation

The correct answer is B. The Agile Manifesto values working software MORE than comprehensive documentation - but doesn't eliminate documentation. The principle means teams should focus on delivering functional value rather than producing excessive documentation. Necessary documentation (user guides, API docs, compliance records) is still created as needed.

Option A misinterprets "over" as "instead of". Agile still values documentation - just not at the expense of working software.

Option C confuses documentation with code comments. These are different concerns.

Option D arbitrarily limits documentation types. The principle is about prioritisation, not restricting documentation categories.
