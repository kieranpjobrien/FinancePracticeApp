# Question

The Agile Manifesto states a preference for "working software over comprehensive documentation." Which interpretation is correct?

## Options

- **A**: Agile teams should produce no written documentation; everything should be in the code.
- **B**: Documentation should be deferred to the end of the project so it's accurate.
- **C**: Working software is valued more than documentation, but documentation needed to deliver and operate the product is still produced.
- **D**: Only end-user documentation should be produced; internal technical documentation is unnecessary.

## Answer

C

## Explanation

The manifesto's "X over Y" pattern means "while there is value in Y, we value X more" — not "Y is forbidden". (C) captures this correctly. (A) misreads "over" as "instead of". (B) describes the waterfall problem agile was reacting against; documentation is produced just-in-time, not at the end. (D) arbitrarily restricts documentation type; the principle is about prioritisation, not category.
