# Question

The team is working on a complex feature that will take three sprints to complete. Stakeholders are asking for visibility into progress. What approach best demonstrates value delivery?

## Options

- **A**: Wait until the feature is complete to show anything
- **B**: Break the feature into incremental slices that deliver partial functionality each sprint, demonstrating progress through working software
- **C**: Provide detailed status reports without working software
- **D**: Show only the final sprint's work

## Answer

B

## Explanation

The correct answer is B. Even large features can be decomposed into increments that deliver partial value or demonstrate progress through working software. This enables feedback, reduces risk, and shows tangible progress. Vertical slicing (thin end-to-end functionality) is preferred over horizontal slicing (complete layers without visible progress).

Option A delays feedback and hides risk.

Option C provides information but not demonstrated value.

Option D ignores the opportunity for earlier validation.
