# Question

What is the primary benefit of iterative delivery compared to single "big bang" delivery?

## Options

- **A**: Less total work is required
- **B**: Frequent delivery enables earlier value realisation, faster feedback, and reduced risk through continuous validation
- **C**: Documentation requirements are eliminated
- **D**: Stakeholder involvement is reduced

## Answer

B

## Explanation

The correct answer is B. Iterative delivery provides multiple benefits: value is realised earlier (not waiting until the end), feedback comes sooner (enabling course correction), risk is reduced (problems surface early), and learning is continuous. Each iteration delivers working functionality that can be evaluated.

Option A is not necessarily true - iterative may require similar or more effort.

Option C is incorrect - documentation is still needed, just approached differently.

Option D is opposite - iterative typically increases stakeholder involvement.
