# Question

The team delivered a feature last sprint, but production issues emerged after deployment. The product owner wants to continue with new features while support wants the team to focus on production issues. How should the PM facilitate this?

## Options

- **A**: Let the PO decide since they own the backlog
- **B**: Treat production issues as high-priority items that affect current users, balance with new development based on severity, and ensure the team has capacity for both
- **C**: Ignore production issues and focus on new features
- **D**: Stop all new development until production is stable

## Answer

B

## Explanation

The correct answer is B. Production issues affect current users - they represent delivered value that's being degraded. Work with PO to prioritise based on severity: critical production issues may warrant immediate focus. Balance: perhaps allocate capacity for both (e.g., 80% new, 20% production). Ensure the team isn't constantly whipsawed between conflicting priorities.

Option A ignores that production issues may override normal backlog prioritisation.

Option C degrades delivered value and damages user trust.

Option D may be excessive if issues are minor.
