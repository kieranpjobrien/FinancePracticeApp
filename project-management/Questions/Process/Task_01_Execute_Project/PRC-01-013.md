# Question

The product owner wants to add a new feature mid-sprint that a key customer has urgently requested. The team is already at capacity. What is the appropriate action?

## Options

- **A**: Automatically reject all mid-sprint changes
- **B**: Add the feature without removing anything
- **C**: Evaluate the feature's value against current sprint items and, if warranted, swap out lower-value work to accommodate it
- **D**: Tell the customer to wait until next year

## Answer

C

## Explanation

The correct answer is C. Agile approaches accommodate change when it delivers value. Evaluate the new request against current work. If the new feature is more valuable than something in the sprint, it may warrant a swap. The sprint capacity remains constant - adding something means removing something of equal or lesser value.

Option A is too rigid and may miss genuine value opportunities.

Option B overcommits the team and threatens sprint completion.

Option D doesn't consider the business value of the urgent request.
