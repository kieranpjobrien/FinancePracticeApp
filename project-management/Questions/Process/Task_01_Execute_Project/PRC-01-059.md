# Question

A mobile app team has shipped three releases in six months. Usage metrics show new features are being used less than 10% by active users. The PO continues planning the next three releases around new features. What should the PM raise?

## Options

- **A**: Suggest reducing release cadence so users have more time to discover and adopt the features that have already shipped.
- **B**: Pause new-feature development until analytics and user research explain why recent shipped features aren't being used.
- **C**: Recommend the team shift toward discovery and measurement cycles before continuing to commit backlog capacity to new features.
- **D**: Ask marketing to run a promotion campaign drawing attention to the recently shipped but under-used feature set.

## Answer

C

## Explanation

Shipping isn't value; adoption and outcomes are. (C) pivots the team toward learning before more building, consistent with empirical process control. (A) spacing releases doesn't address why features aren't landing. (B) a full pause is overcorrection without evidence the pause itself will yield answers. (D) marketing a feature users don't value is treating the symptom, not the root cause.
