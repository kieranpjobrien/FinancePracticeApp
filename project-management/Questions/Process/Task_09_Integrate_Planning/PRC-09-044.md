# Question

A team using Kanban doesn't do iteration planning. A stakeholder asks for a project timeline. How can the PM provide planning visibility without imposing iterations?

## Options

- **A**: Switch to Scrum to enable timeline planning
- **B**: Use flow metrics (cycle time, throughput) to forecast completion ranges
- **C**: Explain that Kanban doesn't support timeline planning
- **D**: Estimate each work item individually and sum to a timeline

## Answer

B

## Explanation

The correct answer is B. Kanban's flow metrics enable probabilistic forecasting. Historical cycle time and throughput data can predict when a scope of work will likely complete, often with confidence intervals. This provides planning visibility appropriate to the methodology.

Option A abandons a potentially appropriate methodology to satisfy a reporting request.

Option C incorrectly states Kanban's capabilities. Flow-based forecasting is well-established.

Option D imposes a practice (individual estimation) that Kanban typically doesn't use and loses the statistical benefits of flow metrics.
