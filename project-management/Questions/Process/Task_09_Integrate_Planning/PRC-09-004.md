# Question

A hybrid IoT project pairs a waterfall hardware team with an agile software team. The hardware team needs frozen software interface specifications by month 3 to start integration, but the software team intends to evolve features iteratively. How should the project manager integrate the two?

## Options

- **A**: Lock down all software requirements by month 3 to give the hardware team the certainty they need.
- **B**: Freeze only the interface contracts (APIs, data formats, protocols) hardware depends on; let other features evolve.
- **C**: Defer hardware integration until the software is feature-complete to avoid friction between the two streams.
- **D**: Run hardware and software as wholly independent projects with separate governance and reporting lines.

## Answer

B

## Explanation

The integration discipline is to separate stable interfaces from evolving features. Lock the contracts that downstream work depends on; leave the rest agile. (A) imposes waterfall on the entire software stream and discards iterative value. (C) often blows the schedule because hardware has long lead times. (D) denies the actual coupling between the streams.
