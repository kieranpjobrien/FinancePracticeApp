# Question

A hybrid project uses waterfall for hardware development and agile for software. The hardware team needs finalised software specifications by month 3 to begin integration testing, but the agile team plans to evolve requirements through iteration. How should the project manager integrate these approaches?

## Options

- **A**: Force the agile team to finalise all specifications by month 3
- **B**: Delay hardware integration testing until software requirements stabilise
- **C**: Identify interface specifications that hardware needs fixed, allow other software requirements to remain flexible
- **D**: Run hardware and software as separate projects with independent timelines

## Answer

C

## Explanation

The correct answer is C. This is interface management in a hybrid project. The PM should identify which specifications are truly fixed dependencies (APIs, data formats, hardware interfaces) and lock those down, while allowing other software features to evolve. This respects both the hardware team's need for stability and the software team's need for flexibility.

Option A (force waterfall on agile) eliminates the benefits of agile for software development. Not all specifications need to be fixed - only those affecting hardware integration.

Option B (delay hardware) may significantly impact the overall schedule. Hardware often has longer lead times, so delaying integration creates larger downstream effects.

Option D (separate projects) ignores the integration dependencies. The projects aren't truly independent if they must work together.
