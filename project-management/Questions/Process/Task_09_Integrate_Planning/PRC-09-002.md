# Question

Why is integrated change control important in project management?

## Options

- **A**: To prevent any changes to the project
- **B**: To ensure changes are evaluated for impact across all project aspects (scope, schedule, cost, quality, risk) before approval
- **C**: To document all stakeholder complaints
- **D**: To speed up the approval process for changes

## Answer

B

## Explanation

The correct answer is B. Projects have interrelated components - a scope change affects schedule, which affects cost, which may create new risks. Integrated change control ensures changes are assessed holistically across all aspects before approval, preventing unintended consequences from partial analysis.

Option A misunderstands the purpose - change control manages changes, it doesn't prevent all changes.

Option C describes issue tracking, not change control.

Option D is wrong - integrated control may actually add time to ensure proper assessment. Speed isn't the goal; informed decisions are.
