# Question

A PM presents an integrated project plan to the steering committee. One executive questions why contingency wasn't added to each activity. Another asks why there's any contingency at all since each estimate should be accurate. How should the PM respond?

## Options

- **A**: Add contingency to each activity to satisfy the first executive
- **B**: Remove contingency to satisfy the second executive
- **C**: Explain how contingency is managed at the appropriate level based on risk assessment
- **D**: Offer two versions of the plan - one with detailed contingency, one without

## Answer

C

## Explanation

The correct answer is C. The PM should educate stakeholders on contingency management. Contingency at activity level encourages use for non-risks. Contingency at project level, sized by risk analysis, provides appropriate buffer without padding every task. Neither executive's extreme view reflects best practice.

Option A and B capitulate to stakeholder preferences without professional judgement.

Option D creates confusion with multiple plan versions and doesn't resolve the underlying misunderstanding.
