# Question

A project using iterative development has completed three sprints. Velocity data suggests the original release date is unachievable with current scope. When should this trigger plan revision?

## Options

- **A**: Immediately, since data indicates the plan is invalid
- **B**: After three more sprints to confirm the trend
- **C**: At the next scheduled planning review
- **D**: Only if stakeholders raise concerns about the timeline

## Answer

A

## Explanation

The correct answer is A. Three sprints provide meaningful velocity data. If the data clearly shows the plan is unachievable, waiting provides no value and delays corrective action. Empirical planning means acting on evidence. The PM should analyse options and engage stakeholders promptly.

Option B delays unnecessarily when evidence is already sufficient.

Option C may be too late depending on when the next review is scheduled. Data-driven planning should be responsive.

Option D is reactive rather than proactive. The PM should raise issues based on data, not wait for others.
