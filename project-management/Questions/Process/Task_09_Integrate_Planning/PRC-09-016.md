# Question

During planning, a PM realises that achieving the desired scope within the fixed deadline will require more resources than budgeted. What does this represent?

## Options

- **A**: A project risk that should be added to the risk register
- **B**: A constraint conflict requiring trade-off decisions
- **C**: Proof that the project should not proceed
- **D**: Normal planning iteration that will resolve itself

## Answer

B

## Explanation

The correct answer is B. The triple constraint (scope, time, cost) creates trade-offs. Fixed deadline and scope with insufficient budget means constraints conflict. The PM must present options: reduce scope, extend deadline, increase budget, or accept higher risk. This is a core integration challenge.

Option A treats a fundamental constraint conflict as just a risk. It's more significant than that.

Option C is premature. Many projects proceed with trade-offs; this doesn't automatically mean failure.

Option D ignores a real problem. Planning won't magically create resources or time.
