# Question

A project manager is creating the project management plan for a hybrid project that uses waterfall for infrastructure and agile for application development. The two tracks have dependencies. What approach should be used for integrated planning?

## Options

- **A**: Create separate plans for each track and coordinate during execution
- **B**: Develop an integrated plan that defines handoff points, synchronization milestones, and coordination mechanisms between the tracks
- **C**: Convert everything to agile to simplify planning
- **D**: Use a predictive approach for both tracks since it's easier to integrate

## Answer

B

## Explanation

The correct answer is B. Hybrid projects require intentional integration planning. The plan should identify where tracks interact (dependencies, handoffs), establish synchronization points (milestones where tracks must align), and define how coordination will happen. This acknowledges both methodologies' strengths while managing their interfaces.

Option A creates two independent plans that don't address integration - the core challenge of hybrid projects.

Option C imposes agile on work that may genuinely benefit from predictive (infrastructure often does). The hybrid approach was presumably chosen for good reasons.

Option D similarly forces a single methodology when hybrid may be appropriate. The challenge is integration, not methodology selection.
