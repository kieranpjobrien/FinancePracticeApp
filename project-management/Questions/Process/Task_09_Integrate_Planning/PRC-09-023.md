# Question

A PM is planning a project using organisational process assets from previous similar projects. The historical data shows those projects consistently underestimated effort. How should this inform the current plan?

## Options

- **A**: Ignore historical data since each project is unique
- **B**: Apply a correction factor to estimates based on historical underestimation patterns
- **C**: Use historical data as-is since it's the best available information
- **D**: Double all estimates to ensure adequate buffer

## Answer

B

## Explanation

The correct answer is B. Historical data's value is in revealing patterns. If past projects consistently underestimated by a measurable amount, applying a correction factor improves accuracy. This is evidence-based planning that learns from organisational experience.

Option A discards valuable information. While projects differ, patterns in estimation accuracy are useful.

Option C repeats known errors. If historical estimates were consistently wrong, using them unchanged perpetuates the problem.

Option D applies arbitrary correction without analysis. The correction should be proportionate to the documented pattern.
