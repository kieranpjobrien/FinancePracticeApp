# Question

Why is the integration knowledge area considered uniquely the project manager's responsibility?

## Options

- **A**: Because integration requires deeper technical expertise than other knowledge areas.
- **B**: Because only the PM is positioned to see the project as a whole and balance trade-offs across all knowledge areas.
- **C**: Because integration involves more stakeholder communication than any other area.
- **D**: Because integration has more processes than any other knowledge area.

## Answer

B

## Explanation

Functional specialists own their domains; the PM is the only role accountable for the whole. That cross-cutting view is what makes integration uniquely the PM's. (A) is wrong; integration is about orchestration, not depth in any one technology. (C) describes communications management. (D) is incorrect on count and does not explain ownership.
