# Question

In an agile project, what serves the function of the project management plan?

## Options

- **A**: The product backlog
- **B**: The sprint backlog
- **C**: A combination of release plan, team agreements, and Definition of Done
- **D**: Agile projects don't have plans

## Answer

C

## Explanation

The correct answer is C. Agile projects are planned; they just plan differently. The release plan provides strategic direction. Team agreements (working agreements, Definition of Ready, Definition of Done) define how work is done. Together with ceremonies and practices, these create a planning framework that serves similar functions to a predictive project management plan.

Option A defines what might be built, not how the project operates.

Option B defines work for one sprint, not the project-level approach.

Option D is a common misconception. Agile emphasises adaptive planning, not absence of planning.
