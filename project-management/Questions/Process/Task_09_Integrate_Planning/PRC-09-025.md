# Question

What distinguishes the project management plan from project documents?

## Options

- **A**: The plan is approved; documents are informal
- **B**: The plan guides execution; documents support planning and execution activities
- **C**: The plan is created first; documents come later
- **D**: The plan belongs to the PM; documents belong to the team

## Answer

B

## Explanation

The correct answer is B. The project management plan (and its subsidiary plans) defines how the project is executed, monitored, and controlled. Project documents (risk register, issue log, stakeholder register, etc.) support activities defined in the plan but don't themselves define how work is done.

Option A oversimplifies. Both plans and documents may be formally approved depending on governance.

Option C isn't necessarily true. Some documents (like stakeholder register) may be created before parts of the plan.

Option D misattributes ownership. Both the plan and documents are project assets with various owners.
