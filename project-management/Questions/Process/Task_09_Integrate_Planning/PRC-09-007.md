# Question

A project involves regulatory compliance requiring specific documentation deliverables. The agile team wants minimal documentation per agile principles. How should the PM integrate these requirements?

## Options

- **A**: Follow agile principles and minimise all documentation
- **B**: Follow regulatory requirements and maximise documentation
- **C**: Tailor the approach - meet regulatory requirements while keeping non-mandated documentation lean
- **D**: Run two separate projects - one agile, one compliance-focused

## Answer

C

## Explanation

The correct answer is C. Effective integration means adapting approaches to context. Regulatory documentation is non-negotiable and should be planned explicitly. For everything else, agile principles of minimal documentation apply. The PM should identify exactly what compliance requires, build those deliverables into the plan, and keep other documentation lean.

Option A ignores real constraints. Regulatory requirements exist regardless of methodology preferences.

Option B overcorrects. Just because some documentation is required doesn't mean all documentation should be maximised.

Option D adds complexity without benefit. Compliance documentation can be integrated into an agile workflow.
