# Question

An agile project team is developing a customer-facing application. The product owner has been making decisions without consulting end users, leading to features that don't match user needs. What should the Scrum Master recommend?

## Options

- **A**: Replace the product owner with someone who understands users better
- **B**: Have the development team validate requirements directly with end users
- **C**: Establish regular user feedback sessions such as sprint reviews with actual users or user testing
- **D**: Document the issue and present it at the next retrospective

## Answer

C

## Explanation

The correct answer is C. Regular user feedback sessions address the root cause - lack of end user input in decision-making. Sprint reviews with actual users or dedicated user testing sessions create structured opportunities for direct stakeholder engagement. This is consistent with agile principles of customer collaboration.

Option A (replace PO) is drastic and doesn't solve the systemic issue. The next PO might have the same problem without user feedback mechanisms.

Option B (dev team validates) undermines the PO role and creates confusion about who owns the product backlog. The solution is better stakeholder engagement, not role confusion.

Option D (discuss at retro) is too passive. While retrospectives are valuable, this issue needs immediate action, not just discussion.
