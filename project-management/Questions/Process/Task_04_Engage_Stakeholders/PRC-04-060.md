# Question

A PM in a weak matrix organisation is struggling to get functional managers to commit their staff reliably to the project. Promised allocations keep slipping when competing functional priorities arise. What should the PM do?

## Options

- **A**: Escalate each slippage to the PMO so functional managers face consistent governance pressure to honour project resource allocations reliably over time.
- **B**: Negotiate formal resource agreements for each key person on the project, with named backups and escalation triggers agreed up-front with managers.
- **C**: Build personal relationships with each functional manager, understand their competing priorities, and co-plan project allocations around shared team calendars each cycle.
- **D**: Request a shift to a stronger matrix for this project so the PM's authority over resource allocation is formally increased for delivery.

## Answer

C

## Explanation

In weak matrix environments, the PM's primary tool is relationship-driven influence; formal escalation and structural change are slow and often ineffective. (C) builds the influence needed to run this project. (A) governance pressure without relationship investment creates resistance. (B) formal agreements help but don't survive real functional pressure without relationship foundation. (D) shifting matrix structure is an org-design change outside PM authority.
