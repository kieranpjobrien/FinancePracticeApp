# Question

An agile project has multiple stakeholder representatives attending sprint reviews. They often disagree among themselves during reviews, causing confusion about feedback. How should this be handled?

## Options

- **A**: Exclude stakeholders from sprint reviews
- **B**: Acknowledge different perspectives, capture all feedback, work with the product owner to reconcile priorities, and communicate back how conflicting feedback is being addressed
- **C**: Let stakeholders fight it out during reviews
- **D**: Only allow one stakeholder to attend

## Answer

B

## Explanation

The correct answer is B. Diverse stakeholder feedback is valuable even when conflicting. Capture: document all perspectives during the review. Reconcile: work with PO offline to prioritise conflicting input based on project objectives. Communicate: close the loop with stakeholders about how their feedback is being addressed. This values all input while maintaining decision-making efficiency.

Option A loses valuable feedback.

Option C creates conflict without resolution.

Option D excludes important perspectives.
