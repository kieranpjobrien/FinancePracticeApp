# Question

A 14-month transformation programme is closing and the team has captured a strong set of lessons learned, including several that contradict an existing project methodology template. The PM wants these lessons to actually change future practice. What is the BEST way to ensure they are used?

## Options

- **A**: Work with the PMO to integrate the relevant lessons into templates, checklists, and onboarding so they shape future projects by default.
- **B**: Publish the lessons learned register in the project archive and notify all programme managers by email.
- **C**: Run a brown-bag presentation for interested project managers and leave the slides on the shared drive.
- **D**: Append the lessons to the existing project methodology document under a "recent learnings" section.

## Answer

A

## Explanation

PMI considers lessons learned a contribution to organisational process assets, and behaviour change happens when the learnings become embedded in the artifacts people actually use. (A) routes the insights through the PMO so they update the defaults future PMs work from, which is how lessons drive change at scale. (B) creates a record but relies on people seeking it out. (C) reaches a self-selected audience and does not survive turnover. (D) buries the lessons in a static document that is rarely re-read, so practice does not change.
