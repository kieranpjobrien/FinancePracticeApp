# Question

A project is transitioning from development to operations. The operations team complains they weren't involved earlier and are unprepared to support the system. What should have been done differently?

## Options

- **A**: Operations shouldn't be involved until the development phase is complete
- **B**: The project should include operations stakeholders from planning phase and define transition criteria early
- **C**: Development team should continue supporting the system indefinitely
- **D**: Operations concerns should be addressed after go-live

## Answer

B

## Explanation

The correct answer is B. Successful transitions require early engagement with receiving teams. Operations should be stakeholders throughout the project - understanding the system being built, providing input on supportability requirements, and preparing for transition. Transition criteria should be defined upfront, not discovered at handover.

Option A (involve at completion) creates the exact problem described. Late involvement means operations is unprepared and may resist accepting the system.

Option C (indefinite dev support) isn't sustainable. Development teams need to move to new projects; operations must eventually own the system.

Option D (address after go-live) is too late. Post-go-live is when support gaps create incidents and user impact.
