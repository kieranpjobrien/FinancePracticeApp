# Question

What is the purpose of a project closure checklist?

## Options

- **A**: To create extra work for the project manager
- **B**: To ensure all necessary closure activities are completed systematically without overlooking important items
- **C**: To delay project completion
- **D**: To satisfy audit requirements only

## Answer

B

## Explanation

The correct answer is B. Closure involves many activities (acceptance, contracts, archives, lessons learned, resource release, etc.) that can be forgotten in the rush to finish. A checklist ensures systematic completion of all necessary activities. It's a practical tool for quality closure, not bureaucratic overhead.

Option A views checklists as burden rather than help.

Option C misrepresents the purpose - checklists help complete closure efficiently.

Option D understates the operational value of complete closure.
