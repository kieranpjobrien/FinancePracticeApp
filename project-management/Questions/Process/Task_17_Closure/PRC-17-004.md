# Question

Which document formally authorises the completion of a project and confirms that resources can be released?

## Options

- **A**: The project charter signed at initiation
- **B**: The final project status report and dashboard
- **C**: The project closure document signed by the sponsor
- **D**: The lessons learned register from closeout

## Answer

C

## Explanation

The signed project closure document (sometimes called the closure report or formal acceptance) is the artefact that formally ends the project and authorises resource release. (A) the project charter authorises the project to begin, not to end. (B) the final project report describes what happened but does not by itself constitute formal authorisation to close. (D) the lessons learned register captures organisational knowledge and is an input to closure, not the authorisation for it.
