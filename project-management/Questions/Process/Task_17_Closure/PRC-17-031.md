# Question

A project has been completed, but stakeholders have different opinions about whether it was successful. Some focus on the product delivered, others on budget overruns, others on missed deadlines. How should the project manager present closure results?

## Options

- **A**: Present only the positive aspects
- **B**: Let stakeholders draw their own conclusions without a formal presentation
- **C**: Present a balanced closure report that addresses all dimensions (scope achieved, schedule performance, cost performance, quality, stakeholder satisfaction), acknowledging both successes and shortfalls with context
- **D**: Only address the negative aspects to show transparency

## Answer

C

## Explanation

The correct answer is C. Project success is multi-dimensional. A balanced closure report presents all aspects with context: what was delivered (scope), how it compared to timeline and budget plans (schedule and cost), quality achieved, and stakeholder perspectives. Providing context (why overruns occurred, what trade-offs were made) enables fair assessment.

Option A is incomplete and not transparent.

Option B abdicates the PM's responsibility to provide closure information.

Option D is unbalanced in the other direction.
