# Question

A project is being transitioned to operations, but the operations team has identified several defects they want fixed before accepting the system. The project team believes these are minor issues that don't affect system functionality significantly. The project is past its planned end date. How should this be resolved?

## Options

- **A**: Force acceptance since the project is overdue
- **B**: Fix all defects regardless of severity to get acceptance
- **C**: Review the defects against acceptance criteria, fix those that violate acceptance criteria, document and categorize others for post-transition resolution, and negotiate conditional acceptance with clear remediation commitments
- **D**: Escalate to legal to interpret the contract

## Answer

C

## Explanation

The correct answer is C. The resolution depends on acceptance criteria. Defects that violate criteria must be fixed before acceptance. Others can be documented for resolution during the warranty/support period. Conditional acceptance (accepting with known defects and remediation plan) is a legitimate approach that enables transition while ensuring issues are addressed.

Option A forces acceptance of potentially unacceptable defects.

Option B may require extensive work for minor issues.

Option D is premature if acceptance criteria provide guidance.
