# Question

A project has delivered all contracted deliverables, but the customer is requesting additional features before signing off on acceptance. The additional features were not in the original scope. What should the project manager do?

## Options

- **A**: Add the features to get the sign-off
- **B**: Refuse any additional work and demand immediate sign-off
- **C**: Document that all contracted deliverables have been completed, obtain formal acceptance for the original scope, and handle additional features as a new project or change request
- **D**: Escalate to legal immediately

## Answer

C

## Explanation

The correct answer is C. Scope creep at closure is common. The PM should demonstrate that contracted deliverables are complete and obtain acceptance for that scope. Additional requests can be addressed through proper channels (change request or new project), not by extending the current project indefinitely. This protects the project while acknowledging the customer's needs.

Option A rewards scope creep and delays closure indefinitely.

Option B is confrontational without offering a path forward for the customer's needs.

Option D is premature if the situation can be resolved through normal project processes.
