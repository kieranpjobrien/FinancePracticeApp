# Question

What is a warranty period in the context of project delivery?

## Options

- **A**: A period when the project can be restarted
- **B**: A defined period after delivery during which the project team or vendor is responsible for fixing defects discovered in the delivered product
- **C**: A time when stakeholders can request new features
- **D**: A budget reserve for future projects

## Answer

B

## Explanation

The correct answer is B. A warranty period is a contractually or organizationally defined period after delivery during which defects are fixed at no additional cost. It provides customers assurance that issues discovered shortly after deployment will be addressed. The warranty is typically limited in duration and scope (defects, not enhancements).

Option A describes project restart, not warranty.

Option C describes scope additions, not defect coverage.

Option D describes financial reserves, not warranty.
