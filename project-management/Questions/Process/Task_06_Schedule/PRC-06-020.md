# Question

An external dependency (vendor deliverable) is on the critical path. The vendor won't commit to a specific date, only a range. How should the PM handle this in scheduling?

## Options

- **A**: Use the optimistic date for planning
- **B**: Plan for the likely date, build contingency for the pessimistic case, maintain close communication with the vendor, and develop alternative approaches if possible
- **C**: Remove the dependency from the schedule
- **D**: Use the pessimistic date and pad all subsequent activities

## Answer

B

## Explanation

The correct answer is B. External dependencies with uncertainty require risk-aware planning. Likely date: reasonable for base planning. Contingency: budget time for delays. Communication: stay close to the vendor for early warning. Alternatives: can you reduce dependency or have backup options? This balances realistic planning with risk management.

Option A ignores risk.

Option C doesn't remove the real dependency.

Option D may be overly conservative and masks the real risk.
