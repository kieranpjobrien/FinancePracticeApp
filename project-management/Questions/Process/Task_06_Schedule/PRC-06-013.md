# Question

A critical path activity is running two days late. The project deadline is fixed. What should the PM do first?

## Options

- **A**: Accept that the project will be late
- **B**: Analyse options to recover: can subsequent critical activities be crashed or fast-tracked? Can scope be reduced? Present options with trade-offs to stakeholders.
- **C**: Immediately add resources to all activities
- **D**: Blame the team for the delay

## Answer

B

## Explanation

The correct answer is B. Critical path delays require immediate analysis. Options: crash (add resources to compress duration), fast-track (parallel activities), scope reduction, or accept delay. Each has trade-offs (cost, risk, functionality). Present options to stakeholders for decision. Don't accept delay without exploring recovery.

Option A gives up without analysis.

Option C wastes resources on non-critical activities.

Option D doesn't solve the problem.
