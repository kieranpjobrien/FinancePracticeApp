# Question

What is a lag in project scheduling?

## Options

- **A**: A delay or waiting time inserted between activities
- **B**: An acceleration of activities
- **C**: A scheduling software feature
- **D**: The last activity in a project

## Answer

A

## Explanation

The correct answer is A. A lag is waiting time between activities. For example, after pouring concrete (predecessor), you might have a 3-day lag for curing before the next activity can start. Lags extend the schedule but reflect real-world constraints.

Option B describes lead, the opposite.

Options C and D misunderstand the scheduling term.
