# Question

The schedule baseline was approved three months ago. Since then several change requests have been approved and incorporated into the working schedule, but the baseline file in the PMIS has not been updated. The latest status report shows the project is "behind baseline." What should the PM do?

## Options

- **A**: Continue to report against the original baseline because it was the version formally approved by the sponsor.
- **B**: Stop comparing to a baseline because the schedule is being actively replanned and the comparison is no longer meaningful.
- **C**: Update the schedule baseline in the PMIS to incorporate the approved changes and report performance against the updated baseline.
- **D**: Maintain a separate "real" baseline for internal use and continue showing the original baseline to stakeholders.

## Answer

C

## Explanation

PMI requires baselines to reflect all approved changes once those changes have passed change control. (C) does that, restoring the integrity of variance reporting. (A) reports against an obsolete baseline that no longer matches what the sponsor has approved, producing misleading variances. (B) abandons baseline reporting rather than maintaining it. (D) keeps two baselines and shows stakeholders the wrong one — a transparency failure.
