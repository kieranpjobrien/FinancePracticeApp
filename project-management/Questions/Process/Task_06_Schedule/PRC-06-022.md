# Question

The schedule baseline was approved three months ago. Since then, several approved changes have been incorporated, but the baseline hasn't been updated. Status reports show the project is behind the original baseline. What should the PM do?

## Options

- **A**: Report against the original baseline since it was approved
- **B**: Update the baseline to reflect approved changes through proper change control, then report against the current approved baseline
- **C**: Abandon baseline reporting
- **D**: Create a secret updated baseline

## Answer

B

## Explanation

The correct answer is B. Baselines should reflect all approved changes. When changes are approved, the baseline should be updated through change control to incorporate them. Reporting against an outdated baseline creates confusion - you appear behind when you're actually on track with approved changes.

Option A reports against an invalid baseline.

Option C loses valuable performance measurement.

Option D lacks transparency.
