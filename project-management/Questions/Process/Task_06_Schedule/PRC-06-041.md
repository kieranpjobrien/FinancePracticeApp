# Question

What is the purpose of the backward pass in critical path method?

## Options

- **A**: To reverse the project
- **B**: To calculate the late start and late finish dates for each activity, working backward from the project end date
- **C**: To undo schedule changes
- **D**: To review past performance

## Answer

B

## Explanation

The correct answer is B. The backward pass moves from project end to start, calculating the latest each activity can start (Late Start) and finish (Late Finish) without delaying the project. Comparing early and late dates reveals float. Activities where early equals late dates are on the critical path.

Options A, C, and D misunderstand "backward pass" in CPM context.
