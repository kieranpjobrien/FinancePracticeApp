# Question

A project has accumulated so many approved scope changes that the original schedule baseline is no longer meaningful for measuring performance. SPI looks disastrous but reflects old scope. The sponsor asks whether to rebaseline. What should the PM recommend?

## Options

- **A**: Keep the original baseline because rebaselining hides poor performance and damages accountability across the project team.
- **B**: Rebaseline immediately using current progress as day zero to restore meaningful performance metrics for stakeholders.
- **C**: Submit a rebaseline through integrated change control with the approved changes rolled into a new baseline version.
- **D**: Run both baselines in parallel and let the steering committee choose which one to consult for performance reporting.

## Answer

C

## Explanation

Rebaselining is a formal change that goes through integrated change control and is used when accumulated approved changes render the baseline non-measurable. (C) is the correct process. (A) protects optics at the cost of measurement. (B) skips change control. (D) creates confused reporting and delegates a governance call that change control is designed to handle.
