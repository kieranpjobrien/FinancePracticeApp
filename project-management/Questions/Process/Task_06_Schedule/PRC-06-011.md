# Question

What is the critical path in a project schedule?

## Options

- **A**: The path with the most resources
- **B**: The longest sequence of dependent activities that determines the minimum project duration
- **C**: The most expensive path
- **D**: The path with the most risks

## Answer

B

## Explanation

The correct answer is B. The critical path is the longest path through the project network, determining the shortest possible project duration. Activities on the critical path have zero float - any delay directly impacts the project end date. Managing the critical path is essential for schedule control.

Options A, C, and D describe other characteristics that may or may not correlate with the critical path.
