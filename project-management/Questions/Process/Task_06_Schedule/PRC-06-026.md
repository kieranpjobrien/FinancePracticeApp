# Question

The agile team's velocity has been: Sprint 1 = 32, Sprint 2 = 28, Sprint 3 = 35, Sprint 4 = 30. The remaining backlog is 150 story points. How many sprints should the PM forecast?

## Options

- **A**: Exactly 5 sprints
- **B**: Approximately 5 sprints (using average velocity of ~31), but communicate as a range (4-6 sprints) given velocity variation
- **C**: 150 sprints
- **D**: Cannot be calculated

## Answer

B

## Explanation

The correct answer is B. Average velocity = (32+28+35+30)/4 ≈ 31. Sprints needed = 150/31 ≈ 4.8. Given velocity variation (28-35 range), forecast 5 sprints but communicate uncertainty: best case ~4 sprints (if velocity is 35+), likely ~5 sprints, conservative ~6 sprints. Ranges communicate appropriate uncertainty.

Option A provides false precision.

Option C miscalculates.

Option D ignores available empirical data.
