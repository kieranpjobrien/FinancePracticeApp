# Question

The project has multiple near-critical paths with very little float. A delay on any of several paths could make them critical. How should the PM manage this schedule risk?

## Options

- **A**: Focus only on the current critical path
- **B**: Monitor all near-critical paths closely, consider adding buffers, maintain contingency plans for multiple paths, and communicate the schedule risk to stakeholders
- **C**: Ignore paths that aren't currently critical
- **D**: Extend the deadline to add float

## Answer

B

## Explanation

The correct answer is B. Multiple near-critical paths create schedule risk - the critical path can shift. Monitor: watch all paths with low float. Buffers: consider adding time contingency. Contingency plans: be ready to respond if any path becomes critical. Communicate: stakeholders should understand the fragility.

Option A ignores paths that could become critical.

Option C misses significant schedule risk.

Option D may not be feasible or necessary with proper management.
