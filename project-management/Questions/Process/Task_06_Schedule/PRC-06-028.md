# Question

The schedule shows the project completing on time, but it assumes all tasks will complete exactly as estimated with no delays. What should the PM recognize about this schedule?

## Options

- **A**: The schedule is reliable since it's based on estimates
- **B**: The schedule is optimistic and lacks contingency - statistically, some tasks will exceed estimates. Build in schedule reserve or communicate the risk.
- **C**: Estimates are always accurate
- **D**: No action needed

## Answer

B

## Explanation

The correct answer is B. Schedules built on point estimates without contingency assume everything goes perfectly - which never happens. Statistically, some tasks will exceed estimates. Either build schedule reserve (buffer) into the plan, or communicate clearly that the schedule assumes best-case and has no margin for delays.

Option A overestimates estimate accuracy.

Option C is demonstrably false from project history.

Option D ignores schedule risk.
