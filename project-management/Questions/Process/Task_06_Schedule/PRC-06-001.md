# Question

A critical path analysis shows that completing Activity D depends on Activities A, B, and C finishing first. Activity A finishes early, Activity B is on schedule, and Activity C is delayed by 5 days. How is the start of Activity D affected?

## Options

- **A**: Activity D can start early because Activity A finished early
- **B**: Activity D starts on schedule because Activity B is on schedule
- **C**: Activity D is delayed by 5 days because it must wait for all predecessors
- **D**: Activity D is delayed, but the exact amount depends on float calculations

## Answer

C

## Explanation

The correct answer is C. When an activity has multiple predecessors with Finish-to-Start dependencies, it cannot start until ALL predecessors complete. Even though A finished early and B is on time, Activity D must wait for Activity C. The 5-day delay in C directly delays the start of D by 5 days.

Option A ignores that D depends on ALL three activities, not just A.

Option B similarly ignores the dependency on C.

Option D overcomplicates - if C is on the critical path (which it is, since it's delaying the project), it has zero float, and the delay directly impacts D.
