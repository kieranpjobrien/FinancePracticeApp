# Question

What is float (slack) in project scheduling?

## Options

- **A**: Extra budget for contingencies
- **B**: The amount of time an activity can be delayed without delaying the project end date or subsequent activities
- **C**: Time allocated for breaks
- **D**: Buffer for quality reviews

## Answer

B

## Explanation

The correct answer is B. Float (or slack) is the flexibility in the schedule - how much an activity can slip without affecting the project end date (total float) or the start of successor activities (free float). Activities on the critical path have zero float.

Options A, C, and D describe different concepts.
