# Question

Stakeholders want a detailed schedule for a project where requirements are still being discovered. The team is using agile methods. How should the PM approach scheduling?

## Options

- **A**: Create a detailed Gantt chart for the entire project
- **B**: Provide a high-level roadmap with release milestones, detail only the near-term iterations, and explain that detailed long-term scheduling will emerge as requirements stabilize
- **C**: Refuse to provide any schedule information
- **D**: Make up detailed estimates to satisfy stakeholders

## Answer

B

## Explanation

The correct answer is B. Agile scheduling uses progressive elaboration. Roadmap: high-level timeline with major milestones. Near-term detail: upcoming iterations are planned in detail. Explain: agile deliberately defers detailed planning until enough is known. This provides appropriate visibility while maintaining agile principles.

Option A creates false precision in uncertain situations.

Option C doesn't meet legitimate stakeholder needs.

Option D is dishonest and will prove wrong.
