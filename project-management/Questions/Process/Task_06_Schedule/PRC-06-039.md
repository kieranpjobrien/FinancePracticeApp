# Question

A capital programme has a schedule of more than 4,000 activities. A full critical-path recalculation in the scheduling tool takes several hours, which is now slowing the weekly steering decisions. What is the BEST way to keep schedule analysis useful without blocking decisions?

## Options

- **A**: Decompose the master schedule into linked sub-networks, run quick analyses on sub-networks for routine decisions, and run the full recalculation weekly.
- **B**: Disable the critical-path engine and rely on each workstream lead's judgement for routine schedule decisions until the master schedule can be simplified and restructured.
- **C**: Reduce the activity count by combining detailed work packages into summary activities so the full critical-path recalculation runs faster on the same overall network.
- **D**: Run a parallel manual schedule alongside the tool and use that as the source of truth for steering decisions because it can be updated far more quickly.

## Answer

A

## Explanation

For very large schedules PMI's guidance is to decompose into manageable sub-networks linked by interface milestones, with full integration runs on a defined cadence. (A) does that. (B) abandons schedule analysis entirely. (C) loses the resolution that makes the schedule useful for control. (D) creates two competing sources of truth and undermines the integrity of the integrated schedule.
