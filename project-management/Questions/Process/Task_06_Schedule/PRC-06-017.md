# Question

What is the key difference between a milestone and an activity in a project schedule?

## Options

- **A**: A milestone represents a significant point or event with zero duration, while an activity is work that consumes time and resources.
- **B**: A milestone is a mandatory deliverable, while an activity is optional work the team may choose to perform.
- **C**: A milestone applies only to predictive projects, while an activity applies only to agile projects.
- **D**: A milestone is reported to executives, while an activity is reported only inside the project team.

## Answer

A

## Explanation

(A) captures the PMI definition: milestones are zero-duration markers (phase complete, deliverable approved, gate passed) while activities are durative work that consume time and resources. (B) is wrong — activities are how work gets done, they are not optional. (C) is wrong — both concepts exist in predictive, agile, and hybrid contexts. (D) confuses scheduling with reporting; both can be reported at any level.
