# Question

What is a lead in project scheduling?

## Options

- **A**: The project manager
- **B**: An acceleration of a successor activity - it can start before its predecessor finishes
- **C**: A delay added between activities
- **D**: The first activity in the project

## Answer

B

## Explanation

The correct answer is B. A lead allows a successor activity to start before its predecessor completes, accelerating the schedule. For example, with a 2-day lead on a Finish-to-Start relationship, the successor can start 2 days before the predecessor finishes. Leads compress the schedule but may increase risk.

Option C describes lag, the opposite.

Options A and D misunderstand the term in scheduling context.
