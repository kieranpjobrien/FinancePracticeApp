# Question

In schedule modelling, what is a "lead"?

## Options

- **A**: An acceleration of a successor activity that allows it to start before its predecessor has finished.
- **B**: A mandated delay that pushes a successor activity to start after its predecessor has finished.
- **C**: A milestone that marks the start of a new project phase or workstream.
- **D**: The earliest activity in a network diagram that has no predecessor.

## Answer

A

## Explanation

(A) is the PMI definition of a lead — for example, "Start B 2 days before A finishes" on a finish-to-start relationship. (B) describes a lag, the opposite. (C) describes a milestone. (D) describes a start activity, not a lead.
