# Question

What is the critical path in project scheduling?

## Options

- **A**: The path with the most resources assigned
- **B**: The longest path through the project network, determining the minimum project duration
- **C**: The path with the highest-priority activities
- **D**: The path identified as most likely to have problems

## Answer

B

## Explanation

The correct answer is B. The critical path is the longest sequence of dependent activities that determines the shortest possible project duration. Activities on the critical path have zero float - any delay in these activities directly delays the project completion.

Option A confuses resource allocation with schedule path. Resource-heavy paths aren't necessarily critical.

Option C confuses priority with schedule dependency. High-priority activities may or may not be on the critical path.

Option D confuses critical with risky. While critical path activities deserve attention, the term "critical" refers to schedule impact, not risk.
