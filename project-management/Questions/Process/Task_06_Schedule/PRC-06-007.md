# Question

What is the purpose of a project milestone?

## Options

- **A**: To assign work to team members
- **B**: To mark significant points or events in the project with zero duration
- **C**: To calculate the project budget
- **D**: To identify project risks

## Answer

B

## Explanation

The correct answer is B. Milestones are significant points in the project timeline - typically completion of major deliverables, phase gates, or key decisions. They have zero duration (they're points, not tasks) and serve as progress markers and communication checkpoints.

Examples of milestones:
- Requirements sign-off
- Design complete
- Testing started
- Go-live date

Option A (assign work) is done through work packages and task assignments, not milestones.

Option C (budget calculation) is done through cost estimating processes.

Option D (identify risks) is done through risk identification processes.
