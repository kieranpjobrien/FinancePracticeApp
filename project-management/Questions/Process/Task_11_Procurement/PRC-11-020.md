# Question

An agile project needs to engage a vendor for a component. Traditional fixed-scope contracts don't fit the adaptive approach. What procurement approach should the PM consider?

## Options

- **A**: Avoid procurement since agile and contracts are incompatible
- **B**: Use a traditional fixed-price contract and manage scope changes through amendments
- **C**: Consider contracts that pay for capacity or outcomes rather than fixed deliverables
- **D**: Complete all agile iterations before engaging the vendor

## Answer

C

## Explanation

The correct answer is C. Agile-friendly contracts can focus on capacity (e.g., dedicated team at monthly rate), outcomes (e.g., payment based on delivered value or working features), or time-boxed engagements with scope flexibility within each period. These align procurement with agile's adaptive nature.

Option A incorrectly assumes incompatibility.

Option B imposes predictive procurement on adaptive development.

Option D delays procurement unnecessarily.
