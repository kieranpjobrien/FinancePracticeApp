# Question

A multi-organisation project includes teams that variously use Slack, Microsoft Teams, and email-only. Important information is repeatedly missed by partners on tools other than the originator's. What should the PM do?

## Options

- **A**: Mandate one common platform across all participating organisations for the duration of the project.
- **B**: Set up automated bridges that mirror messages between Slack and Teams so all conversations appear everywhere.
- **C**: Restrict cross-organisational communication to weekly status meetings only, eliminating the tool-mismatch problem.
- **D**: Establish a small set of shared touchpoints (a project channel and a regular cross-org cadence), define which information must flow through them, and assign liaison roles in each organisation.

## Answer

D

## Explanation

Multi-organisation tool sprawl is solved by agreeing the minimum shared layer plus liaison roles, not by enforcing tool change or eliminating channels. (D) does this. (A) is rarely possible across organisational boundaries and consumes goodwill needlessly. (B) creates noisy duplication and quickly degrades trust in any single channel. (C) starves the project of timely communication.
