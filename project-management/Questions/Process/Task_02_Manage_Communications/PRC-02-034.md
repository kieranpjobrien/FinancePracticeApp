# Question

The project involves regulatory deliverables requiring specific documentation formats and approval chains. The agile team finds these requirements burdensome and wants to skip formal documentation. How should the PM handle this?

## Options

- **A**: Let the team skip documentation since agile values working software over documentation
- **B**: Explain that regulatory requirements are non-negotiable constraints, work with the team to streamline documentation where possible within requirements, and ensure they understand the business necessity
- **C**: Abandon agile to focus on documentation
- **D**: Document everything after the project ends

## Answer

B

## Explanation

The correct answer is B. Regulatory documentation is a constraint, not optional. The Agile Manifesto values working software more than documentation, not zero documentation. Explain: why are these requirements necessary? Streamline: within requirements, are there ways to make documentation less burdensome? Integrate: build documentation into the workflow rather than treating it as separate overhead.

Option A risks regulatory non-compliance.

Option C throws out valuable agile practices unnecessarily.

Option D creates massive effort at the end and risks non-compliance.
