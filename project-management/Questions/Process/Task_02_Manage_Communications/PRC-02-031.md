# Question

Project documentation is becoming outdated as changes occur faster than documentation can be updated. Stakeholders are using stale information. What approach should the PM take?

## Options

- **A**: Stop documenting since it can't keep up
- **B**: Identify which documents are critical and must be current, automate updates where possible, establish clear ownership, and accept that some documents may be point-in-time snapshots
- **C**: Increase documentation staff significantly
- **D**: Make all documents read-only to prevent changes

## Answer

B

## Explanation

The correct answer is B. Not all documentation needs real-time currency. Prioritise: which documents must be current (requirements, decisions)? Which can be periodic snapshots? Automate: use tools that auto-update from authoritative sources. Ownership: assign responsibility for keeping critical docs current. Accept reality: perfect currency for all docs isn't achievable or necessary.

Option A loses valuable project records.

Option C may not be cost-effective or solve the core prioritisation issue.

Option D prevents necessary updates.
