# Question

Project documentation is consistently out of date because changes happen faster than documents are updated. Stakeholders are making decisions from stale information. What should the PM do?

## Options

- **A**: Add a documentation analyst to the team to keep all artefacts current.
- **B**: Mandate that no project change can take effect until the relevant document has been updated.
- **C**: Convert all project documentation into wiki pages so anyone can edit them inline as changes happen.
- **D**: Identify which documents must stay current, assign explicit owners, and automate updates where possible.

## Answer

D

## Explanation

Not every artefact needs real-time accuracy; the PMI move is to differentiate criticality, assign ownership, and use automation where the source data is canonical. (D) does this. (A) adds cost without addressing the lack of structure. (B) blocks legitimate change for a documentation problem. (C) lowers the friction but doesn't address ownership or which docs need to be canonical.
