# Question

A serious data quality incident has occurred during user acceptance testing. The fix may take a week and the PM is worried about how the news will land. How should the PM communicate?

## Options

- **A**: Wait until the fix is in place so the announcement can include the resolution alongside the issue.
- **B**: Notify only the project sponsor for now, agree the messaging together, then communicate to the wider group jointly with the sponsor.
- **C**: Send a brief factual note to the affected stakeholders covering what happened, current impact, mitigations underway, and the expected resolution timeline.
- **D**: Lead the next status meeting with project achievements and address the data issue at the end so it lands in context of overall progress.

## Answer

C

## Explanation

PMI's communication ethic is timely, factual, audience-aware disclosure — especially when stakeholders are affected by the issue. (C) does this directly. (A) delays disclosure of material status. (B) is reasonable for sponsor alignment but the affected stakeholders also need timely information; sponsor sign-off shouldn't gate it. (D) is bury-the-lede and erodes credibility.
