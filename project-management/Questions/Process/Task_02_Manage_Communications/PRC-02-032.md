# Question

When should a project manager use formal written communication versus informal verbal communication?

## Options

- **A**: Formal for everything to maintain professionalism
- **B**: Formal for decisions, contracts, and audit requirements; informal for day-to-day coordination and relationship building
- **C**: Informal for everything to be approachable
- **D**: Only formal with external stakeholders

## Answer

B

## Explanation

The correct answer is B. Match formality to purpose. Formal (written) for: decisions requiring documentation, contractual matters, audit trails, official status. Informal (verbal/chat) for: daily coordination, quick questions, relationship building, preliminary discussions. Using the right level of formality makes communication more effective and efficient.

Options A and C use one level for everything, losing appropriate nuance.

Option D bases formality on relationship rather than purpose.
