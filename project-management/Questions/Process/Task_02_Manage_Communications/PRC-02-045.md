# Question

A project is transitioning from one PM to another. The outgoing PM has significant knowledge about stakeholder relationships, informal agreements, and communication nuances that aren't documented. How should knowledge transfer be handled?

## Options

- **A**: Let the new PM figure it out
- **B**: Schedule dedicated transition time for relationship introductions, documentation of informal agreements, and sharing of communication insights that go beyond formal project documentation
- **C**: Only transfer formal documentation
- **D**: Complete the transition in a single meeting

## Answer

B

## Explanation

The correct answer is B. Effective PM transitions require transferring tacit knowledge, not just documents. Relationship introductions: facilitate introductions to key stakeholders with context. Informal agreements: document any commitments or understandings not in formal records. Communication insights: share what works with different stakeholders. This sets the new PM up for success.

Option A loses valuable knowledge and relationships.

Option C transfers only part of the essential information.

Option D isn't sufficient time for comprehensive knowledge transfer.
