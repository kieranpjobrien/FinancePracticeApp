# Question

Multiple approved changes have accumulated but haven't been implemented due to resource constraints. Some changes are now blocking other work. How should the PM address this?

## Options

- **A**: Implement all approved changes in the order they were approved
- **B**: Prioritise change implementation based on dependencies and business impact
- **C**: Defer all changes until dedicated resources are available
- **D**: Reject some approved changes to reduce the backlog

## Answer

B

## Explanation

The correct answer is B. Approved changes need implementation planning like any other work. When multiple changes are pending, prioritise based on dependencies (changes blocking other work come first), business impact, and available resources. This ensures the most important changes are implemented first.

Option A doesn't consider dependencies or relative importance.

Option C may cause further delays on blocking changes.

Option D invalidates CCB decisions without new information. Approved changes shouldn't be arbitrarily rejected.
