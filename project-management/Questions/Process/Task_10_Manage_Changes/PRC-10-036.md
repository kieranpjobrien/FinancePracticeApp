# Question

A fast-track project has a heavy change request load. The CCB meets weekly but the backlog of changes is growing, some becoming stale. How should the PM address this?

## Options

- **A**: Process changes without CCB approval to clear the backlog
- **B**: Reject changes that have been waiting too long
- **C**: Propose more frequent CCB meetings or delegated approval for categories of changes
- **D**: Prioritise the backlog and present only the most important changes to the CCB

## Answer

C

## Explanation

The correct answer is C. The change control process isn't keeping pace with project needs. Options include more frequent CCB meetings, virtual/asynchronous reviews, or delegating approval for certain change categories (e.g., PM approves changes under $X or within certain scope areas). The process should match project tempo.

Option A bypasses governance.

Option B arbitrarily rejects potentially valid changes.

Option D filters what the CCB sees without authority to do so.
