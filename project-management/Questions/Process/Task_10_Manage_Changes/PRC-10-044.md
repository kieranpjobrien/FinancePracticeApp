# Question

How is change typically managed on an agile project?

## Options

- **A**: With the same formal CCB and integrated change control used on predictive projects.
- **B**: Without any control, since agile principles welcome change at any time.
- **C**: By freezing scope at the start of each release and accepting changes only between releases.
- **D**: Through ongoing product backlog management, where the product owner reprioritises items by value and refines them with the team.

## Answer

D

## Explanation

In agile, the product backlog is the change mechanism: the product owner reorders items based on value as understanding evolves, and refinement keeps items ready for upcoming sprints. (A) imports a predictive control structure that conflicts with agile cadence. (B) confuses welcoming change with the absence of control; the backlog provides the control. (C) is a phase-gate model that contradicts agile's iterative response to change.
