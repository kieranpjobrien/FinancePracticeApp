# Question

Which document defines how changes will be identified, evaluated, approved, and implemented across the project lifecycle?

## Options

- **A**: Change log
- **B**: Change management plan
- **C**: Change control board charter
- **D**: Issue register

## Answer

B

## Explanation

The change management plan, a subsidiary of the project management plan, defines the end-to-end process for handling changes including thresholds, roles, and approval routes. (A) the change log records individual change requests and their status; it is an output of the process, not the governing document. (C) the CCB charter defines that body's authority and membership but not the broader change process. (D) the issue register tracks issues, which are distinct from formal change requests.
