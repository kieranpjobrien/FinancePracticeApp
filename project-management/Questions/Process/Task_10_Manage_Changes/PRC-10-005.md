# Question

Which document defines how changes will be managed throughout the project lifecycle?

## Options

- **A**: Change log
- **B**: Change control board charter
- **C**: Change management plan
- **D**: Issue register

## Answer

C

## Explanation

The correct answer is C. The change management plan (a subsidiary of the project management plan) defines the process for managing changes - how change requests are submitted, evaluated, approved/rejected, and implemented. It establishes thresholds, roles, and procedures.

Option A (change log) is a tracking document that records all change requests and their status. It's an output of change control, not the governing document.

Option B (CCB charter) may define the change control board's authority and membership, but the change management plan defines the overall process.

Option D (issue register) tracks issues, not changes. Issues and changes are related but distinct - an issue may lead to a change request, but they're managed in separate logs.
