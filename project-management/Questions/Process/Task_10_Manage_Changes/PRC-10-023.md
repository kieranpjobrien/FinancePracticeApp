# Question

A project has experienced significant scope creep despite having a change control process. Analysis shows that many small changes were approved individually without considering cumulative impact. What should the PM implement?

## Options

- **A**: Reject all future change requests to stop the creep
- **B**: Require CCB approval for even minor changes
- **C**: Implement periodic reviews of cumulative change impact against original objectives
- **D**: Remove the PM's authority to approve minor changes

## Answer

C

## Explanation

The correct answer is C. The issue is "death by a thousand cuts" - each individual change seemed reasonable but the cumulative effect is scope creep. Periodic reviews that assess total changes against original baseline and objectives catch this pattern. The CCB can then assess whether the cumulative change is acceptable.

Option A overreacts and prevents legitimate changes.

Option B adds overhead without addressing the cumulative assessment gap.

Option D doesn't solve the problem - minor changes individually approved by any authority can still accumulate.
