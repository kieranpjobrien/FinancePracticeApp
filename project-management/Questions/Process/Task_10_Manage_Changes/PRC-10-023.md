# Question

A retrospective on an in-flight programme shows that scope has crept by 18% even though every change followed the change control process. Each change looked small in isolation. What should the PM put in place to address this pattern?

## Options

- **A**: Pause acceptance of new change requests until existing ones are implemented.
- **B**: Require CCB approval on changes regardless of size, removing the PM's delegated tolerance.
- **C**: Add a recurring cumulative-change review that compares the running total of approved changes against the original objectives and baselines.
- **D**: Strip approval authority from the requesters and route all requests through the sponsor.

## Answer

C

## Explanation

The failure mode is "death by a thousand cuts": individually defensible changes accumulate into significant drift. A periodic cumulative-impact review surfaces the trend so the CCB can decide whether the project is still pursuing the originally sanctioned outcomes. (A) blocks legitimate changes and does not address the pattern. (B) increases overhead but does not change how cumulative impact is evaluated. (D) shifts where requests are filed without changing the review pattern that allowed creep.
