# Question

A project manager inherits a project where changes were poorly documented. It's unclear which version of requirements is current. How should the PM establish control?

## Options

- **A**: Start fresh with new documentation ignoring the past
- **B**: Reconstruct the change history as much as possible and establish a new baseline from current state
- **C**: Continue without baseline and manage going forward
- **D**: Halt the project until proper documentation is created

## Answer

B

## Explanation

The correct answer is B. The PM needs to understand the current state to manage the project. Reconstruct what's possible from available records, interviews, and current deliverables. Establish the current state as a new baseline for future change control. This may require stakeholder agreement on what's actually in scope.

Option A ignores potentially valuable historical information.

Option C perpetuates the lack of control.

Option D may be unnecessarily disruptive.
