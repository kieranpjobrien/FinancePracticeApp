# Question

What is a corrective action in project management?

## Options

- **A**: A change request to add new features
- **B**: An action to bring future project performance in line with the project management plan
- **C**: A punishment for team members who make mistakes
- **D**: A decision to cancel the project

## Answer

B

## Explanation

The correct answer is B. A corrective action addresses variance between actual and planned performance. When the project deviates from the plan (behind schedule, over budget, quality issues), corrective actions bring performance back on track. Examples: adding resources to catch up, rework to fix defects, process changes to prevent recurrence.

Option A describes a scope change request, not a corrective action.

Option C misunderstands "corrective" - it's about correcting project performance, not punishing people.

Option D is project termination, not correction.
