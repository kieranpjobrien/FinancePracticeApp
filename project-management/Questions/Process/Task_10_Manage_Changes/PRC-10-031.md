# Question

What is the PRIMARY input to the Perform Integrated Change Control process?

## Options

- **A**: Lessons learned register
- **B**: Change requests
- **C**: Project charter
- **D**: Stakeholder register

## Answer

B

## Explanation

The correct answer is B. Change requests are the primary input to integrated change control. The process exists to evaluate, approve, or reject change requests. Without change requests, there's nothing for the process to control.

Option A informs decisions but isn't the primary input.

Option C is created during initiation and rarely changes.

Option D informs stakeholder analysis but isn't the primary input to change control.
