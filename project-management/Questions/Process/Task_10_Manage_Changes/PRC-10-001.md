# Question

A stakeholder requests a significant change to the project scope. The project has a formal change control process. What should the project manager do FIRST?

## Options

- **A**: Implement the change since the stakeholder has authority
- **B**: Reject the change to protect the baseline
- **C**: Document the change request and assess its impact on scope, schedule, cost, and quality before submitting for approval
- **D**: Immediately convene the change control board for a decision

## Answer

C

## Explanation

The correct answer is C. Before any decision can be made, the change must be properly documented and analyzed. Impact assessment allows the change control board (or decision-maker) to make an informed choice. Rushing to implementation or rejection without analysis is poor practice.

Option A bypasses change control, which exists to ensure informed decisions. Even legitimate requests should go through the process.

Option B rejects without analysis, which isn't fair to the stakeholder or informed decision-making.

Option D skips the impact assessment. The CCB needs impact analysis to make a good decision - convening them without it wastes time.
