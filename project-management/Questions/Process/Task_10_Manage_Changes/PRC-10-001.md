# Question

A senior business stakeholder verbally requests a significant scope addition during a status meeting and says it must be in the next release. The project has an established change control process. What should the project manager do FIRST?

## Options

- **A**: Capture the request as a formal change request and perform an integrated impact assessment on scope, schedule, cost, quality, and risk.
- **B**: Add the work to the next sprint's backlog because the requester has the seniority to authorise it.
- **C**: Convene the change control board immediately so the request can be voted on at the meeting.
- **D**: Decline the request to protect the approved baseline and notify the sponsor of the attempted bypass.

## Answer

A

## Explanation

PMI's change control process requires a documented change request followed by integrated impact analysis before any decision body acts. (A) follows that sequence. (B) bypasses change control regardless of the requester's seniority; authority does not override process. (C) is premature because the CCB needs an impact assessment to make an informed decision, not a verbal request. (D) refuses without analysis and escalates, which assumes bad intent and skips the PM's own duty to evaluate the request.
