# Question

A change request that is "deferred" means:

## Options

- **A**: The change is rejected
- **B**: The change is approved for implementation later
- **C**: The change is set aside for future consideration, neither approved nor rejected
- **D**: The change needs more information before evaluation

## Answer

C

## Explanation

The correct answer is C. A deferred change is neither approved nor rejected. The CCB may defer a change because it's potentially valuable but not appropriate for the current phase, or because other priorities take precedence. Deferred changes can be reconsidered later.

Option A conflates deferral with rejection. Deferred changes haven't been rejected.

Option B conflates deferral with approval. Deferred changes haven't been approved for implementation.

Option D describes an incomplete request, not deferral.
