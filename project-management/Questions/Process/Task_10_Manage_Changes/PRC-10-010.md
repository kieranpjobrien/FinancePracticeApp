# Question

A developer implements a small improvement to the code without submitting a change request, arguing it was an obvious enhancement. What should the PM do?

## Options

- **A**: Accept the improvement since it benefits the project
- **B**: Require the developer to remove the change
- **C**: Document the change retroactively and discuss the importance of following change control
- **D**: Escalate the developer's behaviour to their functional manager

## Answer

C

## Explanation

The correct answer is C. The change exists and may be beneficial, but the process was bypassed. The PM should document what changed (for traceability) and use this as a coaching opportunity about why change control matters - even for improvements. Unapproved changes can cause integration problems, testing gaps, or scope creep.

Option A normalises bypassing change control.

Option B may be impractical and doesn't address the underlying behaviour.

Option D escalates inappropriately for a first offence that's better handled through coaching.
