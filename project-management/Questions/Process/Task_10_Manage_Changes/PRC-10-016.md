# Question

A project baseline was set three months ago. Since then, 15 approved changes have modified the scope. Earned value calculations show poor performance. A stakeholder suggests the original baseline is no longer meaningful. What should the PM do?

## Options

- **A**: Continue using the original baseline since baselines shouldn't change
- **B**: Re-baseline to incorporate all approved changes
- **C**: Report performance against both the original and a revised baseline
- **D**: Evaluate whether re-baselining is appropriate given the extent of approved changes

## Answer

D

## Explanation

The correct answer is D. Significant cumulative change may warrant re-baselining, but this decision requires evaluation. The PM should consider whether the original baseline still provides meaningful performance insight, whether stakeholders understand performance is measured against an outdated plan, and whether re-baselining would mask performance issues or accurately reflect approved scope changes.

Option A may perpetuate meaningless comparisons.

Option B doesn't involve proper evaluation and approval for re-baselining.

Option C adds complexity without necessarily adding value.
