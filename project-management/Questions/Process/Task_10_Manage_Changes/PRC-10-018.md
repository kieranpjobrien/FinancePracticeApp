# Question

When during the project lifecycle is integrated change control performed?

## Options

- **A**: Only during the execution phase when work is actively being delivered.
- **B**: From initiation through closing, whenever change requests arise.
- **C**: Only when a baseline change is required.
- **D**: At scheduled CCB meetings on a fixed cadence.

## Answer

B

## Explanation

Integrated change control is a continuous process active across the entire lifecycle because changes can originate at any point, including during closing. (A) limits the process artificially. (C) ignores changes that affect plans without modifying baselines. (D) confuses the cadence of one mechanism (the CCB meeting) with the broader process, which includes intake, analysis, decision, and update activities.
