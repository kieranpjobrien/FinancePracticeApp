# Question

What should a change request typically include?

## Options

- **A**: Only the description of the desired change
- **B**: Description, justification, impact analysis, and implementation approach
- **C**: The name of the requester and the date submitted
- **D**: Technical specifications for implementing the change

## Answer

B

## Explanation

The correct answer is B. A complete change request enables informed decision-making. It should describe what's requested, why it's needed (justification/business case), what impacts it has on scope, schedule, cost, quality, and risks, and how it would be implemented. This information enables the CCB to make good decisions.

Option A provides insufficient information for evaluation.

Option C captures administrative data but not decision-relevant content.

Option D may be part of implementation approach but isn't sufficient alone.
