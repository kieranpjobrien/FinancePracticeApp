# Question

A project is using configuration management to control product versions. A team member accidentally deployed a previous version to production, overwriting the current version. What should the PM do?

## Options

- **A**: Restore the current version from the configuration management system
- **B**: Accept the previous version if it's stable
- **C**: Investigate how the deployment was possible and implement controls
- **D**: Both A and C

## Answer

D

## Explanation

The correct answer is D. Immediate action is needed to restore the correct version (configuration management should enable this), but the PM should also investigate how the accidental deployment was possible. Were deployment procedures inadequate? Was there insufficient access control? Preventing recurrence is as important as fixing the immediate problem.

Option A addresses immediate recovery but not prevention.

Option B accepts an unintended change without proper evaluation.

Option C addresses prevention but not immediate recovery.
