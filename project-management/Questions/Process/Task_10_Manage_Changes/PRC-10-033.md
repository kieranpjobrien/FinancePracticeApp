# Question

An engineer accidentally redeployed last month's release to production, overwriting the current version. Configuration management captured the prior versions. What should the PM do?

## Options

- **A**: Restore the current version from configuration management and conduct a root-cause review of how the wrong version was deployed.
- **B**: Restore the current version from configuration management and treat the incident as a one-off operator error.
- **C**: Leave the previous version in place since it is stable and avoid the disruption of another deployment.
- **D**: Open a change request to formalise the rollback as the new baseline going forward.

## Answer

A

## Explanation

The PM must address both the immediate impact (restore the correct version) and the systemic question (how was the wrong version deployable in the first place). (B) restores service but ignores prevention, leaving the same hole open. (C) accepts an unauthorised change and abandons the current scope. (D) treats an accidental rollback as a deliberate baseline decision, which it isn't.
