# Question

A defect discovered during integration testing requires corrective action to rework a completed module. The change management plan treats corrective action as an output of Direct and Manage work, not as a change request. The PM is unsure how to document. What should the PM do?

## Options

- **A**: Raise a change request regardless, since any significant rework should always pass through the CCB for approval.
- **B**: Document the corrective action as directed by the change management plan without raising a change request.
- **C**: Document the defect as an issue only, with no corrective action record, until retesting confirms it resolved.
- **D**: Treat the rework informally and focus energy on the fix rather than process documentation at this stage.

## Answer

B

## Explanation

Corrective actions that keep work aligned with the plan are not change requests; the change management plan's treatment is correct and the documentation follows that path. (A) escalates to the CCB work that is expected execution activity, not a change. (C) omits the corrective action record that quality and audit require. (D) skips documentation that is part of how execution is managed.
