# Question

The product owner reprioritises the backlog every sprint. A stakeholder complains this constant change makes planning impossible. How should the PM respond?

## Options

- **A**: Restrict backlog changes to once per month
- **B**: Explain that backlog reprioritisation is normal in agile and planning occurs at different horizons
- **C**: Have the stakeholder work directly with the product owner to understand priorities
- **D**: Implement a formal change control process for backlog changes

## Answer

B

## Explanation

The correct answer is B. In agile, backlog reprioritisation is normal and valuable - it ensures the team works on the most important items as understanding evolves. Planning in agile occurs at multiple levels: release planning provides longer-term visibility while sprint planning provides near-term certainty. The stakeholder needs to understand this different planning model.

Option A contradicts agile principles.

Option C doesn't address the stakeholder's underlying concern about predictability.

Option D imposes predictive controls on an agile practice.
