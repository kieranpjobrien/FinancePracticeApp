# Question

A traditional stakeholder complains that the product owner reprioritises the backlog every sprint, making forward planning impossible from their perspective. What should the agile project manager do?

## Options

- **A**: Ask the product owner to freeze the backlog for a quarter to give the stakeholder predictability.
- **B**: Set the stakeholder's expectations on rolling-wave planning horizons in agile and connect them to release planning artefacts so they can see longer-term direction.
- **C**: Refer the stakeholder directly to the product owner so they can resolve the priorities issue between themselves.
- **D**: Introduce a formal CCB for backlog changes so the stakeholder can approve reprioritisations.

## Answer

B

## Explanation

The right move is to address the stakeholder's underlying need (forward visibility) using agile-native tools: release planning, roadmaps, and forecast ranges. (A) imposes predictive constraints on agile and reduces the team's ability to respond to value. (C) hands off without educating the stakeholder on how planning works in this delivery model. (D) introduces governance that conflicts with the agile mindset and slows down value delivery.
