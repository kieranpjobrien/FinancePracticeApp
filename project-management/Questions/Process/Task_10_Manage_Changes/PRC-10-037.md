# Question

What is the primary purpose of version control in a project?

## Options

- **A**: To standardise the tools team members use to access project content.
- **B**: To track changes to project items over time and enable retrieval of prior versions.
- **C**: To enforce that only CCB-approved changes are made to deliverables.
- **D**: To keep the project on schedule by reducing rework time.

## Answer

B

## Explanation

Version control records the history of changes to artifacts (documents, code, configurations) and allows teams to retrieve any prior version. (A) describes tool standardisation, a different concern. (C) describes change control, which uses version control but is not the same thing. (D) is a possible benefit, not the purpose.
