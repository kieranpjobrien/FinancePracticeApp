# Question

Midway through a Scrum sprint delivering a healthcare records module, a regulator announces a rule change that affects a feature scheduled for this sprint. The product owner wants to swap the sprint backlog immediately. What should the PM advise the team?

## Options

- **A**: Swap the sprint backlog immediately because regulatory changes override normal sprint discipline at this point.
- **B**: Keep the current sprint backlog unchanged because regulatory impact can be planned for the next sprint.
- **C**: Use the abnormal termination path to end the sprint and replan around the regulatory requirement properly.
- **D**: Informally add the regulator's work mid-sprint and extend the sprint to absorb the new scope.

## Answer

C

## Explanation

Scrum provides an abnormal sprint termination for exactly this situation; it preserves discipline while responding to material external change. (A) swaps the backlog without using the prescribed Scrum mechanism. (B) delays response to a regulatory change that should be handled promptly. (D) violates the timebox and blurs the control boundary.
