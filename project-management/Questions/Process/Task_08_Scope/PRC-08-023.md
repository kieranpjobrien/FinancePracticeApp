# Question

The product owner ranks the backlog purely on customer-facing value. The development team argues that without addressing accumulating technical debt, future delivery will collapse. How should the project manager facilitate the discussion?

## Options

- **A**: Coach the team to translate the cost of debt — slower delivery, higher defect rate, future re-engineering — into business terms the PO can compare against feature value.
- **B**: Defer to the PO because the PO owns prioritisation and accountability for outcomes.
- **C**: Defer to the team because they have direct visibility of the technical reality.
- **D**: Set up a separate parallel debt backlog the team manages without PO involvement.

## Answer

A

## Explanation

Technical debt is a real cost, but it competes with feature value on the same backlog. The PM's job is to make the trade-off legible so the PO can make a single, informed prioritisation. (B) ignores legitimate technical input. (C) bypasses the PO's accountability. (D) hides debt from the PO and produces parallel queues that both sides will fight over.
