# Question

Which technique involves creating a simplified working model of the product to gather feedback on requirements?

## Options

- **A**: Benchmarking
- **B**: Prototyping
- **C**: Document analysis
- **D**: Focus groups

## Answer

B

## Explanation

The correct answer is B. Prototyping creates a tangible model of the expected product before full development. This allows stakeholders to interact with a working representation, making it easier to identify requirements they might not have articulated and to validate understanding of stated requirements.

Option A (benchmarking) compares practices to other organisations to identify best practices or standards.

Option C (document analysis) reviews existing documentation to extract requirements.

Option D (focus groups) brings qualified stakeholders together for discussion but doesn't involve building a working model.
