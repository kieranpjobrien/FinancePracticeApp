# Question

After decomposing a work package into activities, the team realises two members have done overlapping work because they each interpreted the package differently. Which underlying gap most likely allowed this to happen?

## Options

- **A**: The schedule did not assign clear start dates for each activity.
- **B**: The work package was too large and should have been split further.
- **C**: A single owner had not been designated for the work package.
- **D**: The WBS dictionary entry did not contain a precise scope description and boundaries for the work package.

## Answer

D

## Explanation

The WBS dictionary is what makes a WBS box unambiguous: scope of work, boundaries, deliverables, acceptance criteria, owner. Without a clear entry, well-meaning teammates fill in the gaps differently. (A) addresses timing, not meaning. (B) might also help but smaller packages without descriptions are equally interpretable. (C) is one possible control, but PMI's standard answer for shared understanding is the dictionary, not single-person ownership.
