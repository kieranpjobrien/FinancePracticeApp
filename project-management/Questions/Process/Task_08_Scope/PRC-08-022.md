# Question

A project team decomposed a work package into activities, but different team members interpreted the work package differently, resulting in duplicate effort. What could have prevented this?

## Options

- **A**: More detailed project schedule
- **B**: Clearer WBS dictionary entries with explicit scope descriptions
- **C**: Smaller work packages to reduce ambiguity
- **D**: Assigning a single person to each work package

## Answer

B

## Explanation

The correct answer is B. The WBS dictionary provides detailed descriptions of each WBS element including scope of work, boundaries, and acceptance criteria. Clearer dictionary entries would have established a shared understanding of what the work package included.

Option A addresses timing of work, not scope clarity.

Option C might help but doesn't address the root cause. Smaller packages can still be ambiguous without clear descriptions.

Option D reduces collaboration and may not be practical. Clear definitions enable multiple people to work effectively on shared understanding.
