# Question

A stakeholder requests a new feature midway through a project. The feature aligns with the project objectives but wasn't in the original scope. What should the project manager do FIRST?

## Options

- **A**: Add the feature to the backlog since it aligns with objectives
- **B**: Reject the request since it's outside the approved scope
- **C**: Evaluate the impact on schedule, cost, and resources through the change control process
- **D**: Ask the sponsor to decide whether to include the feature

## Answer

C

## Explanation

The correct answer is C. All scope change requests should be evaluated through the integrated change control process. This assesses impacts on the triple constraint (scope, time, cost) and other project factors before making a decision. Even beneficial changes need proper impact assessment.

Option A bypasses change control. Even good ideas need impact analysis before commitment.

Option B is premature rejection. The change control process exists to evaluate requests, not automatically reject them.

Option D escalates without analysis. The sponsor needs impact information to make an informed decision.
