# Question

Halfway through a regulated banking project, a stakeholder asks for a new feature that aligns with project objectives but is outside the approved scope baseline. What should the project manager do FIRST?

## Options

- **A**: Run the request through integrated change control and assess impact on schedule, cost, quality, and risk before a decision.
- **B**: Add the feature to the team's backlog so they can pick it up in the next iteration once capacity permits.
- **C**: Decline the request because it falls outside the approved baseline and divert stakeholder attention back to committed scope.
- **D**: Ask the sponsor to make the call without performing additional analysis first so the answer can come quickly from leadership.

## Answer

A

## Explanation

Even valuable, on-strategy changes belong inside change control so impacts are visible and authorised. (B) effectively commits the team without analysis or approval. (C) refuses without process — change control exists precisely for this case. (D) escalates without giving the sponsor the analysis they need to decide; the PM owes them an impact assessment, not a raw question.
