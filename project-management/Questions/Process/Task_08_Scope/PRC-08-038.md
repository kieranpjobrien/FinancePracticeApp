# Question

What does the "100% rule" refer to in WBS development?

## Options

- **A**: 100% of team members should be involved in creating the WBS
- **B**: The WBS must include 100% of work defined by the project scope
- **C**: Each work package should be 100% complete before starting the next
- **D**: WBS elements should be decomposed until they are 100% clear

## Answer

B

## Explanation

The correct answer is B. The 100% rule states that the WBS must include 100% of the work defined by the project scope - no more, no less. Each level must represent 100% of the work in its parent element. This ensures nothing is missing and nothing extra is included.

Option A describes participation, not a WBS principle.

Option C describes sequential execution, which isn't required or always appropriate.

Option D misinterprets the rule. Clarity is important but isn't what the 100% rule addresses.
