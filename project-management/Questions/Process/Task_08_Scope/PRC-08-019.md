# Question

What is the purpose of a WBS dictionary?

## Options

- **A**: To define the abbreviations and terms used in the WBS
- **B**: To provide detailed information about each WBS element including work description, acceptance criteria, and responsible party
- **C**: To list all the deliverables in alphabetical order for easy reference
- **D**: To map WBS elements to project schedule activities

## Answer

B

## Explanation

The correct answer is B. The WBS dictionary provides detailed supporting information for each WBS component. This includes scope description, deliverables, activities, milestones, resources, cost estimates, acceptance criteria, and technical references. It ensures everyone understands what each work package entails.

Option A describes a glossary, not a WBS dictionary.

Option C misunderstands the purpose. The WBS dictionary provides depth of information, not just reorganisation.

Option D describes mapping done during schedule development, not the WBS dictionary's purpose.
