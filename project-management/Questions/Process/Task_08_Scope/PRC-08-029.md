# Question

Following the launch of an MVP, users flood the team with feature requests. The PO asks the project manager how to manage scope across upcoming releases. What should the PM recommend?

## Options

- **A**: Capture every request in a structured product backlog and prioritise based on business value, strategic alignment, and effort.
- **B**: Implement the requests with the highest user vote count first to maximise customer satisfaction.
- **C**: Hold the original product roadmap unchanged so the team is not destabilised by new input.
- **D**: Implement all requests to demonstrate responsiveness to the user base.

## Answer

A

## Explanation

A managed product backlog is the agile mechanism for absorbing demand without losing focus or strategy. (B) optimises for one signal (volume) that can mistake the loudest users for the most valuable. (C) ignores valuable post-launch learning, which is the whole point of the MVP. (D) is unbounded scope absorption with no value lens.
