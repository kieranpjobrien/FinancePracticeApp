# Question

An MVP (Minimum Viable Product) has been defined for a new software product. After launch, users request many additional features. How should the PM manage scope for subsequent releases?

## Options

- **A**: Add all requested features to maintain customer satisfaction
- **B**: Prioritise features based on user feedback volume
- **C**: Use a product backlog to capture requests and prioritise based on value and strategic alignment
- **D**: Stick to the original product roadmap regardless of user feedback

## Answer

C

## Explanation

The correct answer is C. A product backlog captures all potential work and enables prioritisation based on multiple factors: business value, user need, strategic fit, dependencies, and effort. This structured approach ensures the most valuable features are delivered without committing to everything or ignoring feedback.

Option A leads to scope creep and unfocused product development.

Option B uses one metric (volume) that may not reflect actual value. Loudest users aren't always most important users.

Option D ignores valuable market feedback. Roadmaps should evolve based on learning.
