# Question

A project has explicit scope exclusions documented. A stakeholder argues that one exclusion was a mistake and the excluded feature is essential. The project is 40% complete. What should the PM do?

## Options

- **A**: Add the feature since the stakeholder says it's essential
- **B**: Maintain the exclusion since it was documented and approved
- **C**: Treat this as a change request and evaluate the impact of including the previously excluded feature
- **D**: Ask the original authors of the scope document to explain why it was excluded

## Answer

C

## Explanation

The correct answer is C. Documented exclusions can be reconsidered if circumstances or understanding change. The proper path is through change control: evaluate the impact of including the feature, considering that 40% of the project was planned without it. Present options and let decision-makers choose.

Option A bypasses change control.

Option B treats documentation as unchangeable. Exclusions were decisions that can be revisited through proper process.

Option D provides context but doesn't move toward resolution. Historical rationale is useful but doesn't determine whether the feature should now be included.
