# Question

A product owner regularly changes user story priorities mid-sprint, causing the team to switch focus and leave work incomplete. What should the Scrum Master do?

## Options

- **A**: Allow the changes since the PO owns the backlog
- **B**: Explain that sprint scope should be stable once committed, and changes should wait for sprint planning
- **C**: Escalate to management to control the product owner
- **D**: Have the team ignore the PO's requests

## Answer

B

## Explanation

The correct answer is B. While the PO owns the backlog, sprint commitment represents a short-term agreement. Constant mid-sprint changes prevent the team from achieving flow state and completing quality work. The SM should coach the PO on sprint stability - new priorities can be added to the backlog for next sprint planning, not disrupting current work.

Option A misunderstands PO authority. PO owns backlog prioritisation but shouldn't disrupt active sprints except for true emergencies.

Option C is premature and damages the relationship. Coaching should come before escalation.

Option D creates conflict and doesn't address the underlying issue through education.
