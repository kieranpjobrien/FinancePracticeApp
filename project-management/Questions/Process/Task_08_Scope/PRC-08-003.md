# Question

A project manager discovers that the team has been implementing features based on verbal requests from stakeholders that weren't in the approved scope. The features add value but weren't formally approved. What should the PM do?

## Options

- **A**: Remove the unapproved features to maintain scope integrity
- **B**: Document the features as a scope change, assess impact, and submit for formal approval retroactively
- **C**: Accept the features since they add value and continue
- **D**: Reprimand the team for not following process

## Answer

B

## Explanation

The correct answer is B. The features already exist, so removing them wastes the effort. However, they need formal recognition to align baseline, manage expectations, and prevent future informal additions. Retroactive documentation brings the situation under control without destroying completed value.

Option A wastes completed work over process. The value exists - destroying it doesn't help.

Option C ignores the process breakdown and doesn't update baselines, leaving the project in an inconsistent state.

Option D focuses on blame rather than solution. The immediate need is regularizing the situation; process improvement can follow.
