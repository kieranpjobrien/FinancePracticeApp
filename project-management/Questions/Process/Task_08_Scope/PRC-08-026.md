# Question

A project team is struggling to estimate a complex feature because they've never built anything like it. The product owner needs a commitment for planning purposes. What approach should the PM recommend?

## Options

- **A**: Provide a rough estimate with high contingency and commit to refining it
- **B**: Use a spike (time-boxed research) to reduce uncertainty before estimating
- **C**: Estimate based on similar features from other projects
- **D**: Break the feature into smaller pieces that are easier to estimate

## Answer

B

## Explanation

The correct answer is B. A spike is a time-boxed investigation to gain knowledge needed for estimation. For novel work with high uncertainty, investing time in learning before committing improves estimate accuracy and reduces risk. The spike itself is estimable, and findings inform the actual feature estimate.

Option A commits despite known uncertainty. High contingency signals the estimate isn't reliable.

Option C uses analogous estimation but may not apply if the feature is truly novel.

Option D may help but doesn't address the fundamental lack of knowledge about the technical approach.
