# Question

A vendor delivers a component that meets all specified requirements but doesn't integrate well with other system components. The integration problems weren't covered in requirements. Who bears responsibility?

## Options

- **A**: The vendor, since they should have anticipated integration needs
- **B**: The project team, since integration requirements should have been specified
- **C**: Both parties share responsibility and should work together to resolve the issues
- **D**: The PM, since they should have identified the gap during requirements review

## Answer

B

## Explanation

The correct answer is B. Requirements define contract scope. If integration requirements weren't specified, the vendor fulfilled their contractual obligation. The project team (including the PM) is responsible for ensuring requirements are complete. This is a lesson learned about requirements completeness for future procurements.

Option A holds the vendor to unstated requirements, which isn't fair or contractually sound.

Option C sounds collaborative but the vendor delivered what was specified. Expecting them to share resolution cost for unspecified requirements isn't reasonable.

Option D focuses blame narrowly. While the PM has responsibility, so does the entire team involved in requirements development.
