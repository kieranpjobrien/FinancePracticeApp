# Question

In agile, what is the PRIMARY tool for managing project scope?

## Options

- **A**: Project scope statement
- **B**: Work breakdown structure
- **C**: Product backlog
- **D**: Requirements traceability matrix

## Answer

C

## Explanation

The correct answer is C. The product backlog is the single source of requirements for any changes to be made to the product in agile. It's an ordered list of everything that might be needed, owned by the product owner, and constantly refined and reprioritised based on feedback and changing needs.

Option A is used in predictive approaches to define scope boundaries.

Option B decomposes work hierarchically but isn't the primary agile scope management tool.

Option D tracks requirements to deliverables but isn't the primary tool for managing what's in scope.
