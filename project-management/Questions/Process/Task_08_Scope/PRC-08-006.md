# Question

A project manager notices that the team is consistently delivering more functionality than specified in user stories - adding extra features they think users will want. Stakeholders are pleased with the extras, but the project is falling behind schedule. What should the PM address?

## Options

- **A**: Praise the team for their initiative and customer focus
- **B**: Adjust the schedule to accommodate the additional work
- **C**: Address the scope creep by reinforcing that work should match defined requirements, with extras going through backlog prioritisation
- **D**: Reduce testing time to compensate for the additional development

## Answer

C

## Explanation

The correct answer is C. This is gold-plating - adding unrequested features. Even if stakeholders appreciate extras now, undisciplined scope expansion causes schedule delays and may add features that don't provide proportional value. Extras should be captured as backlog items and prioritised against other work.

Option A (praise) reinforces problematic behaviour. Initiative should be channelled into backlog suggestions, not unplanned additions.

Option B (adjust schedule) rewards scope creep without addressing root cause. The pattern will continue, causing further delays.

Option D (reduce testing) trades quality for scope - a poor trade that creates downstream problems (defects, rework, user dissatisfaction).
