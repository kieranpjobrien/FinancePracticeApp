# Question

A regulatory change requires adding new compliance features to a project. The sponsor says there's no additional budget, but the features are legally required. How should the PM proceed?

## Options

- **A**: Add the features and absorb the cost impact within existing contingency
- **B**: Present options for reducing other scope to accommodate mandatory compliance features
- **C**: Proceed without the features and document the regulatory risk
- **D**: Delay the project until additional funding is approved

## Answer

B

## Explanation

The correct answer is B. When mandatory scope increases without budget increase, something must give. The PM should present trade-off options: reduce other scope, extend timeline, or accept quality/risk trade-offs. The sponsor can then make an informed decision about how to accommodate the mandatory work within constraints.

Option A may not be possible if contingency is insufficient, and using contingency for scope changes rather than risks is inappropriate.

Option C creates legal liability. Mandatory compliance isn't optional regardless of budget.

Option D may not be viable if there's a regulatory deadline, and doesn't present alternatives.
