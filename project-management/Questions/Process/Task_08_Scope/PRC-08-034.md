# Question

The project charter specified a mobile app for iOS. Midway through development, the sponsor casually mentions in a meeting that Android support is also expected. What should the PM do?

## Options

- **A**: Add Android development to the project since the sponsor mentioned it
- **B**: Clarify the scope formally with the sponsor and submit a change request if Android is required
- **C**: Continue with iOS only since that's what the charter specified
- **D**: Propose a follow-on project for Android support

## Answer

B

## Explanation

The correct answer is B. The PM needs to clarify whether this is a new requirement or a misunderstanding about original scope. If Android is genuinely required, it's a significant scope change requiring formal change control with impact analysis. Casual mentions in meetings shouldn't drive scope changes.

Option A accepts scope expansion without analysis or formal approval.

Option C ignores potentially critical information. The sponsor may have always expected Android, indicating a charter gap.

Option D assumes a separate project is the solution before understanding whether Android should have been in scope originally.
