# Question

The project charter approves an iOS-only mobile app. In the middle of a status meeting the sponsor refers casually to "Android users" as if Android support were obviously included. What should the PM do FIRST?

## Options

- **A**: Add Android delivery to the team's plan because the sponsor has clearly expressed an expectation.
- **B**: Privately confirm with the sponsor whether Android is required, and if so submit a change request describing scope, cost, and schedule impact.
- **C**: Continue iOS-only and note in the minutes that the charter does not include Android.
- **D**: Spin up a separate Android project so the existing one is not destabilised.

## Answer

B

## Explanation

A single off-hand remark is not authorisation, but ignoring it would also be wrong if a real expectation exists. The PM clarifies first, then runs any actual scope change through change control. (A) commits the team without analysis. (C) records a discrepancy but does not resolve it. (D) presumes a solution before knowing the question.
