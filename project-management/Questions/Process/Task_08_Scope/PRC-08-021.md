# Question

What is the difference between product scope and project scope?

## Options

- **A**: Product scope is defined by the customer; project scope is defined by the PM
- **B**: Product scope describes features of the product; project scope describes work needed to deliver the product
- **C**: Product scope is fixed; project scope can change
- **D**: Product scope is detailed; project scope is high-level

## Answer

B

## Explanation

The correct answer is B. Product scope defines the features and functions that characterise the product, service, or result. Project scope defines all the work required to deliver the product with the specified features. Product scope focuses on "what" is being built; project scope focuses on "how" it will be built.

Option A misattributes ownership. Both scopes involve multiple stakeholders.

Option C is incorrect. Both scopes can change through proper change control processes.

Option D confuses detail level. Both can be detailed or high-level depending on the project phase.
