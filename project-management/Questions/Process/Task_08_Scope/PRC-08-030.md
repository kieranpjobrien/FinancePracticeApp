# Question

Scope creep is BEST described as:

## Options

- **A**: Approved changes to the project scope
- **B**: Uncontrolled expansion of scope without adjustments to time, cost, and resources
- **C**: The natural evolution of requirements during project execution
- **D**: Adding features that improve project quality

## Answer

B

## Explanation

The correct answer is B. Scope creep refers to uncontrolled changes or continuous growth in project scope without corresponding adjustments to schedule, budget, or resources. It typically occurs when changes are made without formal change control, leading to project overruns.

Option A describes legitimate scope changes through change control, not scope creep.

Option C normalises what should be controlled. While requirements may evolve, changes should go through proper processes.

Option D describes potential gold plating. Adding features without approval is scope creep regardless of intent.
