# Question

A project delivers a data analytics platform. Users report that while the data is accurate, generating reports takes too long. The team argues accuracy was the quality requirement. How should the PM view this situation?

## Options

- **A**: The team delivered what was specified; performance should be a change request
- **B**: Quality requirements were incomplete; performance should have been specified as a quality attribute
- **C**: Users need to adjust their expectations since accuracy was prioritised
- **D**: Add performance testing to validate future releases meet unstated expectations

## Answer

B

## Explanation

The correct answer is B. Quality encompasses multiple attributes including performance, usability, reliability, and accuracy. If performance wasn't specified as a quality requirement, that's a gap in requirements elicitation. Quality means fit for purpose - if users can't use the platform effectively due to speed, it's not fit for purpose regardless of accuracy.

Option A is technically correct but misses the point. The goal is satisfied users, not contractual compliance. This reveals a planning gap.

Option C blames users for an incomplete specification. Quality professionals should anticipate user needs.

Option D addresses future releases but doesn't acknowledge the current situation represents incomplete quality planning.
