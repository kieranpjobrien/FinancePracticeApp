# Question

A software development team is experiencing high defect rates. The project manager analyses the defects and finds that 80% of bugs originate from three modules written by one developer who recently joined the team. What quality management approach should the PM implement?

## Options

- **A**: Assign the defective modules to a more experienced developer
- **B**: Implement mandatory code reviews for all code before merging, with particular attention to the new developer's work
- **C**: Add additional testing phases to catch defects before release
- **D**: Replace the developer with someone more experienced

## Answer

B

## Explanation

The correct answer is B. Code reviews are a quality assurance practice that prevents defects from entering the codebase. For a new developer, reviews also serve as mentoring and knowledge transfer. This addresses the root cause (lack of familiarity with codebase/standards) rather than just catching defects downstream.

Option A (reassign modules) doesn't develop the new team member and creates a bottleneck where only certain people can work on certain code.

Option C (more testing) catches defects but doesn't prevent them. It's more expensive to find and fix defects later in the process. Quality should be built in, not inspected in.

Option D (replace developer) is extreme for what appears to be an onboarding issue. New developers need time and support to learn the codebase.
