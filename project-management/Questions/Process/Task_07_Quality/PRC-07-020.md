# Question

The sponsor of a new software product asks you to "track code quality" but has not defined what code quality means. What should the project manager do FIRST?

## Options

- **A**: Facilitate a working session with the sponsor and key stakeholders to define which quality attributes (performance, reliability, maintainability, security) matter for this product and how they will be measured.
- **B**: Adopt industry-standard code metrics such as cyclomatic complexity, code coverage, and duplication and report against them from the next sprint.
- **C**: Ask the development team which metrics they consider meaningful for code quality and use those as the project's measures.
- **D**: Review past projects in the organisation, copy the metric set used by the most successful one, and apply it here.

## Answer

A

## Explanation

PMI defines quality as conformance to stakeholder needs, so the first step is to make the sponsor's "code quality" measurable in terms the stakeholders care about. (A) does that. (B) jumps to a generic metric set that may not reflect what the sponsor values. (C) lets the team pick the metrics they would be measured against, which is a conflict of interest and disconnected from stakeholder priorities. (D) imports another project's context that may not match this product's needs.
