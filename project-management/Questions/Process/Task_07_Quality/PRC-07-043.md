# Question

A project manager inherits a project where quality problems have caused significant rework. Analysis shows the team has been "gold plating" - adding unrequested features that introduce complexity and bugs. What should the PM do FIRST?

## Options

- **A**: Implement strict change control to prevent unauthorised additions
- **B**: Discuss with the team why they've been adding unrequested features
- **C**: Remove all non-required features from the current deliverables
- **D**: Document the quality issues and present them to leadership

## Answer

B

## Explanation

The correct answer is B. Before implementing solutions, understand the root cause. The team may be adding features because requirements are unclear, stakeholders are making informal requests, or developers believe they're adding value. Understanding why helps choose the right intervention.

Option A might be appropriate but doesn't address why gold plating is happening. If requirements are genuinely unclear, strict change control won't help.

Option C may be necessary but could remove valuable functionality or create other problems. Understand the situation first.

Option D documents symptoms without addressing causes. The PM should investigate before escalating.
