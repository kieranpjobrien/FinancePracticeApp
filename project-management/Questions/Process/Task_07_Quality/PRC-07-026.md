# Question

A software team practices continuous integration with automated testing. A developer's code passes all automated tests but another developer notices it doesn't handle an edge case that could cause data corruption. What quality practice should be strengthened?

## Options

- **A**: Increase code coverage requirements for automated tests
- **B**: Implement peer code reviews as part of the Definition of Done
- **C**: Add a manual testing phase before deployment
- **D**: Require developers to document all edge cases before coding

## Answer

B

## Explanation

The correct answer is B. Peer code review caught an issue that automated tests missed. This demonstrates the value of human review for identifying logic errors, edge cases, and design issues that automated tests don't cover. Making it part of the DoD ensures it happens consistently.

Option A might help but code coverage measures quantity, not quality of tests. You can have 100% coverage and still miss edge cases if tests don't check the right conditions.

Option C adds delay without the focused expertise that peer review provides. Manual testing at the end is less efficient than catching issues during development.

Option D front-loads effort but developers may not anticipate all edge cases. Peer review leverages collective knowledge during implementation.
