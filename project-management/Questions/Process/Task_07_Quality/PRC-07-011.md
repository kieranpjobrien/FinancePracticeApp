# Question

A defect-trend analysis shows the team is spending roughly 30% of every sprint reworking code that failed test. Root-cause analysis traces most of the defects to ambiguous and incomplete user stories. What should the project manager prioritise to address this?

## Options

- **A**: Increase test automation coverage so defects are caught earlier in the build pipeline.
- **B**: Add a dedicated QA engineer to the team to triage and categorise defects faster.
- **C**: Tighten the acceptance criteria template so completed work is rejected unless every criterion is provably met.
- **D**: Invest in prevention activities — story refinement, examples and acceptance-test-driven development, and prototyping — so requirements are unambiguous before development starts.

## Answer

D

## Explanation

PMI's cost-of-quality model places prevention as the cheapest place to spend on quality, especially when the root cause is upstream of development. (D) attacks the cause — ambiguous requirements — directly. (A) and (B) improve detection, which is a more expensive place to invest than prevention and does not stop the defects being introduced. (C) tightens the gate after the work is done; the developers still build the wrong thing first.
