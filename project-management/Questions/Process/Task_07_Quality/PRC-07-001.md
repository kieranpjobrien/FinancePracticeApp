# Question

During sprint review, stakeholders approve a feature but express concerns about minor usability issues that weren't in the acceptance criteria. The team believes the feature meets the Definition of Done. What should the project manager recommend?

## Options

- **A**: Mark the feature complete since it meets the agreed criteria
- **B**: Reject the feature and require the team to fix usability issues before accepting
- **C**: Accept the feature as complete and create new backlog items for usability improvements
- **D**: Update the Definition of Done to include usability criteria going forward

## Answer

C

## Explanation

The correct answer is C. The feature legitimately meets the agreed criteria - changing the rules retroactively is unfair to the team. However, stakeholder feedback is valuable. Creating new backlog items captures the improvements for prioritization. This respects both the process (DoD) and stakeholder input.

Option A technically correct but ignores valuable feedback. The PM should ensure feedback is captured, not dismissed.

Option B moves the goalposts after the fact, which is demoralizing and unfair. The team delivered what was agreed.

Option D is good practice for future work but doesn't address the immediate situation. Both C and D might be appropriate, but C directly addresses this feature.
