# Question

A project uses automated testing with 85% code coverage. A major bug escapes to production in an area with 100% coverage. Stakeholders question the value of automated testing. How should the PM respond?

## Options

- **A**: Acknowledge that automated testing has limitations and isn't a guarantee
- **B**: Increase the code coverage target to catch more issues
- **C**: Explain that coverage measures quantity of code tested, not quality of tests, and review test effectiveness
- **D**: Add manual testing to complement automated testing

## Answer

C

## Explanation

The correct answer is C. Code coverage measures which lines of code are executed during tests, not whether tests verify the right conditions. 100% coverage with poor assertions catches nothing. The PM should help stakeholders understand this distinction and lead an effort to improve test quality, not just quantity.

Option A is true but doesn't educate stakeholders or improve the situation.

Option B addresses the wrong metric. The covered code had the bug - more coverage wouldn't help.

Option D might help but doesn't address the root issue that existing tests weren't verifying the right things.
