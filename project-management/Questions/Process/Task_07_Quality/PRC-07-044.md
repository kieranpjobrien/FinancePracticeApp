# Question

A project measures 85% automated test coverage. A serious defect ships to production in a module that had 100% line coverage. The CIO asks the PM whether automated testing is actually working. What is the best response?

## Options

- **A**: Acknowledge that automated testing has limits and supplement with more exploratory manual testing.
- **B**: Raise the coverage target to 95% to reduce the chance of similar misses.
- **C**: Explain that coverage measures code executed, not the quality of assertions, and lead a review of test design.
- **D**: Concede that automated testing alone cannot guarantee quality and recommend a return to phased manual QA.

## Answer

C

## Explanation

Coverage and test quality are different things. A test can execute every line and assert nothing meaningful. The right response educates the stakeholder and targets the actual root cause: weak assertions. (A) is true but does not explain the failure or lead to improvement. (B) chases the wrong metric — the missed defect was in 100%-covered code. (D) over-corrects by abandoning a valuable practice rather than fixing how it is applied.
