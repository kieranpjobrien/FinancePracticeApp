# Question

An agile team discovers a critical bug in production that was missed during testing. The retrospective reveals the team didn't have clear acceptance criteria for that user story. What should the PM facilitate?

## Options

- **A**: Add more testing phases to the Definition of Done
- **B**: Require product owner sign-off before any story is marked complete
- **C**: Improve the Definition of Ready to ensure stories have clear acceptance criteria before sprint planning
- **D**: Implement a dedicated QA phase after each sprint

## Answer

C

## Explanation

The correct answer is C. The root cause is stories entering development without clear acceptance criteria. The Definition of Ready ensures stories meet minimum standards before the team commits to them. Preventing unclear stories from entering sprints addresses the root cause.

Option A adds testing but doesn't address the root cause. You can't effectively test without clear criteria to test against.

Option B adds a checkpoint but doesn't ensure criteria exist upfront. It could create bottlenecks without solving the underlying problem.

Option D contradicts agile principles of building quality in. Adding a separate QA phase doesn't address why criteria were unclear.
