# Question

A project team repeatedly finds the same types of defects in their code reviews. The PM notices patterns in what's being caught. What continuous improvement action should the PM recommend?

## Options

- **A**: Replace code reviews with automated static analysis tools
- **B**: Create a checklist of common issues to prevent them from being introduced
- **C**: Assign the most experienced developer to review all code
- **D**: Add more reviewers to each code review session

## Answer

B

## Explanation

The correct answer is B. Identifying patterns enables prevention. A checklist based on frequently found issues helps developers avoid making those mistakes in the first place. This shifts effort from detection to prevention, which is more cost-effective.

Option A may help but tools and humans catch different issues. Automated tools complement but shouldn't replace human review for logic and design issues.

Option C creates a bottleneck and doesn't address the root cause. One person reviewing everything doesn't scale and doesn't prevent defects from being introduced.

Option D increases detection resources without addressing prevention. More reviewers don't help if the same mistakes keep being made.
