# Question

A project has strict quality requirements due to safety implications. The team proposes using AI-generated code to accelerate development. What quality consideration should the PM prioritise?

## Options

- **A**: AI-generated code should be prohibited for safety-critical systems
- **B**: Establish rigorous review and testing processes for all code regardless of source
- **C**: Require AI-generated code to pass additional quality gates compared to human-written code
- **D**: Accept AI code only for non-critical components of the system

## Answer

B

## Explanation

The correct answer is B. Quality standards should be based on requirements and risk, not code origin. All code in a safety-critical system needs rigorous review and testing. Human-written code can have bugs too. The focus should be on verification processes that ensure quality regardless of how code was created.

Option A may be overly restrictive. The source matters less than verification that requirements are met.

Option C creates inconsistent standards. If AI code needs more scrutiny, that suggests the standards for human code may be insufficient.

Option D assumes AI code is inherently riskier. With proper review, AI-assisted development can meet quality standards anywhere in the system.
