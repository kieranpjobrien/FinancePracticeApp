# Question

A safety-critical avionics project's developers want to use AI code-generation tools to accelerate delivery. The chief engineer worries that AI-produced code may not meet DO-178C standards. What is the project manager's best position on quality?

## Options

- **A**: Disallow AI-generated code in this product because the regulatory regime cannot accommodate it.
- **B**: Permit AI-generated code only in non-flight-critical components and require manual rewrite for the rest.
- **C**: Require AI-generated code to clear additional bespoke quality gates beyond the standard pipeline.
- **D**: Hold all code to the same regulator-mandated review, traceability, and verification regardless of how it was produced.

## Answer

D

## Explanation

PMI quality philosophy and certification regimes such as DO-178C are concerned with evidence of conformance to requirements, not with the author of the code. The right answer is to apply the existing rigorous controls uniformly. (A) is overly broad and unsupported by the standard. (B) creates two unequal pipelines and assumes AI code is intrinsically less safe. (C) introduces inconsistent standards that imply the human-written baseline is insufficient — if it is, the human standard should rise too.
