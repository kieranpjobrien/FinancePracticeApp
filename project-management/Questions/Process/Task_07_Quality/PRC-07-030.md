# Question

A project delivering a mobile app has a quality objective of "95% user satisfaction." During testing, the team focuses only on functional requirements. What quality dimension is being overlooked?

## Options

- **A**: Reliability - the app should work consistently
- **B**: Usability - how easy and pleasant the app is to use
- **C**: Performance - the app should respond quickly
- **D**: All of the above contribute to user satisfaction

## Answer

D

## Explanation

The correct answer is D. User satisfaction is influenced by multiple quality dimensions beyond functional correctness. An app that works correctly but is slow (performance), crashes occasionally (reliability), or is confusing to navigate (usability) will not achieve 95% satisfaction.

Options A, B, and C are all correct individually but incomplete. Focusing on any single dimension while ignoring others would miss the holistic nature of user satisfaction.

The team should identify which quality attributes drive satisfaction for their users and include appropriate testing for each dimension, not just functional testing.
