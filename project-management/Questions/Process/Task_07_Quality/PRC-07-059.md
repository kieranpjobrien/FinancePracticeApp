# Question

A Scrum developer has added an extra feature during implementation because she thought the user "would probably want it." It works, but it was never in the story and was not prioritised. The product owner is unaware. What should the PM advise?

## Options

- **A**: Surface the addition to the PO, discuss acceptance and inclusion, and reinforce that backlog authority sits with the PO.
- **B**: Accept the addition silently since it works and adds value, and note it in the sprint review for awareness only.
- **C**: Instruct the developer to remove the feature since it was not prioritised and risks confusing the user acceptance test.
- **D**: Add the feature to the backlog retrospectively and prioritise it in the next sprint so the work is not lost altogether.

## Answer

A

## Explanation

Gold-plating bypasses the product owner's prioritisation authority; the right action is transparent surfacing and reinforcement of the PO role. (A) is correct. (B) normalises backlog drift. (C) wastes work without PO input. (D) legitimises an unauthorised change through process gymnastics.
