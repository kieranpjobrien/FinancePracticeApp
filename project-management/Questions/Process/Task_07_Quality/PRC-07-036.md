# Question

A hybrid project has both agile development teams and waterfall infrastructure teams. The agile teams use Definition of Done while infrastructure uses phase-gate quality reviews. Integration testing keeps finding interface issues. What should the PM do?

## Options

- **A**: Require both teams to use the same quality approach
- **B**: Add integration testing to the agile Definition of Done
- **C**: Create cross-functional quality checkpoints where both teams verify interfaces together
- **D**: Assign a dedicated integration team responsible for interface quality

## Answer

C

## Explanation

The correct answer is C. The issue is at the boundary between teams. Cross-functional checkpoints bring both teams together to verify interfaces work correctly before full integration. This addresses the root cause (disconnected quality approaches at integration points) without forcing methodology changes.

Option A ignores that different approaches suit different work types. Forcing uniformity may reduce effectiveness for one or both teams.

Option B only addresses one side. Infrastructure teams would still proceed to integration without agile team involvement.

Option D adds resources but may create a "throw it over the wall" mentality where teams don't own integration quality themselves.
