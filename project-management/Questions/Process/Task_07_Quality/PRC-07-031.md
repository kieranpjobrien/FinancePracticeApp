# Question

A Scrum team's Definition of Done includes "all tests pass" and "code reviewed by another team member." A story was marked done after a 30-second approval where the reviewer never ran the code locally; a defect later reached production. What should the project manager facilitate FIRST?

## Options

- **A**: Update the Definition of Done with explicit reviewer steps such as "reviewer must check out the branch and run the tests locally" so the requirement is unambiguous in future.
- **B**: Convene the team to discuss what "code reviewed" actually means in practice and whether the spirit of the Definition of Done was followed in this case.
- **C**: Add a separate independent QA gate after code review so a second control catches what a perfunctory review may have missed in the future.
- **D**: Configure the build pipeline so pull requests cannot be merged unless additional automated checks (mutation tests, coverage gate) pass.

## Answer

B

## Explanation

Before adding new process or new tools, PMI's continuous improvement and Scrum's empirical inspect-and-adapt cycle expect the team to first understand the gap between the rule and the practice. (B) does that and is the prerequisite to any of the other actions. (A) prescribes a fix without first confirming what the team's shared understanding currently is. (C) and (D) add controls on top of a process the team is not yet executing as intended.
