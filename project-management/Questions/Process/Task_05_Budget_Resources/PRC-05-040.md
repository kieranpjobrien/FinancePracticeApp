# Question

The organisation is implementing time tracking software that requires detailed effort logging. Team members complain it's administrative overhead that reduces productive time. How should the PM balance tracking needs with team concerns?

## Options

- **A**: Force detailed tracking regardless of impact
- **B**: Explain the value of tracking data for project management and organisational decisions, work with the team to streamline logging where possible, and advocate for reasonable tracking requirements
- **C**: Abandon all time tracking
- **D**: Log time on behalf of team members

## Answer

B

## Explanation

The correct answer is B. Time tracking provides valuable data but has costs. Explain value: how is data used? What decisions does it enable? Streamline: can tracking be simplified while meeting needs? Advocate: if requirements are excessive, raise concerns. Balance: accept some overhead for necessary data while minimising burden. This addresses both organisational and team needs.

Option A ignores legitimate concerns.

Option C loses valuable management data.

Option D is inappropriate and produces inaccurate data.
