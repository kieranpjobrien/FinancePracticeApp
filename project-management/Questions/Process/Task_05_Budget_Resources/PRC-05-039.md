# Question

The project's burn rate is 25% above plan because of unexpected technical complexity in a core component. The team is productive but the work simply requires more effort than was estimated. The sponsor asks the PM for "options." What should the PM bring to the next meeting?

## Options

- **A**: A request for additional budget equal to the projected EAC overrun, with justification.
- **B**: A recommendation to reduce scope to fit the remaining budget, with a list of features to defer.
- **C**: A re-baselined schedule that spreads the remaining effort over a longer duration to lower the monthly burn.
- **D**: Several options — additional funding, scope reduction, schedule extension, and hybrids — each with cost, schedule, and value impact and a recommended choice.

## Answer

D

## Explanation

When the sponsor explicitly asks for "options," PMI expects a decision package — multiple alternatives with trade-offs and a clear PM recommendation — not a single answer. (D) does that. (A), (B), and (C) each present only one of the trade-offs the sponsor asked to compare, so any of them on its own would pre-empt the sponsor's decision. The PM's job is to enable the choice, not to make it unilaterally.
