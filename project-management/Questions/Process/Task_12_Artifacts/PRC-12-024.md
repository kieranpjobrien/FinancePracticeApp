# Question

Three months into a project, the lessons learned register has no new entries even though several notable incidents have occurred. The team says they don't have time to stop and document lessons. What should the project manager do?

## Options

- **A**: Add a lessons learned agenda item to the existing sprint retrospective or phase-gate review.
- **B**: Raise team non-compliance with the functional manager so it is addressed in individual performance reviews.
- **C**: Maintain the lessons learned register personally based on the PM's observations each week.
- **D**: Wait until project close and run a dedicated lessons-learned workshop to capture everything at once.

## Answer

A

## Explanation

PMI treats lessons learned as a continuous practice and expects the PM to remove friction rather than add processes. (A) uses ceremonies the team is already running, captures lessons while memories are fresh, and requires minimal extra time. (B) turns a process-design problem into a performance issue and damages trust. (C) loses the team's perspective, which is the whole point of lessons learned. (D) is classic "lessons learned at the end" — by then context is gone and the lessons cannot help the current project.
