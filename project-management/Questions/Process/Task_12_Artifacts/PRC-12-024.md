# Question

A project manager notices that the lessons learned register hasn't been updated since the project began three months ago, despite several significant events occurring. Team members say they're too busy with project work to document lessons. What approach should the project manager take?

## Options

- **A**: Wait until project completion to gather all lessons learned at once
- **B**: Document the lessons learned personally without team input
- **C**: Integrate lessons learned capture into existing project ceremonies and processes to minimize additional overhead
- **D**: Escalate to management that the team is not complying with documentation requirements

## Answer

C

## Explanation

The correct answer is C. Making lessons learned part of regular activities (retrospectives, phase reviews, milestone meetings) removes the perception of extra burden. Brief capture during these events, when information is fresh, is more effective than trying to recall lessons later or adding separate documentation sessions.

Option A risks losing valuable details as memories fade. End-of-project lessons learned often miss important context.

Option B loses the team's perspective and insights. Lessons learned should capture multiple viewpoints.

Option D treats it as a compliance issue rather than addressing the root cause (perceived overhead).
