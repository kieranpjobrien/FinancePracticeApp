# Question

What is the PRIMARY purpose of maintaining a configuration management system for project artifacts?

## Options

- **A**: To archive all documents at project completion
- **B**: To control versions and changes to project documents, ensuring the correct version is available and changes are tracked
- **C**: To restrict access to project documents
- **D**: To automate document creation

## Answer

B

## Explanation

The correct answer is B. Configuration management controls the versions and changes to project artifacts (documents, code, deliverables). It ensures everyone works from the correct version, changes are tracked and authorized, and the history of changes is preserved. This prevents confusion from multiple versions and enables audit trails.

Option A describes archiving, which is one aspect but not the primary purpose.

Option C describes access control, which may be part of the system but isn't its primary purpose.

Option D describes automation, which isn't configuration management's purpose.
