# Question

A project manager joins an ongoing project and discovers that project documentation is scattered across multiple platforms - some in email, some in a shared drive, some in a project management tool, and some only in team members' notes. What should be the project manager's approach?

## Options

- **A**: Accept the current state and adapt to it
- **B**: Immediately delete all distributed documents and start fresh
- **C**: Consolidate artifacts into an organized repository, establish documentation standards, and communicate the approach to the team
- **D**: Assign a team member to find documents when needed

## Answer

C

## Explanation

The correct answer is C. Scattered documentation creates risk of lost information, version confusion, and wasted time searching. Consolidating into an organized repository, establishing standards for future documentation, and communicating expectations creates a sustainable system. This should be done systematically, not disruptively.

Option A accepts an inefficient and risky situation that will continue to cause problems.

Option B risks losing important information and creates disruption without a transition plan.

Option D treats symptoms rather than the root cause and creates dependency on one person.
