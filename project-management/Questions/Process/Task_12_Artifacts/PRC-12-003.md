# Question

A new analyst joins the project and asks the PM where to find the current schedule, risk register, and stakeholder list. The PM realises these documents live across email threads, two SharePoint sites, and several personal drives in different versions. What should the PM do?

## Options

- **A**: Send the analyst direct links to the most recent copies the PM is aware of so they can start work today.
- **B**: Maintain a separate index spreadsheet that lists where each artifact lives so people can find them.
- **C**: Ask each functional lead to circulate their own current copies of relevant documents to new joiners.
- **D**: Establish a single project repository with version control and access protocols, migrate the artifacts in, and retire the duplicate locations.

## Answer

D

## Explanation

PMI's approach to artifact management calls for a single source of truth so every team member, present and future, accesses current information. (A) helps one person and leaves the structural problem for the next joiner. (B) creates an index over a problem rather than fixing it. (C) institutionalises duplicate copies and keeps version drift alive.
