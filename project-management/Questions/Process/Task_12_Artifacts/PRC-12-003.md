# Question

A new team member joins an ongoing project and asks the project manager where to find the current project schedule, risk register, and stakeholder list. The PM discovers these documents are scattered across emails, personal drives, and multiple SharePoint sites with varying versions. What should the PM do?

## Options

- **A**: Send the new team member links to the most recent versions they can find
- **B**: Establish a centralised project repository with version control and migrate all artifacts
- **C**: Ask each team member to maintain their own copies of relevant documents
- **D**: Create a document index spreadsheet listing where each artifact is stored

## Answer

B

## Explanation

The correct answer is B. Effective artifact management requires a single source of truth with version control. A centralised repository ensures everyone accesses current versions, maintains document history, and supports project knowledge management. This is a process improvement that benefits the entire project.

Option A (send links) is a short-term fix that doesn't address the underlying problem. The next person to join will face the same issue.

Option C (personal copies) worsens the problem. Multiple copies without central control leads to version confusion and inconsistent information.

Option D (index spreadsheet) is marginally better but still requires manual maintenance and doesn't solve version control. It's a workaround, not a solution.
