# Question

A SaaS team ships features each sprint, but user-facing documentation lags by four sprints and customers are filing tickets about missing help articles. What should the project manager recommend?

## Options

- **A**: Pause feature work for one sprint to clear the documentation backlog before resuming.
- **B**: Hire a contract technical writer to handle the historical backlog while the team continues to ship features.
- **C**: Update the team's definition of done so a feature is not complete until its documentation ships with it.
- **D**: Move all product documentation to a community wiki so users can author the missing articles themselves.

## Answer

C

## Explanation

PMI's "build quality in" principle says the standard for "done" should prevent debt from accumulating. (C) fixes the system so the backlog cannot grow again. (A) clears the historic backlog but does nothing to stop a new one forming next sprint. (B) addresses the symptom and creates handoff overhead between writer and developers. (D) abdicates a vendor responsibility and damages customer trust.
