# Question

A project produces technical deliverables that require accompanying user documentation. The technical team completes features faster than documentation can be written, creating a growing documentation backlog. What approach should the project manager recommend?

## Options

- **A**: Deliver features without documentation and catch up later
- **B**: Slow down feature development to match documentation pace
- **C**: Include documentation as part of the definition of done for features, so features aren't considered complete until documentation is ready
- **D**: Outsource all documentation to a separate team

## Answer

C

## Explanation

The correct answer is C. Including documentation in the definition of done ensures documentation is created alongside features rather than as a separate trailing activity. This integrates documentation into the delivery flow and prevents backlog accumulation. It may slightly slow individual features but maintains overall quality.

Option A creates technical debt in documentation and risks delivering unusable features.

Option B may unnecessarily slow development if documentation can be integrated into the workflow.

Option D may work but creates coordination overhead and potential knowledge gaps between developers and documentation writers.
