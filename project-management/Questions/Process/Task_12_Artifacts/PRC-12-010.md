# Question

What is the purpose of establishing a project baseline?

## Options

- **A**: To create a historical record that cannot be changed
- **B**: To provide an approved reference point against which project performance can be measured and compared
- **C**: To lock the project scope so no changes can occur
- **D**: To document the initial project estimate only

## Answer

B

## Explanation

The correct answer is B. A baseline is an approved version of a work product (scope, schedule, cost) that serves as a reference point. It enables performance measurement by comparing actual results against the plan. Baselines can be updated through formal change control when approved changes occur.

Option A is incorrect because baselines can be updated through proper change control processes.

Option C is incorrect because baselines don't prevent changes; they provide a reference point for measuring the impact of changes.

Option D is too narrow - baselines capture the approved plan, not just initial estimates, and cover multiple dimensions (scope, schedule, cost).
