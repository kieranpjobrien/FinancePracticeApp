# Question

An organization is running multiple related projects as part of a program. Each project maintains its own documentation repository with different structures and conventions. Program-level reporting is difficult because information must be manually gathered and reconciled from each project. What should the program manager recommend?

## Options

- **A**: Allow each project to continue with their current practices for autonomy
- **B**: Establish program-level documentation standards and a common repository structure while allowing project-specific extensions where needed
- **C**: Consolidate all project documentation into a single repository managed by the program office
- **D**: Hire additional staff to compile program reports manually

## Answer

B

## Explanation

The correct answer is B. Program-level standards enable efficient reporting and cross-project visibility while project-specific extensions accommodate legitimate differences. This balances program needs (consistency for reporting) with project needs (flexibility for unique aspects).

Option A perpetuates inefficiency and difficulty in program oversight.

Option C may create excessive centralization and not account for legitimate project-specific documentation needs.

Option D treats symptoms rather than the root cause and creates ongoing costs.
