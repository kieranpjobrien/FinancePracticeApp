# Question

A programme manager runs five related projects, each with its own repository structure and naming conventions. Programme-level reporting each month takes three days of manual consolidation. What should the programme manager do?

## Options

- **A**: Mandate a single programme-wide document repository and migrate all existing artifacts over one weekend.
- **B**: Hire a programme coordinator whose role is to collect and reconcile artifacts each month.
- **C**: Define programme-level standards for shared artifacts and reports, and let each project retain project-specific structures beneath them.
- **D**: Continue the current consolidation approach and focus effort on improving the reporting template instead.

## Answer

C

## Explanation

PMI expects tailoring at programme level — shared artifacts need shared standards, but projects keep legitimate local differences. (C) fixes the reporting pain without over-centralising. (A) is too aggressive and ignores project context; migrations at programme scale fail when imposed top-down. (B) buries the problem under permanent manual effort instead of solving it. (D) polishes the symptom (the template) and leaves the three-day consolidation in place.
