# Question

During a project audit, it is discovered that several approved changes were implemented without updating the project baseline documents. The project is 60% complete. What should the project manager do FIRST?

## Options

- **A**: Continue the project and update documents at the end
- **B**: Perform configuration verification to identify all discrepancies between approved changes and current documentation, then update baselines accordingly
- **C**: Roll back all changes that weren't properly documented
- **D**: Report the team members responsible to senior management

## Answer

B

## Explanation

The correct answer is B. Configuration verification (or audit) compares the documented configuration with the actual product/deliverables. The PM must first understand the full scope of discrepancies before taking corrective action. Once identified, baselines can be properly updated to reflect approved changes.

Option A allows the problem to persist, increasing risk of confusion and errors for the remaining 40% of the project.

Option C is impractical at 60% completion and would cause significant rework. The changes were approved, just not documented properly.

Option D focuses on blame rather than solving the problem. Process improvement should be addressed, but first the immediate documentation gap needs resolution.
