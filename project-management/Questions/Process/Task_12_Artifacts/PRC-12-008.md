# Question

A mid-project audit reveals that several CCB-approved changes were implemented in the deliverables but never reflected in the baseline documents. The project is 60% complete. What should the PM do FIRST?

## Options

- **A**: Continue execution and bring the documentation into line during project closure when the team has more bandwidth.
- **B**: Roll back every change that is not already reflected in the baseline documents and re-implement them through a clean process.
- **C**: Identify and discipline the team members responsible for not updating the baselines so the gap does not recur.
- **D**: Run a configuration verification (audit) to identify every discrepancy between approved changes and current baseline documents, then update the baselines and reconcile the configuration record.

## Answer

D

## Explanation

The PM must first establish the full extent of the gap between the documented configuration and the actual configuration before correcting it. (A) lets the gap grow and increases risk for the remaining 40%. (B) is impractical at 60% complete and discards approved work. (C) addresses behaviour without first fixing the immediate documentation problem; coaching follows once the configuration is restored.
