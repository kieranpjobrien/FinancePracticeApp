# Question

A project team using an agile approach captures decisions and action items during daily standups on a whiteboard that is photographed and shared. Some important decisions from early sprints are now difficult to find in the photo archives. How should the team improve their artifact management?

## Options

- **A**: Stop using the whiteboard and switch to formal documentation
- **B**: Continue the current practice since agile values working software over documentation
- **C**: Transfer key decisions and action items to a searchable digital log while maintaining the whiteboard for daily collaboration
- **D**: Assign someone to memorize all decisions

## Answer

C

## Explanation

The correct answer is C. This preserves the collaborative benefits of the whiteboard for daily use while ensuring important decisions are captured in a searchable format. Agile values working software over comprehensive documentation, but this doesn't mean no documentation - it means right-sized documentation that serves a purpose.

Option A loses the collaborative benefits of the visual, physical tool that works well for daily interaction.

Option B misapplies the agile value - important decisions should be findable. The manifesto says "over" not "instead of."

Option D is impractical and creates a single point of failure.
