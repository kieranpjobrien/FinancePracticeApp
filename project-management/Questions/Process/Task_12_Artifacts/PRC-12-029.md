# Question

A systems integration project uses three contractors, each with their own documentation tools. The contract requires a consolidated project documentation package at go-live. The PM is joining during planning. What should the PM do FIRST?

## Options

- **A**: Build documentation standards, required formats, and review checkpoints into each contractor's statement of work before execution starts.
- **B**: Let each contractor use their native tool and appoint a documentation lead to merge everything before go-live.
- **C**: Mandate that all contractors adopt the prime contractor's documentation tool for the duration of the project.
- **D**: Accept the varying formats and plan a dedicated documentation workstream for the final two sprints.

## Answer

A

## Explanation

Procurement documents are the correct lever for controlling contractor outputs, and the planning phase is the time to use them. (A) sets expectations when they can actually be enforced. (B) treats a contractual issue as a post-hoc integration problem and accepts rework risk. (C) is disruptive, may trigger change requests from contractors, and does not address underlying standards. (D) defers the problem to the end, which is when integration effort explodes.
