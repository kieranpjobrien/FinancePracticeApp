# Question

A project involves multiple contractors who each maintain their own documentation systems. The contract requires all project artifacts to be delivered to the client at project completion. The project manager is concerned about integration and consistency of the final documentation package. What should be done?

## Options

- **A**: Allow contractors to use their own systems and compile everything at the end
- **B**: Require all contractors to use the same documentation system as the prime contractor
- **C**: Establish documentation standards, deliverable formats, and periodic documentation reviews in the contracts and project governance
- **D**: Hire a documentation specialist at project end to organize everything

## Answer

C

## Explanation

The correct answer is C. Contractual requirements and governance processes should establish documentation expectations upfront: required formats, naming conventions, metadata standards, and review checkpoints. This ensures contractors work toward a consistent final package throughout the project, not just at the end.

Option A risks discovering integration issues too late to address them efficiently.

Option B may not be practical if contractors have different established systems and the learning curve would impact delivery.

Option D treats symptoms at the end rather than preventing problems throughout.
