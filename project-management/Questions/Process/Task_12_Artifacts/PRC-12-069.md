# Question

A new data protection officer reviews the project and finds the stakeholder register holds personal data that is no longer required, kept "just in case". The project is still six months from closure. What should the PM do?

## Options

- **A**: Retain the current register content since the project is still running and personal data may prove useful later anyway.
- **B**: Work with the DPO to minimise data held to what is currently required, apply retention rules, and document basis.
- **C**: Delete all personal data elements from the register immediately and rebuild any fields the team specifically asks for later.
- **D**: Transfer the register to the DPO's custody so their team manages personal data while the project manages other artifacts.

## Answer

B

## Explanation

PMI treats compliance and data minimisation as non-negotiable obligations handled with the right control owners. (B) applies the data protection principle of minimisation and keeps a documented basis. (A) retains unnecessary personal data against policy. (C) over-deletes without a structured review and will disrupt legitimate uses. (D) splits custody of one artifact across two owners and creates governance confusion.
