# Question

An agile team is using a digital Kanban board to track work items. A stakeholder requests access to view project progress but is unfamiliar with agile tools. What is the BEST approach for managing this artifact access?

## Options

- **A**: Deny access since the stakeholder doesn't understand the tool
- **B**: Provide read-only access to the Kanban board and offer a brief orientation on how to interpret the information
- **C**: Create separate reports manually for this stakeholder
- **D**: Ask the stakeholder to attend all daily standups instead

## Answer

B

## Explanation

The correct answer is B. Project artifacts should be accessible to appropriate stakeholders. Providing read-only access maintains transparency while preventing unauthorized changes. The brief orientation ensures the stakeholder can meaningfully use the information, supporting their engagement without creating extra reporting overhead.

Option A excludes a legitimate stakeholder from project information they may need.

Option C creates manual overhead and potential for information to become outdated or inconsistent with the actual board.

Option D may not be practical for the stakeholder's schedule and doesn't address their need to access information on demand.
