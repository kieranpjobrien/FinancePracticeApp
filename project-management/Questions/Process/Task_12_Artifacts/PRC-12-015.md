# Question

An agile team maintains a product backlog as a living document that changes frequently. A stakeholder complains that they can never find a stable version to review for planning purposes. How should the product owner address this?

## Options

- **A**: Stop making changes to the backlog
- **B**: Create periodic snapshots of the backlog at key milestones while maintaining the dynamic working version
- **C**: Provide the stakeholder with a different, static document instead
- **D**: Remove the stakeholder's access to reduce confusion

## Answer

B

## Explanation

The correct answer is B. This balances agile's need for a dynamic, evolving backlog with the stakeholder's legitimate need for stable reference points. Periodic snapshots (such as at sprint boundaries or release planning) provide stable versions for planning while the working backlog continues to evolve.

Option A undermines agile's responsiveness to change. The backlog should evolve as understanding grows.

Option C creates duplicate documentation that may diverge from the actual backlog, causing confusion about what's really planned.

Option D excludes a stakeholder who has legitimate planning needs.
