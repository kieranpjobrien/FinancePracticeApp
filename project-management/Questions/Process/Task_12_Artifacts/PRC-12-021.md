# Question

A project has been running for 18 months with inconsistent documentation practices. Different team members have used varying naming conventions, folder structures, and tools. A new executive sponsor requests a comprehensive project history for a board presentation. The project manager estimates it would take two weeks to compile and organize the scattered documentation. What is the BEST course of action?

## Options

- **A**: Tell the sponsor the information isn't available
- **B**: Compile what can be gathered quickly and note gaps, while simultaneously establishing documentation standards for the remainder of the project
- **C**: Delay other project work to spend two weeks organizing historical documents
- **D**: Create new documentation that summarizes the project history based on team members' memories

## Answer

B

## Explanation

The correct answer is B. This pragmatically addresses both the immediate need (providing available information to the sponsor with transparency about gaps) and the underlying problem (establishing standards going forward). Perfect historical records aren't achievable, but the project can improve from this point on.

Option A fails to provide any value to the sponsor when some information is recoverable.

Option C may impact project delivery for historical documentation that has limited ongoing value.

Option D risks inaccuracies from memory-based reconstruction. It's better to acknowledge gaps than fabricate history.
