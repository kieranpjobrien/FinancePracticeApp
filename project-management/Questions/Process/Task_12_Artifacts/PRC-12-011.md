# Question

A project team member accidentally overwrites an important document with an earlier version. The document contained significant updates from the past two weeks. The team uses a basic shared folder without version control. What should the project manager do to prevent this in the future?

## Options

- **A**: Restrict document editing to the project manager only
- **B**: Implement a version control system that maintains document history and allows recovery of previous versions
- **C**: Require team members to keep personal backup copies
- **D**: Send warning emails about being careful with documents

## Answer

B

## Explanation

The correct answer is B. Version control systems automatically maintain document history, enabling recovery of any previous version. This is a systematic solution that prevents the problem regardless of individual mistakes. Modern document management tools make this straightforward to implement.

Option A creates a bottleneck and doesn't scale. The PM shouldn't be the only person who can edit documents.

Option C puts the burden on individuals and creates inconsistency. Personal backups may not be current or accessible when needed.

Option D addresses behavior but doesn't prevent accidents. People will still make mistakes; the system should be resilient to errors.
