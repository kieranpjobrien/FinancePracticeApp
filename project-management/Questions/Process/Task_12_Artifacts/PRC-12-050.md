# Question

Midway through a predictive energy project, the PM notices engineers have been editing the baseline scope document directly to reflect approved changes rather than creating new versions. Traceability of original scope is now unclear. What should the PM do?

## Options

- **A**: Restore the original baseline, reissue versioned change records for each prior amendment, and reinforce version-control rules with engineering.
- **B**: Leave the current document as a clean working reference and simply start versioning the baseline from this point onward afterwards.
- **C**: Ask each engineer to submit a memo describing which edits they made so the history can be reconstructed from memory.
- **D**: Report the finding in the next status report so the steering committee can decide what corrective action to take.

## Answer

A

## Explanation

PMI's configuration management discipline requires the baseline to be immutable and changes to flow through versioned records. (A) restores traceability and prevents recurrence. (B) accepts a permanent loss of the original baseline. (C) relies on fallible memory and produces a weaker record than the version-control system already could. (D) defers action the PM is accountable for taking now.
