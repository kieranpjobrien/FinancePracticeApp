# Question

An issue resolution was implemented and appeared to work. Two weeks later, the same symptoms reappear. What should the project manager ensure happens?

## Options

- **A**: Implement the same fix again
- **B**: Close the issue as a new, separate issue
- **C**: Reopen the original issue, investigate why the fix didn't hold, and ensure the actual root cause is addressed this time
- **D**: Accept that some issues are permanent

## Answer

C

## Explanation

The correct answer is C. Recurring symptoms after a "fix" indicate the root cause wasn't actually addressed. The original issue should be reopened (maintaining history and context), and deeper investigation should identify why the fix failed and what the actual root cause is. This prevents an endless cycle of temporary fixes.

Option A may provide temporary relief but won't solve the underlying problem.

Option B loses context and history from the original investigation.

Option D accepts ineffective resolution without further effort.
