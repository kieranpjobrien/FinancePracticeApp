# Question

What information should be captured in an issue log entry?

## Options

- **A**: Only the issue description
- **B**: Issue description, priority, owner, status, target resolution date, and actual resolution details
- **C**: Only issues that affect the schedule
- **D**: Team member names who caused the issue

## Answer

B

## Explanation

The correct answer is B. A comprehensive issue log captures key information enabling management: what the issue is, how urgent/important, who's responsible for resolution, current status, when it needs to be resolved, and how it was ultimately addressed. This enables tracking, accountability, and learning.

Option A lacks information needed for management and tracking.

Option C arbitrarily excludes issues affecting other dimensions (cost, quality, etc.).

Option D focuses on blame rather than resolution.
