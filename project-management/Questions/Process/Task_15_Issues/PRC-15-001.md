# Question

A critical technical issue is discovered that will delay the project by two weeks if not resolved. The team has identified a workaround that allows development to continue, but the underlying issue remains. What should the project manager do?

## Options

- **A**: Implement the workaround and close the issue
- **B**: Implement the workaround to continue progress while tracking the underlying issue for proper resolution
- **C**: Stop all work until the underlying issue is properly resolved
- **D**: Escalate immediately to the project sponsor

## Answer

B

## Explanation

The correct answer is B. This balances maintaining progress (via workaround) with responsible issue management (tracking for proper resolution). Workarounds are legitimate short-term tactics, but issues they mask should remain open for eventual resolution to prevent accumulating technical debt.

Option A closes the issue prematurely. The underlying problem remains and may cause future problems if not properly tracked and resolved.

Option C is overly cautious when a valid workaround exists. Stopping all work for two weeks is unnecessary if development can continue safely.

Option D is premature - the team has a solution. Escalation should occur when the team cannot resolve issues, not for every problem.
