# Question

A team member identifies an issue but doesn't want it logged because they think it makes their work look bad. How should the project manager respond?

## Options

- **A**: Respect their wish and don't log it
- **B**: Log it anonymously without attribution
- **C**: Explain that issue logging is about project health, not blame, and that identifying issues is valuable - then ensure the issue is properly logged
- **D**: Report the team member for trying to hide problems

## Answer

C

## Explanation

The correct answer is C. This is a cultural issue about how issues are perceived. The PM should address the concern directly: issues should be logged for project success, not to assign blame. People who identify issues early are helping the project. Creating a safe environment for issue reporting improves project health.

Option A allows an issue to go unmanaged.

Option B addresses the symptom but not the underlying culture issue.

Option D punishes issue identification, making the culture worse.
