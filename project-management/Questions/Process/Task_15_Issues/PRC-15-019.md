# Question

An agile team uses a visual impediment board during daily standups. Team members add impediments and remove them when resolved, but there's no formal log or historical record. The project manager wants visibility into patterns. How should this be balanced?

## Options

- **A**: Replace the visual board with a formal issue tracking system
- **B**: Maintain both the visual board for daily team use and a lightweight log for tracking patterns and history
- **C**: The visual board is sufficient for agile teams
- **D**: Have the PM maintain a separate log without the team's knowledge

## Answer

B

## Explanation

The correct answer is B. The visual board serves the team's real-time needs effectively. Adding a lightweight log (even just periodically capturing what's on the board) provides the historical data needed for pattern analysis without undermining the team's agile practices. Both needs can be satisfied.

Option A may slow the team's agile responsiveness.

Option C doesn't provide historical data for pattern analysis.

Option D creates a hidden system that may diverge from reality.
