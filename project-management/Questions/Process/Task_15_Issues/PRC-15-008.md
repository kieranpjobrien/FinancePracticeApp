# Question

Three open issues are competing for the same scarce specialist. Issue A blocks a critical-path deliverable due in seven days. Issue B affects customer satisfaction but has no fixed deadline. Issue C is annoying the team but already has a documented workaround. How should the project manager prioritise?

## Options

- **A**: Address them in the order they were logged so newer issues do not jump the queue.
- **B**: Ask the team to vote on which issue to tackle first to keep them engaged.
- **C**: Escalate all three to the sponsor and let them set the priority order.
- **D**: Address Issue A first because it is high impact and time-critical, evaluate Issue B's customer impact next, and let Issue C's workaround keep it lower in the queue.

## Answer

D

## Explanation

PMI prioritises issues on impact and urgency. (D) applies that lens cleanly: critical-path-and-near-deadline first, customer-impact second, workaround-protected last. (A) ignores priority entirely. (B) hands a project management call to a vote. (C) escalates operational decisions the PM should be making.
