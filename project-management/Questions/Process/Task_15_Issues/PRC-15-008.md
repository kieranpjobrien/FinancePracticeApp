# Question

Multiple issues are competing for the same limited resources. Issue A affects a critical path deliverable due next week. Issue B affects customer satisfaction but has no immediate deadline. Issue C is causing team frustration but has a known workaround. How should the project manager prioritize?

## Options

- **A**: Address issues in the order they were reported
- **B**: Let the team decide which issue to work on
- **C**: Prioritize based on impact and urgency: Issue A first (critical path, time-sensitive), then assess whether Issue B's customer impact warrants attention before Issue C which has a workaround
- **D**: Escalate all issues to the sponsor for prioritization

## Answer

C

## Explanation

The correct answer is C. Issue prioritization should consider impact (how much harm if not addressed) and urgency (how quickly action is needed). Issue A is both high impact (critical path) and high urgency (due next week). Issue B's priority depends on customer impact severity. Issue C has a workaround reducing urgency. This systematic approach allocates resources effectively.

Option A doesn't consider impact or urgency.

Option B abdicates PM responsibility for prioritization.

Option D escalates operational decisions unnecessarily.
