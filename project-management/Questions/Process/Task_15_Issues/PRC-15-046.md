# Question

On a hybrid healthcare platform project, three production defects logged as issues all trace back to the same misunderstood API contract. The Scrum team wants to patch each one individually within the sprint. What should the project manager do first?

## Options

- **A**: Pause the patches, run a root cause analysis on the shared API contract, and address the underlying misunderstanding.
- **B**: Approve all three patches so the sprint commitment remains intact and revisit the contract documentation during the next retrospective.
- **C**: Escalate the recurring defects to the product owner so business stakeholders can decide whether the sprint goal remains realistic.
- **D**: Log a single consolidated issue covering all three defects and allow the team to choose the patching sequence themselves.

## Answer

A

## Explanation

PMI prioritises root cause analysis over symptom patching when a pattern emerges across multiple issues. (A) prevents the fourth, fifth, and sixth defect from appearing by correcting the contract itself. (B) repeats the same rework and violates continuous improvement. (C) prematurely escalates before the PM has analysed the common cause. (D) reorganises the log without solving anything.
