# Question

How often should the issue log be reviewed?

## Options

- **A**: Only when new issues are added
- **B**: Annually
- **C**: Regularly - at least at each status meeting - to monitor progress, update priorities, and ensure nothing falls through the cracks
- **D**: Only at the end of the project

## Answer

C

## Explanation

The correct answer is C. Regular review ensures issues are being actively managed: progress is being made, priorities are current, no issues have stalled or been forgotten. The frequency depends on project pace - fast-moving projects may review daily; others may review weekly. Integration with status meetings ensures consistent attention.

Option A only reviews when adding, not managing existing issues.

Option B is far too infrequent for active issue management.

Option D is far too late - issues should be managed throughout the project.
