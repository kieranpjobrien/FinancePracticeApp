# Question

In the daily standup a developer mentions an issue they hit yesterday and quietly resolved by themselves. You realise nothing about it was added to the issue log. What should the project manager emphasise?

## Options

- **A**: Since the developer resolved it the same day, no log entry is necessary; the issue log is for unresolved problems only.
- **B**: All issues should be logged regardless of resolution status to support visibility, pattern detection, and lessons learned.
- **C**: Verbal reporting in standup satisfies the issue management process for items the team handles internally.
- **D**: Only issues that affect the schedule baseline need to be entered in the formal log.

## Answer

B

## Explanation

The issue log is also a data source: it captures patterns, supports audit, and feeds lessons learned. (B) preserves that. (A) loses every quickly resolved issue, which is exactly the data needed to spot recurring patterns. (C) substitutes ephemeral verbal reporting for the documented record. (D) carves up the log on the wrong axis and drops most entries.
