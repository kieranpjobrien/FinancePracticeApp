# Question

A team member reports an issue verbally during a standup but the project manager notices it was never added to the issue log. When asked, the team member says it's been resolved informally. What should the project manager emphasize?

## Options

- **A**: Since it's resolved, no further action is needed
- **B**: All issues should be logged regardless of resolution status to maintain visibility, track patterns, and capture lessons learned
- **C**: Only major issues need to be logged
- **D**: Verbal reporting is sufficient for issues

## Answer

B

## Explanation

The correct answer is B. Issue logging serves multiple purposes: tracking current issues, identifying patterns (like the recurring issue problem), providing audit trails, and capturing lessons learned. Even quickly resolved issues should be logged to maintain complete records and enable analysis of issue patterns.

Option A loses valuable information about what issues occurred.

Option C makes arbitrary distinctions and loses pattern data.

Option D doesn't provide documentation or enable pattern analysis.
