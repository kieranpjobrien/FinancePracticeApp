# Question

The same category of defect keeps reappearing every two or three sprints. Each occurrence is fixed individually but the pattern continues. What should the project manager do?

## Options

- **A**: Continue fixing each occurrence as it arises, since the defects are getting resolved each time.
- **B**: Run a root cause analysis on the recurring defects, treat the underlying cause as the real issue, and put a permanent fix in place.
- **C**: Make a single team member the long-term owner of this defect class so future occurrences are routed to them automatically.
- **D**: Accept the recurrence as inherent to the system and price it into the team's capacity planning each sprint.

## Answer

B

## Explanation

Recurring defects are a classic root cause signal. (B) is the PMI response: stop firefighting and fix the underlying cause using techniques like the 5 Whys or fishbone analysis. (A) entrenches waste. (C) just routes the symptom more efficiently. (D) makes failure a planning assumption rather than something to remove.
