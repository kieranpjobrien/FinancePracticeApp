# Question

A project manager notices that the same type of issue keeps recurring across multiple sprints. The team addresses each occurrence individually but the root problem persists. What should the project manager do?

## Options

- **A**: Continue fixing each occurrence as it arises
- **B**: Conduct root cause analysis to identify the underlying problem causing the recurring issues and implement a permanent fix
- **C**: Accept that some issues will always recur
- **D**: Assign one team member to be responsible for this type of issue

## Answer

B

## Explanation

The correct answer is B. Recurring issues indicate an underlying root cause that isn't being addressed. Root cause analysis (using techniques like 5 Whys, fishbone diagrams, or Pareto analysis) identifies the source of the problem. Addressing the root cause prevents recurrence rather than repeatedly fixing symptoms.

Option A perpetuates wasted effort on repeated fixes.

Option C accepts inefficiency without attempting improvement.

Option D assigns responsibility but doesn't solve the underlying problem.
