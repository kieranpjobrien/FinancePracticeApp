# Question

A Scrum team delivering a customer portal is told the parent organisation requires a formal change request for any scope modification. The team argues this conflicts with their agile cadence of refining the backlog every sprint. What should the project manager do?

## Options

- **A**: Tailor the governance approach so backlog refinement within the agreed release scope stays with the team, while changes that affect contract scope, budget, or release date go through formal change control.
- **B**: Apply the organisational change control process to every backlog item to ensure consistency with corporate governance standards.
- **C**: Ask the sponsor to formally exempt the team from change control so the Scrum framework can operate as intended by its authors.
- **D**: Have the product owner approve all backlog changes silently and only raise the formal change request if the team is later audited.

## Answer

A

## Explanation

PMI guidance on tailoring is that governance should fit the delivery approach without abandoning organisational accountability. (A) draws the right line: in-sprint refinement is a team activity, but anything that breaches the baseline still needs CCB review. (B) drowns the team in paperwork and defeats the purpose of an agile cadence. (C) is a unilateral exemption request that ignores legitimate enterprise governance needs. (D) hides changes from governance and is an integrity failure.
