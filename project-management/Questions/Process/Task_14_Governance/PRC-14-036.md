# Question

A project involves developing a product that will be used by multiple client organizations, each with their own governance requirements for products they deploy. The project must satisfy internal organizational governance plus varying external client governance requirements. How should this complexity be managed?

## Options

- **A**: Pick one client's governance requirements to follow
- **B**: Follow only internal governance since clients aren't paying yet
- **C**: Map all governance requirements to identify common elements and variations, establish a governance framework that satisfies core common requirements, with documented processes for meeting client-specific requirements during deployment
- **D**: Renegotiate with clients to accept standard governance

## Answer

C

## Explanation

The correct answer is C. With multiple governance requirements, analysis reveals common elements (satisfy once) and variations (require client-specific processes). The framework handles common requirements efficiently while planning for client-specific adaptations. This is more sustainable than trying to satisfy every variation during development.

Option A ignores other clients' legitimate requirements.

Option B ignores requirements that will affect deployment success.

Option D may not be possible if clients have regulatory or organizational mandates.
