# Question

A project governance structure defines that all scope changes must be approved by the CCB. However, the agile team is making small adjustments to user stories during sprints to better meet customer needs. The PMO argues these are scope changes requiring CCB approval. How should this be resolved?

## Options

- **A**: Stop all story refinement and get CCB approval for everything
- **B**: Ignore the PMO and continue current practices
- **C**: Work with the PMO to define the threshold between normal agile refinement (within team authority) and scope changes requiring CCB approval (typically affecting budget, timeline, or contract)
- **D**: Remove the project from PMO oversight

## Answer

C

## Explanation

The correct answer is C. This is a definitional issue about what constitutes a "scope change." Normal agile refinement within the approved scope/budget/timeline is different from changes that affect baselines. Defining the threshold (e.g., refinement within sprint goals vs. adding features beyond the release scope) resolves the conflict while maintaining appropriate governance.

Option A makes agile impossible to practice.

Option B creates ongoing conflict with the PMO.

Option D doesn't resolve the underlying issue and may not be possible.
