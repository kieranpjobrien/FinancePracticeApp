# Question

A vendor delivering a project to a client has its own governance — change control, status format, escalation path — that differs from the client's governance. Both organisations want their model to apply. What should the project manager do?

## Options

- **A**: Adopt the client's governance in full because the client owns the outcome.
- **B**: Adopt the vendor's governance in full because they are doing the work.
- **C**: Run both governance models in parallel so neither party feels overruled.
- **D**: Build an integrated governance framework that maps the two processes where they overlap and defines clear interfaces where they must stay separate.

## Answer

D

## Explanation

PMI expects governance to satisfy both parties' legitimate needs while removing duplication. (D) is the right design — unify what can be unified, name the interfaces for the rest. (A) and (B) ignore the other party's contractual and organisational requirements. (C) creates duplication and contradictory decisions; running parallel governance is exactly the symptom the question describes.
