# Question

A project team consistently delivers what was planned but business value is questioned. Stakeholders feel the delivered features don't address their most important problems. The team is using Scrum with detailed user stories and acceptance criteria. What methodology practice might help?

## Options

- **A**: More detailed acceptance criteria
- **B**: Longer sprints to deliver more complete features
- **C**: Implement practices that ensure continuous focus on value, such as impact mapping, regular value reviews, and outcome-based metrics rather than just output metrics
- **D**: More frequent sprint planning

## Answer

C

## Explanation

The correct answer is C. The gap is between output (features delivered) and outcome (business value). Practices like impact mapping connect features to goals, value reviews assess whether delivered work achieves intended outcomes, and outcome metrics measure actual impact rather than just completion. This shifts focus from "building things right" to "building the right things."

Option A addresses story clarity but not value prioritization.

Option B doesn't address the fundamental value question.

Option D adds ceremony but doesn't address value focus.
