# Question

A programme has four workstreams: data migration (well-defined, sequential), custom development (uncertain, iterative), training development (depends on system features), and change management (continuous). What approach should the programme manager use?

## Options

- **A**: Match each workstream to the approach that fits it (predictive for migration, agile for development, dependency-driven for training, adaptive for change), with shared programme-level governance.
- **B**: Apply a single methodology across all workstreams to keep governance simple and reporting consistent.
- **C**: Sequence the workstreams so each finishes before the next begins, removing the need to coordinate different approaches.
- **D**: Let each workstream lead choose their own methodology and integrate at quarterly programme reviews.

## Answer

A

## Explanation

Programme-level tailoring matches approach to work and uses shared governance to coordinate. (A) does that. (B) forces inappropriate methods on at least some workstreams. (C) wastes elapsed time and breaks dependencies that need parallel work. (D) hands off methodology choice without shared governance, which is exactly how programmes lose alignment.
