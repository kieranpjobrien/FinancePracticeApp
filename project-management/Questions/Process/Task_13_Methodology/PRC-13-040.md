# Question

A project involves multiple work streams: data migration (well-defined, sequential), custom development (uncertain, iterative), training development (dependent on system features), and change management (ongoing, adaptive). What methodology structure makes sense?

## Options

- **A**: Use the same methodology for all work streams
- **B**: Let each work stream choose independently
- **C**: Apply appropriate methodologies to each work stream (predictive for migration, agile for development, parallel for training with development dependencies, adaptive for change management) with integrated governance
- **D**: Sequence everything to avoid complexity

## Answer

C

## Explanation

The correct answer is C. Complex programs often have work streams with different characteristics. Matching methodology to each stream's nature (predictive for known work, agile for uncertain work, etc.) optimizes each stream while integrated governance ensures they align. This requires more sophisticated program management but delivers better outcomes.

Option A forces inappropriate methods on some streams.

Option B risks misalignment between interdependent streams.

Option D extends timelines unnecessarily and doesn't change the fundamental complexity.
