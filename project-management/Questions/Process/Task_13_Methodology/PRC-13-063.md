# Question

An IoT project combines hardware design (long lead time, fixed tooling) with firmware (iterative) and cloud services (cloud-native continuous delivery). The organisation is pushing one methodology for consistency. What should the PM propose?

## Options

- **A**: Use predictive end-to-end to align with hardware lead times, since hardware is the longest pole in the tent.
- **B**: Use Scrum end-to-end for consistency and stretch hardware work across multiple sprints as needed to fit each iteration.
- **C**: Use Kanban end-to-end so each component can flow at its own natural pace without artificial sprint boundaries imposed.
- **D**: Tailor: predictive for hardware, Scrum for firmware, continuous delivery for cloud, with aligned integration milestones.

## Answer

D

## Explanation

PMI's tailoring principle permits different methodologies for different work types under shared integration governance. (D) fits each component. (A) forces iteration-friendly software into rigid gates. (B) applies sprints to hardware tooling that cannot sprint. (C) loses the synchronisation needed at integration points.
