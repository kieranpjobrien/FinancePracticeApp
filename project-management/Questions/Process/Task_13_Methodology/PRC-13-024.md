# Question

A government agency is modernizing a 30-year-old system. The existing system processes millions of transactions daily and must remain operational throughout the transition. Documentation of the current system is limited. Requirements for the new system include both replicating existing functionality and adding new capabilities. What methodology approach is MOST appropriate?

## Options

- **A**: Big bang waterfall replacement
- **B**: Pure agile greenfield development
- **C**: Phased approach with discovery iterations to understand existing functionality, followed by incremental migration with parallel running of old and new systems
- **D**: Complete documentation of the old system before starting development

## Answer

C

## Explanation

The correct answer is C. This complex situation requires multiple elements: discovery work to understand undocumented existing functionality, incremental migration to manage risk, and parallel running to ensure the critical system remains operational. This hybrid approach manages the risks inherent in replacing a mission-critical legacy system.

Option A is extremely high risk for a system processing millions of daily transactions.

Option B doesn't account for the need to replicate existing, poorly-documented functionality.

Option D may never complete - discovering everything about a 30-year-old system is impractical without iterative exploration.
