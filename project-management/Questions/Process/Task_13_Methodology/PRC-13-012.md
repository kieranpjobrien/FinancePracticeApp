# Question

A company is migrating from an old ERP system to a new one. The core data migration and system configuration are well-defined, but the business process changes and user adoption will require iteration based on employee feedback. The project has a firm go-live date due to the old system's license expiration. Which approach is BEST?

## Options

- **A**: Pure waterfall to meet the deadline
- **B**: Pure agile to accommodate learning
- **C**: Hybrid with predictive planning for migration milestones and agile sprints for process refinement and training, all working toward the fixed deadline
- **D**: Two separate projects - one waterfall for migration, one agile for adoption

## Answer

C

## Explanation

The correct answer is C. This recognizes that different project elements have different characteristics. The firm deadline requires predictive planning for critical path items (migration, go-live). The need for iteration based on feedback suits agile for process and training work. Integrating both within a single hybrid approach maintains coordination while applying appropriate methods.

Option A doesn't accommodate the learning needed for adoption.

Option B may not provide the rigor needed to meet a firm deadline for technical migration.

Option D creates coordination challenges and may not be practical.
