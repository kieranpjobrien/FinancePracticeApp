# Question

A project manager notices that their agile team has excellent velocity metrics but stakeholder satisfaction is low. Stakeholders complain that the team delivers features but they're not what was really needed. What methodology aspect needs attention?

## Options

- **A**: Increase velocity further
- **B**: Focus on more detailed requirements documentation
- **C**: Improve stakeholder collaboration and feedback integration - velocity without value delivery suggests a disconnect between output and outcomes
- **D**: Change to waterfall for more control

## Answer

C

## Explanation

The correct answer is C. High velocity with low satisfaction indicates the team is efficiently delivering the wrong things. This is a collaboration and feedback problem - the agile principle of "customer collaboration over contract negotiation." The team needs better mechanisms for understanding stakeholder needs and validating that deliveries actually meet those needs.

Option A optimizes the wrong metric - delivering wrong things faster doesn't help.

Option B may help but doesn't address the fundamental collaboration gap.

Option D doesn't solve the problem of understanding true needs.
