# Question

A startup is building a product for a market that does not yet exist. The investor wants quarterly demonstrations and the founders need to test multiple value propositions quickly. Which delivery approach is MOST suitable?

## Options

- **A**: Lean startup MVP cycles with agile delivery, validating each hypothesis before committing further investment.
- **B**: Waterfall delivery with extensive market research up front so the first release lands fully validated.
- **C**: Scrum with a strict release cadence so the investor sees a working product every sprint.
- **D**: Detailed requirements specification before any development to satisfy investor governance.

## Answer

A

## Explanation

The defining trait is unknown product-market fit, which is what lean startup is built for. (A) tests value before scaling effort. (B) over-invests in research before the market direction is even known. (C) gives a delivery cadence but does not address validation of value. (D) specifies a product no one has confirmed will be wanted.
