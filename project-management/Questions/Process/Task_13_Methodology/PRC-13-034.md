# Question

What is the main characteristic of Kanban that distinguishes it from Scrum?

## Options

- **A**: Kanban has more documentation requirements
- **B**: Kanban uses continuous flow with WIP limits instead of fixed-length iterations
- **C**: Kanban requires more team members
- **D**: Kanban is only for manufacturing

## Answer

B

## Explanation

The correct answer is B. Kanban emphasizes continuous flow rather than timeboxed iterations. Work items move through stages (columns on a Kanban board) as capacity allows, controlled by Work In Progress (WIP) limits rather than sprint commitments. This makes Kanban more flexible for work that can't easily fit into fixed cycles.

Option A is incorrect - Kanban typically has minimal prescribed documentation.

Option C is incorrect - team size isn't a distinguishing factor.

Option D is incorrect - while Kanban originated in manufacturing, it's widely used in software development and other fields.
