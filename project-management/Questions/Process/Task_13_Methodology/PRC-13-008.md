# Question

An organization wants to develop a new mobile application. The core infrastructure and security components are well-defined and must meet specific compliance standards. However, the user-facing features need to evolve based on market feedback. What approach should the project manager recommend?

## Options

- **A**: Pure predictive for the entire project
- **B**: Pure agile for the entire project
- **C**: Hybrid approach using predictive for infrastructure/security and agile for user features
- **D**: Defer methodology selection until requirements are clearer

## Answer

C

## Explanation

The correct answer is C. This project has two distinct parts with different characteristics. Infrastructure and security components have stable requirements and compliance needs (suited to predictive). User-facing features need market feedback and evolution (suited to agile). A hybrid approach applies the right methodology to each component.

Option A ignores the need for flexibility in user features.

Option B doesn't provide the rigor needed for compliance-critical infrastructure.

Option D delays an important decision that affects project planning and team structure.
