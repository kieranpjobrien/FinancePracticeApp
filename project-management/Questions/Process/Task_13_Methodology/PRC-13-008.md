# Question

A new mobile banking app has two distinct workstreams: a core authentication and security platform with regulator-defined controls, and customer-facing features the product team wants to evolve based on weekly user testing. What approach should the PM recommend?

## Options

- **A**: A predictive delivery approach for the entire project so the regulator sees a single plan.
- **B**: Pure Scrum for the entire project, treating the regulator-defined controls as backlog items.
- **C**: A hybrid approach: predictive planning and gating for the security platform, agile iteration for customer-facing features.
- **D**: Begin with predictive planning for both workstreams, and decide whether to switch the customer features to agile after the first release.

## Answer

C

## Explanation

The two parts have genuinely different characteristics, and PMI says match approach to work. (C) does that cleanly. (A) imposes ceremony on customer features that need fast feedback. (B) lets agile cadence drive regulatory deliverables, which usually fails compliance reviews. (D) defers a decision that should be made now and risks rework when the team changes course.
