# Question

A project involves building a mobile app with unclear user requirements but a fixed regulatory compliance deadline in 6 months. The organisation typically uses waterfall. What approach should the project manager recommend?

## Options

- **A**: Pure waterfall with detailed upfront requirements gathering
- **B**: Pure Scrum with 2-week sprints and evolving requirements
- **C**: Hybrid approach with fixed compliance milestones and iterative feature development
- **D**: Kanban with continuous flow and no time-boxing

## Answer

C

## Explanation

The correct answer is C. The project has characteristics requiring both predictability (fixed deadline, compliance) and flexibility (unclear requirements). A hybrid approach allows iterative development to discover requirements while maintaining milestone discipline for compliance activities. This tailors the methodology to project needs.

Option A (pure waterfall) is risky with unclear requirements. Detailed upfront planning may produce specifications that don't match actual user needs, leading to rework.

Option B (pure Scrum) may not provide enough predictability for the fixed deadline. Without milestone planning, the team may not prioritise compliance-critical features appropriately.

Option D (Kanban) provides flexibility but limited predictability. Without time-boxing or milestones, meeting the fixed deadline is harder to manage.
