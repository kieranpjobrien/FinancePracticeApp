# Question

A project involves integrating with legacy systems that have fixed integration windows and strict change control, while also developing new features that need rapid iteration with user feedback. What approach is MOST appropriate?

## Options

- **A**: Use waterfall for the entire project to match the legacy system constraints
- **B**: Use agile for the entire project to enable rapid iteration
- **C**: Use a hybrid approach with predictive planning for integration work and agile iterations for new feature development
- **D**: Divide into two separate projects with different methodologies

## Answer

C

## Explanation

The correct answer is C. Different parts of the project have different characteristics - legacy integration benefits from predictive planning (known requirements, fixed windows), while new features benefit from agile iteration. A hybrid approach matches methodology to work characteristics rather than forcing one approach on everything.

Option A loses the benefits of iteration for new features where it would add value.

Option B may create friction with legacy systems that need predictable schedules and formal change control.

Option D creates unnecessary overhead and integration challenges from having separate projects.
