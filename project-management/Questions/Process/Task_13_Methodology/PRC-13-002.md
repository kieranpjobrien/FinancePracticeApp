# Question

A project must integrate with a legacy ERP that only allows changes during a quarterly maintenance window, while also developing a new customer-facing portal where the business wants weekly user feedback. What is the MOST appropriate approach?

## Options

- **A**: Use Scrum across the entire project so all work moves at the same cadence and rhythm for the team.
- **B**: Run two predictive workstreams aligned to the quarterly ERP windows so all dependencies remain predictable and stable.
- **C**: Use a hybrid approach: predictive planning for the ERP integration, agile iterations for the customer portal work.
- **D**: Split the work into two separate projects with separate PMs, budgets, and governance structures for each stream.

## Answer

C

## Explanation

PMI's tailoring guidance points directly here: methodology should follow the nature of the work. (C) lets each part of the project use what fits. (A) is a poor fit for the ERP work that must respect change windows. (B) loses the rapid feedback cycle the portal needs. (D) introduces governance overhead and dependency-management cost without solving the underlying methodology mismatch.
