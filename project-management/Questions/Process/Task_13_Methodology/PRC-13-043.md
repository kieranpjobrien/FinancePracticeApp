# Question

An organization wants to try agile but is concerned about losing visibility into project progress and costs. They currently rely heavily on Gantt charts and earned value metrics. What approach would help them transition?

## Options

- **A**: Continue using traditional metrics exclusively
- **B**: Abandon all traditional metrics and use only burndown charts
- **C**: Implement agile metrics (burndown, velocity, cycle time) alongside translated traditional views (roadmaps showing planned features, cost tracking aligned with iterations) to maintain visibility during transition
- **D**: Delay agile adoption until comfort with new metrics develops

## Answer

C

## Explanation

The correct answer is C. During transition, maintaining familiar visibility mechanisms while introducing agile metrics helps stakeholders stay informed. Roadmaps can provide timeline visibility, iteration-based cost tracking enables financial oversight, while burndown charts and velocity provide team-level agile metrics. Over time, reliance on traditional metrics may decrease.

Option A doesn't enable the agile transition.

Option B may create a visibility gap that concerns stakeholders.

Option D indefinitely delays beneficial change.
