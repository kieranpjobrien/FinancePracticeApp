# Question

A project started with agile methodology but is now facing challenges: the customer is rarely available for feedback, requirements are actually more stable than anticipated, and stakeholders prefer comprehensive upfront documentation. What should the project manager consider?

## Options

- **A**: Continue with agile since changing methodology mid-project is disruptive
- **B**: Force stakeholders to participate in agile ceremonies
- **C**: Assess whether the initial methodology choice was appropriate and consider transitioning to a more predictive approach that fits the actual project characteristics
- **D**: Document the challenges but continue as planned

## Answer

C

## Explanation

The correct answer is C. Methodology selection should be reconsidered when project characteristics differ significantly from initial assumptions. If requirements are stable, customer availability is limited, and stakeholders prefer upfront documentation, a more predictive approach may actually fit better. The PM should make this assessment objectively.

Option A prioritizes consistency over effectiveness.

Option B attempts to force fit when the methodology may not be appropriate.

Option D acknowledges problems but doesn't address them.
