# Question

A project is using predictive methodology but encountering significant scope creep. Stakeholders keep requesting changes even though requirements were supposedly locked. What might this indicate about methodology fit?

## Options

- **A**: The project needs stricter change control
- **B**: Requirements may not have been as well-understood as assumed, suggesting a more adaptive approach might have been appropriate
- **C**: The stakeholders are being unreasonable
- **D**: The project should be cancelled

## Answer

B

## Explanation

The correct answer is B. Persistent change requests often indicate that requirements weren't truly stable - a key assumption for predictive approaches. When stakeholders continue to request changes despite "locked" requirements, it may indicate genuine uncertainty that an adaptive approach would handle better. The PM should evaluate whether methodology fit is contributing to the challenge.

Option A treats symptoms rather than considering root cause.

Option C blames stakeholders without examining whether their changes reflect legitimate learning.

Option D is premature without understanding the underlying cause.
