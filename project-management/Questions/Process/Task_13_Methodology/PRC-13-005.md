# Question

Which factor is MOST important when selecting a project methodology?

## Options

- **A**: The methodology the project manager is most familiar with
- **B**: The methodology the organisation has used for the longest time
- **C**: The level of uncertainty and need for flexibility in the project
- **D**: The methodology that requires the least documentation

## Answer

C

## Explanation

The correct answer is C. Methodology selection should be driven by project characteristics. Projects with high uncertainty and evolving requirements benefit from adaptive (agile) approaches. Projects with stable, well-understood requirements may suit predictive (waterfall) approaches. The key factor is matching methodology to project context.

Option A (PM familiarity) prioritises convenience over appropriateness. PMs should develop competence in multiple approaches.

Option B (organisational history) is inertia, not analysis. What worked before may not suit this project's specific characteristics.

Option D (least documentation) optimises for the wrong thing. Some projects need more documentation (compliance, safety-critical), others less.
