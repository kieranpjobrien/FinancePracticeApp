# Question

A project must deliver an MVP in six months and continue iterating after launch. The organisation funds projects through quarterly stage gates that release the next tranche of budget. How should the project be structured?

## Options

- **A**: Adopt a predictive plan that aligns each phase to a quarterly gate so funding decisions match the gates exactly.
- **B**: Run agile delivery and request removal of the quarterly stage gates, since they conflict with iterative funding.
- **C**: Run agile delivery underneath, aligning significant increments and demos with the quarterly gates so leadership can make funding calls based on evidence.
- **D**: Run agile until the MVP launches, then convert to predictive delivery to align with the post-launch funding cycle.

## Answer

C

## Explanation

PMI expects governance and delivery method to coexist; agile fits inside stage gates if increments are aligned. (C) gives the funder real evidence at each gate while preserving iterative delivery. (A) hides the iterative reality behind a predictive plan. (B) treats governance as the obstacle rather than designing for it. (D) flips methodologies for no good reason and loses the iterative benefit just when post-launch feedback matters most.
