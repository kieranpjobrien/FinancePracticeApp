# Question

Which characteristic makes a project MOST suitable for an agile approach?

## Options

- **A**: Requirements are fully defined and unlikely to change
- **B**: Requirements are expected to evolve and customer feedback is critical
- **C**: The project has strict regulatory sequencing requirements
- **D**: The technology is mature and well-understood

## Answer

B

## Explanation

The correct answer is B. Agile approaches excel when requirements are uncertain or evolving and when customer/stakeholder feedback is needed to guide development. Iterative delivery allows for learning and adaptation based on feedback.

Option A describes conditions favoring predictive approaches - stable requirements allow detailed upfront planning.

Option C describes sequential dependencies better suited to predictive methods.

Option D describes low uncertainty, which doesn't require agile's adaptive capabilities (though agile can still work).
