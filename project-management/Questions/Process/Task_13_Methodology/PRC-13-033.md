# Question

A project team wants to adopt test-driven development (TDD) and continuous integration (CI) practices from Extreme Programming (XP). They are currently using Scrum. What is the BEST approach?

## Options

- **A**: Abandon Scrum and fully adopt XP
- **B**: Keep using Scrum as the project framework while incorporating XP engineering practices that complement it
- **C**: These practices cannot be mixed
- **D**: Create a completely new methodology combining elements

## Answer

B

## Explanation

The correct answer is B. Scrum and XP are complementary. Scrum provides project/product management framework (sprints, roles, ceremonies), while XP provides engineering practices (TDD, CI, pair programming). Many teams successfully use Scrum's framework with XP's technical practices. This isn't mixing methodologies inappropriately - it's using the strengths of each.

Option A abandons a working framework unnecessarily.

Option C is incorrect - these practices complement each other well.

Option D adds unnecessary complexity when proven combinations exist.
