# Question

A hardware development project involves both physical component manufacturing (with long lead times) and embedded software development (with more flexibility). The components must integrate at specific points. What methodology approach addresses these different rhythms?

## Options

- **A**: Use the hardware timeline for everything since it's the constraint
- **B**: Use agile for everything to maximize flexibility
- **C**: Apply predictive planning for hardware manufacturing milestones while using agile iterations for software, with integration points serving as synchronization gates
- **D**: Develop hardware and software completely separately

## Answer

C

## Explanation

The correct answer is C. Hardware and software have inherently different characteristics in this scenario. Hardware's long lead times and physical constraints suit predictive planning. Software's flexibility enables agile iterations. Integration points serve as gates where both streams must align. This hybrid approach respects the different natures of each discipline.

Option A constrains software unnecessarily when it could be more adaptive.

Option B ignores hardware's physical constraints and long lead times.

Option D ignores the need for integration and risks discovering incompatibilities late.
