# Question

A connected-vehicle programme builds physical sensor units (12-week manufacturing lead times) and the embedded software that runs on them. Hardware and software must integrate at three pre-set test events. What approach should the PM use?

## Options

- **A**: Predictive planning for hardware milestones, agile iterations for the embedded software, with the integration test events serving as synchronisation gates.
- **B**: Apply the hardware schedule across the entire programme so the software cadence matches it exactly.
- **C**: Treat hardware and software as two separate projects so each can be optimised independently.
- **D**: Run the entire programme in two-week sprints, with hardware lead-time risk owned by the supplier.

## Answer

A

## Explanation

PMI's hybrid pattern fits exactly: each workstream uses what suits it, integration points are explicit. (A) gives hardware the predictability it needs and lets software iterate. (B) wastes the software flexibility and slows learning. (C) loses the integration discipline that this product depends on. (D) ignores the physical reality of 12-week lead times — sprints do not change shipping schedules.
