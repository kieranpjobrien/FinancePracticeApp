# Question

Three weeks into post-handover hypercare on a predictive healthcare platform, operations has logged 14 support calls where the documented procedure did not resolve the incident. The PM is still accountable until hypercare ends. What should the PM do?

## Options

- **A**: Close each incident individually by providing a workaround verbally to operations, since hypercare is specifically designed to bridge such transient gaps temporarily.
- **B**: Analyse the 14 incidents for documentation gaps, update the affected procedures, re-verify with operations, and track open gap closures through hypercare exit.
- **C**: Treat each support call as out of scope for hypercare and direct operations to raise change requests against the documentation for normal backlog handling.
- **D**: Request a hypercare extension from the sponsor so the team has more time to resolve the ongoing documentation gaps encountered during operations calls.

## Answer

B

## Explanation

Hypercare exists precisely to find and correct handover gaps before the project withdraws, so PMI expects structured gap closure, not triage. (B) upgrades the documentation where it actually fails. (A) leaves the documented process wrong and relies on hidden knowledge. (C) denies operations the improvement loop hypercare is designed to provide. (D) extends time without fixing the underlying gaps.
