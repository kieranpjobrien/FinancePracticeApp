# Question

A project's lessons learned sessions consistently capture the same types of issues (scope creep, unclear requirements, resource constraints) but subsequent projects continue experiencing the same problems. What is likely failing?

## Options

- **A**: The lessons learned are too detailed
- **B**: The team should stop capturing lessons learned since they're not working
- **C**: Capturing lessons is working, but there's no effective mechanism to apply them - the organization needs better processes for incorporating lessons into future project planning
- **D**: The wrong people are attending lessons learned sessions

## Answer

C

## Explanation

The correct answer is C. This is a classic organizational learning failure: lessons are captured but not applied. The problem isn't the capture process - it's the application process. Organizations need mechanisms to ensure lessons are reviewed during project initiation and planning, and that systemic issues are addressed organizationally.

Option A doesn't explain why the same issues recur.

Option B abandons the practice rather than fixing the implementation gap.

Option D may contribute but doesn't explain why known lessons aren't applied.
