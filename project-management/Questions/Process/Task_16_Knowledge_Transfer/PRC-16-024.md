# Question

An organization has a lessons learned repository with thousands of entries from past projects. Project managers rarely use it because finding relevant lessons is difficult and many entries are outdated or poorly written. How should this be improved?

## Options

- **A**: Delete the repository and start fresh
- **B**: Add more lessons without changing the process
- **C**: Improve the repository with better categorization, search, quality standards for new entries, curation to remove or update outdated content, and integration into project initiation processes so reviewing relevant lessons becomes standard practice
- **D**: Make using the repository mandatory

## Answer

C

## Explanation

The correct answer is C. A knowledge repository's value depends on findability, quality, and usage. Improvements should address all three: better categorization and search for findability, quality standards and curation for content quality, and process integration for usage. Making it a natural part of project initiation increases adoption.

Option A loses potentially valuable historical knowledge.

Option B worsens the existing problems.

Option D mandates usage without making it useful.
