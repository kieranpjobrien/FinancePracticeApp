# Question

The PMO maintains a lessons-learned repository with thousands of entries from past projects. Project managers rarely consult it because content is hard to find, often outdated, and inconsistent in quality. What should be done?

## Options

- **A**: Improve categorisation and search, set quality standards, curate stale content, and embed a lessons-review step into project initiation.
- **B**: Make consulting the repository a mandatory project management policy with audit checks on every project to enforce compliance with it.
- **C**: Archive the existing entries and start a new repository with stricter content rules from the outset to raise the baseline quality.
- **D**: Encourage every PM to add more entries from their current projects to refresh the content base and dilute older, outdated entries.

## Answer

A

## Explanation

(A) addresses findability, quality, and adoption together, which is what the repository actually needs. (B) mandates use of a tool that is not yet useful. (C) discards real lessons because the system around them is broken. (D) layers more content onto an already-broken structure.
