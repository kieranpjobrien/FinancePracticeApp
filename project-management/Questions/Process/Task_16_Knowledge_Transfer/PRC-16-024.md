# Question

The PMO maintains a lessons-learned repository with thousands of entries from past projects. Project managers rarely consult it because content is hard to find, often outdated, and inconsistent in quality. What should be done?

## Options

- **A**: Improve the repository with better categorisation and search, set quality standards for new entries, curate or retire stale content, and embed a "review relevant lessons" step into the project initiation process.
- **B**: Make consulting the repository a mandatory project management policy with audit checks for compliance.
- **C**: Archive the existing entries and start a new repository with stricter rules from the outset.
- **D**: Encourage every PM to add more entries from their current projects to refresh the content base.

## Answer

A

## Explanation

(A) addresses findability, quality, and adoption together, which is what the repository actually needs. (B) mandates use of a tool that is not yet useful. (C) discards real lessons because the system around them is broken. (D) layers more content onto an already-broken structure.
