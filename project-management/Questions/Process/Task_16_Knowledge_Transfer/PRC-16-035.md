# Question

A project has multiple workstreams with specialized knowledge in each. If one workstream lead is unavailable, work on that stream stops because no one else understands it. What knowledge transfer approach would address this single-point-of-knowledge risk?

## Options

- **A**: Accept the risk since specialization is efficient
- **B**: Have each lead create documentation
- **C**: Implement cross-training where each workstream has at least one backup person with sufficient knowledge to continue work, supplemented by documentation and regular knowledge-sharing between streams
- **D**: Combine all workstreams into one team

## Answer

C

## Explanation

The correct answer is C. Single points of knowledge are a risk. Cross-training develops backup capability for each specialty. The backup doesn't need expert-level knowledge, just enough to continue work or triage issues. Documentation supports this, and regular knowledge sharing maintains the cross-training over time.

Option A accepts significant continuity risk.

Option B helps but documentation alone may not enable someone to continue work.

Option D loses specialization benefits and may not be practical.
