# Question

From a knowledge transfer perspective, what is the primary benefit of pair programming?

## Options

- **A**: It guarantees a measurable productivity improvement by doubling effective output per developer hour.
- **B**: It eliminates the need for separate code documentation since the second developer knows the code.
- **C**: It enables real-time sharing of techniques, context, and design reasoning between two developers as they work.
- **D**: It is most useful as a way to onboard junior developers under direct supervision.

## Answer

C

## Explanation

Pair programming's main knowledge benefit is the in-the-moment exchange of tacit understanding. (C) captures that. (A) misstates the productivity claim and is not the knowledge-transfer benefit. (B) overstates by claiming documentation becomes unnecessary. (D) understates the practice; experienced engineers also learn from each other.
