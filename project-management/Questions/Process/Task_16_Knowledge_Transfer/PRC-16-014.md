# Question

What is pair programming's benefit from a knowledge transfer perspective?

## Options

- **A**: It produces code twice as fast
- **B**: It enables real-time knowledge transfer as developers work together, sharing techniques, context, and understanding
- **C**: It eliminates the need for documentation
- **D**: It is only useful for training junior developers

## Answer

B

## Explanation

The correct answer is B. Pair programming creates continuous knowledge transfer as two developers share their thinking, techniques, and understanding in real-time. This transfers tacit knowledge (how to approach problems, coding patterns, system context) that's difficult to document. Both participants learn from each other.

Option A mischaracterizes pair programming's goal - it's about quality and knowledge, not speed.

Option C is incorrect - documentation still has value.

Option D understates the benefit - experienced developers also learn from each other.
