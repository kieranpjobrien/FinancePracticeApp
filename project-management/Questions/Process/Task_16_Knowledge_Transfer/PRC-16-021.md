# Question

The project's technical documentation is detailed, but engineers say it takes them too long to find what they need. What approach best improves knowledge accessibility?

## Options

- **A**: Add training sessions on speed-reading and navigation patterns so engineers can move through documentation faster.
- **B**: Improve information architecture: clear taxonomy, search, navigation, and short overviews that link out to detailed sections.
- **C**: Significantly reduce documentation depth so the remaining material is small enough to read end-to-end.
- **D**: Add additional how-to documentation to cover any task that engineers reportedly cannot quickly find existing content for.

## Answer

B

## Explanation

The problem is findability, which is solved with structure. (B) is the right intervention. (A) addresses the user rather than the artefact. (C) loses information to make the body smaller without making it more navigable. (D) adds more content to a system that already has a navigation problem.
