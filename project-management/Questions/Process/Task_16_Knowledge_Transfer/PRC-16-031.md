# Question

A team uses a wiki for project documentation. Over time, different pages have conflicting information because updates weren't applied consistently. New team members don't know which information to trust. How should this be addressed?

## Options

- **A**: Delete the wiki and start over
- **B**: Add more content to clarify
- **C**: Conduct a wiki cleanup to identify and resolve conflicts, establish ownership for key pages, implement review processes for updates, and mark pages with "last verified" dates
- **D**: Tell people to use the most recent page

## Answer

C

## Explanation

The correct answer is C. Inconsistent knowledge bases erode trust. A systematic cleanup resolves existing conflicts. Page ownership ensures someone is accountable for accuracy. Review processes prevent future conflicts. Verification dates indicate currency. Together, these restore trust in the knowledge base.

Option A loses potentially valuable content and doesn't prevent recurrence.

Option B adds more content without resolving contradictions.

Option D doesn't resolve contradictions and assumes recency equals accuracy.
