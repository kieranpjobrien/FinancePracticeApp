# Question

A project created extensive runbooks for operational procedures. Six months after deployment, the operations team reports the runbooks are outdated because changes were made that weren't reflected in documentation. What process improvement is needed?

## Options

- **A**: Recreate the runbooks from scratch
- **B**: Accept that documentation becomes outdated
- **C**: Establish a process where documentation updates are part of the change management process - no change is complete until documentation is updated
- **D**: Have operations update documentation when they notice issues

## Answer

C

## Explanation

The correct answer is C. Documentation decay is prevented by making documentation updates part of the change process itself. If changes can't be deployed without documentation updates (similar to how code can't be deployed without passing tests), documentation stays current. This is a process change, not just a documentation task.

Option A fixes the symptom but doesn't prevent recurrence.

Option B accepts declining value without addressing root cause.

Option D is reactive rather than preventive.
