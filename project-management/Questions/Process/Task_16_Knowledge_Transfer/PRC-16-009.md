# Question

A project manager wants to capture lessons learned throughout the project rather than just at the end. How should this be implemented?

## Options

- **A**: Wait until the project end for a comprehensive review
- **B**: Hold a brief lessons learned session after each major milestone or phase, capturing insights while they're fresh and making them available to the ongoing project
- **C**: Assign one person to observe and document lessons
- **D**: Only capture negative lessons (things that went wrong)

## Answer

B

## Explanation

The correct answer is B. Capturing lessons learned throughout the project (at milestones, phase gates, or sprint retrospectives) yields better quality insights - memory is fresh and details are accurate. These lessons can also benefit the current project if captured early enough. This continuous approach is superior to relying on end-of-project memory.

Option A loses detail and the ability to improve the current project.

Option C limits perspective to one observer's viewpoint.

Option D misses valuable positive lessons (what worked well).
