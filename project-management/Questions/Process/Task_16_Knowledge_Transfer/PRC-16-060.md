# Question

A Scrum team delivering an enterprise collaboration platform ships features each sprint, but operations consistently reports that handover artefacts for each increment are thin. Operations is losing trust. What should the PM do?

## Options

- **A**: Create a separate documentation team that catches up after each sprint to supplement the development team's transfer artefacts before operations accepts.
- **B**: Instruct the Scrum Master to add documentation tasks to each sprint backlog as separate items, prioritised after functional backlog items complete.
- **C**: Work with the product owner and team to extend the Definition of Done to include handover artefacts so each increment is transfer-ready.
- **D**: Keep shipping features at current pace and batch a dedicated documentation sprint every quarter, so operations receives a periodic consolidated artefact update.

## Answer

C

## Explanation

Scrum makes transfer quality sustainable by embedding it in the Definition of Done, so every increment is ready for its recipient. (C) solves the problem at source and respects Scrum accountabilities. (A) creates a separate team working from secondhand context. (B) leaves documentation optional behind features. (D) accepts quarterly gaps where operations has no fresh artefacts.
