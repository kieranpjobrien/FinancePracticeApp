# Question

A team member has developed a useful automation script that significantly improves a common task. They've shared it with colleagues informally, but it's not documented or officially maintained. What should the project manager facilitate?

## Options

- **A**: Let it continue as informal sharing
- **B**: Tell the team member to stop using unofficial tools
- **C**: Formalize the tool by documenting it, adding it to the team's standard toolset, establishing maintenance ownership, and sharing it more broadly in the organization
- **D**: Let IT take over the tool completely

## Answer

C

## Explanation

The correct answer is C. Useful grassroots innovations should be captured and formalized. Documentation ensures the tool can be understood and maintained. Adding to the standard toolset legitimizes it. Ownership ensures ongoing maintenance. Broader sharing spreads the benefit. This turns an informal improvement into organizational knowledge.

Option A risks losing the tool if the creator leaves or the script breaks.

Option B loses a valuable improvement.

Option D may separate the tool from its context and creators unnecessarily.
