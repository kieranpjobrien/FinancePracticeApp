# Question

A project involved learning from several failures before finding approaches that worked. The team is reluctant to document the failures in detail because they're embarrassed. However, these failures contain valuable lessons for avoiding the same mistakes. How should the project manager approach this?

## Options

- **A**: Don't document the failures to protect the team
- **B**: Require detailed documentation of every mistake
- **C**: Reframe failure documentation as "learning documentation" focused on what was discovered rather than who failed, capture the insights in ways that help others avoid the same path, and emphasize the value of hard-won knowledge
- **D**: Document failures anonymously without team involvement

## Answer

C

## Explanation

The correct answer is C. The framing matters significantly. "What we learned" is more acceptable than "what we failed at." Focus on insights (don't try approach X because Y happens) rather than narrative of failure. Emphasize the value - these lessons were expensive to learn and can save others the same cost. This captures the knowledge while respecting the team.

Option A loses valuable hard-won knowledge.

Option B may create defensiveness and poor-quality documentation.

Option D loses context the team could provide and may not capture full learnings.
