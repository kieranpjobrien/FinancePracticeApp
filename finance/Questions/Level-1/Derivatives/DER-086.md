# Question

A recombining binomial tree is characterized by:

## Options

- **A**: Different up and down factors in each period
- **B**: An up move followed by down returns to the same price as down then up
- **C**: Prices that never return to previous levels

## Answer

B

## Explanation

Recombining tree structure:

**Recombining property:**
- ud = du (path independent)
- Order of moves doesn't matter
- Reduces computational complexity

**Standard assumption:**
- u × d = 1
- Example: u = 1.1, d = 0.909
- Up then down = Down then up = Same price

**Node calculation:**
- Period 0: S
- Period 1: Su, Sd
- Period 2: Suu, Sud (=Sdu), Sdd
- Only 3 nodes in period 2, not 4

**Benefits:**
- Fewer nodes to calculate
- More efficient pricing
- Still captures price dynamics

**Example:**
- S = $100, u = 1.1, d = 0.909
- Suu = $121
- Sud = Sdu = $100
- Sdd = $82.64

**Common mistakes:**
- Option A: Standard model uses constant factors
- Option C: Recombining means prices can repeat
