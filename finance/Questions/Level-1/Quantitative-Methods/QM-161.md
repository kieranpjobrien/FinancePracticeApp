# Question

Expected Shortfall (Conditional VaR) differs from VaR because it:

## Options

- **A**: Only considers the best-case scenarios
- **B**: Measures the average loss in the tail beyond the VaR level
- **C**: Uses a fixed percentage threshold regardless of confidence level

## Answer

B

## Explanation

Expected Shortfall (ES), also called Conditional VaR (CVaR), measures the expected loss given that the loss exceeds the VaR threshold. While VaR only tells us the minimum loss in the worst x% of cases, ES tells us the average loss in those worst cases. ES is a coherent risk measure and better captures tail risk, which is why Basel III requires banks to use ES rather than VaR.

**Common mistakes:**
- Option A is the opposite; ES focuses on worst-case (tail) scenarios
- Option C confuses the calculation; both measures use a specified confidence level
