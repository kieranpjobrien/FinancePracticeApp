# Question

Look-ahead bias in backtesting a trading strategy occurs when:

## Options

- **A**: Historical data is too short for meaningful analysis
- **B**: The strategy uses information that was not available at the time the trade would have been made
- **C**: Trading costs are excluded from the performance calculation

## Answer

B

## Explanation

Look-ahead bias occurs when a backtest inadvertently uses information that would not have been known at the time of the trading decision. Examples include using restated financial data (before restatement was published), using point-in-time index constituents (before additions/deletions), or using economic data before its release date. This bias inflates backtested performance and leads to disappointment in live trading.

**Common mistakes:**
- Option A describes a limitation related to sample size, not look-ahead bias
- Option C describes a different issue related to realistic performance estimation
