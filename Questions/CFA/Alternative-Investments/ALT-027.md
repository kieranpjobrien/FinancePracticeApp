# Question

Backfill bias occurs in alternative investment databases when:

## Options

- **A**: Funds report returns late each quarter
- **B**: Funds add historical track records when first entering the database
- **C**: Databases fail to update NAV calculations

## Answer

B

## Explanation

Backfill bias explained:

**What happens:**
- Fund decides to join database
- Adds prior years' performance
- Only successful funds tend to join
- Historical returns inflate averages

**Mechanism:**
- New fund has 3-year track record
- Strong performance → joins database
- Weak performance → stays private
- Database only gets winners' histories

**Combined with survivorship bias:**
- Winners enter with good history
- Losers never enter OR drop out
- Double inflation effect
- Database overstates returns

**Magnitude:**
- Estimated 1-3% annual overstatement
- Combined biases: 4-6%+ overstatement
- Significant for investment decisions

**Mitigation:**
- Use only post-entry returns
- Require immediate reporting
- Track database additions

**Common mistakes:**
- Option A: Late reporting is different issue
- Option C: NAV calculation errors are separate
