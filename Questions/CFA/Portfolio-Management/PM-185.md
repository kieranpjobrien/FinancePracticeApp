# Question

VaR backtesting involves:

## Options

- **A**: Forecasting future VaR based on historical patterns
- **B**: Comparing realized losses to VaR estimates to assess model accuracy
- **C**: Increasing VaR limits after periods of low volatility

## Answer

B

## Explanation

VaR backtesting:

**Purpose:**
- Validate VaR model accuracy
- Check if exceptions match confidence level
- Identify model weaknesses

**Process:**
- Compare daily P&L to VaR estimate
- Count "exceptions" (losses exceeding VaR)
- Compare exception rate to expected

**Expected exceptions:**
- 95% VaR: Expect 5% exceptions (~13 per year)
- 99% VaR: Expect 1% exceptions (~2.5 per year)

**Traffic light approach (Basel):**
- Green: Exceptions within acceptable range
- Yellow: Some concern, possible issues
- Red: Too many exceptions, model rejected

**Actions if failing:**
- Revise model assumptions
- Update parameters
- Consider alternative approaches

**Common mistakes:**
- Option A: Backtesting is retrospective, not forecasting
- Option C: This describes procyclical behavior, not backtesting
