# Question

On a hybrid government grants-portal project, the project manager wants high confidence that the development team, the policy authors, and the eventual case workers all genuinely share the same understanding of how an application will be assessed, before build begins. Past projects have shown that all three groups can sit through the same briefing, sign off, and still go on to build, write, and process applications differently. Which two techniques best achieve the project manager's aim? (Choose two.)

## Options

- **A**: Walk all three groups through realistic worked examples of applications end to end and confirm they agree on each outcome
- **B**: Build a low-fidelity prototype of the assessment flow and review it jointly so differing interpretations become visible
- **C**: Have the policy authors run a briefing session presenting the assessment rules to the developers and case workers
- **D**: Compile a shared glossary of the assessment terms and circulate it to all three groups for reference
- **E**: Hold a workshop where each group lists its open questions about the rules and the policy authors answer them

## Answer

A, B

## Explanation

Confirming shared understanding requires actively surfacing where the three groups interpret the same rules differently, which worked examples (A) and a jointly reviewed prototype (B) do by making abstract rules tangible and forcing each group to react to a concrete case where mismatches show up. Option C is a clear presentation, but it transmits the rules one way; the audience can absorb them and still hold divergent readings that are never tested. Option D builds useful shared vocabulary, yet agreeing on term definitions does not guarantee the groups apply them the same way to a real application. Option E is genuinely helpful and surfaces known uncertainties, but it only addresses questions the groups already know to ask, missing the silent misinterpretations each group does not realise it holds. C, D, and E all assist, but only A and B expose hidden mismatches before build.
