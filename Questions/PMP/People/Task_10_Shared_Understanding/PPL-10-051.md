# Question

A fully remote team across four time zones relies heavily on written async communication. Misunderstandings have surfaced repeatedly in retrospectives, mostly from terse chat messages that were read with different tones than intended. What should the PM do?

## Options

- **A**: Move all substantive discussions to scheduled video meetings so tone and intent are conveyed directly, and keep chat only for brief operational coordination.
- **B**: Establish shared communication norms — when to use which channel, tone expectations, async-response etiquette, and confirmation patterns — and revisit them regularly as a team.
- **C**: Ask each team member to add more context to every chat message and require a read-acknowledgement before the sender assumes the message has been received fully.
- **D**: Appoint a communications lead in each time zone to review outgoing chat messages for tone before they are sent to anyone outside that particular zone.

## Answer

B

## Explanation

Shared understanding in a remote context is a set of explicit norms the team writes and maintains together; vague niceness rules and lone fixes don't hold up at scale. (A) forces synchronous meetings across four time zones, which breaks sustainable pace and assumes sync is always better. (C) adds burden per message without building a shared pattern. (D) introduces a gatekeeper that slows communication and does not teach the team anything durable.
