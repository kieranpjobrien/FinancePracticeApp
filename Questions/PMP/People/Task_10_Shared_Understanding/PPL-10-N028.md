# Question

On a hybrid insurance-claims modernisation programme, business analysts describe requirements in customer-journey terms while engineers describe them in API and data-model terms. In review meetings each group nods along, but later each delivers or signs off on something the other did not expect. The project manager concludes that both sides believe they understand each other when in fact they do not. What is the best action?

## Options

- **A**: Adopt a single requirements template both groups must use so requirements are expressed consistently across the programme
- **B**: Appoint a business-systems analyst to sit with both groups and translate each side's requirements into the other's language
- **C**: Have each group document its interpretation of the disputed requirements and circulate both for the other to review and comment on
- **D**: Facilitate joint sessions where the groups turn requirements into shared concrete examples and a common glossary, then confirm by each playing back the other's understanding

## Answer

D

## Explanation

The failure is illusory agreement: each group uses its own language and assumes a mutual understanding that does not exist, so the fix must force the hidden mismatch into the open and confirm real convergence — which shared examples, a common glossary, and explicit playback achieve in the room. Option B is the strongest distractor and a common real-world move, but it makes one person the conduit and single point of failure; understanding is not built directly between the groups, so the nodding resumes whenever the translator is absent. Option A standardises the format of requirements but not their meaning; two groups can complete the same template and still picture different things. Option C surfaces interpretations in writing, which helps, but it is asynchronous and passive, so each side can skim the other's document and re-confirm a false agreement without the live reconciliation that exposes gaps. All four are legitimate; only D reliably reveals and closes the comprehension gap.
