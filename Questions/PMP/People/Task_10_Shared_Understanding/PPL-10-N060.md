# Question

During planning for a regional rollout of a new point-of-sale system, the project manager listed key assumptions and emailed them to the operations, IT and retail stakeholders, asking for objections. None objected. Weeks later it emerged that retail and IT had each silently assumed a different answer to who would train store staff — an assumption the project manager had never listed. What does this most strongly indicate the project manager should do differently?

## Options

- **A**: Facilitate a workshop where stakeholders surface and examine their own and each other's unstated assumptions together, rather than only reacting to a pre-written list
- **B**: Map the cross-team responsibility boundaries (e.g. a RACI) with the stakeholders so handoffs like training have a clearly agreed owner
- **C**: Walk through the assumption list live with all stakeholders in one meeting and have each confirm their understanding aloud
- **D**: For each assumption, identify which stakeholder owns it and have that owner validate it with the others

## Answer

A

## Explanation

The damaging assumption was one nobody wrote down because each party silently held a different version of it, so the fundamental lesson is that reviewing any pre-authored artefact — however it is confirmed — cannot catch what no one thought to state. Facilitating stakeholders to surface their own and each other's hidden assumptions together (A) is built to expose those unspoken gaps and is the most direct corrective. Mapping responsibility boundaries (B) would in fact have caught this particular training-ownership gap and is a strong, concrete practice, but it addresses one *category* of assumption (who-does-what) and would still miss unstated assumptions of other kinds; it treats a symptom class rather than the underlying habit of relying on an incomplete list. Walking the list through live (C) and assigning owners to validate each item (D) both improve how the existing list is confirmed, yet they remain anchored to assumptions someone already articulated — the very limitation that failed here — so neither reliably surfaces the assumptions no one has voiced.
