# Question

What is a user story mapping session primarily designed to achieve?

## Options

- **A**: Build a shared visual model of the user journey and how features fit together to deliver value.
- **B**: Convert business requirements into formally estimated story-point values for the release backlog.
- **C**: Decide which developers will own which stories across an upcoming release of the product.
- **D**: Produce a detailed technical design for the system that the team will then implement and test.

## Answer

A

## Explanation

User story mapping arranges stories along a user journey so the team and stakeholders can see how individual stories combine into end-to-end value and identify slices for early releases. (B) describes estimation, which is a separate activity (planning poker, t-shirt sizing, etc.). (C) describes work assignment, which Scrum teams self-organise around at the sprint level. (D) describes technical design, which is downstream of and informed by the story map but not its purpose.
