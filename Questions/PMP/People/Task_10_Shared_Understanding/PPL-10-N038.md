# Question

A predictive project to deliver a custom laboratory information system relies on an external vendor. The vendor's first deliverable technically conforms to the written specification but does not do what the lab managers actually needed, because the spec was open to interpretation. The next deliverable, a sample-tracking module, is about to begin against spec language that is similarly open. As the project manager, what is the best first step to keep it from repeating the gap between the words and what the lab managers need?

## Options

- **A**: Run a joint session where the lab managers and the vendor walk through concrete end-to-end scenarios for the sample-tracking module and confirm the same understanding of intent before the vendor builds
- **B**: Have the vendor produce a short design walkthrough or mock-up of how they intend to build the module, and have the lab managers review it before development starts
- **C**: Hold a requirements workshop with the lab managers to capture detailed, example-based acceptance criteria for the module, then issue those to the vendor as the basis for the work
- **D**: Add a formal acceptance test, defined jointly with the lab managers, that the module must pass before it is signed off

## Answer

A

## Explanation

The deliverable met the words but not the intent because the lab managers and the vendor never built a shared understanding of what was needed, so the strongest first step puts both parties in the same room to reconcile intent on the upcoming module before any build, using concrete scenarios that expose hidden assumptions on both sides at once. Option B is genuinely useful and closely related, but a vendor-produced walkthrough surfaces only the vendor's interpretation for the client to react to; it is one-sided and can still miss needs the lab managers never thought to mention, whereas a joint scenario walk builds the understanding bilaterally and earlier. Option C improves the written basis and is good practice, but criteria captured from the lab managers alone and then handed over reintroduce the same one-way transmission that caused the original gap, since the vendor's reading of those criteria is never confirmed. Option D defines a clear bar but verifies intent only after the module is built, reducing surprise without preventing the wasted effort that upfront alignment avoids. All four are legitimate steps a competent project manager might take; A most directly builds the shared understanding of intent at the point it matters.
