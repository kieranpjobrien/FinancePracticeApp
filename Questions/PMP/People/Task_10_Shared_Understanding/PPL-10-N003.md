# Question

A project manager is starting a predictive infrastructure project with a team drawn from four departments that have never worked together. In the first week, members keep proposing conflicting technical approaches because each assumes a different end goal for the system. The approved charter and a written objectives statement already exist and have been circulated. What should the project manager do first to establish a common foundation?

## Options

- **A**: Run a chartering or context-setting workshop that has the team co-author a fresh shared statement of the goal and guiding principles
- **B**: Have the team build a high-level solution architecture together so the differing end-goal assumptions become visible in the design
- **C**: Facilitate a working session where the team walks through the charter's objectives and success criteria and each member restates them in their own words, surfacing where their assumptions diverge
- **D**: Map each department's stake and expected outcome so the conflicting goals each member is optimising for are made explicit

## Answer

C

## Explanation

Every option builds shared understanding, so the test is sequencing — what gives the firmest foundation first. An authoritative charter and objectives already exist, so the immediate gap is not missing direction but unverified comprehension of it; walking through the existing objectives and having members restate them (C) directly tests and corrects each person's mental model against the agreed source, and exposes exactly where assumptions split. Co-authoring a fresh goal statement (A) is valuable but risks re-litigating decisions the charter has already settled and is better suited to when no agreed goal exists. Building the architecture together (B) and stakeholder/outcome mapping (D) are useful follow-ons that will land far better once the team shares a confirmed understanding of the goal — done first, they channel the same unaligned assumptions into a design argument or a stakeholder map. C establishes the baseline the others depend on.
