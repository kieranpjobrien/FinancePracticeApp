# Question

A project manager runs an agile team delivering a media-streaming feature set. The five developers are each split 50% across this product and two other projects, and the resource manager pulls them onto urgent work elsewhere without notice. Velocity swings wildly, context-switching is high, and the team says it can never settle into a rhythm. The work is ongoing and strategically important, but the other two projects are also live and the resource manager guards the shared model because it spreads scarce specialists. The project manager is preparing a case to take to the resource manager. What should the project manager advocate for?

## Options

- **A**: Securing the five developers as a dedicated, stable team allocated full-time to this product so they can settle into a sustainable pace.
- **B**: Agreeing a protected core of two or three developers dedicated to this product while the rest stay shared, balancing stability against the demand on scarce specialists.
- **C**: Agreeing a fixed reassignment protocol — a notice period and a per-sprint cap on pull-offs — so the team keeps its members within each sprint even if shared overall.
- **D**: Negotiating a longer sprint length so the disruption from reassignments is absorbed within fewer, larger planning cycles.

## Answer

A

## Explanation

The instability is caused by fragmented, fluctuating allocation, and an agile team performs best when it is dedicated and stable; advocating for the whole team to be allocated full-time removes the root cause rather than cushioning it, which the work's strategic importance justifies (A). A protected core is a reasonable compromise, but it leaves part of the team still churning and the product only partly stabilised — a fallback if full dedication is refused, not the position to argue for first when the work warrants a stable team (B). A notice period and pull-off cap reduce the surprise and protect each sprint, but the developers are still divided across three products, so the context-switching and the inability to find a rhythm largely persist (C). Lengthening sprints merely changes the cadence over which the same disruption lands; it hides the variability inside bigger cycles instead of stabilising the team (D). The structural fix the situation calls for is a dedicated, stable team.
