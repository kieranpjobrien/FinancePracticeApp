# Question

A skills assessment on a hybrid automotive-software project reveals gaps between the team's current competencies and what the next two releases demand. The project manager wants to close the gaps responsibly while keeping delivery moving. Which THREE actions best build the team's capability for the work ahead? (Select THREE.)

## Options

- **A**: Create individual development plans that pair targeted learning with upcoming work that exercises the new skills.
- **B**: Use real backlog items as deliberate stretch assignments with mentoring, so skills grow through doing the work.
- **C**: Engage a specialist for the first affected release with an explicit brief to coach team members while delivering.
- **D**: Send the team on an intensive external certification course in the new skills before the affected releases begin.
- **E**: Prioritise the gapped skills onto your most capable engineers so the affected releases land safely, and develop the others later.

## Answer

A, B, C

## Explanation

Capability builds best when development is anchored to the actual work: development plans tied to upcoming tasks (A), mentored stretch assignments on real backlog items (B), and a specialist who coaches while covering the immediate need (C) all grow the team where the skills will be applied and as the work happens. (D) front-loaded external training is not worthless, but knowledge learned in a classroom away from the project's context decays before it is used and pulls the team off delivery; it is weaker than learning embedded in the work. (E) concentrating the gapped skills on the strongest engineers may protect the next releases, but it deepens a key-person dependency and explicitly postpones developing everyone else — the opposite of building team-wide capability.
