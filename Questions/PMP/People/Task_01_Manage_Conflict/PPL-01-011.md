# Question

There is rising tension between developers and the business analyst. Developers say requirements are unclear; the BA says developers don't read what's written. What should the project manager do?

## Options

- **A**: Have the BA produce more detailed requirement templates to remove ambiguity.
- **B**: Add a technical lead as a reviewer between the BA and the developers.
- **C**: Run joint requirement-refinement sessions where developers and the BA work through stories together.
- **D**: Set up a formal defect log so requirements issues are tracked back to source.

## Answer

C

## Explanation

(C) is collaborating: the conflict is rooted in one-way communication, so the fix is direct dialogue and shared ownership of the artefact — exactly what backlog refinement provides. (A) assumes the BA's docs are the problem and adds overhead without changing the dynamic. (B) inserts a middleman rather than fixing the relationship. (D) tracks symptoms but doesn't change the underlying interaction.
