# Question

A team is struggling to complete work within sprints. Analysis shows requirements are frequently clarified or changed mid-sprint, after work has already started. What improvement should the Scrum Master prioritise?

## Options

- **A**: Lengthen sprints so there's more time to absorb mid-sprint changes.
- **B**: Add a formal change control gate for any mid-sprint requirement change.
- **C**: Add more developers to increase capacity.
- **D**: Improve backlog refinement so stories are well-defined before sprint planning.

## Answer

D

## Explanation

(D) attacks the root cause — stories are entering the sprint without enough clarity. Refinement makes acceptance criteria explicit and questions answered before the team commits. (A) adds time but doesn't prevent the churn. (B) adds bureaucracy without curing the upstream problem. (C) adds people to a process problem (Brooks's Law) and rarely improves throughput.
