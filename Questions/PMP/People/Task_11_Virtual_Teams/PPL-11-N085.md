# Question

A virtual construction-design team has accumulated six collaboration tools over two years: two chat apps, two document repositories, a wiki and a separate tracker. New members cannot tell where current information lives, and the same decision is sometimes recorded in three places with conflicting detail. The project manager is asked to sort this out. What is the best first step?

## Options

- **A**: Write a clear guide mapping which tool to use for which purpose and circulate it to the whole team so people know where each kind of information belongs.
- **B**: Agree with the team a single authoritative source for each type of information — decisions, documents, conversations — and consolidate onto the fewest tools that cover those needs.
- **C**: Choose the most capable single platform and migrate everything into it, retiring the other five tools to remove all ambiguity.
- **D**: Add an index page that links out to where each piece of information currently lives across the six tools, so people have one place to start searching.

## Answer

B

## Explanation

The root issue is duplicated, conflicting homes for information, which leaves no authoritative version and confuses newcomers. Establishing one source of truth per information type and consolidating onto the minimum viable toolset attacks both the duplication and the sprawl at once, with the team's buy-in on what lives where (B). A usage guide (A) documents the current mess without reducing it — six tools and three copies of a decision remain, just now annotated. Forcing everything into one platform (C) sounds clean but is heavy-handed: some tools may serve distinct, legitimate needs, and a wholesale migration risks breaking workflows and losing history; "fewest that cover the needs" is wiser than "exactly one." An index of links (D) is a signpost over the duplication, helping people find conflicting copies faster without resolving which is correct. The best first step removes the duplication and defines authority, not merely the navigation.
