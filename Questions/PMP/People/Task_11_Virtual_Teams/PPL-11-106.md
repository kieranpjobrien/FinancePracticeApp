# Question

A newly formed remote agile team is wasting time hunting for current information: the latest designs are emailed around, decisions are buried in chat threads, and the backlog lives in a spreadsheet that several people keep their own copies of. Members regularly act on out-of-date versions. What should the project manager do first?

## Options

- **A**: Establish a single shared toolset with one agreed source of truth for the backlog, decisions and artefacts, and confirm everyone can access it.
- **B**: Ask every member to email a daily summary of the latest status so the whole team starts each day from the same information.
- **C**: Appoint one team member as information coordinator to consolidate the scattered versions and circulate an authoritative copy each morning.
- **D**: Add a standing agenda item to the daily stand-up to verbally reconcile which version of each artefact is now current.

## Answer

A

## Explanation

The root problem is the absence of a shared, authoritative information environment, which a virtual team depends on; establishing one source of truth everyone can reach fixes the cause directly. (B) layers more email onto an email-driven problem and still leaves no canonical version. (C) creates a manual bottleneck and single point of failure instead of letting the tool hold current state. (D) spends scarce synchronous time patching version confusion that should not exist once a single source of truth is in place.
