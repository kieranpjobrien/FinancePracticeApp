# Question

Members of a remote pharmaceutical-documentation team keep emailing different versions of the same spreadsheet to one another, and people regularly act on outdated figures. The PM wants to stop the version confusion. What should the PM do?

## Options

- **A**: Move the spreadsheet to a shared collaborative tool that everyone edits in one place, and agree it is the single source of truth.
- **B**: Ask everyone to add a version number and date to the filename each time they send the spreadsheet, so the latest copy is easy to identify.
- **C**: Nominate one person to hold the master spreadsheet and have everyone email their changes to that person to merge in.
- **D**: Set a rule that the spreadsheet is only circulated once a week, so there are fewer competing versions in flight at any time.

## Answer

A

## Explanation

Version confusion on a distributed team is solved by having one authoritative copy that everyone works in, not many copies in transit, so a shared collaborative tool agreed as the single source of truth removes the problem at its root (A). (B) still produces multiple copies and merely makes the newest one easier to spot — people will keep working from stale ones. (C) creates a single bottleneck and a manual merge step that lags reality and breaks when that person is offline across time zones. (D) reduces how often versions diverge but does nothing about the fact that they diverge at all; the underlying many-copies model is the issue.
