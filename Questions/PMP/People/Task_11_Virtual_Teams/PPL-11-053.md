# Question

A virtual programme has two sub-teams in different cities, each strongly bonded internally but with almost no cross-team interaction. Recent integration defects have been traced to assumptions each sub-team made in isolation. The sub-team leads insist the silo is efficient. What should the PM do?

## Options

- **A**: Keep the sub-teams as they are and hold a weekly integration review where the PM personally reconciles each interface and assumption between them.
- **B**: Have the two sub-team leads run a joint daily standup covering only integration points, and leave each sub-team's internal standup entirely untouched.
- **C**: Rotate one or two members between the sub-teams each iteration so direct relationships and shared context build naturally across the integration boundary.
- **D**: Introduce cross-team working agreements covering interfaces, assumptions and joint sessions for design work that touches both sub-teams.

## Answer

D

## Explanation

PMI's approach to cross-team integration is explicit working agreements plus shared design sessions at the boundary — this addresses the assumption-making root cause. (A) makes the PM the integration point, which does not scale and hides the problem. (B) is a useful tactic but alone does not address design-time assumptions. (C) is a slower relationship-building move; it helps over time but does not directly fix the interface process today.
