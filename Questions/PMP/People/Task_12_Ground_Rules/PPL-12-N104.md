# Question

A distributed agile team across three time zones is co-creating its working agreement. On meeting times, the earliest zone wants all ceremonies in their morning and the latest zone wants them in their afternoon; no single fixed slot is equitable, and the team is split on what to write down. Members ask the Scrum Master to suggest a way forward they can then agree. Which suggestion should the Scrum Master put to the team?

## Options

- **A**: A single fixed slot placed where it inconveniences the fewest members, accepting that one zone bears a standing cost for the sake of routine.
- **B**: A rotation that moves the ceremony time across the zones on a set cycle, so the inconvenience is shared rather than borne by one group.
- **C**: A move to asynchronous ceremonies, removing the live slot altogether so no zone is ever disadvantaged by timing.
- **D**: A fixed daily overlap window the team commits to, within which it later settles the specific ceremony times.

## Answer

B

## Explanation

The team has explicitly asked for a proposal to ratify, so suggesting one is facilitation, not imposition; the test is which option best balances fairness with the real-time collaboration ceremonies need. A shared rotation spreads the unavoidable cost evenly while keeping live ceremonies, and it is the arrangement distributed teams most readily accept as fair, so (B) is strongest. (A) is a legitimate stability-first choice and easy to administer, but it permanently disadvantages one zone, which is the very thing the deadlock is about. (C) protects everyone's hours but trades away the synchronous interaction that planning and other ceremonies depend on, over-correcting a scheduling issue into a collaboration loss. (D) sets a useful constraint yet leaves the actual dispute — which times inside the window — unresolved, so the same stall would recur the moment the team tried to fill it in.
