# Question

A predictive satellite-manufacturing programme has six component teams that each set their own ground rules at kickoff. At programme integration, friction has emerged: teams use different definitions of "blocker," escalate on different timelines, and hand off work in incompatible formats, slowing integration badly. Each team's internal rules work fine in isolation. What should the programme manager do?

## Options

- **A**: Impose one standard set of ground rules across all six teams to eliminate every inconsistency between them.
- **B**: Direct each team to adopt the ground rules of the team whose integration record is strongest, for consistency.
- **C**: Facilitate the teams to agree a small set of shared cross-team ground rules for the interfaces between them, while leaving each team's internal norms intact.
- **D**: Leave the teams' rules untouched and address each integration clash individually as it arises through the issue log.

## Answer

C

## Explanation

The friction occurs specifically at the interfaces between teams — shared terms, escalation timing, and handoff formats — not inside any one team, so the fix targets the seams while preserving each team's autonomy over its own internal ways of working. (C) does this by co-creating a lean set of cross-team agreements just for the interactions that span teams. (A) over-standardises, discarding internal norms that work well and imposing rules where no shared problem exists, which breeds resistance and needless disruption. (B) forces five teams to abandon functioning agreements in favour of another's, conflating one team's integration success with the suitability of all its internal rules. (D) is purely reactive, leaving the structural mismatch in place so the same categories of clash recur indefinitely.
