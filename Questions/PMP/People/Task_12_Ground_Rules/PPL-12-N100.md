# Question

A hybrid renewable-energy software team has jotted four statements onto a single shared page and asks the project manager to file each in the artefact where it will be properly owned and enforced. Match each statement (left) to the artefact it belongs in (right).

## Items

- **1**: "We address it together as a team whenever someone misses a stand-up two days running."
- **2**: "A story is not Done until it has automated test coverage and passes the security scan."
- **3**: "We aim to cut average defect-resolution time by twenty per cent over the next quarter."
- **4**: "The project will deliver a working customer portal that meets the agreed acceptance criteria by Q3."

## Matches

- **A**: Working agreement
- **B**: Definition of Done
- **C**: Team improvement goal (retrospective action)
- **D**: Project charter / scope statement

## Answer

1-A, 2-B, 3-C, 4-D

## Explanation

Each statement is owned by a different artefact according to what it governs. Statement 1 is a behavioural norm about how the team responds to a member's conduct, which belongs in the working agreement (A). Statement 2 defines when a product increment is complete and releasable, the role of the Definition of Done (B). Statement 3 is a measurable performance-improvement target the team sets to better its own process, a retrospective improvement goal (C). Statement 4 states the project's high-level objective and deliverable, which sits in the project charter or scope statement (D). The distinction matters because filing any of these in the wrong place — for instance treating a completeness criterion as a behavioural norm, or an improvement goal as a Done criterion — means it is reviewed by the wrong people at the wrong time and is no longer reliably enforced.
