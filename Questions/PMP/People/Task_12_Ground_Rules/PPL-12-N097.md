# Question

A project manager is supporting an agile media-production team through the life of its working agreement. At four different points, a different action keeps the agreement healthy and genuinely owned. Match each moment in the team's journey (left) to the most appropriate project-manager action (right).

## Items

- **1**: The brand-new team is about to define how it will work together.
- **2**: Eight sprints on, several rules no longer match how the team actually works.
- **3**: A member quietly admits afterwards she disagreed with a rule the group adopted enthusiastically.
- **4**: Two new joiners keep breaking norms they were never told about.

## Matches

- **A**: Revisit and revise the agreement in a retrospective so it reflects current reality.
- **B**: Facilitate the team to co-create the agreement so the members own it from the start.
- **C**: Add a walkthrough of the agreement to onboarding so newcomers learn the norms.
- **D**: Create a safe way to voice the dissent and revisit the rule before it hardens.

## Answer

1-B, 2-A, 3-D, 4-C

## Explanation

Each moment calls for a distinct intervention in the working agreement's lifecycle. Moment 1 is creation, where the project manager facilitates co-creation so the team owns the agreement from the outset (B). Moment 2 is drift, addressed by revisiting and revising the agreement in the retrospective, the dedicated event for inspecting ways of working (A). Moment 3 reveals suppressed dissent that threatens real ownership, so the response is to make the objection safe to voice and revisit the rule before it sets (D). Moment 4 is an onboarding gap, fixed by building the agreement into how new members are oriented (C). Swapping any pairing fails — for example, treating drift as an onboarding problem, or answering hidden dissent by simply re-running creation — because each situation has a different root cause and only one of these actions resolves it.
