# Question

A distributed agile team building a travel-booking platform spans three continents with little overlap in working hours. During working-agreement creation, a member proposes the rule "we respond to each other quickly." Two iterations later, complaints arise: one member expected replies within an hour, another assumed "by the next working day" was fine, and a blocker sat unanswered for sixteen hours. The project manager is asked to help. What is the best way to address the underlying problem?

## Options

- **A**: Ask members to assume positive intent and give each other grace, since response speed will naturally improve as the team gels.
- **B**: Facilitate the team to rewrite the rule into an observable standard, e.g. "non-urgent messages answered within one working day; blockers flagged and answered within four working hours."
- **C**: Mandate that all blockers be raised in the daily stand-up rather than asynchronously, so nothing waits between meetings.
- **D**: Introduce a shared status indicator so members can see who is online and direct urgent questions only to those available.

## Answer

B

## Explanation

The failure traces to a vague, unmeasurable rule that each person interpreted differently; an effective ground rule must be specific and observable so everyone holds the same expectation. (B) converts the aspiration into a concrete standard with distinct paths for routine versus blocking messages, which is exactly what the situation demands. (A) addresses tone but leaves the ambiguity that caused the sixteen-hour gap fully intact. (C) defers all blockers to a single daily event, which for a team with almost no overlapping hours can add nearly a day's latency — worse than the problem. (D) helps people find who is available but still defines no agreed response expectation, so the same mismatch can recur.
