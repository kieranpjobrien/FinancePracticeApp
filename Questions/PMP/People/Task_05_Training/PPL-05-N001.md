# Question

On a 14-month satellite-ground-station upgrade, three engineers on the telemetry workstream have strong RF backgrounds but no experience with the ground-control software the contract mandates. A competency review has already confirmed the specific gap: they can size and validate telemetry parameters but cannot configure or troubleshoot the control software unaided. The first telemetry package is due in nine weeks. The vendor's certified course is well regarded but runs only on a fixed date seven weeks out and takes a full week, returning the engineers with two weeks to spare but no hands-on practice on the project's actual configuration. One telemetry lead already fluent in the software is on the team. How should you close the gap?

## Options

- **A**: Book all three engineers onto the certified course on its fixed date, since formal vendor certification is the most defensible evidence that they are competent on the mandated software.
- **B**: Have the three engineers pair with the fluent telemetry lead on the project's real configuration tasks from now, and use the certified course later in the programme once the first package has shipped.
- **C**: Pair the three with the fluent lead on the real configuration now AND hold their places on the certified course on its fixed date, so they gain hands-on competence before the package and formal certification before later work.
- **D**: Ask the resource managers to move the bulk of the first package's control-software configuration to the fluent lead, with the three engineers shadowing, so the deadline is protected while they learn.

## Answer

C

## Explanation

The gap is already sized, so this turns on which development approach best fits a hard nine-week deadline where the only course returns the engineers with no practice on the actual configuration. (C) gives them hands-on competence on the real system before the package via pairing with the in-house expert, and keeps the certified course for its evidential value later — both needs met without either crowding the deadline. (A) relies solely on a generic course that finishes with only two weeks' buffer and no project-specific practice, leaving real competence unproven just before delivery. (B) builds the right hands-on skill but discards the certification the contract values and the engineers will need for later control-software work. (D) protects the deadline by concentrating the work on one person with the others merely shadowing — that is staffing around the gap rather than closing it, and it overloads the lead while leaving the three still not independently competent. The fine margin is between (B) and (C): both build real skill now, but (C) also preserves the contractually valued certification at no cost to the deadline.
