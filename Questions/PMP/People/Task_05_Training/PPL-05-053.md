# Question

A minor security incident on your project exposed credentials in a commit history. The root cause analysis shows the developer did not know the secure-handling policy, and three other team members admit similar uncertainty. The incident is contained but the sponsor wants assurance. What should the project manager do FIRST?

## Options

- **A**: Work with security to run targeted secure-coding and secret-handling training for the team, then add pre-commit tooling and spot checks to confirm application.
- **B**: Issue a disciplinary warning to the developer involved and send a short reminder email to the rest of the team about following the existing policy.
- **C**: Add secure-coding training to the organisation's annual compliance programme and rely on that yearly refresher to prevent the same issue recurring.
- **D**: Engage a third-party penetration testing firm to continuously monitor the codebase for exposed secrets and flag any new ones automatically.

## Answer

A

## Explanation

PMI's preferred response to a skill-caused incident is to close the capability gap and reinforce it with supporting controls, tied to the team's actual work. (A) does exactly that, targeted and timely. (B) treats a systemic gap as an individual failing and relies on a weak mechanism. (C) defers a live issue to a yearly cadence and does not address the current project. (D) adds expensive detection on top of an unresolved cause without building the knowledge needed to prevent recurrence.
