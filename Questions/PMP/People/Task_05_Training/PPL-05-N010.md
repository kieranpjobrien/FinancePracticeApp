# Question

Halfway through a manufacturing-ERP implementation, 40 warehouse staff must start using a new handheld-scanner workflow at go-live in two weeks. The staff work rotating shifts and cannot all be released from the floor together. The vendor's package is a two-day classroom course; your controls lead can instead build short device-side sessions, and a few supervisors have already learned the workflow. Go-live is fixed and all 40 must be able to run the workflow unaided on day one. Which approach should you take?

## Options

- **A**: Run short hands-on sessions at the workstations on each shift using the live scanners, and check each operator can complete the core workflow unaided before signing them off.
- **B**: Deliver the device-side sessions on each shift, then judge readiness from go-live floor performance and add support where errors appear.
- **C**: Have the supervisors who already know the workflow run their teams through it during quieter periods, using a one-page guide from the vendor.
- **D**: Run the vendor's two-day classroom course in shift-friendly cohorts so every operator receives the complete curriculum before go-live.

## Answer

A

## Explanation

All four reach the floor in different ways; the deciding factor is which gets every operator demonstrably able to run the workflow before a fixed go-live. (A) delivers hands-on practice on the real devices per shift *and* confirms competence operator-by-operator before sign-off, so no one starts day one unverified. (B) uses the same good delivery method but defers the readiness check to live performance, meaning gaps surface as errors during go-live rather than before it. (C) leverages willing supervisors but they are not confirmed competent to teach, the one-pager is thin for a hands-on task, and consistency across shifts is unmanaged. (D) gives full content but a classroom format is a poor fit for a brief, hands-on, device-based task and still proves nothing about each operator's ability. The fine margin is (A) vs (B): both train well at the workstation, but only (A) verifies competence before the workflow goes live.
