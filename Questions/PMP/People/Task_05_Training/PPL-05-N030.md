# Question

You are mobilising a team for a twelve-month programme that uses several specialised technologies, each needed in a different phase — some early, some only late. People will develop as the work proceeds, phase boundaries may move, and members will be assigned to phases as they open. You have a week before the first phase starts and want to set up how the team's capability will be tracked through the programme. Which approach should you put in place?

## Options

- **A**: Run a skills audit now to record everyone's current capability across the relevant technologies, giving you an accurate baseline to plan the early phases from.
- **B**: Map each member's current proficiency against the competencies each phase requires, with target dates, and keep that mapping updated as people develop and phases shift.
- **C**: Define the competencies each role on the programme is expected to hold, and use those role profiles to drive training and phase assignments.
- **D**: Have each member self-assess their proficiency in each technology the programme uses, repeating the survey at the start of each phase.

## Answer

B

## Explanation

The capability picture has to connect individuals to phase-specific requirements and stay current as people grow and phases move, which is what should shape how it is tracked. (B) is best: a proficiency-versus-requirement mapping with dates, kept live, exposes the gap for each upcoming phase and supports both just-in-time training and assignment decisions throughout. (A) a baseline audit is accurate the day it is taken but is a snapshot against no requirement, so it informs the early phases yet gives no forward view and decays as people develop. (C) role profiles state what a role should know, not what specific individuals can do today, so they cannot reveal a named person's gap or readiness for the next phase. (D) repeated self-assessment refreshes the data but is subjective, uncalibrated against the actual phase requirements, and still leaves you matching capability to need by hand. Mapping real proficiency to phase requirements and maintaining it is the approach that tracks capability the way the programme needs.
