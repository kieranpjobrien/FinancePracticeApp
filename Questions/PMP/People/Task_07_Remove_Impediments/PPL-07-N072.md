# Question

A project manager on a hybrid laboratory-relocation project notices that team members keep mentioning small blockers in passing — a missing key, a delayed delivery, an unanswered query — but these are not recorded anywhere, so some linger for days before anyone acts. The team already runs a brief daily check-in and uses a shared task tool. What should the project manager do to address this?

## Options

- **A**: Add a standing impediments item to the existing daily check-in so blockers are raised, owned, and followed up each day.
- **B**: Create a visible impediment log or board in the shared task tool where any blocker is captured, assigned an owner, and tracked to closure.
- **C**: Set up a dedicated weekly impediments meeting to work through all outstanding blockers as a group.
- **D**: Ask team members to flag blockers to the project manager directly so they can be triaged and assigned quickly.

## Answer

B

## Explanation

The underlying problem is that impediments are invisible and unowned, so they drift; the most durable fix is a persistent, visible log or board where every blocker is captured, given an owner, and tracked to closure regardless of when it surfaces (B). (A) adding an impediments slot to the daily check-in helps raise blockers and is a good complement, but a once-a-day verbal prompt with nothing recorded still lets items slip between check-ins and offers no shared view of what is open and who owns it — visibility, not just a moment to speak, is what was missing. (C) a weekly meeting batches resolution and can leave a blocker sitting for days, the very failure being fixed. (D) funnelling everything through the project manager creates a bottleneck and keeps the blockers out of the team's shared view. The discrimination is between standing, owned visibility (B) and approaches that surface blockers only periodically (A, C) or privately (D).
