# Question

A project manager on a predictive data-centre fit-out arrives to find three blockers at once: a delayed cooling-unit delivery affecting work due in three weeks, a permit query holding up cabling work scheduled to start tomorrow, and a minor tooling shortage a team can work around for now. With limited time this morning, how should the project manager proceed?

## Options

- **A**: Address the blockers in the order they were reported to be fair to each team that raised one.
- **B**: Start with the cooling-unit delivery because long-lead equipment is usually the hardest to recover.
- **C**: Tackle the permit query first because it blocks work starting tomorrow, then the cooling delivery, then the tooling shortage.
- **D**: Resolve the tooling shortage first because it is the quickest to close and clears one item off the list.

## Answer

C

## Explanation

With several blockers competing for limited time, the project manager should triage by urgency and impact: the permit query threatens work starting tomorrow, so it is most time-critical and comes first, followed by the three-week cooling delivery, with the workable-around tooling shortage last (C). (A) first-come-first-served ignores that the blockers have very different deadlines and consequences. (B) the cooling unit matters but has three weeks of runway, so prioritising it over tomorrow's permit work misreads urgency. (D) doing the easiest item first to "clear the list" optimises for tidiness, not for protecting imminent work. The margin is prioritising by impact and timing (C) rather than by order, lead time alone, or ease.
