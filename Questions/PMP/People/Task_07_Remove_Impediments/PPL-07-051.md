# Question

Your agile team runs two-week sprints but the organisation's procurement cycle for any new tool or subscription takes six to eight weeks minimum. Several sprints have been impeded by waiting for approvals on small-dollar items essential for delivery. What should the project manager do?

## Options

- **A**: Have the team buy the items personally and submit expense claims later so the sprint work does not stall waiting for the formal procurement process to complete.
- **B**: Absorb the procurement delay into the sprint planning by only committing to stories whose tooling is already in place for the whole two-week sprint window.
- **C**: Engage the procurement function to co-design a fast-track path for low-value project tooling, using an agreed threshold, a pre-approved catalogue, and named delegated approvers.
- **D**: Escalate every blocked procurement request individually to the sponsor so that each one is expedited on its own merits through executive sponsorship each time.

## Answer

C

## Explanation

PMI addresses a systemic mismatch between delivery cadence and support-function cadence by co-designing a repeatable fast path, not by workarounds, scope constraint, or one-off escalation. (C) is that path. (A) breaches procurement policy and can create audit and expense issues. (B) shapes scope around the constraint rather than fixing it. (D) consumes sponsor bandwidth on individual cases and does not produce a durable fix.
