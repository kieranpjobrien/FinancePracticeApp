# Question

On a predictive offshore-wind construction project, a key structural engineer assigned to the project is being repeatedly pulled away by her functional manager to firefight on an unrelated programme, costing the project roughly two days a week and threatening a design milestone. The matrix is weak, so the functional manager controls her assignments. The project manager has a signed resource agreement naming this engineer. What should the project manager do?

## Options

- **A**: Reallocate the engineer's project tasks to other team members so the milestone can still be met despite her reduced availability.
- **B**: Escalate immediately to the project sponsor and the functional manager's director to enforce the resource agreement.
- **C**: Meet the functional manager with the resource agreement and the milestone impact, and agree how the engineer's committed time will be protected.
- **D**: Adjust the project schedule to reflect the engineer's actual availability and rebaseline the design milestone.

## Answer

C

## Explanation

There is a signed commitment being breached and a named owner — the functional manager — whom the project manager can address directly, so the correct first step is a fact-based conversation pairing the agreement with the milestone impact to restore the committed time (C). (A) redistributing her work may be infeasible given her specialist role and concedes the resource without first defending it. (B) jumping to the director skips the responsible manager and applies disproportionate force before the normal channel has been used. (D) rebaselining accepts the slip before testing whether the commitment can be honoured. The margin is between engaging the accountable manager with evidence (C) and prematurely working around, over-escalating, or surrendering the milestone.
