# Question

On a predictive aerospace-components project, the project manager keeps clearing the same blocker: drawings arrive at the machining cell with missing tolerances, machining stops, and the project manager chases the design office to supply the data each time. Each instance is resolved within a day, but it has happened on six work packages and is eroding the schedule margin. What is the project manager's most effective response?

## Options

- **A**: Continue resolving each missing-tolerance instance promptly, as the one-day turnaround has kept disruption contained.
- **B**: Add a schedule buffer to each remaining machining work package to absorb the expected tolerance delays.
- **C**: Instruct the machining cell to proceed using assumed tolerances and correct any parts later if the values differ.
- **D**: Investigate why drawings reach machining without complete tolerances and work with the design office to fix the handoff so the blocker stops recurring.

## Answer

D

## Explanation

A blocker that recurs identically across six work packages is a process defect, not a series of incidents, so the most effective response is to find why incomplete drawings keep reaching machining and fix the design-to-machining handoff at source (D) — removing the cause rather than the symptom. (A) prompt firefighting has limited the damage but guarantees the blocker keeps returning and consuming margin. (B) buffering every package merely budgets for the waste instead of eliminating it. (C) machining to assumed tolerances on aerospace components invites scrap, rework and a serious quality risk. The hard insight is to treat the recurrence as a signal to address root cause through the handoff process (D), not to keep absorbing or masking it.
