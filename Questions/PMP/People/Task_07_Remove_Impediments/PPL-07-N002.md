# Question

On a predictive plant-automation project in a strong matrix, the only engineer qualified to sign off PLC safety logic keeps being pulled by his functional manager into production firefighting, causing repeated slippage. You have agreed a plan with that manager twice, and each time an urgent plant issue has overridden it within days. The next safety sign-off gate is in three weeks and is on the critical path. The functional manager genuinely controls this engineer and is accountable for plant uptime. What is the BEST course of action?

## Options

- **A**: Quantify the schedule and cost impact of the gate slipping, and bring it to the sponsor framed as a competing-priorities decision between project delivery and plant firefighting, with a specific ask to protect the engineer's time through the gate.
- **B**: Have another conversation with the functional manager, this time agreeing a written commitment with named backfill for firefighting so the engineer is only pulled as a genuine last resort.
- **C**: Negotiate with the functional manager to secure a fixed, smaller block of the engineer's time each week dedicated solely to the sign-off package, accepting reduced firefighting cover during that window.
- **D**: Start cross-training a second engineer on the safety logic now and ask the certifying authority whether a co-sign arrangement could let that engineer share the gate sign-off.

## Answer

A

## Explanation

This is a genuine fine-margin call: B, C and D are all things a competent PM might do, and each is a real attempt to address the conflict — the issue is that the root cause sits above the PM's authority. Two agreed plans have already been overridden because the functional manager has a legitimate, competing accountability (plant uptime) that outranks the project at peer level, so only the sponsor can arbitrate the priority — making A the strongest move (A). (B) is a more robust version of a peer agreement, but it still depends on the same manager honouring it against the same overriding pressure that has twice won out. (C) is a sensible negotiation that might partly help, but it again relies on peer goodwill and trades away firefighting cover the manager may simply refuse to give. (D) is good long-term resilience and worth starting, but a second engineer cannot realistically become gate-ready and certified within three weeks, so it does not protect this gate. The distinguishing factor is that the conflict has demonstrably exceeded peer resolution and needs the one person who owns both priorities.
