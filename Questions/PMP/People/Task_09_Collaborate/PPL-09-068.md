# Question

An agile team building a payments feature for a fintech keeps clashing with the central security team, who they see as a gate that blocks releases late with little warning. The security team, for their part, says they are pulled in only at the end and then blamed for delays. Both groups are willing to work better together but have fallen into a hand-off-and-blame pattern. Which TWO actions would most help the two teams genuinely collaborate? (Select TWO.)

## Options

- **A**: Invite a security representative into the team's backlog refinement and reviews so security considerations are shaped in early, alongside the work.
- **B**: Agree shared, visible security acceptance criteria up front so both teams build toward the same definition of "secure enough" from the start.
- **C**: Add a formal security sign-off gate just before release with a documented checklist the team must complete to pass.
- **D**: Have the project manager relay the security team's requirements to the development team in writing to keep the two groups from clashing directly.
- **E**: Track the number of security findings raised per release on a dashboard so the team can see how often security blocks them.

## Answer

A, B

## Explanation

The teams are stuck in a late hand-off, so collaboration improves by pulling security in early and giving both a shared target. (A) embeds a security voice in the team's own events so concerns surface while work is still shapeable, and (B) creates a jointly owned definition of done for security that removes the end-of-line surprise; both turn a gate into a partnership. (C) reinforces the very late-gate pattern that is causing the friction. (D) inserts the PM as a go-between, which keeps the two teams apart rather than collaborating directly. (E) measures conflict without changing the way of working, and framing findings as "blocks" can deepen the blame dynamic.

## Anti-patterns avoided

- C, D, and E all manage the symptom; the margin is integrating security early and sharing the target versus gating, relaying, or measuring.
