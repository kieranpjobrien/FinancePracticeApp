# Question

A client stakeholder feels his input is consistently ignored by the project team and has escalated his frustration to the PM's manager. What should the PM do FIRST?

## Options

- **A**: Apologise to the manager and commit that all client input will be implemented going forward without exception.
- **B**: Set up a recurring forum with the client to capture input, route it into decisions, and report back.
- **C**: Explain to the manager that the team has been declining the client's suggestions because they are not feasible right now.
- **D**: Ask the manager to handle the client relationship directly while the PM focuses on project delivery instead.

## Answer

B

## Explanation

The stakeholder needs a real channel for input and visible closure on what happens to it, which is what (B) provides. (A) trades a real fix for a promise no one can keep. (C) defends the team's behaviour rather than addressing the stakeholder's experience. (D) hands off a relationship the PM owns and signals the issue is not the PM's to solve.
