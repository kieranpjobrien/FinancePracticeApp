# Question

Two agile teams in a telecoms programme depend on each other: Team A consumes an API that Team B owns, and B's frequent unannounced changes keep breaking A's integration. The teams have grown irritable in stand-up. The project manager brings both sides together to negotiate a working agreement that lets them keep moving without one team being held hostage by the other. Which TWO commitments would form the soundest basis for that agreement? (Select TWO.)

## Options

- **A**: Team B will give advance notice of breaking API changes and maintain the current version for an agreed window so Team A can adapt in its own cadence.
- **B**: The teams will agree a shared contract for the interface and a lightweight change process both follow, so neither side alters it unilaterally.
- **C**: Team A will freeze its integration work and simply consume whatever Team B ships, accepting rework whenever B changes the interface.
- **D**: Team B will route every proposed API change through the project manager for approval before implementing it, so the PM controls the interface.
- **E**: The two teams will be merged under one backlog so the dependency disappears, removing the need for any inter-team agreement.

## Answer

A, B

## Explanation

A durable inter-team agreement combines advance notice with a maintained version window (A) and a jointly owned interface contract with an agreed change process (B), so changes become predictable and neither team can unilaterally break the other. (C) makes Team A absorb all the pain and rework, which is not an agreement but a capitulation that leaves the underlying problem live. (D) inserts the PM as a bottleneck on every technical change, which does not scale and pulls decision-making away from the teams who own the work. (E) is a heavy structural reorganisation proposed to dodge a problem that a working agreement can solve directly, and merging teams brings its own costs.
