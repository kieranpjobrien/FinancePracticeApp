# Question

A project manager at a regional water utility is about to enter talks with the operations department over how much of its maintenance crew can be released to the project during the upcoming outage window. The operations head has mentioned, informally, that being left short-handed during an unplanned failure is what worries her. The project manager has half a day before the meeting and has already mapped the schedule's crew needs. How is that half day best spent?

## Options

- **A**: Work out the smallest crew the project can run with and the point past which the schedule cannot recover, and frame two release options that would each leave operations a reserve for unplanned failures.
- **B**: Pull the outage-failure history and crew callout records so the discussion can rest on what cover operations has actually needed in past windows rather than on each side's assertion.
- **C**: Pre-brief the operations head one-to-one to surface any further concerns and signal that the project means to protect her cover, so the meeting opens on a cooperative footing.
- **D**: Reconfirm with the contract-labour desk that short-notice agency crew are available within budget, so the project holds a fallback if operations will not release enough of its own people.

## Answer

A

## Explanation

With limited prep time and the schedule's needs already mapped, the highest-value move is to convert that into negotiating substance: the true minimum, the unrecoverable threshold, and a couple of release options that also leave operations the reserve she is worried about — this equips the project manager to trade flexibly while protecting both sides' core interests, which is the heart of negotiation preparation. The others are each genuinely useful and would form part of thorough preparation, but rank below A for this meeting. (B) gathers objective data, which strengthens the case but is a supporting input rather than the core position the manager must walk in holding. (C) builds the relationship and may surface more interests, yet pre-empting the meeting risks anchoring before the project's own floor is even set. (D) secures a fallback, but firming up an alternative is worth most once the project knows the minimum it actually needs from operations — which A establishes first.
