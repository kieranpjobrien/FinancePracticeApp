# Question

Two department heads are blocking the project charter. One insists the new reporting system go live first in their region; the other insists their region must be first. Both say they cannot wait, and rollout capacity allows only one region in the first wave. The project manager wants to break the deadlock with a proposal both can accept. What is most useful for the PM to explore before proposing a sequence?

## Options

- **A**: The specific reason each region needs to be first — a regulatory deadline, a customer event, a competing initiative — since the real drivers may differ in ways that allow a sequence neither stated position revealed.
- **B**: Each region's operational readiness to absorb the system first, so the wave order can be set by which region can actually go live without rework.
- **C**: Whether the first wave could be scoped down to a single high-impact module delivered to both regions at once, so neither head has to wait for the headline capability.
- **D**: The delivery cost and risk of each possible sequence, so the PM can recommend the order that is cheapest and least disruptive to execute.

## Answer

A

## Explanation

Principled negotiation works on interests rather than positions, and the most valuable first move is to surface why each region needs to be first; the underlying drivers frequently differ (a hard deadline versus a launch event), which reveals options to satisfy both that the competing "we must be first" positions conceal. (B) is a sound input but addresses feasibility, not the heads' actual needs — a region could be ready yet have no real urgency, so readiness alone can settle on the wrong basis. (C) is a creative option worth testing, but proposing a solution before understanding each party's interest risks solving the wrong problem if, say, one region needs the full system rather than one module. (D) optimises the PM's delivery convenience rather than the stakeholders' underlying requirements, and a sequence that is cheap to run but misses a regulatory date is not actually the better deal.
