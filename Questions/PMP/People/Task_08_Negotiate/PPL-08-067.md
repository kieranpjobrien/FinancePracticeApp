# Question

A project manager is negotiating a statement of work with a vendor that will supply a development squad to an agile programme. The vendor is accustomed to fixed-scope, fixed-price engagements with detailed upfront requirements, whereas this programme works from an evolving backlog with priorities reset each iteration. The project manager wants terms that let the relationship work the way the programme actually runs while still protecting the buyer. Which provisions should the project manager press for in the agreement? (Choose three.)

## Options

- **A**: A capacity-and-rate basis funding a stable squad per iteration, rather than a price fixed to a frozen scope agreed at signing.
- **B**: A definition-of-done and acceptance criteria the parties agree to apply to delivered increments, so quality is testable each iteration.
- **C**: A clause allowing the buyer to re-prioritise the backlog at each iteration boundary without a formal contract variation each time.
- **D**: A fixed total contract value tied to a baselined scope agreed at signing, with every backlog change after that handled through formal variations, to keep the buyer's spend predictable.
- **E**: A termination-for-convenience clause letting the buyer exit at any iteration boundary on short notice if value is not being delivered, so the buyer is not locked in.

## Answer

A, B, C

## Explanation

To make a contract fit agile delivery, the buyer wants to fund capacity rather than frozen scope (A), to anchor quality in an agreed definition of done and acceptance criteria applied each iteration (B), and to retain the freedom to re-prioritise at iteration boundaries without a contract variation every time (C) — together these let the relationship run the way the programme works while still protecting the buyer. The two distractors are deliberately tempting buyer protections. (D) does give cost predictability, but baselining total value to a frozen scope and routing every change through a formal variation reintroduces exactly the fixed-requirements model that fights an evolving backlog — it directly contradicts C and would generate constant variations. (E) is a genuinely sensible protection and a buyer may well want it, but it governs exit, not how the engagement runs iteration to iteration; it is not one of the three provisions that make the working relationship fit agile delivery, which is what the question asks for.
