# Question

A PM mentors stakeholders using a deliberate progression of approaches as a mentee develops. Match each mentoring need (left) to the technique best suited to it (right). Each technique is used once.

## Items

- **1**: A mentee can perform a task but lacks confidence and needs to see it done well before attempting it.
- **2**: A mentee has the capability and the PM wants them to reason their own way to a solution.
- **3**: A mentee is partway through a new responsibility and benefits from doing it with the PM alongside, sharing the work.
- **4**: A mentee is ready to perform independently and needs the chance to do so with support on standby.

## Matches

- **A**: Pose open, probing questions and let the mentee work out the answer themselves.
- **B**: Model the task while the mentee observes, then debrief what made it effective.
- **C**: Delegate the responsibility fully, remaining available if the mentee needs help.
- **D**: Co-perform the task together so the mentee leads parts while the PM supports the rest.

## Answer

1-B, 2-A, 3-D, 4-C

## Explanation

The four needs map onto a progression from showing to releasing, and matching them correctly mirrors how support fades as readiness grows. Need 1 is a confidence gap, so (B) modelling lets the mentee see it done before trying, lowering the barrier. Need 2 is about developing independent reasoning in a capable mentee, so (A) probing questions draws the answer from them rather than supplying it. Need 3 is a partial-readiness stage, so (D) co-performing keeps the PM alongside while the mentee leads parts. Need 4 is full readiness, so (C) delegating with support on standby grants real ownership while a safety net remains. Mismatching collapses the progression — modelling for the independent-ready mentee (4) would under-stretch them, and delegating fully to the unconfident one (1) would overwhelm; the developmental art is fitting the support level to the mentee's current stage.
