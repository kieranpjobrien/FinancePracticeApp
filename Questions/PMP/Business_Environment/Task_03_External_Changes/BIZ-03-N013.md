# Question

A predictive project is building a manufacturing line. A new mandatory carbon-reporting regime is announced requiring the organisation to disclose embodied emissions of new capital assets from next financial year, after this line is due to be operational. The line is in scope, but emissions data was never a project requirement, and detailed design is well advanced. The PM has confirmed with the sustainability lead that the regime will apply. What should the project manager do first?

## Options

- **A**: Raise a change request to add emissions-monitoring instrumentation to the line's design and rebaseline accordingly.
- **B**: Log a risk that the delivered line may not support the organisation's future reporting obligation, and review it with the sponsor.
- **C**: Work with the sustainability lead to determine exactly what emissions data and design features the regime requires of an asset like this line.
- **D**: Confirm with the sponsor whether meeting the new regime is the project's responsibility or an operational matter for after handover.

## Answer

C

## Explanation

The PM cannot yet act sensibly because what the regime demands of this asset is unknown, so the first step is to establish the specific data and design requirements with the responsible sustainability stakeholder (C). Raising a change request for instrumentation (A) commits to one solution before the requirement is defined and may add the wrong capability. Logging a risk for later review (B) is reasonable to do but stops short of finding out what is actually required, which is what makes any later response meaningful. Clarifying ownership with the sponsor (D) is a fair question, but the obligation attaches to the asset this project is building, so understanding the requirement comes first and informs that ownership discussion rather than substituting for it.
