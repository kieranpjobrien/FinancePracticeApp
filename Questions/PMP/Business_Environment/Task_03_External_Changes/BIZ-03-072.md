# Question

Customer research mid-project reveals that the target segment has shifted expectations: they now want an embedded API instead of a standalone portal. The project is building a standalone portal. What should the PM recommend?

## Options

- **A**: Continue the portal build because research is directional and requirements must remain stable mid-execution.
- **B**: Pivot the architecture now to embedded API delivery so that the product matches the newer research findings.
- **C**: Validate the shift, quantify scope and benefit impact, and present options to the sponsor.
- **D**: Finish the portal build, then commission a new follow-on project to add the embedded API capability later.

## Answer

C

## Explanation

A claimed customer shift requires validation and a quantified options pack before pivoting (C). (A) dismisses research as directional and ignores product-market fit. (B) pivots on a single data point without validation. (D) defers responding to a known shift and commits to building the wrong thing first.
