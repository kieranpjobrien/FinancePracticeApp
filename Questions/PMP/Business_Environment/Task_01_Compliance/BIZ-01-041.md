# Question

A government client's contract requires WCAG 2.1 AA accessibility conformance. The development team has implemented features without explicit accessibility checks because "nobody complained in user testing." What should the PM do?

## Options

- **A**: Add accessibility testing to the definition of done and schedule remediation of existing features.
- **B**: Run a single accessibility audit at user acceptance testing to catch issues before launch.
- **C**: Rely on the test team to flag accessibility issues as they encounter them.
- **D**: Document the conformance gap as a known issue and proceed to delivery.

## Answer

A

## Explanation

Contractual accessibility obligations are part of done, not a late-stage check (A). (B) defers conformance to a single late audit and risks expensive rework. (C) is informal and depends on testers noticing issues that automated and manual accessibility tests would catch. (D) accepts a contractual breach.
