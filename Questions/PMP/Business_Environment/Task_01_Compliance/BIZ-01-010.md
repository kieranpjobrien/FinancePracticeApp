# Question

An agile team proposes skipping the security compliance test in the upcoming sprint to hit a launch date, arguing the new features are low risk. What should the project manager do?

## Options

- **A**: Defer to the team's judgement on risk and allow the skip for one sprint only.
- **B**: Escalate to the sponsor and request approval to bypass the security gate.
- **C**: Allow the skip but log a formal risk in the risk register against the team lead.
- **D**: Reduce sprint scope so the security tests can run within the agreed timebox.

## Answer

D

## Explanation

Compliance gates do not bend to deadlines, but scope can. (D) preserves the security test by tailoring sprint contents — a normal agile lever. (A) hands a compliance decision to the team, which is not theirs to make. (B) implies sponsors can override security gates; they generally cannot. (C) accepts the violation and assigns blame, which is theatre, not risk management.
