# Question

A project manager is planning compliance for a medical-device software project and encounters four distinct situations during planning. Match each situation to the most appropriate compliance response.

## Items

- **1**: A regulatory clause is ambiguous and could be read two ways, each with different design implications.
- **2**: An obligation is clear, but no one on the project has been made responsible for meeting it.
- **3**: A required compliance activity is fully understood but has not been reflected anywhere in the schedule or budget.
- **4**: The team must be able to prove, after launch, that a particular control was applied to every release.

## Matches

- **A**: Assign a named owner accountable for the obligation.
- **B**: Seek an authoritative interpretation from legal or the regulator before committing to a design.
- **C**: Define and retain the evidence and records that demonstrate the control was applied.
- **D**: Incorporate the activity into the schedule and cost baselines so it is planned and resourced.

## Answer

1-B, 2-A, 3-D, 4-C

## Explanation

An ambiguous clause with design consequences needs an authoritative reading before the project commits, so situation 1 maps to seeking interpretation (B). A clear obligation with no responsible party is an ownership gap, so situation 2 maps to assigning an owner (A). A understood activity absent from plans must be built into the baselines to be resourced and tracked, so situation 3 maps to incorporating it into schedule and cost (D). The need to prove post-launch that a control was consistently applied is met by defining and retaining evidence, so situation 4 maps to evidence and records (C). Each response addresses a different failure mode — interpretation, accountability, planning, and provability — so swapping any pairing leaves one gap unaddressed.
