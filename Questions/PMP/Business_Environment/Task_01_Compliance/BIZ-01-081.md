# Question

A hybrid project is building a payments gateway that must remain compliant with an industry security standard whose requirements are revised roughly annually. The project will run for eighteen months, so the standard will almost certainly change before launch. The project manager is designing how the project will stay compliant across that period. Which measures form a sound approach to managing this evolving obligation? (Select all that apply.)

## Options

- **A**: Establish a mechanism to monitor the standard for published revisions throughout the project.
- **B**: Route material changes to the standard through change control to assess and incorporate their impact.
- **C**: Assign a named owner responsible for tracking the standard and triggering the response when it changes.
- **D**: Freeze the requirements to the version current at project start to avoid mid-flight disruption.
- **E**: Plan a single compliance check against the latest version only in the final month before launch.

## Answer

A, B, C

## Explanation

An evolving obligation across an eighteen-month project demands active monitoring for revisions (A), a controlled route to assess and incorporate material changes so the product tracks the standard (B), and a named owner who is accountable for watching it and triggering the response (C) — together these keep compliance current as the standard moves. Freezing to the start version (D) guarantees the gateway could launch against a superseded standard and fail certification. A single late check (E) discovers any divergence when remediation is most expensive and schedule-critical, defeating the purpose of planning for change in the first place.
