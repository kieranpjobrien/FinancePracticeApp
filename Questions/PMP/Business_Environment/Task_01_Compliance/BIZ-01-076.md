# Question

A predictive project must comply with an accessibility standard, a data-residency rule, and a financial-records-retention obligation. The steering committee complains that compliance status is only ever reported as a vague "on track" and they cannot tell whether the project is genuinely meeting its obligations. The project manager wants to give the committee something more meaningful each reporting period, in a form a governance body can actually act on. Which actions would make the compliance reporting genuinely informative? (Select all that apply.)

## Options

- **A**: Define a specific, verifiable measure for each obligation and report actual performance against it.
- **B**: State, for each obligation, the evidence that supports the reported status and where it is held.
- **C**: Flag any obligation trending towards non-compliance early, with the action being taken to recover it.
- **D**: Roll the three obligations into a single overall compliance percentage so the committee gets one clear number each period.
- **E**: Attach the full compliance test logs and audit working papers to every report so the committee has the complete underlying detail.

## Answer

A, B, C

## Explanation

Meaningful compliance reporting rests on verifiable measures the committee can read against each obligation (A), traceable evidence that substantiates each status rather than asserting it (B), and forward-looking flags that show emerging problems and recovery actions while there is still time to act (C). Rolling the obligations into a single percentage (D) is the tempting trap — it looks crisp, but a blended number hides which obligation is failing and can read as healthy while one obligation sits in breach, which is exactly the loss of visibility the committee is objecting to. Dumping the full test logs and working papers into every report (E) over-corrects in the other direction, burying the governance body in operational detail it has neither the time nor remit to parse and obscuring the status it actually needs. The discrimination on D and E is between informing a governance decision at the right altitude and either over-aggregating or over-loading the report.
