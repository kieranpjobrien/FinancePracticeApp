# Question

A hospital rolled out an electronic triage system three weeks ago, targeting a reduction in average patient wait times. At the steering committee, the emergency-department director reports that waits feel noticeably shorter and asks the committee to declare the benefit achieved so the team can be released to the next initiative. The benefits management plan defines the wait-time metric and recorded a pre-implementation baseline, and the first formal benefit review is scheduled for month three. What should the project manager do?

## Options

- **A**: Pull the current wait-time data and compare it against the recorded baseline now, so the committee's declaration rests on the agreed metric rather than the director's impression.
- **B**: Hold to the month-three review date in the benefits plan, explaining that three weeks is too short to separate a real shift from normal day-to-day variation.
- **C**: Support declaring the benefit, treating the director's front-line read as a valid early leading indicator while formal confirmation follows at the scheduled review.
- **D**: Ask the analytics team to add a control comparison against a similar department that did not change, so any measured improvement can be attributed to the new system rather than seasonal demand.

## Answer

A

## Explanation

A benefit is confirmed against the agreed metric and baseline, and both already exist, so the proportionate response to a request to declare success now is to run that comparison now — it converts the director's impression into evidence and lets the committee decide on data rather than assertion (A). Holding rigidly to the month-three date (B) is defensible governance and guards against over-reading three weeks of noise, but it leaves an active declaration request unanswered when the data needed to test it is already available; deferring the question is weaker than answering it with the baseline. Accepting the director's read as a leading indicator (C) is tempting because front-line staff are close to the work, but a subjective impression is not a leading indicator of the wait-time benefit — it is anecdote standing in for the very metric the plan defines, and endorsing a declaration on it redefines how the benefit is judged. Adding a control-department comparison (D) is methodologically the strongest way to prove attribution and worth doing for the formal review, but as the immediate move it over-engineers a steering-committee response, takes longer to stand up, and is not required to test the basic claim against the existing baseline. The fine line is A versus B: both respect the metric, but A uses the data on hand to answer the question put to the committee, while B postpones it.
