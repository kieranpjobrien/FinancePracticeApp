# Question

A programme manager is consolidating the benefits forecasts of three concurrent projects for a board paper. Project A (process automation) claims £1.2m in productivity savings. Project B (workforce restructuring) claims £900,000 from reducing headcount in the same department Project A automates. On review, the programme manager realises both projects are counting the value released by removing the same roles. What should the programme manager do before submitting the consolidated forecast?

## Options

- **A**: Sum both figures, since each project independently planned and will independently deliver its own scope.
- **B**: Submit only the larger of the two figures and discard the smaller project's benefit claim entirely.
- **C**: Reconcile the overlap so the released value is counted once at programme level, and reflect each project's distinct contribution without double-counting.
- **D**: Ask each project to halve its claim so the totals appear conservative to the board.

## Answer

C

## Explanation

When two projects claim value from the same underlying change, the programme must reconcile the overlap so the benefit is counted once, while still recognising each project's genuine, distinct contribution to achieving it (C). A double-counts: adding £1.2m and £900,000 would tell the board the programme releases value it cannot deliver twice, inflating the case. B arbitrarily discards a real contribution rather than disentangling the shared and distinct portions, understating one project and misattributing the benefit. D fabricates a "conservative" figure by halving claims with no analytical basis, which is neither accurate nor defensible and misrepresents both the overlap and each project's actual share.
