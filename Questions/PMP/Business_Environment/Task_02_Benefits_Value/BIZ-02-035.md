# Question

A senior stakeholder requests a new feature that is technically feasible but does not contribute to any benefit in the approved business case. The team has the capacity. What should the PM do?

## Options

- **A**: Accept the request because capacity exists and the requester is a senior stakeholder.
- **B**: Reject the request outright because it falls outside the approved project scope baseline.
- **C**: Ask the stakeholder to articulate the value, then route it through change control.
- **D**: Add the feature to a future release once the current programme has formally closed out.

## Answer

C

## Explanation

A request without an articulated benefit must be evaluated for value before consuming capacity (C). (A) treats seniority as a substitute for value justification. (B) shuts down a possibly valuable idea without analysis. (D) defers without understanding whether the work is worth doing at all.
