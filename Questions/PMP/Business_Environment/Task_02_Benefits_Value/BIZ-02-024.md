# Question

Your project delivered its full scope but the expected benefits cannot materialise because a sister project it depended on was cancelled. What should the project manager do?

## Options

- **A**: Close the project as a success because all planned features were delivered as agreed in scope.
- **B**: Quietly repurpose the delivered system so it can operate without the cancelled project's data feed.
- **C**: Record the dependency failure, lay out options for the system, and escalate to governance.
- **D**: Document the other project's cancellation as the root cause and proceed to formal project closure.

## Answer

C

## Explanation

The system is orphaned — a governance decision is needed on repurpose, decommission, or invest (C). (A) closes on outputs when outcomes are impossible. (B) is a major scope change without approval. (D) assigns blame without making a forward decision.
