# Question

An automation project was justified by an expected 20% reduction in claim processing time. Three months after go-live, measured improvement is only 12%. What should the project manager recommend?

## Options

- **A**: Close the project as successful because the deliverable went live on schedule and within budget.
- **B**: Restate the business case to set the new baseline at 12% so future variance is measured fairly.
- **C**: Investigate the gap, identify root causes, and propose follow-on actions to recover the missing benefit.
- **D**: Allow another twelve months of operations data before drawing conclusions about the gap.

## Answer

C

## Explanation

A material benefits gap deserves investigation, not redefinition or delay (C): low adoption, missing training, or an over-stated baseline could be at play. (A) confuses delivery with value. (B) is benefits manipulation that hides the shortfall. (D) is sometimes valid but should not be the default response to a 40% miss.
