# Question

A start-up's agile team has released a minimum viable version of a budgeting feature to test the hypothesis that users will link their bank accounts to receive automated insights. After four weeks, the data shows strong sign-ups but only 12% of users actually linking an account — far below the 50% the value case assumed. The product owner proposes pressing on to build the full set of planned insight features, reasoning that more features will increase linking. The PM is asked for a view. What should the PM advise?

## Options

- **A**: Investigate why linking is so low and address that barrier before investing in more insight features, since the unvalidated assumption undermines the value of everything built on top of it.
- **B**: Proceed to build the full insight feature set, because a richer product is more likely to persuade users to link their accounts.
- **C**: Lower the value case's linking assumption to 12% and continue the plan unchanged, accepting the measured rate as the new baseline.
- **D**: Pause the whole initiative, since a 12% linking rate against a 50% assumption shows the product concept has failed.

## Answer

A

## Explanation

The MVP exists to validate the riskiest assumption, and the data has falsified it — at 12% linking, every downstream insight feature delivers value to only a fraction of the assumed base, so the priority is to learn why linking fails and fix that barrier before building more on the broken assumption (A). B bets that more features will fix an adoption problem whose cause is unknown; it doubles the investment on top of an unvalidated foundation, which is exactly what validated learning warns against. C accepts the low rate as fact and presses on, but quietly re-baselining to 12% without understanding the cause locks in a fivefold benefit shortfall rather than attempting to recover it. D over-reacts: one early metric below target signals a barrier to investigate, not a failed concept, and abandoning outright forfeits a feature that may work once the linking obstacle is understood. The fine margin is A versus C: both register that 50% was wrong, but only A treats the gap as a problem to diagnose and close, whereas C resigns to it.
