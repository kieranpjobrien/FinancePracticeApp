# Question

A hybrid programme delivering a manufacturer's e-commerce replatform has a fixed go-live for the peak trading quarter. With six weeks left, the team finds it can deliver either an improved checkout (forecast to lift conversion, value realised continuously from launch) or a new product-recommendation engine (forecast larger annual value, but which the analytics team confirms cannot be tuned to deliver that value until three months of live transaction data have accumulated). Only one will fit before the freeze; the other slips to the following release. The product owner asks the PM which to prioritise for this release. What is the strongest basis for the recommendation?

## Options

- **A**: Prioritise the recommendation engine, because its forecast annual value is larger and capturing the bigger benefit should drive sequencing.
- **B**: Prioritise the checkout improvement, because its value begins accruing at launch whereas the engine's value is gated for three months regardless of when it ships.
- **C**: Prioritise the checkout improvement, because it is lower risk and more likely to be delivered cleanly before the freeze.
- **D**: Split the engine into a thin first slice for this release and defer the checkout, to begin accumulating the data the engine needs sooner.

## Answer

B

## Explanation

Sequencing for value compares not just benefit size but *when* each benefit can be realised: the checkout pays from launch, while the engine's value is locked for three months by the data-accumulation constraint, so shipping the engine in this release buys nothing extra during the peak quarter the freeze targets (B). A fixes on the larger headline figure and ignores that the engine cannot convert that figure into realised value in the window that matters, leaving peak-quarter benefit on the table. C reaches the right release but on a delivery-risk argument; the decisive factor here is the timing of value realisation, not relative risk, and an examiner expects the benefit logic. D begins the data clock earlier but sacrifices the checkout's immediate, certain peak-quarter benefit to slightly advance value that is gated anyway — a poor trade given the engine still cannot deliver until the data matures. The fine margin is B versus C: both choose the checkout, but only B rests on the realisation-timing of the benefit rather than on delivery confidence.
