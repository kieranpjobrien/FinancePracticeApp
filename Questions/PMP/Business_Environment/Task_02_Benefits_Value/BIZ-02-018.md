# Question

After several Scrum sprints, the product owner reports that features work well in isolation but fail to deliver the promised productivity gains because they don't integrate with the existing user workflow. What should the project manager facilitate?

## Options

- **A**: Pause new feature development and re-prioritise integration and UX work in the next sprint.
- **B**: Continue the current backlog ordering to maintain delivery cadence.
- **C**: Add capacity to the team so more features can be delivered each sprint.
- **D**: Ask users to adapt their workflow around the delivered features.

## Answer

A

## Explanation

Empirical agile means responding to feedback that points at the real bottleneck (A): integration, not feature count. (B) keeps producing low-value output. (C) treats velocity as the constraint when it is direction. (D) shifts the cost of poor design onto users.
