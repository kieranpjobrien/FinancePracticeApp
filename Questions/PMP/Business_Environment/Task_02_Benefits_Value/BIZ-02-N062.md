# Question

On an agile project building a retailer's inventory app, two backlog items compete for the next sprint and only one will fit. A barcode-scanning feature is forecast to deliver a large lifetime saving but cannot pay anything back until a depot hardware upgrade lands next quarter. A stock-count reconciliation feature is forecast to deliver a smaller lifetime saving but would start cutting write-offs from the day it ships. Both are roughly the same build effort. The product owner asks the PM which ordering basis best protects value. What should the PM advise?

## Options

- **A**: Sequence the reconciliation feature first, because it begins realising value immediately whereas the scanning feature's larger benefit is gated until next quarter regardless of when it ships.
- **B**: Sequence the scanning feature first, because its larger forecast lifetime saving is the bigger benefit and the greater prize should lead the backlog.
- **C**: Sequence whichever feature the highest-paying stakeholder group is asking for first, to keep the strongest sponsors engaged and supportive.
- **D**: Sequence by build effort, taking whichever feature can be completed and shipped soonest so the team maintains delivery momentum.

## Answer

A

## Explanation

Value sequencing weighs not just the size of a benefit but when it can actually be realised: the reconciliation feature starts cutting write-offs the day it ships, while the scanning feature's larger saving is locked until the depot hardware lands next quarter — so shipping scanning now buys no earlier value and forfeits the reconciliation savings in the meantime (A). B fixes on the larger lifetime figure and ignores that the benefit cannot convert to realised value this quarter, leaving certain near-term savings on the table for a prize that is gated anyway. C orders by stakeholder influence rather than value, which is a legitimate engagement concern but not the basis for protecting benefit — letting the loudest sponsor set the sequence can actively destroy value. D optimises for throughput, but here both items cost the same effort, so "soonest to build" does not discriminate and momentum is not the value question being asked. The fine margin is A versus B: both are real benefits, but only A accounts for realisation timing — the deciding factor when one benefit is larger yet cannot pay back until later.
