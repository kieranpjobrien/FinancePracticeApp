# Question

A university is introducing a single shared student-records system to replace spreadsheets each faculty maintains independently. Two faculties are cooperative. The largest faculty's administrators resist, saying the new system removes flexibility they rely on for unusual course structures. The project manager has confirmed the new system can in fact handle those structures with a configuration the administrators have not seen. What is the best response?

## Options

- **A**: Run a working session with the administrators to map their real-world cases into the system and let them see their needs met, adjusting the design where genuine gaps emerge.
- **B**: Send the administrators documentation showing the configuration that supports their course structures, resolving the misunderstanding.
- **C**: Escalate to the provost to mandate adoption, since the objection is based on a misunderstanding rather than a real limitation.
- **D**: Grant the faculty an exemption to keep its spreadsheets for the unusual courses while adopting the system for everything else.

## Answer

A

## Explanation

The resistance is rooted in a belief that the system cannot handle their work; the most durable way to shift it is to let them experience their own cases working in the system and co-resolve any true gaps, which builds both understanding and ownership (A). Sending documentation (B) addresses the fact but not the trust and loss-of-control concern, and is easy to dismiss without hands-on proof. Mandating adoption (C) overrides rather than resolves the concern and is likely to harden resistance in the very group whose buy-in matters most. An exemption (D) concedes the standardisation objective and entrenches the parallel spreadsheets the project exists to eliminate.
