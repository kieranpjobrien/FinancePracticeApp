# Question

A PM is running the Collect Requirements process for a predictive water-treatment upgrade. Operators, regulators, maintenance staff, and a vendor hold partly conflicting needs, and the PM wants a complete, traceable, agreed set of requirements before the scope statement is drafted. All of the actions below are plausible. Which three best support producing a sound, agreed requirements set at this stage? (Choose three.)

## Options

- **A**: Run facilitated workshops that bring the conflicting groups together to surface and reconcile competing requirements.
- **B**: Record each requirement with attributes and acceptance criteria so it can be traced and later verified.
- **C**: Prioritise the requirements with the stakeholders so trade-offs are explicit if scope later has to be constrained.
- **D**: Interview each stakeholder group separately and have the PM consolidate their inputs into a single requirements list.
- **E**: Circulate a questionnaire to all stakeholders and adopt the requirements that the majority rank highest.

## Answer

A, B, C

## Explanation

Facilitated workshops (A) bring conflicting parties into one room to reconcile competing needs, which is the technique best suited to this situation. Documenting attributes and acceptance criteria (B) makes each requirement traceable and verifiable and underpins the traceability matrix. Prioritising with stakeholders (C) makes trade-offs explicit so later scope decisions are deliberate. (D) separate interviews are a legitimate elicitation method, but consolidating conflicting inputs privately leaves the contradictions unresolved and unowned, so it is weaker than reconciling them openly in workshop. (E) a questionnaire can gather breadth, but resolving genuinely conflicting technical and regulatory requirements by majority vote can override mandatory regulator or safety needs, so it does not produce a sound agreed set.
