# Question

A predictive systems integration project is about to enter user acceptance testing. The PM realises three deliverables have no documented acceptance criteria because they were added via approved change requests that only captured cost and schedule impact. What should the PM do?

## Options

- **A**: Proceed into user acceptance testing and let testers define acceptance criteria as issues emerge.
- **B**: Cancel the upcoming user acceptance testing window until all deliverables are fully specified.
- **C**: Work with stakeholders to define and approve acceptance criteria before user acceptance testing begins.
- **D**: Adopt the original project's acceptance criteria as the default standard for the new deliverables.

## Answer

C

## Explanation

Acceptance criteria must exist before validation begins; agreeing them now protects the UAT window from scope disputes. (A) sets up subjective pass/fail calls and likely rework. (B) abandons a scheduled window instead of doing the short work needed to prepare. (D) applies criteria that were written for different deliverables and may not fit.
