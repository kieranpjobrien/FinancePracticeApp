# Question

On a predictive utilities upgrade, a contractor delivers a substation work package and declares it complete, but the project manager's inspector and the contractor disagree over whether a particular earthing test was ever required for that package. Both sides are reading from the approved scope baseline. The customer acceptance walkthrough is in two days. What should the project manager do first?

## Options

- **A**: Check the WBS dictionary entry for that work package to see whether the earthing test is part of its defined acceptance criteria.
- **B**: Raise a change request to add the earthing test so the package is unambiguously covered before the walkthrough.
- **C**: Ask the contractor to perform the earthing test now as a precaution, since an untested earth could fail customer acceptance.
- **D**: Bring the disagreement to the customer at the walkthrough and let them decide whether the test is needed.

## Answer

A

## Explanation

The dispute is about what the approved scope already requires for this package, and the WBS dictionary is the component of the scope baseline that records each work package's detailed description and acceptance criteria; consulting it settles whether the test is in or out before anyone changes anything. (A) reads the existing baseline rather than altering it. (B) assumes the test is out of scope and adds it through change control when it may already be required, risking unnecessary change and cost. (C) performs work whose necessity is still unknown, which could be gold-plating or wasted effort if the dictionary excludes it. (D) outsources a question the baseline can answer internally and risks surfacing an unresolved internal disagreement to the customer at the acceptance event.
