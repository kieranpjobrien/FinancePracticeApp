# Question

A hybrid smart-metering rollout runs an agile firmware stream alongside a predictive field-installation stream. Mid-sprint, the agile team changes a firmware data format in response to genuine user feedback. The installation stream has already printed 50,000 configuration cards built on the old format and is due to begin field deployment in ten days. The agile product owner maintains the change is clearly better for users and wants it shipped in the current sprint. What should the project manager do first?

## Options

- **A**: Ask the firmware team to release the change behind a version flag so the old format keeps working for the printed cards, while the new format is available going forward.
- **B**: Raise the change as a formal change request and route it through the project's integrated change control before either stream acts on it further.
- **C**: Have the installation stream pause its deployment preparation until the firmware team's sprint completes and the final data format is confirmed.
- **D**: Assess how the format change affects the printed cards and the imminent deployment, and convene both stream leads to decide on a coordinated, controlled course of action.

## Answer

D

## Explanation

In a hybrid project a change in one stream can ripple into another, and the project manager's first need is the cross-stream impact picture plus a joint decision by the people who own each stream — that is the move that makes any subsequent action informed and controlled. (D) provides exactly that. (A) backward-compatible versioning is very likely the *right resolution* and a strong distractor, but committing to it before the impact is assessed and the installation lead is consulted presupposes the solution and may overlook constraints the installation stream can see (for instance whether the meters can hold both formats). (B) integrated change control is appropriate for a cross-stream scope change and will probably follow, but raising the request before the impact is understood asks the change authority to decide without the analysis, and treating it as the *first* step skips the joint fact-finding that frames the request. (C) pausing the installation stream is a candidate protective action, but halting an imminent deployment is costly and premature before the impact is even sized — it may prove unnecessary if the change can be versioned or deferred. The fine margin is that A, B and C are all plausible next steps, yet each presumes something the impact assessment and joint decision in D must establish first.
