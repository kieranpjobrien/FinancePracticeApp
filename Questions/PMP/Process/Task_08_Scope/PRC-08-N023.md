# Question

While building the WBS for a predictive offshore wind substation project, the project manager has a single work package, "subsea cable installation", estimated at 1,400 hours over three months. The work is subcontracted, can only proceed during a narrow weather window, and the team cannot yet give a confident estimate or describe how they would track it once underway. The subcontractor has not yet been engaged. The schedule and procurement leads both need this package to firm up. What should the project manager do first?

## Options

- **A**: Decompose the package into smaller work packages reflecting its distinct activities, so each can be estimated and progress can be measured against the weather window.
- **B**: Bring the prospective subcontractor in to detail and estimate the work, then carry their breakdown straight into the WBS as the work packages.
- **C**: Hold the package at its current level and attach a planning-package note that it will be elaborated once the weather window and vessel schedule are confirmed.
- **D**: Add the package to the WBS dictionary with explicit assumptions and acceptance criteria so the large package is at least well defined and controllable as it stands.

## Answer

A

## Explanation

The package fails the test that governs decomposition — it cannot yet be confidently estimated or monitored — so the PM's first move is to break it into smaller components that can be, which is what unblocks both the schedule and procurement leads. (A) does that. (B) is tempting because the subcontractor holds the delivery expertise, but handing them the decomposition outright cedes the WBS to a party not yet under contract and risks a structure built around their commercial interests rather than the project's control needs; the PM owns the WBS and should decompose first, then validate against the subcontractor's input. (C) a planning package is the right tool when work genuinely cannot be elaborated yet, but here the obstacle is that no one has done the decomposition, not that the information is unavailable, so deferring simply postpones a doable task and leaves the leads without estimates. (D) a WBS dictionary entry adds definition but not manageability: documenting assumptions on a 1,400-hour package that still cannot be estimated or tracked does not resolve the underlying problem, which is granularity. The fine margin is that B, C and D are each plausible WBS actions, but only A addresses why the package is unworkable.
