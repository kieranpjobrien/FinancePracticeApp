# Question

A predictive office-relocation project depends on the old floor being cleared of legacy furniture before the project's fit-out crew can begin. The facilities department has formally committed to disposing of that furniture on its own budget and to a fixed date, and the project plan assumes the floor is empty when fit-out starts. The project manager wants the scope statement to record this correctly so that disposal is clearly not the project's work, while still flagging the project's reliance on it happening on time. How should the project manager capture furniture disposal?

## Options

- **A**: List furniture disposal under project exclusions, and record the project's reliance on facilities completing it by the agreed date as an assumption.
- **B**: List furniture disposal only under project exclusions, since stating it is out of scope is sufficient to protect the boundary.
- **C**: Record furniture disposal as an assumption that facilities will clear the floor on time, without an explicit exclusion.
- **D**: Add furniture disposal to the WBS as a deliverable owned by facilities so its completion is tracked inside the project's scope baseline.

## Answer

A

## Explanation

Two distinct things are true and the scope statement should record both: disposal is outside the project's work (an exclusion, which fixes the boundary and pre-empts "I assumed you'd handle it"), and the project's schedule rests on facilities completing it on time (an assumption, which makes that dependency visible and trackable). (A) captures both, so neither the boundary nor the dependency is lost. (B) sets the boundary but drops the reliance the PM explicitly wants flagged, leaving a real schedule dependency invisible. (C) flags the dependency but, without an exclusion, leaves the boundary ambiguous and invites the assumption that the project owns disposal. (D) is the trap: placing the work in the WBS pulls it inside the scope baseline and implies the project is accountable for delivering it, which contradicts the decision that facilities owns it; an item another organisation owns belongs in exclusions and assumptions, not as a project work package.
