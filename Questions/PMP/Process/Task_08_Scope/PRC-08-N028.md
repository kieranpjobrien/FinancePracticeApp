# Question

A vendor is delivering an agile fixed-price project to a client who believed agile meant "we keep adding features as we learn." By sprint five the client's accumulated new requests, if all built, would consume roughly 40% more capacity than the contract funds, and several of the new requests are clearly more valuable than items still sitting in the original backlog. The client is a strategic account the vendor wants to retain. As the project manager, what is the best response?

## Options

- **A**: Accept the higher-value new requests into the backlog and hold a commercial discussion to extend the contract to fund the resulting net increase in scope.
- **B**: Keep the new requests in a separate parking list and deliver the originally agreed backlog first, then propose the new items as a funded follow-on once the contracted scope is complete.
- **C**: Add the new requests now and protect the fixed price by reducing the depth of testing and refinement across the backlog so the extra features fit the funded capacity.
- **D**: Help the client reprioritise the whole backlog by value within the funded capacity, swapping the higher-value new requests in for lower-value original items, and flag that anything beyond the funded capacity is a separate commercial conversation.

## Answer

D

## Explanation

On a fixed-price agile engagement the flexibility the client values operates by *re-ordering within the funded envelope*, not by enlarging it for free — and because some new requests outrank original items, the value-maximising move is to trade them in, not bolt them on. (D) does this and correctly reserves any genuine net increase for a commercial conversation, honouring both the agile principle and the contract. (A) is the strongest distractor: proposing a contract extension is a legitimate path for true net-new scope, but reaching for it first skips the free, immediate value available by reprioritising within the existing budget, and pushes a commercial negotiation the client may resent before the backlog has even been re-ranked. (B) rigidly delivering the original backlog first ignores that agile exists to incorporate better-value learning as it emerges; parking clearly superior items to build inferior ones wastes funded capacity on lower value. (C) silently funding the additions by cutting quality degrades the deliverable the fixed price was agreed for and trades a visible scope conversation for a hidden quality compromise. The margin is fine because A and B are both defensible commercially, but only D captures the available value within the current contract before invoking any change to it.
