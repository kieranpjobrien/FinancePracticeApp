# Question

On an agile content-management project, the product owner has ordered the backlog by business value and the top items are clearly the highest-value features. Two sprints running, though, the team has delivered mostly lower-ranked items, explaining that each top feature depends on a shared content-model component that is large, only partly built, and blocks the high-value work until it is finished. Business stakeholders see low-value output and are losing confidence. What should the Scrum Master do first?

## Options

- **A**: Ask the product owner to re-order the backlog to match what the dependencies currently allow the team to build, so the order reflects reality.
- **B**: Help the team and product owner break the shared component into thin slices that can be delivered alongside the highest-value features, so value-ordered work starts flowing sooner.
- **C**: Have the team build out the full shared component first as a foundation, then deliver the high-value features in value order in the following sprints.
- **D**: Bring the stakeholders into the next review to explain the dependency, so they understand why high-value items have not yet appeared.

## Answer

B

## Explanation

The real constraint is a coarse dependency that is starving the value-ordered work, so the most effective first move is to attack that dependency by slicing it thin enough to ship incrementally with the features that need it, which is how agile reconciles value order with technical reality. (B) restores value-based flow rather than merely explaining or reshuffling around the blockage. (A) re-orders the backlog away from value to fit the dependency, which abandons the product owner's value sequencing and rewards the impediment instead of removing it. (C) builds a large foundation up front before any high-value increment ships, reintroducing the big-batch, deferred-value pattern agile exists to avoid and prolonging the stakeholders' concern. (D) communicating the dependency is worth doing and may follow, but on its own it manages perception without changing the flow of value, so it is not the first action that resolves the underlying problem.
