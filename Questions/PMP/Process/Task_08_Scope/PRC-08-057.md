# Question

A vendor delivering a payroll integration claims configuring thirty-two legacy leave types is out of scope, while the client insists it was always implied. The original scope statement lists only "leave type configuration" without specifying which types. What should the PM do?

## Options

- **A**: Direct the vendor to configure all thirty-two leave types because that is what the client clearly expected.
- **B**: Accept the vendor's position because ambiguous wording should be read in favour of the vendor.
- **C**: Split the difference and ask the vendor to configure half the leave types without additional charge.
- **D**: Facilitate a working session to document the agreed list and raise a change request for any additions.

## Answer

D

## Explanation

Ambiguity in the scope statement is resolved by joint clarification and formal change control for any agreed additions. (A) imposes the client's interpretation without negotiation or change approval. (B) accepts the vendor's narrow reading without attempting reconciliation. (C) splits the difference arbitrarily instead of clarifying intent and impact.
