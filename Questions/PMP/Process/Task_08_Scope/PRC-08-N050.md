# Question

A project manager on a predictive defence-logistics programme has received a change request to add a deliverable. A junior planner asks which documents they should pull to work out, specifically, *what scope* the change adds or alters, before the wider cost and schedule analysis is handed to the planning and finance leads. Which artefacts answer that specific question? (Select THREE.)

## Options

- **A**: The approved project scope statement, to see whether the change alters stated deliverables, boundaries or exclusions.
- **B**: The work breakdown structure, to identify which deliverables and branches the change adds, removes or restructures.
- **C**: The WBS dictionary, to check the affected work packages' definitions and acceptance criteria.
- **D**: The requirements traceability matrix, to trace which originating requirements the change satisfies or contradicts.
- **E**: The project management plan's scope management plan, to read the procedure for how a scope change is processed.

## Answer

A, B, C

## Explanation

The question asks which artefacts tell you what scope the change actually adds or alters. The scope statement, the WBS and the WBS dictionary are precisely the documents that define the project's deliverables, structure and work-package detail, so comparing the request to them shows what scope moves. (A), (B) and (C) each answer a distinct part of that scope question. (D) the traceability matrix is genuinely useful in a change assessment and links the change to requirements, but it records requirement-to-deliverable relationships rather than defining the scope content itself, so it supports the assessment without being where the changed scope is read off. (E) the scope management plan tells you *how* to process a scope change, not *what* scope the change contains; it governs the procedure, not the content. The margin is between the artefacts that hold the defined scope and adjacent documents that describe or trace it.
