# Question

On a predictive HR-systems implementation, three departments describe the same payroll process in conflicting ways and each is convinced its own version is the correct one. The project manager has already run two facilitated workshops; both ended with the departments restating their positions and no convergence, because each believes a different regulation governs the process and none will concede. Design is due to start in a fortnight. What should the project manager do first?

## Options

- **A**: Convene one more facilitated workshop, this time with the departments' managers present, to push the groups to settle on a single agreed version.
- **B**: Obtain an authoritative ruling from the payroll-compliance or legal function on which regulatory interpretation actually applies, then take the reconciled requirements back to the departments.
- **C**: Document all three versions as alternative requirements in the baseline and let design proceed against the option each department uses, deferring the choice.
- **D**: Escalate to the project sponsor to direct which department's version will be treated as authoritative so design can begin on time.

## Answer

B

## Explanation

Facilitation has already been tried twice and failed because the disagreement is not a matter of preference that more dialogue will reconcile; it turns on a factual question of which regulation governs, which no amount of further workshopping among equally convinced peers can settle. Resolving the underlying fact through the authoritative compliance source removes the actual blocker, after which the requirements can be agreed (B). (A) repeats the technique that has already failed twice and assumes adding managers will break a deadlock rooted in a regulatory fact, not in seniority. (C) bakes mutually exclusive requirements into the baseline and pushes an unresolved conflict into design, which is far more expensive to unpick. (D) a sponsor can impose a decision, but directing a winner without establishing which interpretation is legally correct may force the project to build the wrong, non-compliant process; the sponsor needs the compliance answer first.
