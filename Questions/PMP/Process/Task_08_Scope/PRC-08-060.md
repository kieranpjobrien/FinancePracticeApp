# Question

On a Scrum team delivering a customer service portal, the product owner has been swapping stories in and out of the active sprint mid-iteration based on shifting executive priorities. Velocity and morale are both dropping. What should the PM raise?

## Options

- **A**: Stable sprint scope is a Scrum discipline; changes should happen at sprint boundaries, not mid-sprint.
- **B**: The team should adjust to priority changes because responding to change is an agile value.
- **C**: Extend sprint length so fewer boundary changes are needed for the shifting executive priorities.
- **D**: Freeze the product backlog for a quarter so the product owner can focus only on current sprint stories.

## Answer

A

## Explanation

Scrum protects the sprint goal once committed; priority changes belong between sprints so the team can deliver reliably. (B) quotes the Agile Manifesto to justify chaos; responding to change does not mean perpetual disruption. (C) makes sprints longer, which amplifies rather than solves the churn problem. (D) over-corrects by freezing the backlog, which is equally non-agile.
