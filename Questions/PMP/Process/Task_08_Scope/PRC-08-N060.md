# Question

A predictive water-utility project has received a formal change request to add an extra treatment stage. The project manager is at the point of evaluating the change so the change control board can later make a disposition. Which actions belong to assessing this change before it is approved? (Select THREE.)

## Options

- **A**: Analyse the change's impact on schedule, cost, quality, scope and risk across the affected workstreams.
- **B**: Use the requirements traceability matrix to identify which existing requirements and deliverables the change would touch.
- **C**: Evaluate options for delivering the extra stage and recommend a disposition with the supporting impact analysis.
- **D**: Update the scope, schedule and cost baselines to reflect the extra treatment stage so the team can begin.
- **E**: Brief the operations and regulatory stakeholders that the extra treatment stage is being added.

## Answer

A, B, C

## Explanation

Evaluating a change before disposition means establishing its full impact and forming a recommendation, all without yet altering the project. (A) is the cross-area impact analysis that informs the decision. (B) traces the change to the requirements and deliverables it disturbs so nothing is missed. (C) weighs options and produces the recommended disposition with evidence, which is precisely the assessment the board acts on. (D) is the seductive distractor: updating the baselines is a real and necessary activity, but it happens only after the board approves the change, so doing it now is exactly the uncontrolled change Control Scope exists to prevent. (E) is also legitimate work, but communicating that the stage "is being added" both pre-empts a decision not yet made and belongs to managing communications after approval, not to assessing the request. The margin is between actions that build the assessment and actions that are valid only once the change is authorised.
