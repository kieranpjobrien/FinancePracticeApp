# Question

An external auditor will review a hybrid pharmaceutical-manufacturing project. For any approved specification, the auditor must be able to reconstruct which version was in force when a particular batch was released and to satisfy themselves that the record of it is trustworthy. The project manager is hardening how specifications are managed before the audit, and several of the measures under consideration are good practice in their own right. Which two practices most directly enable the auditor to make that determination? (Choose two.)

## Options

- **A**: Retain a dated, immutable version history so the exact content of a specification on any past date can be reproduced.
- **B**: Record an approval and effective date against each specification version, tying it to the period during which it governed.
- **C**: Restrict edit rights on specifications to a single document controller to reduce the chance of accidental changes.
- **D**: Maintain a master index listing the current version number of every specification.
- **E**: Store all specifications in a single repository so the auditor has one place to look.

## Answer

A, B

## Explanation

The auditor's question is temporal and integrity-based: what did this spec say at release, and is it unchanged? An immutable dated version history lets the exact past content be reproduced and proves it has not been altered (A), and an approval/effective date on each version pins which version was authoritative at the batch's release moment (B); together they answer the question directly. (C) limiting edit rights lowers the risk of error but provides no historical record, so it cannot demonstrate what a spec said in the past. (D) a current-version index shows today's state, not the state at a prior release date. (E) a single repository aids retrieval but says nothing about historical content or integrity over time, which is the crux of the audit.
