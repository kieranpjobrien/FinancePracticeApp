# Question

An audit of a hybrid programme finds that project artifacts exist but are hard to rely on: people cannot find the latest version, some logs contradict each other, and a key register has not been updated in weeks. The project manager wants to raise the standard of artifact management. Three of the statements below are sound guiding principles; two are plausible-sounding but flawed. Which three should guide the improvements? (Choose three.)

## Options

- **A**: Artifacts should be kept current, so they reflect the actual state of the project when consulted.
- **B**: There should be a single source of truth for each artifact, avoiding conflicting duplicate copies.
- **C**: Artifacts should be accessible to those who need them, with the current version easy to locate.
- **D**: Once an artifact is baselined, it should change only through formal change control, never by direct edit.
- **E**: Every artifact should be brought to the same high level of formality and detail, so the set is consistent.

## Answer

A, B, C

## Explanation

Sound artifact management keeps information current (A), maintains one authoritative source per artifact to prevent contradictions (B), and makes the right version easy for the right people to find (C). (D) is half-right and therefore tempting: baselines should indeed change only through change control — but stated as a blanket rule it overreaches, because many living artifacts (logs, registers, the very register flagged as stale here) are meant to be updated continuously, not frozen behind a change board; applied universally it would worsen the staleness the audit found. (E) sounds like a consistency virtue but is a flaw: formality should be tailored to each artifact's purpose and audience, and forcing everything to maximum detail creates waste and obscures what matters — uniformity of *level* is not the goal, fitness for purpose is.
