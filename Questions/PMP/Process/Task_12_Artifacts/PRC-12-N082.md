# Question

On a predictive water-treatment plant project, the design lead emails the project manager: "I've already updated the approved piping drawing to fix a clash the contractor flagged, and reissued it to the site so they aren't held up." The same pipe route appears in several other approved drawings that other trades are working from. The project manager must decide what to address first. What is the project manager's most important concern?

## Options

- **A**: The reissued drawing carries the same version identifier as the superseded one, so the site cannot tell which copy is current.
- **B**: The drawings held by other trades still show the old route and may now conflict with the reissued one, and no impact assessment was done before the change was released.
- **C**: The fix was released to the site before the contractor produced evidence that the clash was real rather than a modelling artefact.
- **D**: The change reached the project manager after the fact by email instead of being raised through the project's change request process up front.

## Answer

B

## Explanation

The question turns on which consequence carries the project's substantive risk, and several options name genuine lapses. The deepest one is that an approved drawing was altered and pushed out while several other approved drawings sharing that pipe route were never checked or updated — so trades may now be building from mutually inconsistent documents, a physical integrity problem already in motion (B). (D) names the process failure that allowed this — the change skipped its authorisation route — but the email at least surfaced it; the unassessed cross-drawing conflict is the harm that flows from it and the thing the PM must contain first. (A) a duplicated version identifier is a real configuration defect, yet it is a labelling problem secondary to drawings that disagree on the route itself. (C) verifying the clash is sound, but even a genuine clash would not make an uncontrolled, unassessed reissue safe, so it is not the gravest concern here.
