# Question

A project manager takes over a mid-flight project and finds artifacts spread across email threads, a shared drive, the PM tool, and individual team members' OneNote notebooks. What should the PM do FIRST?

## Options

- **A**: Run a working session with the team to map where each artifact lives, then agree a single source of truth and migration plan.
- **B**: Migrate every document into the central PM tool over the weekend so the team starts Monday on a clean unified baseline.
- **C**: Leave existing artifacts exactly where they live today and require all new artifacts to be created in the PM tool from now on.
- **D**: Ask each team member to send their personal notes and notebooks to the PM, who will then file them centrally for everyone.

## Answer

A

## Explanation

PMI emphasises understanding the current state and engaging the team before changing it. (A) inventories what exists, why it lives where it does, and gets team buy-in for a single repository — the first step in any artifact rationalisation. (B) is disruptive, may break links and references, and risks losing context held by the team. (C) leaves the original problem in place and creates a two-system world that is even harder to navigate. (D) puts the PM in a clerical role and does not establish any standard for ongoing artifact management.
