# Question

A construction project is two weeks from handover. The operations team that will run the facility says it cannot accept the asset without as-built drawings, the equipment register, warranty documents, and maintenance manuals. These exist but are scattered across the PM's drive, the contractor's portal, and engineers' inboxes, and there is no documented agreement on exactly what the handover package must contain. What should the project manager do first?

## Options

- **A**: Confirm with the operations team the exact set of artifacts and acceptance criteria required for turnover, then locate and consolidate them into the agreed package.
- **B**: Inventory what already exists across the three locations so the project manager knows what is in hand before approaching the operations team.
- **C**: Direct the contractor to assemble the as-built drawings, register, and warranties from its portal into the turnover package, since most originate with them.
- **D**: Start consolidating the documents already in the PM's possession into a single turnover folder to build momentum against the deadline.

## Answer

A

## Explanation

The artifacts exist but the *requirement* does not — there is no agreed definition of a complete, acceptable package — so the first step is to fix the acceptance target with the receiving team; everything else is gathering against an unknown bar. That makes (A) correct. (B) is sensible early work but inventorying against no agreed list risks collecting the wrong set and still leaves acceptance undefined. (C) delegates much of the assembly correctly but presumes the contents are settled; without agreed criteria the contractor may deliver an incomplete or mismatched package. (D) produces visible progress yet, started before the required set is known, risks effort on documents that do not match what operations will accept.
