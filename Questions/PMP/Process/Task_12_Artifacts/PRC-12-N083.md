# Question

Midway through a hybrid medical-device project, testing flags that a delivered feature behaves in a way the regulatory affairs team says was never permitted. Before deciding whether to rework the feature or challenge the test result, the project manager has to establish why the feature exists and what else would be affected if it changed. Every artifact is current and well stored, yet assembling this picture is taking the team days of manual cross-referencing. Which artifact, had it been kept, would most directly have spared that delay?

## Options

- **A**: A requirements traceability matrix maintained across the requirement, design, build, and test artifacts.
- **B**: A decision log capturing, for each contested feature, the rationale and who approved building it that way.
- **C**: A change log recording every modification made to the requirements set since the baseline was set.
- **D**: A configuration management index identifying the controlled current version of every requirement, design, and test item.

## Answer

A

## Explanation

Each of these artifacts is real and useful, so the discriminator is which one answers the two questions the PM actually faces: where did this feature come from, and what is downstream of it if we touch it. Only the requirements traceability matrix holds the standing links from a requirement out to its design, build, and test items, letting the PM both trace the feature to its source and read off everything affected by a change in a single lookup (A) — precisely the cross-referencing the team is now doing by hand. (B) a decision log explains why a feature was built a certain way, valuable for the regulatory conversation but silent on which requirement and tests connect to it. (C) a change log shows how requirements evolved over time, not the static web of relationships between a requirement and its downstream items. (D) configuration management guarantees each artifact is the correct version but records no relationships among them, so the manual cross-referencing would remain.
