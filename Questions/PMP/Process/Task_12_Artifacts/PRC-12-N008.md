# Question

A new project manager inherits a struggling hybrid project. She finds that issues are tracked in three places: a spreadsheet kept by the previous PM, a channel in the team chat tool, and verbal items raised in stand-up that are never written down. Items fall through the cracks and stakeholders complain that the same problems recur. What should she do first?

## Options

- **A**: Ask the team to be more disciplined about recording issues in the existing spreadsheet.
- **B**: Consolidate the issues into a single authoritative issue log with clear ownership and status, and agree one place where all issues are henceforth recorded.
- **C**: Close all open issues and start the log fresh, so the new process is not weighed down by legacy clutter.
- **D**: Escalate the disorganised issue tracking to the sponsor as a governance failure by the previous project manager.

## Answer

B

## Explanation

The problem is a fragmented artifact with no single source of truth, so consolidating into one issue log with owners and a single agreed capture point directly fixes why items are lost. (A) urging discipline does not resolve the fact that there are three competing locations, so items will still scatter. (C) wiping open issues to start clean discards unresolved problems and their history, which is exactly what is causing recurrence. (D) blaming the predecessor to the sponsor does nothing to recover the lost issues and is not the first, constructive step.
