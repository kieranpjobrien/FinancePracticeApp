# Question

Which is the primary benefit of using a standard status report template across a programme?

## Options

- **A**: It removes the need for status meetings between the PM and sponsor.
- **B**: It guarantees that variances will be detected before they become issues.
- **C**: It produces consistent structure so readers know where to find information.
- **D**: It replaces the need for a separate project schedule artifact.

## Answer

C

## Explanation

Standard templates create predictability for both authors and readers, which is exactly the point of a programme-wide template. (A) is wrong because templates do not replace the dialogue that happens in status meetings. (B) is wrong because templates display variance but do not detect it on their own. (D) is wrong because reports describe progress against the schedule, they do not substitute for the schedule itself.
