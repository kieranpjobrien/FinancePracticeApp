# Question

A team member saves an updated design document over the prior version in a shared network folder, losing two weeks of edits. The folder has no version history. What should the project manager do to prevent recurrence?

## Options

- **A**: Require all design documents to be peer reviewed before saving to the shared folder.
- **B**: Move project documents to a repository that automatically retains version history and allows rollback.
- **C**: Ask team members to save dated copies (e.g. design_v3_2026-04-17.docx) each time they edit.
- **D**: Restrict edit rights to the document owner and require others to suggest changes by email.

## Answer

B

## Explanation

Effective artifact management relies on tooling, not discipline. (B) addresses the root cause systemically — any overwrite can be rolled back regardless of who made the mistake. (A) adds a review step but does not recover lost content if a save-over still happens. (C) relies on every team member remembering to follow a convention; it is fragile and creates a messy folder. (D) creates a single point of failure and slows collaboration without solving the underlying version-history gap.
