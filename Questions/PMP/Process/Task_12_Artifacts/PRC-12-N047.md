# Question

On a predictive avionics project a safety requirement is changed. Weeks later a design engineer is found to have built to the superseded version. Investigating, the project manager confirms the change went through the change control board, the requirements document in the repository shows the new wording, and the engineer was on the distribution list for the change notice. Yet the requirements traceability matrix still linked the old requirement to its original design elements and test cases, and that is what the engineer consulted. To prevent a recurrence, what should the project manager treat as the primary failure to correct?

## Options

- **A**: The traceability matrix is not maintained as part of change implementation, so an approved requirement change does not propagate to its linked design and test items.
- **B**: The change notice relied on a distribution list, so receipt and action by each affected engineer were never confirmed.
- **C**: The repository holds the requirements document and the matrix as separate artifacts, so the two can present different versions of the same requirement.
- **D**: The change board approved the requirement change without a defined impact analysis across the dependent artifacts.

## Answer

A

## Explanation

The engineer relied on the matrix precisely because that is the artifact meant to link a requirement to its design and tests; the breakdown is that implementing the approved change never updated those links, so (A) is the primary failure. (D) is tempting and related — an impact analysis would have *listed* the dependent items — but analysis without the matrix being updated still leaves the stale links the engineer trusted; the matrix maintenance is the gap that bites. (B) is a real weakness, yet the engineer was notified and the failure occurred at the artifact he consulted, not in the notification. (C) describes normal practice: requirements documents and traceability matrices are legitimately separate artifacts — the fault is not that they are separate but that one was not kept in step when the change was made.
