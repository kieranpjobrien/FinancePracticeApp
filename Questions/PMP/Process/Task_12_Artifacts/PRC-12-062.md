# Question

The organisation is migrating from one project management tool to another next quarter. The PM's project has twelve months of artifacts including a rich risk register and decision log. What should the PM do?

## Options

- **A**: Export everything to static PDFs before migration and treat those PDFs as the historical record after the transition completes.
- **B**: Wait until the migration finishes and then ask the team to recreate the critical artifacts directly in the new project tool.
- **C**: Plan the migration carefully: map fields between tools, validate critical artifacts post-migration, and keep the old tool read-only temporarily.
- **D**: Start keeping duplicate records in both tools now so the project has continuity no matter which tool ultimately prevails.

## Answer

C

## Explanation

PMI treats tool transitions as a configuration-management event requiring planning, validation, and temporary redundancy. (C) preserves integrity and traceability. (A) loses dynamic structure and queryability that live artifacts provide. (B) discards twelve months of history and relies on recall. (D) creates dual maintenance overhead and divergence risk without a transition plan.
