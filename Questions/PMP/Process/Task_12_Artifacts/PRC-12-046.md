# Question

A hybrid infrastructure programme stores architecture diagrams in a wiki, code in Git, and decision logs in email threads. Auditors cannot trace why a key design choice was made six months ago. What should the programme manager do FIRST?

## Options

- **A**: Assign an analyst to reconstruct the missing design decision history from the email archives before the auditors return.
- **B**: Define a single authoritative location for each artifact type, with versioning rules, and migrate current working artifacts there.
- **C**: Ask the auditors to accept the wiki as the system of record and close the audit finding on that basis.
- **D**: Escalate to the sponsor that the audit feels unfair given how the tools were originally set up at project start.

## Answer

B

## Explanation

PMI treats configuration management as establishing the single source of truth and versioning before reconstruction can help. (B) fixes the root cause by defining authoritative storage and traceability. (A) treats the symptom by patching history without preventing recurrence. (C) ignores the real governance gap. (D) is premature escalation that deflects rather than resolves.
