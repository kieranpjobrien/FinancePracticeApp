# Question

On a predictive avionics project, a stakeholder requests a change to a single safety requirement. The project manager wants to understand every design element, test case, and deliverable that the change would touch before the change board decides. Which artifact best supports this and what should the project manager do with it?

## Options

- **A**: Consult the requirements traceability matrix to trace the requirement through to its linked design, build, and test artifacts, and present the impact to the change board.
- **B**: Review the work breakdown structure to find which work packages mention the requirement.
- **C**: Search the issue log for any past entries referencing the safety requirement.
- **D**: Ask the test lead to estimate, from experience, which test cases the change would affect.

## Answer

A

## Explanation

The requirements traceability matrix exists precisely to link each requirement to its source, design, build, and verification artifacts, so it gives the complete, auditable impact set the change board needs. (B) the WBS organises deliverables and work packages but does not map individual requirements to design and test elements. (C) the issue log records problems, not requirement-to-artifact relationships, so it would miss most affected items. (D) relying on the test lead's recall is incomplete and error-prone for a safety-critical change when a traceability artifact is available.
