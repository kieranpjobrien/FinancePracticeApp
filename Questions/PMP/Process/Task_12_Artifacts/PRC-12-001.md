# Question

What is the PRIMARY purpose of a configuration management system for project artifacts?

## Options

- **A**: To control versions and changes so the correct version is available and all changes are traceable.
- **B**: To archive every project document at the end of the project for future reference and audit purposes.
- **C**: To restrict access to sensitive project documents to authorised users only and enforce security policy.
- **D**: To automate the creation of routine project documents and reduce manual effort across the delivery team.

## Answer

A

## Explanation

Configuration management exists to identify, control, and report on the state of project items so everyone works from the right version and changes are tracked end-to-end. (B) describes archiving, which is one downstream activity within CM. (C) describes access control, a related but separate concern. (D) describes document automation, which CM systems may support but is not their purpose.
