# Question

On a predictive hospital construction project, a site engineer builds a load-bearing wall from a drawing she pulled off a shared network folder last week. A subsequent inspection finds the wall is in the wrong position; the drawing had been superseded two revisions earlier, but the old file was still in the folder. The project manager needs to prevent a recurrence. What is the most effective action?

## Options

- **A**: Issue a memo instructing all staff to confirm with the design lead before using any drawing from the shared folder.
- **B**: Establish a configuration management system that controls revisions, marks superseded documents, and defines how the current approved version is identified and issued.
- **C**: Move all current drawings into a separate "approved" folder and delete the superseded versions from the network.
- **D**: Add a mandatory revision-check step to the daily site huddle so the team reviews the latest drawing register each morning.

## Answer

B

## Explanation

The root cause is the absence of controlled configuration management: there is no reliable mechanism that identifies the current approved version and removes obsolete ones from circulation, so (B) addresses the system rather than the symptom. (A) relies on individual diligence and a single point of contact, which does not scale and will fail again under time pressure. (C) deleting superseded versions destroys the revision history needed for traceability and audit, and a one-off cleanup does not govern future changes. (D) a daily check is a useful habit but still depends on people manually reconciling versions rather than the system guaranteeing only the current document is issued.
