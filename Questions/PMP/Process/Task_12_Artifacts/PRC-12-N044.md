# Question

A project manager is configuring the document repository for a cross-organisational infrastructure programme involving a client, two contractors, and a regulator. Commercial annexes must stay confidential between the PM and the client; design drawings must be available to all parties and always reflect the latest approved version; and the regulator must be able to verify, after the fact, exactly which drawing version was current on any given date. Which actions should the project manager take when configuring the repository? (Choose three.)

## Options

- **A**: Apply role-based permissions so each stakeholder group can reach only the artifacts appropriate to its role.
- **B**: Hold one controlled copy of each design drawing and grant all parties read access to the current approved version.
- **C**: Preserve a dated version history for each drawing so the version current on any past date can be reconstructed for the regulator.
- **D**: Give the two contractors edit access to the drawings so they can post their own revisions directly and keep the repository current.
- **E**: Mirror the confidential annexes into a separate client-only repository as a backup in case repository permissions are misconfigured.

## Answer

A, B, C

## Explanation

The three stated needs map directly to three controls: role-based permissions to protect the confidential annexes (A), a single controlled copy of each drawing so every party sees the same current version (B), and a retained dated version history so the regulator can reconstruct what was current on any date (C). (D) sounds efficient but lets contractors publish revisions without the approval step that "latest *approved* version" requires, breaking configuration control over the drawings everyone relies on. (E) is plausible-sounding defence-in-depth, but duplicating confidential annexes into a second store multiplies the places they can leak and creates a copy that can fall out of sync — the correct safeguard is to get the permissions in (A) right and audit them, not to spread the sensitive data.
