# Question

Multiple Scrum teams contribute to one product release. Dependencies between teams are managed in ad-hoc Slack threads. Release dates slip and blame flies between teams. What governance pattern should the programme manager introduce?

## Options

- **A**: Appoint a release director who personally assigns cross-team dependencies and decides the sequence of each release centrally going forward.
- **B**: Publish a shared dependency board with a joint coordination forum and explicit escalation rules for unresolved cross-team dependencies.
- **C**: Ask each team to absorb its own dependencies by negotiating directly with the other teams in Slack as issues emerge.
- **D**: Extend release cycles from monthly to quarterly so teams have more time to find and resolve dependencies between themselves.

## Answer

B

## Explanation

PMI's scaled-agile governance makes cross-team dependencies visible and assigns resolution rules. (B) does both. (A) centralises decisions that belong with the teams. (C) formalises the ad-hoc pattern that is already failing. (D) extends cadence without fixing the coordination mechanism.
