# Question

You discover that several technical decisions affecting your project's architecture have been made informally by senior engineers, outside the agreed technical governance process. The decisions appear sound, but nothing is documented and the architecture board never reviewed them. What should the project manager do FIRST?

## Options

- **A**: Document the existing decisions retroactively and update the project artefacts so the formal record matches the current reality.
- **B**: Talk with the engineering leads to understand why the process is being bypassed, then adjust governance to be fit-for-purpose.
- **C**: Raise a formal performance concern with the engineering manager so the senior engineers stop bypassing governance in future.
- **D**: Direct that no further technical decisions be made until each one has been formally approved by the architecture board in session.

## Answer

B

## Explanation

When a process is being bypassed by competent people, the process is usually mis-sized. (B) treats the bypass as a signal and tailors governance, then secures the documentation that was missing. (A) papers over the symptom without addressing why the process isn't being used. (C) jumps to a discipline frame before understanding the cause. (D) freezes work to enforce a process that the team has already shown to be unworkable in its current form.
