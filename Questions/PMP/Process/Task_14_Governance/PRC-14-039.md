# Question

During initiation, project governance was defined as just the sponsor and project manager. As planning progresses, the project's complexity has grown significantly and the current structure is clearly too thin. What should the project manager do?

## Options

- **A**: Leave the structure as defined in the charter; changing governance mid-project tends to create more confusion than it actually solves.
- **B**: Wait until the first significant issue forces a governance conversation so the need is obvious to everyone involved in the project.
- **C**: Propose an updated governance structure that matches the project's actual complexity, with additional stakeholders and oversight cadence.
- **D**: Ask the sponsor to personally absorb the additional oversight work rather than expanding the governance group with new members.

## Answer

C

## Explanation

Governance is meant to be right-sized for the project and can be revisited as understanding improves. (C) is the proactive tailoring PMI expects. (A) holds on to an initiation-era structure that no longer fits. (B) lets preventable issues occur first, which is exactly what governance is meant to avoid. (D) overloads the sponsor and still doesn't build the structure the project needs.
