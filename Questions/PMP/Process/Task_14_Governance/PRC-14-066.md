# Question

A project manager is setting up the governance framework for a hybrid programme that will replace a utility's customer billing system over two years. The sponsor asks what the framework must actually define so that it is more than a meeting schedule. Which elements belong in the governance framework? (Select THREE.)

## Options

- **A**: The decision rights that specify who is accountable and who is consulted for each category of decision.
- **B**: The escalation paths and the tolerance thresholds that trigger a decision being raised to a higher authority.
- **C**: The detailed task estimates and resource assignments for the first three months of delivery.
- **D**: The assurance and review arrangements that independently confirm the project is being run as intended.
- **E**: The day-to-day sprint board and the team's self-selected definition of done for each user story.

## Answer

A, B, D

## Explanation

A governance framework exists to allocate authority and provide oversight, so its core content is who decides what (decision rights), when a matter must move up (escalation paths and tolerances), and how the organisation independently checks the project is on track (assurance and review). (A), (B) and (D) are precisely these structural elements. (C) is operational planning output that lives in the schedule and resource plan, not in the governance framework; estimates change constantly and are not a governance artefact. (E) is team-level delivery mechanics that the team owns under whatever autonomy governance grants it — putting the sprint board and definition of done into the governance framework would confuse delivery execution with oversight, and these are not decisions the governance bodies make.
