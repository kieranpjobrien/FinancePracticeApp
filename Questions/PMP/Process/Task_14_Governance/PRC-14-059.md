# Question

A project manager is starting an agile project to build an internal expense-management tool. The stakeholders, eager to see working software, are pressing the team to begin the first sprint immediately, and several argue that a lightweight agile effort can settle decision rights, escalation paths, and oversight as questions actually come up. The project manager must decide how to handle governance relative to the push to start delivering. What should the project manager do?

## Options

- **A**: Define the governance structure — decision authority, escalation, and oversight bodies — as the project is set up, so it is in place before the first delivery decisions arise, then begin sprinting.
- **B**: Start the first sprint now to satisfy the stakeholders, and establish decision rights and escalation incrementally as concrete questions surface during early sprints.
- **C**: Run the first sprint to learn how the team and product actually behave, then design the governance after the first release so it fits what was observed.
- **D**: Stand up the full oversight bodies and decision processes first and hold off starting delivery until that governance is complete and ratified.

## Answer

A

## Explanation

Governance should be in place at the outset so decision rights, escalation paths, and oversight bodies are defined before the decisions and conflicts they govern begin to appear — and on an agile project this matters more, not less, because fast delivery generates those decisions quickly. Setting governance up as part of project start-up and then sprinting satisfies both the need for early clarity and the stakeholders' wish to get moving. (B) is the most tempting distractor because "incremental" sounds agile, but letting authority and escalation emerge only as questions surface leaves the team without a decision framework during the formative early sprints, which is exactly when costly disputes can take root. (C) goes further down the same path, deferring governance until after a release and exposing the whole first increment to ungoverned decision-making. (D) errs the opposite way, treating full governance as a gate that blocks all delivery; right-sizing means establishing the essential structure quickly and starting, not halting value delivery until a complete apparatus is ratified.
