# Question

A predictive project to build a new distribution centre is one component of a larger supply-chain transformation programme. The project's own steering committee approves a design change that optimises the distribution centre's throughput. Weeks later, the programme board discovers the change has invalidated an integration assumption shared with two other projects in the programme, forcing rework elsewhere. The project manager is asked how to prevent recurrence. What governance improvement best addresses the cause?

## Options

- **A**: Raise the project steering committee's approval threshold so larger design changes require more senior project-level sign-off.
- **B**: Require the project to circulate all approved design changes to the other projects as a courtesy notification after approval.
- **C**: Define that design changes affecting programme-level shared assumptions or interfaces must be referred to the programme board, not decided at the project tier alone.
- **D**: Merge the three projects' steering committees into one combined committee that approves every change across the programme.

## Answer

C

## Explanation

The failure is a decision-tier mismatch: a change with programme-wide consequences was decided within a single project's authority, where no one was positioned to see the cross-project impact. The fitting correction is to define that changes touching shared programme assumptions or interfaces are escalated to the programme board, which owns the integration view, while purely local changes stay at the project tier (C). Raising the project-level threshold (A) keeps the decision inside the project tier that could not see the programme impact, so the same blind spot remains. Post-approval courtesy notification (B) informs the other projects only after the damaging decision is made, too late to prevent the rework. Merging all three steering committees into one (D) is a heavy structural change that drags every trivial local decision to the programme level, sacrificing project-tier autonomy to solve a problem that targeted escalation handles precisely.
