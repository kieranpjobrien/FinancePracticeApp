# Question

A project manager on a predictive infrastructure programme encounters a recurring pattern: vendors keep asking who has the authority to approve interface changes between two work packages, and the answer is taking days to establish each time. Different team leads give conflicting answers. What should the project manager do first to address this?

## Options

- **A**: Escalate each interface change to the sponsor so a single named authority resolves every dispute consistently.
- **B**: Define and publish a decision-rights matrix that names who approves each decision type, including interface changes.
- **C**: Instruct the team leads to reach consensus among themselves before any vendor query is answered.
- **D**: Add an agenda item to the next steering committee meeting to discuss vendor frustration with response times.

## Answer

B

## Explanation

The repeated question is not "what is the answer to this change?" but "who is authorised to decide?" — a gap in the governance structure's decision rights. A decision-rights matrix (often expressed as RACI or a delegation-of-authority table) names the accountable party for each decision type up front, so the answer no longer has to be reconstructed each time (B). Escalating every interface change to the sponsor (A) overloads the sponsor with operational decisions that should be delegated and does not fix the underlying ambiguity. Requiring consensus among team leads (C) is what is already failing and has no defined tie-breaker. Raising vendor frustration at the steering committee (D) treats a symptom as a discussion topic without establishing the clear authority that would remove the delay.
