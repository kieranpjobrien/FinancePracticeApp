# Question

A project manager is tailoring the governance structure for a small, three-month hybrid pilot to test a new mobile check-in feature for a hotel chain. The pilot has one team, a modest budget, and a single business owner. The organisation's default governance model assumes a year-long programme with a full steering committee, monthly gates, and a change control board. Which considerations should guide how the project manager tailors the governance for this pilot? (Select THREE.)

## Options

- **A**: The level of oversight should be proportionate to the pilot's size, risk, and complexity rather than copied wholesale from the programme-scale default.
- **B**: Even when streamlined, the governance must still define who holds decision authority and how issues escalate, so accountability is never left undefined.
- **C**: The structure should be set at the programme-default level now and pared back later if the pilot proves it can run with less, so nothing essential is missed at the start.
- **D**: Any tailoring should remain consistent with the organisation's governance principles and any mandatory standards that apply regardless of project size.
- **E**: Because the pilot is short, its governance should be designed to match whichever delivery approach — predictive or adaptive — the pilot ends up using most.

## Answer

A, B, D

## Explanation

Tailoring governance means right-sizing oversight to the project's context while preserving the essentials. Oversight should be proportionate to size, risk, and complexity (A); even a lean structure must still define decision authority and escalation so accountability is explicit (B); and tailoring must stay within the organisation's governing principles and any non-negotiable mandatory standards (D). (C) sounds prudent but inverts the tailoring task: starting at the heavyweight default "to be safe" loads a three-month pilot with the very over-governance proportionate tailoring exists to prevent, and oversight is meant to be sized to risk up front, not imposed in full and trimmed later. (E) names a real influence on governance design but the wrong driver here: governance is shaped chiefly by the project's risk, complexity, and stakeholder context and by the organisation's principles — letting the eventual delivery cadence dictate the oversight, especially "whichever it ends up using most," puts method ahead of the size and risk that should govern a small pilot's structure.
