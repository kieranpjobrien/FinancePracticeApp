# Question

A project manager has been asked to stand up governance for a small, low-risk internal project to digitise a paper expense-claim form. The organisation's standard governance template prescribes a full steering committee, monthly assurance reviews, and a formal change control board. Applying it in full would require more senior time than the project itself consumes. What should the project manager do?

## Options

- **A**: Apply the standard template in full, because consistency with the organisation's governance approach takes priority over efficiency.
- **B**: Tailor the governance to the project's size and risk, retaining the essential controls and escalation paths but scaling down the bodies and review frequency to match.
- **C**: Run the project with no formal governance at all, since it is small and low-risk and any oversight would be disproportionate.
- **D**: Ask the sponsor to reclassify the project as larger so that the full governance template is clearly justified.

## Answer

B

## Explanation

Governance should be proportionate to a project's size, risk and complexity; the right response to an over-heavy template is to tailor it, keeping the essential controls and escalation routes while right-sizing the bodies and cadence (B). (A) elevates uniformity over proportionality and would burn senior time the project cannot warrant. (C) over-corrects into having no oversight, removing even the minimal controls and escalation a small project still needs and abandoning the principle that some governance is always required. (D) is a dishonest workaround that distorts the project's classification to force-fit a process, rather than adapting the process to the project as tailoring properly allows.
