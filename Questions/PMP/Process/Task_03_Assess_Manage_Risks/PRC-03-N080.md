# Question

On a hybrid clinical-trial data-management project, a risk has a 20% probability of causing a $250K loss if it occurs. A proposed mitigation would cut the probability to 5% but costs $60K to implement and does not change the loss size if the risk still occurs. The sponsor wants responses justified on expected-value grounds. What should the project manager recommend?

## Options

- **A**: Implement the mitigation, because it reduces expected loss by more than its cost, yielding a net expected benefit.
- **B**: Implement the mitigation, because reducing probability from 20% to 5% materially lowers exposure and is worth the cost.
- **C**: Accept the risk without mitigation, because the mitigation cost exceeds the expected loss it would remove.
- **D**: Accept the risk and instead hold the full $250K as contingency reserve to cover the loss if it occurs.

## Answer

C

## Explanation

Expected loss before mitigation = 0.20 × 250,000 = $50K; after mitigation = 0.05 × 250,000 = $12.5K, so the mitigation removes only $37.5K of expected loss yet costs $60K. On the sponsor's expected-value basis the spend is not justified, so accepting the risk is the right recommendation (C). (A) asserts a net benefit, but the cost ($60K) exceeds the expected loss avoided ($37.5K), so the arithmetic is wrong. (B) reaches the mitigation decision by emphasising the probability drop while disregarding that the cost outweighs the benefit, which the sponsor said must govern. (D) reserving the full $250K confuses the maximum possible loss with the $50K expected exposure and grossly over-reserves. The decision turns on comparing the $37.5K reduction in expected loss against the $60K cost.
