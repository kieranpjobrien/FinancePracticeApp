# Question

While planning responses, a project manager identifies a threat that a national data-residency bill, if passed, would force the company's data onto in-country servers. The project's application runs on the shared corporate data centre that three other programmes also use, and where workloads physically sit is decided by enterprise IT under its own infrastructure budget. The project manager has analysed the threat and must record a response strategy. Which is most appropriate?

## Options

- **A**: Avoid the threat by re-architecting the project's own application to run from a compliant in-country region, removing this project's exposure.
- **B**: Escalate the threat so the response is owned at the level that controls the shared infrastructure and budget affected.
- **C**: Mitigate the threat now by funding early migration of the shared data centre's relevant workloads to a compliant location.
- **D**: Accept the threat and hold project contingency to cover the project's share of any relocation once the bill becomes law.

## Answer

B

## Explanation

The lever that resolves this threat, where the shared workloads physically sit, is controlled by enterprise IT and the exposure is common to several programmes, so no single project can own an effective response; the correct strategy is to escalate to the level that controls the infrastructure and budget, where it is then managed. (B) fits. (A) re-architecting only this project's application is plausible but does not remove the shared exposure, may be infeasible on shared infrastructure, and could duplicate or conflict with an enterprise solution. (C) funding migration of shared workloads spends on assets and decisions the project does not own or control. (D) project contingency is sized for project-level risk, not an enterprise relocation shared across programmes, so accepting and reserving for it misplaces the response.
