# Question

On a hospital IT project, qualitative analysis is complete and the high-priority threats are ranked. Reviewing the ranked list, the project manager suspects that many of the top threats share a few common root causes—possibly one legacy vendor and one ageing data centre—and wants to confirm where exposure is concentrated so a few targeted responses could address several risks at once. What should the project manager do next?

## Options

- **A**: Categorise the threats using the risk breakdown structure to group them by source and confirm which root causes account for the most exposure.
- **B**: Run a probability and impact reassessment to re-rank the threats and make sure the most severe ones are correctly at the top.
- **C**: Develop an individual response plan for each high-priority threat in ranked order, starting from the most severe.
- **D**: Perform a Monte Carlo simulation to model the combined effect of the top threats on the overall schedule and cost.

## Answer

A

## Explanation

The project manager already has a ranked list and a specific hypothesis: that exposure clusters around a few shared sources. Confirming that calls for grouping the threats by source, which the risk breakdown structure does—revealing whether one vendor or facility drives many risks, so a single response can tackle several (A). (B) re-ranks by severity, but ranking is already done and severity does not show common causes. (C) jumps to planning one-by-one responses and would miss the chance to address several risks with one shared mitigation—the very thing the project manager is trying to find. (D) quantifies aggregate schedule and cost impact but tells you nothing about which sources the risks share. Grouping by source is the step that tests the project manager's concentration hypothesis.
