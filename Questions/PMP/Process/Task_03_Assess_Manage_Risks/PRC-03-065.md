# Question

A requested scope change would add $80K of certain value but introduces a new risk: a 25% probability of $200K schedule-recovery cost if integration fails. Other integration risks are independent of this change. What should the PM recommend on expected value grounds alone?

## Options

- **A**: Approve the change because expected added value exceeds the expected cost of the new integration-failure risk component by a clear margin.
- **B**: Reject the change because expected cost of the new risk ($50K) is less than value, but appetite and residual exposure matter here too.
- **C**: Reject the change because expected cost of the new risk ($50K) exceeds the value gained from the proposed scope addition for the project.
- **D**: Approve the change only if insurance or a transfer mechanism can be found for the integration-failure risk before any formal approval.

## Answer

A

## Explanation

Expected cost of the new risk = 0.25 × 200,000 = $50K. Expected added value = $80K. Net expected value = +$30K, so on EMV grounds approve. (A) applies the arithmetic correctly. (B) gets the arithmetic right but flips the conclusion. (C) inflates the expected cost incorrectly. (D) demanding insurance on every marginal risk isn't proportionate and isn't required by EMV analysis alone.
