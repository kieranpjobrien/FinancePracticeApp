# Question

On a transport ticketing project, each high-priority threat in the risk register already has an agreed response plan and a named risk owner. Reviewing recent status, the project manager finds that two owners have not updated their risks for several weeks, the early-warning triggers for those risks were never defined, and one owner has since rolled off the project. What should the project manager do first?

## Options

- **A**: Work with the affected owners to define the missing triggers and reassign the orphaned risk, so each high-priority threat has a current owner who knows what condition will require action.
- **B**: Take personal ownership of the two stale risks and the orphaned one so they are actively watched until the next review.
- **C**: Add the lapsed risk updates as a standing agenda item and raise the issue of overdue owners at the next weekly status meeting.
- **D**: Escalate the lapse in risk monitoring to the sponsor, since owners failing to maintain high-priority risks is a governance concern.

## Answer

A

## Explanation

Naming an owner is not enough—an owner can only act if a trigger tells them when to, and an orphaned risk has no owner at all. The first move is to close those specific gaps: define the missing triggers and reassign the risk so monitoring is real, not nominal (A). (B) makes the project manager a single point of failure for several risks and treats the symptom (no one watching) without fixing the cause (no triggers, no owner). (C) defers active risks to a routine meeting and only flags lateness rather than restoring monitoring. (D) escalates before the project manager has used the authority they already have to fix an in-project gap; escalation may follow if owners cannot be made effective, but it is not the first step.
