# Question

While planning responses for a retail point-of-sale rollout, the project manager has agreed a set of risk responses with the team but notices that several risks in the register have no named individual responsible for monitoring them and carrying out the response if needed. What should the project manager do to close this gap?

## Options

- **A**: Assign a risk owner to each affected risk so a specific person is accountable for monitoring it and implementing its response.
- **B**: Keep the project manager as the default owner of all risks to maintain a single point of accountability.
- **C**: Assign ownership only to the highest-priority risks, since lower-priority risks rarely require active monitoring.
- **D**: Leave ownership unassigned until each risk actually materialises, then nominate whoever is best placed at that time.

## Answer

A

## Explanation

Each identified risk should have a designated risk owner responsible for tracking it and executing the agreed response, which is exactly the gap described. (B) making the project manager own every risk is impractical and weakens accountability for individual risks that others are better placed to watch. (C) leaving lower-priority risks unowned means no one is watching for their triggers, so they can escalate unnoticed. (D) deferring assignment until a risk occurs removes any monitoring and the early-warning value of having an owner. Naming an owner per risk is the standard remedy.
