# Question

A project manager is opening risk identification for a complex airport-baggage-system upgrade and wants techniques that will surface risks the core team has not already thought of. Which approaches are well suited to broadening the set of identified risks beyond the team's existing assumptions? (Select all that apply.)

## Options

- **A**: Facilitated brainstorming and interviews with operations staff, vendors, and frequent stakeholders who see the system from different angles.
- **B**: A prompt list or risk-category framework used to systematically probe each category for overlooked threats and opportunities.
- **C**: Reviewing the risk register from a comparable past baggage-system project as a source of analogous risks.
- **D**: Calculating the expected monetary value of the risks currently in the register to decide which to fund.
- **E**: Building a tornado diagram from the current quantitative model to rank the risks already identified.

## Answer

A, B, C

## Explanation

Identification is about discovering risks, so techniques that bring in diverse perspectives (A), systematically probe categories (B prompt lists), and draw on comparable projects (C analogous/lessons-learned) all expand the set. (D) EMV is analysis and prioritisation of already-identified risks, not identification. (E) a tornado diagram is a sensitivity-analysis output that ranks known risks; it cannot reveal risks not already in the model. Both D and E presuppose the risks are already found, which is the opposite of the goal here.
