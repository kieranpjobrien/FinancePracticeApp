# Question

A logistics project has a contingency plan ready for the risk that a port strike disrupts inbound shipments. The risk register notes that a union ballot result is the trigger condition. The ballot has just returned in favour of strike action, and a strike is now likely within ten days. What should the project manager do?

## Options

- **A**: Execute the contingency plan now, because the agreed trigger condition has been met.
- **B**: Wait until the strike actually begins before executing the contingency plan, to avoid acting prematurely.
- **C**: Reassess the probability and impact of the risk and update its score before deciding whether to act.
- **D**: Convene the change control board to approve the contingency plan before it can be implemented.

## Answer

A

## Explanation

A trigger (warning sign) is defined in advance precisely so that the contingency plan is executed when it occurs; the ballot result is the stated trigger, so the plan should run now while there is still lead time. (B) waiting for the strike itself defeats the purpose of an early-warning trigger and squanders the response window. (C) re-scoring is unnecessary because the team already agreed this trigger warrants action; analysis here only delays response. (D) a pre-approved contingency plan does not require fresh CCB approval to execute when its trigger fires. The key is recognising that the trigger, not the event, is the action point.
