# Question

An agile team developing a multiplayer game identifies eleven new risks during a release-planning workshop, and the list now feels unwieldy. The team has capacity to plan responses for only a few risks this release and must decide where to direct that effort. What is the most appropriate next step?

## Options

- **A**: Rate each risk's probability and impact on an agreed scale, then focus response planning on the highest-rated few and watch-list the rest.
- **B**: Build a cost-and-schedule model for each risk so every one carries a monetary figure before the team decides which to action.
- **C**: Take the risks in the order they were raised during the workshop and plan responses down the list until this release's capacity is used up.
- **D**: Have the team plan a brief response for all eleven now so that none is left without a plan, accepting that each gets only shallow attention.

## Answer

A

## Explanation

A quick qualitative pass—rating probability and impact on an agreed scale—is the lightweight way to sort a long list so scarce response capacity goes to the risks that matter most, while the remainder stay visible on a watch list (A). Each alternative is a reasonable-sounding way to cope with too many risks, which is the point of discrimination. (B) reaches for quantitative modelling before any prioritisation; numbers on every risk are heavier than a release workshop needs and still don't tell the team where to start. (C) does ration effort to capacity, but order-of-raising is not a measure of severity, so it risks spending the budget on minor risks while a serious one waits. (D) refuses to prioritise at all, spreading limited capacity so thin that a critical risk receives the same shallow treatment as a trivial one. The call turns on assessing severity to focus effort versus over-analysing, using an arbitrary order, or diluting attention across everything.
