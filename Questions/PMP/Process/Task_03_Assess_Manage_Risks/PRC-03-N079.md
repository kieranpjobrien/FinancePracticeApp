# Question

A project manager for an offshore wind-cable installation is documenting responses for a high-impact weather risk. The team has agreed a contingency plan that will be executed if a defined trigger occurs, and the project manager now wants the response documentation to be robust. Which of the following correctly describe sound elements of this response planning? (Select all that apply.)

## Options

- **A**: Defining a clear trigger condition that signals when the contingency plan should be executed.
- **B**: Preparing a fallback plan to be used if the primary contingency plan proves ineffective.
- **C**: Estimating and reserving contingency for the residual exposure that remains after the planned response.
- **D**: Treating the contingency plan as the project's baseline schedule so the team works to it by default.
- **E**: Removing the original risk from the register once the contingency plan has been written.

## Answer

A, B, C

## Explanation

Robust response planning defines a trigger that fires the contingency plan (A), prepares a fallback plan for cases where the primary response fails (B), and reserves contingency for the residual risk left after responding (C). (D) a contingency plan is conditional and executed only if its trigger occurs; it is not the working baseline. (E) writing a response does not retire the risk; it stays in the register and is monitored until it can no longer occur. The correct options reflect that responses are conditional, layered, and leave residual exposure to be funded.
