# Question

A complex airport-baggage-system project encountered a serious integration failure mid-project that was eventually overcome through an innovative workaround. As the project closes, the project manager documents both the failure and the workaround in detail and lodges them in the organisation's repository. A team member, conscious that everyone is about to move on, asks what the lasting point of this is. Which justification best captures the primary reason this is done at closure?

## Options

- **A**: It gives the team a fair, shared account of how the failure was handled, so individuals are not later judged unfairly for a problem they collectively solved.
- **B**: Stored as an organisational process asset, it lets future projects anticipate the same failure and reuse the proven workaround, improving their outcomes.
- **C**: It feeds the close-out performance review, evidencing that the project overcame a major issue and so met its objectives despite the setback.
- **D**: It strengthens the organisation's relationship with the integration vendor by formally crediting the joint problem-solving that resolved the failure.

## Answer

B

## Explanation

All four describe genuine by-products of documenting a hard-won lesson, which is what makes this discriminating. But the primary, enduring reason lessons are captured at closure is organisational: once stored as a process asset they improve future projects by warning against the repeat failure and supplying a reusable solution (B). (A) is a real benefit of a fair, blameless record but is about this team's standing, not future projects. (C) uses the record to evidence this project's success — a backward-looking reporting use, not the forward value to the organisation. (D) is a relationship benefit that may follow, but crediting the vendor is incidental to why the lesson is documented and retained. The forward, cross-project value is the defining purpose.
