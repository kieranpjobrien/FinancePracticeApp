# Question

A hybrid infrastructure programme is closing its final phase and handing the product to a managed-service operator. The project manager is assembling the closure package and wants each closure outcome recorded in the correct artefact. Match each closure outcome (left) to the artefact where it primarily belongs (right).

## Items

- **1**: The agreed criteria the operator used to formally accept the delivered service
- **2**: Insights about what worked and what to do differently, captured for future programmes
- **3**: The confirmation that all procurement obligations have been met and contracts can be settled
- **4**: The arrangements for support, warranty cover, and the operator's responsibilities after go-live

## Matches

- **A**: Lessons learned register / repository
- **B**: Formal acceptance / sign-off record
- **C**: Transition (handover) plan
- **D**: Procurement closure documentation

## Answer

1-B, 2-A, 3-D, 4-C

## Explanation

Each outcome maps to the artefact built to hold it. Acceptance criteria and the operator's confirmation of acceptance live in the formal acceptance record (1-B), which is the evidence the product met requirements. Reusable insights belong in the lessons learned repository so future programmes can draw on them (2-A). Confirmation that contractual obligations are discharged sits in the procurement closure documentation, which underpins final settlement (3-D). Post-go-live support, warranty, and responsibility arrangements are defined in the transition plan that governs the handover to operations (4-C).
