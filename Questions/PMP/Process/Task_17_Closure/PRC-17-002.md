# Question

A hybrid product-development programme has just completed its design phase and is about to begin the build phase, which uses a different supplier and a partly new team. The programme runs phase reviews between stages. What should the project manager ensure happens as the design phase formally closes?

## Options

- **A**: Confirm the phase deliverables are accepted, capture lessons, and update artefacts so the build phase starts from an agreed baseline.
- **B**: Defer all phase documentation until the entire programme finishes so a single, consolidated closure report can be produced.
- **C**: Disband the design team immediately and rely on the build supplier to reconstruct any context they need from the contract.
- **D**: Skip the phase review because the programme is still active and a review is only required at final programme closure.

## Answer

A

## Explanation

Closing a phase mirrors closing a project in miniature: deliverables are accepted, lessons are captured while memory is fresh, and artefacts are brought to an agreed state so the next phase has a clean starting baseline. (B) loses time-sensitive knowledge and delays decisions that the build phase depends on. (C) destroys design context that the new supplier will need, raising rework risk. (D) misunderstands phase governance — reviews occur at each gate, not only at the very end.
