# Question

A project manager on a hybrid logistics-platform project is working four open issues, each at a different point. For each, identify the next action that does the most to move it forward.

## Items

- **1**: A high-priority issue with a named owner and a target date has sat untouched for two weeks; the owner says they have repeatedly asked another department's manager to approve the spend the fix needs, and been ignored.
- **2**: An interface defect was logged, fixed and closed last month; a near-identical defect has now been logged again, and the same fix has just been re-applied and the new instance closed.
- **3**: A finance lead's blocked sign-on has been resolved by IT, the lead confirms they can now complete validation, and the issue still shows "in progress" with two follow-up actions listed.
- **4**: A team member reports in stand-up that a slow shared environment has been costing the team time for several days; it is not on the issue log and no one is tracking it.

## Matches

- **A**: Log it with an owner, priority and target date so it can be tracked and worked.
- **B**: Escalate it to a governance body that can authorise the decision the owner cannot obtain.
- **C**: Confirm the agreed outcome is met, clear the open actions, and close it.
- **D**: Investigate the common cause across both instances before relying on the latest fix.

## Answer

1-B, 2-D, 3-C, 4-A

## Explanation

Item 1 is already logged and owned; what blocks it is an approval the owner has chased without success and the PM cannot compel, so the move that helps is escalation to a body that can authorise it (B) — not re-logging or re-assigning. Item 2 has been "fixed and closed" twice, so re-applying the same patch treats the symptom; the action with leverage is investigating why it recurs before trusting the latest fix (D). Item 3 is resolved in substance but left open with stale actions, so the value is in verifying the outcome, clearing the actions and closing it (C). Item 4 is a live problem nobody is tracking, so it must first be logged and owned before anything else can manage it (A). The discrimination is in state, not category: each item could superficially suggest "just log it" or "just close it", but the action that actually advances each one differs.
