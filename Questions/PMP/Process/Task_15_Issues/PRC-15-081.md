# Question

On a predictive automotive-components project, a quality engineer emails the project manager to report that a calibration jig on the assembly line has failed and is holding up final inspection of a batch due for shipment on Friday. The project manager agrees this needs to be worked. What should the project manager do first to manage this?

## Options

- **A**: Add it to the risk register and assign a probability and impact rating so it can be quantified.
- **B**: Record it in the issue log with a clear description, assign an owner, and set a target resolution date.
- **C**: Ask the quality engineer to keep working the problem and report back once the jig is fixed.
- **D**: Raise a change request to move the Friday shipment date so the batch is not recorded as late.

## Answer

B

## Explanation

A present problem that threatens a deliverable belongs in the issue log, where it can be described, owned and given a target date so it is actively tracked to closure. (A) treats a realised event as something that *might* happen; the risk register is for future uncertainties, not a jig that has already failed. (C) leaves the problem informal and untracked, with no owner of record or date, which is exactly what issue management exists to prevent. (D) jumps to changing a baseline before the issue is even logged or its real impact assessed, and a date slip may not be necessary if the jig is repaired in time.
