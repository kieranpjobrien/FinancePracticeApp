# Question

A project manager is sorting several newly surfaced items into the correct register. Each one is real and warrants attention; the task is to record only those that are ISSUES — a condition already affecting the project and needing action now — rather than risks. Which TWO should go in the issue log? (Select TWO.)

## Options

- **A**: A supplier has formally served notice that prices will rise 8% in 30 days; the increase has not yet taken effect but is now contractually committed.
- **B**: A key subject-matter expert resigned yesterday and the project has no identified replacement, halting the design work she led.
- **C**: A regulator has published a draft rule, open for consultation, that if enacted next year would change the project's reporting obligations.
- **D**: The integration testing environment went down this morning and is currently blocking the scheduled test cycle.
- **E**: A second vendor has warned that skilled testers are getting scarce and it may not be able to staff the next phase as planned.

## Answer

B, D

## Explanation

The give-away the question deliberately blurs is that "committed" or "highly likely" is not the same as "already affecting the project". (B) and (D) are conditions that have materialised and are impeding work *right now* — the SME has gone and design is halted; the environment is down and testing is blocked — so they are issues to be owned and worked immediately. (A) is the trap: the price rise is certain and imminent, but it has not yet occurred and its effect is still in the future, so it is a near-certain risk to plan for, not a present issue. (C) is a possible future rule change still in consultation, and (E) is a forecast staffing shortfall — both uncertain and forward-looking, hence risks. The test is not how likely or committed the event is, but whether its impact is being felt now.
