# Question

On a hybrid government grants-portal project, a key business stakeholder has been raising the same complaint at every steering meeting — that a reporting feature behaves incorrectly — but the development team's testing shows the feature working exactly as specified. The complaint reappears each fortnight, tempers are fraying, and the sponsor has asked the project manager to "make this go away." What should the project manager do?

## Options

- **A**: Meet the stakeholder to observe precisely how they use the feature and reconcile their expectation with the specification.
- **B**: Close the issue, citing the passing test results as evidence the feature is correct.
- **C**: Raise a defect and ask the team to change the feature to match the stakeholder's description.
- **D**: Escalate the stakeholder's conduct to the sponsor as an unreasonable and repeated demand.

## Answer

A

## Explanation

A recurring complaint that conflicts with passing tests usually signals a gap between the stakeholder's expectation and the documented specification, so the project manager must engage directly, observe the actual usage, and reconcile the two before concluding anything (A). Closing the issue on test evidence alone ignores a genuine perception problem and guarantees the complaint returns next fortnight (B). Changing the feature to match the description without understanding why the specification and expectation diverge risks reworking conforming functionality and may breach the agreed requirement (C). Escalating the stakeholder's conduct treats a legitimate, if mis-framed, concern as misbehaviour, which damages the relationship and still leaves the underlying mismatch unresolved (D).
