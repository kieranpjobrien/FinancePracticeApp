# Question

A scrum master returns from leave to find four open impediments on the board: a flaky test that intermittently fails the build, a licence renewal for a tool the team uses daily that lapses tomorrow, a request to tidy the team's wiki, and a developer waiting on access to a reporting database to start a story planned for later this sprint. Limited time is available before the next ceremony. Which impediment should the scrum master address first?

## Options

- **A**: The tool licence lapsing tomorrow, because it will halt the whole team's daily work if it expires.
- **B**: The flaky test, because an unreliable build erodes confidence in the team's pipeline.
- **C**: The database access, because it blocks a story already committed for this sprint.
- **D**: The wiki tidy-up, because it is quick and clears an item from the board.

## Answer

A

## Explanation

Triaging by impact and urgency together, the lapsing licence is the only impediment that is both imminent and capable of stopping the entire team's daily work, so it must be addressed first to prevent a far larger blockage (A). The flaky test genuinely hurts pipeline confidence but it is intermittent and degrades rather than halts work, so it can follow (B). The database access blocks one developer on a story planned for later in the sprint, giving it more slack than a team-wide stoppage tomorrow (C). The wiki tidy-up is low impact, and clearing it merely because it is quick lets a trivial task crowd out an urgent, high-impact one (D).
