# Question

On a hybrid satellite-ground-station project, an open issue has had a named owner and a target date for three weeks. The owner reports the only viable fix requires reallocating budget from another work package, a decision that sits above the project manager's delegated financial authority. The target date passes today. What should the project manager do?

## Options

- **A**: Close the issue as unresolvable at project level and capture it as a constraint for the next phase.
- **B**: Reassign the issue to a more senior team member who may be able to resolve it without extra budget.
- **C**: Extend the target date and ask the owner to keep searching for a fix that stays within current funding.
- **D**: Escalate the issue to the governance body that can authorise the budget reallocation, with the options and a recommendation.

## Answer

D

## Explanation

When resolving an issue needs a decision beyond the PM's authority, the correct move is to escalate it to the body that can make that decision, framed with options and a recommendation. (B) reassigns the work, but no individual on the team can grant budget the PM cannot grant, so it does not address the real blocker. (C) keeps the issue stalled by repeatedly chasing a fix that the owner has already shown is not viable within funding. (A) abandons a live, material issue rather than routing it to the authority that can actually unblock it, which is not how an unresolved issue exceeding authority should be handled.
