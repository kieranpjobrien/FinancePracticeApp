# Question

A project manager reviews the issue log and finds several issues that have shown a status of "in progress" for weeks with no change. Each is well described and is marked as being worked by the project team. At stand-up, team members each assume one of the others is handling them. Which adjustment best improves the likelihood that these issues get resolved?

## Options

- **A**: Assign a single accountable owner to each issue, with a defined next action and a due date.
- **B**: Set a standing weekly slot where the whole team reviews these issues together and agrees what to do next on each.
- **C**: Break each issue into smaller sub-issues so the work is less daunting and easier to pick up.
- **D**: Have each issue's progress reported to you directly each day so you can chase whoever is closest to it.

## Answer

A

## Explanation

The issues are well described and being "worked by the team", yet each person assumes another is on it — the gap is accountability, not clarity or effort. Naming one accountable owner per issue, with a concrete next action and a date, is what converts diffuse responsibility into a tracked, closeable item. (B) creates a recurring forum but shared review without a named owner can leave each issue still belonging to "the team". (C) makes the work smaller but smaller unowned tasks are still unowned. (D) routes status to the PM but "chasing whoever is closest" never fixes the fact that no one is actually accountable. The discrimination is recognising that ownership, not visibility, description or task size, is the missing element.
