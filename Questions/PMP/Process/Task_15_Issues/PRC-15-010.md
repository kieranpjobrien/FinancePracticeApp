# Question

Which set of fields best represents the minimum content of an issue log entry?

## Options

- **A**: Description, priority, assigned owner, status, target resolution date, and resolution details.
- **B**: Description and the team member responsible for causing the issue, so root causes are clearly attributed.
- **C**: Description and current status, with all other fields handled verbally during standups.
- **D**: Description, the affected work package, and the requested change request number, since every issue feeds change control.

## Answer

A

## Explanation

A useful issue log captures what the issue is, how urgent it is, who owns it, where it stands, when it is due, and how it was ultimately resolved. (A) covers that. (B) is blame-oriented and skips most management fields. (C) loses traceability the moment standup ends. (D) wrongly equates issues with change requests; not every issue triggers one.
