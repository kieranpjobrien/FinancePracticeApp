# Question

A project manager inherits a project where issues are surfaced in meetings but rarely resolved, and stakeholders complain that nothing seems to move. The PM has limited time to change the process and wants the changes that will most directly restore momentum. Which THREE actions best fit that aim? (Select THREE.)

## Options

- **A**: Maintain a single, visible issue log reviewed on a regular cadence, with each entry's status and next action kept current.
- **B**: Assign every open issue a single accountable owner with a defined next action and a date to report back.
- **C**: Define clear escalation criteria so issues that stall past a threshold are promptly raised to the right level.
- **D**: Require every issue to be analysed to full root cause and a formal corrective-action plan before it can be worked.
- **E**: Capture every issue in a detailed central register with rich metadata, reviewed in depth at the monthly steering committee.

## Answer

A, B, C

## Explanation

The give-away the question conceals is that the project's failure is momentum — issues surface but nobody owns or moves them — so the fixes must create ownership, visibility, and a forcing function. A current single log (A), single-owner accountability with a report-back date (B), and escalation criteria that unstick stalled items (C) attack exactly that. (D) is good discipline for significant problems but mandating full root-cause analysis and a corrective-action plan *before any work* adds front-loaded process that would slow resolution further — the opposite of what a "nothing moves" project needs. (E) improves the record but pushes substantive review to a monthly cadence; rich metadata reviewed once a month does not give the frequent, owner-driven follow-through that restores movement.
