# Question

How frequently should the project manager review the issue log?

## Options

- **A**: Once at the start of each phase and once at the end, with no review in between.
- **B**: Regularly through the project, typically at each status meeting, to monitor progress, refresh priorities, and prevent stalls.
- **C**: Only when stakeholders specifically request a review of open issues.
- **D**: Annually, or whenever the project has been running long enough to accumulate enough issues to make a review worthwhile.

## Answer

B

## Explanation

The issue log is a living document and needs regular attention. (B) reflects standard PMI practice. (A) leaves the log unmanaged for most of each phase. (C) makes review reactive to stakeholder pressure. (D) is far too infrequent for any project of meaningful pace.
