# Question

A project manager on a predictive ERP-deployment project is coaching a new team member on where different items belong. Match each item the team member describes to the correct project artefact or destination.

## Items

- **1**: A finance lead is currently unable to log in to the test system, halting their validation work today.
- **2**: A possible future shortage of trained testers next quarter that has not yet happened.
- **3**: A stakeholder's request to add two new report types not in the agreed requirements baseline.

## Matches

- **A**: Risk register
- **B**: Issue log
- **C**: Change control process

## Answer

1-B, 2-A, 3-C

## Explanation

A present problem stopping work today (1) is a current issue and belongs in the issue log (B). A possible future event that has not yet occurred (2) is an uncertainty and belongs on the risk register (A). A request that expands the agreed scope (3) is a change and must go through the change control process (C) for impact assessment and a decision. The distinction turns on timing and nature: issues are now, risks are potential, and scope additions are changes.
