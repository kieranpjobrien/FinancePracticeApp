# Question

At closure of a predictive cloud migration, the PM discovers operations has been given access credentials to production systems but not to the monitoring dashboards, configuration repositories, or incident runbook portal. What should the PM do?

## Options

- **A**: Grant operations production credentials only for now, since other tools can be provisioned individually as operations encounters specific incidents requiring them later.
- **B**: Escalate the access gap to the security team and ask them to issue appropriate credentials across all tools during weeks following project closure now.
- **C**: Ask the development team to continue handling monitoring and configuration changes on behalf of operations until the necessary tool access is provisioned fully.
- **D**: Treat the tooling access as part of handover scope, map every tool operations needs, and confirm credentials before formally closing the project.

## Answer

D

## Explanation

Knowledge transfer is incomplete without the tools that turn knowledge into action, and the PM is responsible for a full operational handover. (D) scopes the handover properly and confirms readiness. (A) leaves operations unable to see or change what they own. (B) escalates scope the PM should already own. (C) perpetuates development dependency the handover is meant to end.
