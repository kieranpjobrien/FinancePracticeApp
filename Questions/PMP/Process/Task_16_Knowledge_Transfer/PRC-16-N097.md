# Question

A project manager set up a shared repository for a hybrid pharmaceutical-manufacturing project, but six months in the team has stopped trusting it and drifted back to personal copies. Looking into it, the PM finds documents are going stale because no one is responsible for updating them, and even where content is current, people cannot tell which version is the live one. The PM wants the repository trusted again. Which TWO measures will most directly fix the underlying problems? (Select TWO.)

## Options

- **A**: Assign a named owner to each area of content, with a defined review cadence, so documents are kept current and someone is accountable for them.
- **B**: Apply consistent version and status labelling (e.g. draft / approved / superseded) so anyone can immediately identify the current document.
- **C**: Audit the repository now to remove or archive the out-of-date documents so only current material remains.
- **D**: Restrict edit rights so only designated authors can change documents, reducing uncontrolled or accidental edits.
- **E**: Add a clear folder structure and search so people can find the right document quickly.

## Answer

A, B

## Explanation

The discriminator is which measures fix the two stated *causes* — content going stale because nobody owns it, and no way to tell what is current — rather than treating symptoms or adjacent issues. (A) installs ownership and a review rhythm, the direct remedy for staleness and the missing accountability. (B) makes currency visible through status labelling, so the "which version is live" problem disappears. (C) a clean-up is genuinely useful and tempting, but it fixes today's stale content without preventing recurrence; without owners it will be stale again in months. (D) tightening edit rights improves change control and is sound governance, but the team's problem was neglect, not uncontrolled editing, so it does not address why documents went stale or were unidentifiable. (E) structure and search help findability, which was not the failure described — people could find documents, they just could not trust or date them. The fine margin is the root-cause pair (A, B) versus the worthwhile-but-adjacent fixes (C, D, E): only ownership-with-cadence and currency-labelling remove the two causes named.
