# Question

A project manager inherits a programme where a well-structured knowledge repository already exists, but nobody uses it. Team members say it is faster to message a colleague than to search the repository, and they are not confident the content is current. The PM wants the repository to become the team's default source. Which action will most improve this?

## Options

- **A**: Assign clear ownership for each content area with a review cadence, and integrate the repository into the team's existing workflow so updating it is part of completing work, not a separate task.
- **B**: Run a training session demonstrating the repository's search features so the team can find content faster than messaging a colleague.
- **C**: Introduce a rule that questions answerable from the repository must not be asked in the team chat.
- **D**: Migrate the repository to a more modern platform with better search and tagging to improve the user experience.

## Answer

A

## Explanation

The two stated barriers are stale content (a trust problem) and the repository being separate from how work happens (a friction problem); fixing both is what drives adoption. (A) addresses currency through owned, reviewed content and reduces friction by embedding updates into the workflow. (B) tackles only discoverability, not the trust issue, and the team's real complaint is confidence in the content. (C) mandates behaviour without making the repository trustworthy, so people comply grudgingly or work around it. (D) treats a tooling symptom; a better platform with the same neglected, stale content still will not be used.
