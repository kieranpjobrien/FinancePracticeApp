# Question

A predictive infrastructure project has completed. The PM drafts lessons learned but the PMO says similar lessons have been filed from previous projects and nothing has changed. The sponsor wants genuine uptake this time. What should the PM do?

## Options

- **A**: Publish the lessons to the repository as required by closure procedure and note the PMO concern in the closure pack narrative accordingly.
- **B**: Skip the lessons learned exercise since identical content has been filed before, and prioritise the team's remaining closure workload instead now.
- **C**: Work with the PMO to identify why past lessons did not change practice, then co-design a dissemination and adoption path.
- **D**: Expand the lessons document with more detail and examples so future readers find the material more compelling than prior entries have been.

## Answer

C

## Explanation

PMI treats lessons learned as an organisational learning input, which means dissemination and uptake matter as much as capture. (C) addresses the actual failure point: adoption. (A) files the lessons the same way that has already failed. (B) abandons the capture step entirely. (D) assumes volume solves an adoption problem that is structural, not narrative.
