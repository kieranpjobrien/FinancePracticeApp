# Question

A project manager overseeing a hybrid rail-signalling programme must respond to four continuity situations arising in the same week. Match each situation to the response that best fits it. Each response is used once.

## Items

- **1**: A specialist who keeps the team's tricky calibration step working leaves in two weeks; one named engineer will own that step for the next year and must be able to perform it.
- **2**: A region that has finished construction is being handed to a separate team that will operate it day to day once it goes live next month.
- **3**: An engineer joining tomorrow will chair a review and needs to know what is currently open and what is being watched on the programme right now.
- **4**: The programme closes in three months and the PMO wants what was learned to be available to programmes that start afterwards.

## Matches

- **A**: A structured operational-readiness handover with runbooks and joint working sessions
- **B**: Hands-on pairing on the live task with the departing specialist
- **C**: Finalised lessons learned archived into the organisation's repository
- **D**: A walkthrough of the current risk register, issue log, and recent status reports

## Answer

1-B, 2-A, 3-D, 4-C

## Explanation

Each situation is matched by the knowledge type and time horizon it presents, and the tempting mis-maps are what make it discriminate. (1) one named person must be able to *perform* a tricky, person-held step within two weeks — hands-on pairing on the live task (B) builds doing-ability while the source is present; archiving lessons (C) would only describe it. (2) a finished scope passing to a separate operating team is an operational transition needing readiness artefacts and joint sessions (A), not the single-person pairing of (1). (3) an incoming person needs current-state awareness for tomorrow's review, fastest from the live risk, issue, and status artefacts (D); pairing or a handover would be too slow and aimed at the wrong knowledge. (4) durable reuse by future programmes after closure is served by finalised lessons in the shared repository (C). The traps are using pairing (B) for the current-state need (3), or sending the live operational handover (2) to a lessons archive — each applies a mechanism built for a different horizon.
