# Question

In an agile programme's retrospectives the team raises the same minor tweaks each sprint while the deeper recurring problems never come up, though they surface readily in side conversations and one-to-ones. The project manager has noticed this began under the previous manager and that the team gives fuller, more candid input when assessments are clearly not in play. The project manager wants the retrospectives themselves to start producing the real issues. What should the project manager do first?

## Options

- **A**: State plainly that retrospective content stays out of any individual assessment, keep it that way, and visibly act on what gets raised so the team sees candour is safe and useful.
- **B**: Run a short anonymous input round before each retrospective so the harder issues can be tabled without anyone being identified as the source.
- **C**: Have a neutral facilitator from outside the team run the retrospectives so members feel less exposed when speaking.
- **D**: Restructure the retrospective with a format that drives from surface symptoms to underlying causes, so the deeper problems are reached by design.

## Answer

A

## Explanation

The tell is that candour appears everywhere except the retrospective and tracks to a change of manager — the constraint is trust in how retrospective input gets used, not the technique. (A) rebuilds that trust at the source by separating the forum from assessment and proving it through consistent, non-punitive follow-through; once safe, the issues already aired privately can surface openly. (B) gets problems onto the table but via anonymity, which works around the distrust rather than resolving it and tacitly confirms that being named is risky. (C) lowers exposure by changing who runs the session, yet the team stays guarded as long as it doubts where its words end up. (D) is a fine method for a team that already speaks freely, but better structure cannot extract what people are choosing to withhold. Restoring trust must come before facilitation tricks, anonymity, or format changes can bite.
