# Question

A Scrum team has built an analytics tool, but user adoption three months post-launch is only 20 percent. The users received demos and documentation at handover. Business outcomes depend on wider adoption. What should the PM do?

## Options

- **A**: Run another round of demos to the same user group so the tool's capabilities are reinforced through repeated live exposure now.
- **B**: Add more written documentation addressing the features users appear to be ignoring, based on usage telemetry collected since launch day.
- **C**: Request the sponsor mandate adoption across the user population so that usage rises to justify the original business case projections again.
- **D**: Diagnose adoption barriers through interviews, address motivation and ability gaps using a change model like ADKAR, and iterate support.

## Answer

D

## Explanation

Low adoption after training typically reflects missing change-management components, not missing information. (D) diagnoses where users actually sit on the adoption curve and treats the real barrier. (A) repeats content that already failed to produce adoption. (B) adds documentation when demos did not solve the problem. (C) forces compliance instead of earning engagement.
