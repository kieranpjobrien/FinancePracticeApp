# Question

A project manager on a two-year hospital construction project has historically held a single lessons-learned workshop at project closeout. The sponsor notes that previous projects captured rich insights too late to act on them, and that staff turnover meant key contributors had already left by closeout. The PM wants to change the approach for this project. What is the most effective change?

## Options

- **A**: Capture lessons learned continuously throughout the project — at the end of each phase and at significant milestones — feeding actionable items back into the current project where possible.
- **B**: Bring the closeout lessons-learned workshop forward to the midpoint of the project so more contributors are still present.
- **C**: Require every team member to maintain a personal lessons log and submit it before they leave the project.
- **D**: Hold two lessons-learned workshops, one at the end of each year, to capture insights while memories are fresh.

## Answer

A

## Explanation

Lessons learned deliver the most value when captured continuously and fed back in time to benefit the project that generated them, which also avoids the turnover problem. (A) embeds capture into the phase and milestone cadence so insights are both timely and usable now. (B) and (D) are improvements over a single closeout session but remain periodic snapshots that still miss insights occurring between events and act too late for many. (C) gathers raw input but personal logs submitted at exit are not reviewed, validated, or acted on, so the knowledge does not re-enter the project.
