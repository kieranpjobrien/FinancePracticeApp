# Question

A software integration project has a fixed go-live date. The only database specialist is over-allocated: two activities need her in the same week. One of those activities is on the critical path; the other has four days of free float. The project manager wants to keep the committed end date. What should the project manager do?

## Options

- **A**: Shift the activity that has free float later within its float so it no longer overlaps the specialist's critical-path work.
- **B**: Apply resource levelling across both activities and accept the revised completion date the levelling produces.
- **C**: Move the critical-path activity later so the two no longer overlap, and re-baseline the schedule to the new finish.
- **D**: Split the specialist's time across both activities in the shared week so each progresses at half rate.

## Answer

A

## Explanation

Because one activity carries free float, the conflict can be resolved by delaying that activity within its float — this is resource smoothing, and it preserves the critical path and the committed date. (B) levelling does not respect the float limit and is used when the resource ceiling, not the date, is the hard constraint; here it would needlessly push the finish. (C) delaying the critical-path activity directly moves the end date the PM is trying to protect, and there is no float there to absorb it. (D) half-rating a single specialist across two tasks slows the critical-path work, likely breaching the date, and adds context-switching loss — float on the other activity makes this unnecessary.
