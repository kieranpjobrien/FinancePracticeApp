# Question

You run a stable Scrum team of seven that has been delivering at a consistent velocity for six sprints. The portfolio manager wants to pull one developer to help an unrelated initiative for "three or four weeks." What should the project manager do?

## Options

- **A**: Agree to the request because portfolio-level priorities outrank a single team's velocity on a project.
- **B**: Decline the request to protect team stability and the established sprint velocity baseline for the project.
- **C**: Quantify the velocity and delivery impact and present the trade-offs to the portfolio manager first.
- **D**: Bring the issue to the project sponsor and ask them to defend the current team composition.

## Answer

C

## Explanation

PMI expects the PM to enable informed decisions through transparent impact analysis rather than capitulating, refusing, or escalating. (C) gives the portfolio manager the data — likely velocity drop, sprint goals at risk, delivery date impact — so the trade-off can be made consciously. (A) accepts a change that affects the project baseline without analysing the impact. (B) is unilaterally protectionist and ignores legitimate portfolio needs. (D) escalates prematurely; the PM should engage the portfolio manager directly first and only involve the sponsor if the issue cannot be resolved at that level.
