# Question

A two-year transformation programme is approved with a high-level budget. Only the first six months of work is well understood; later phases depend on outcomes that are not yet known. Finance wants a single fixed cost baseline for the whole programme now so it can lock the annual capital plan. What should the project manager propose?

## Options

- **A**: Produce a fully detailed bottom-up estimate for all 24 months now so finance has the single fixed figure it has asked for.
- **B**: Use the approved high-level number as the firm programme baseline and manage any later divergence through the change control process.
- **C**: Baseline the near-term work in detail and carry the later phases as a planning estimate with stated ranges, refining them through rolling-wave planning as each phase approaches.
- **D**: Baseline only the first six months as the committed figure and present the later phases as a single point estimate equal to the remainder of the approved high-level budget.

## Answer

C

## Explanation

Rolling-wave planning lets the imminent work be estimated to a baseline level of confidence while distant, poorly understood work is held as a ranged planning estimate and progressively elaborated — this gives finance a defensible figure without inventing false precision. (A) a detailed bottom-up estimate for undefined future work fabricates certainty that does not exist and will churn constantly. (B) treating an unvalidated high-level number as firm guarantees a flood of change requests and undermines the baseline's credibility. (D) is closest to right because it does baseline the known work, but collapsing the unknown phases to a single point figure reintroduces exactly the false precision rolling-wave avoids; the later work should carry a stated range, not a spurious point value, until it is elaborated.
