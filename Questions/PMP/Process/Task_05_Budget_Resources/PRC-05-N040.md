# Question

A project manager reviews the resource histogram for the build phase and sees demand for test engineers briefly spike to three times the steady-state level, then drop back. The team's senior engineers are already committed elsewhere during the spike, the project finish date must hold, and hiring to the peak would leave new engineers idle on either side of it. The project manager is deciding how to handle the spike. What is the best planning response?

## Options

- **A**: Check whether the spike activities carry enough float to be redistributed across a wider window, and if so spread them so demand stays nearer the steady state without moving the finish date.
- **B**: Level the load by delaying part of the spike work until engineers free up, accepting a later finish if the activities cannot absorb the shift within their float.
- **C**: Staff to the peak by bringing on short-term contract engineers for the spike window, holding both scope and the finish date unchanged.
- **D**: Subcontract the spike scope as a discrete package to a specialist test house, transferring the peak demand off the project's own resource pool.

## Answer

A

## Explanation

The peak is transient and the finish date must hold, so the project manager first tests whether the spike activities have float to be redistributed and, if they do, smooths them across a wider window — keeping demand near a sustainable level without extending the schedule (A). (B) levelling delays the work to fit available resources and will push the finish date if there is insufficient float, which the date constraint here forbids as a first choice — levelling is the fallback when float runs out, not the opening move. (C) contracting to the peak is a legitimate option but adds cost and onboarding for a brief spike and is only justified once an in-schedule smoothing solution is ruled out. (D) subcontracting the package transfers the demand but fragments the test scope and adds interface and quality-oversight burden for a short spike — a heavier intervention than the situation needs before float has even been checked.
