# Question

The resource histogram for an upcoming phase shows the test team over-allocated above its capacity line in three separate weeks. The over-allocated tasks all sit on the critical path, and the phase-end milestone is contractually fixed and cannot move. Bringing in additional testers is not an option for this phase. How should the project manager best resolve the over-allocation?

## Options

- **A**: Apply resource smoothing, shifting the over-allocated work only within its available float so the team stays within capacity and the finish date is unaffected.
- **B**: Apply resource levelling, adjusting start and finish dates to respect the capacity limit even though this is likely to extend the critical path and the milestone.
- **C**: Re-estimate the test activities with shorter durations so the same work fits within the team's available weekly capacity.
- **D**: Reduce the test scope for the over-allocated weeks so demand falls under the capacity line while holding the milestone.

## Answer

A

## Explanation

The constraint is a fixed milestone the project manager cannot breach, so the optimisation technique chosen must preserve the end date: resource smoothing moves work only within available float and never delays the critical path, which is exactly what a fixed milestone demands. (B) resource levelling respects the capacity limit but allows the dates to move, and on critical-path work that pushes the very milestone the scenario says is fixed — the right technique for the wrong constraint. (C) shortening durations to force a fit is not a resource-optimisation response; it manufactures a schedule by assumption and understates the work. (D) cutting test scope on critical safety-of-delivery work to relieve a capacity peak trades a resource problem for a quality risk and is not an optimisation of the existing plan.
