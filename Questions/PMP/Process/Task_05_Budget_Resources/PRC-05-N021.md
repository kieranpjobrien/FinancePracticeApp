# Question

On a predictive instrumentation project, the resource histogram shows the lead controls engineer over-allocated during a window where two work packages overlap. One of the two packages sits on the critical path with zero float; the other has several days of free float. The engineer is the only person qualified for both. The sponsor has made the committed finish date the project's top priority. How should the project manager resolve the over-allocation?

## Options

- **A**: Move the float-bearing package later within its free float so it no longer overlaps the critical-path package, keeping the engineer within capacity and the finish date intact.
- **B**: Delay whichever package resolves the over-allocation soonest, accepting the resulting shift to the finish date as the cost of keeping the engineer within capacity.
- **C**: Keep both packages on their planned dates and authorise overtime for the engineer across the overlap so neither package moves.
- **D**: Split the critical-path package's activities so part of its work shifts off the overlap window, relieving the engineer without touching the other package.

## Answer

A

## Explanation

Only the non-critical package carries float, so shifting that package within its free float removes the overlap without moving the critical path or the finish date — the sponsor's priority. (B) describes levelling without regard to which package has float; delaying the critical-path package, or delaying the non-critical one beyond its float, would push the finish date when a float-only shift was available. (C) overtime keeps the dates but adds cost and fatigue to solve what free float can absorb for nothing, and sustained overtime on a sole-qualified resource carries its own delivery risk. (D) splitting the critical-path package re-times critical work and risks the finish date, whereas the slack to exploit lies entirely in the other package.
