# Question

A predictive offshore-survey project is 40% complete and running at CPI 0.90. The overrun to date came from a vessel mobilisation problem that has been fully resolved; the remaining survey work is routine and the team expects it to run to the original estimates. The schedule is on plan. The project manager is preparing the estimate at completion (EAC) for the steering group. Which TWO actions are most appropriate?

## Options

- **A**: Forecast the remaining work at the budgeted rate and present EAC = AC + (BAC − EV), since the driver of the past overrun no longer applies.
- **B**: Present EAC = BAC / CPI as the primary forecast so the steering group sees the outcome if current cost efficiency were to persist.
- **C**: Include EAC = BAC / CPI alongside the primary forecast as a conservative bound, and state the assumption that separates the two.
- **D**: Hold EAC at BAC for now, since at 40% complete the cost data is too immature to support re-forecasting the budget.
- **E**: Reset the cost baseline to current actuals so the reported variance reflects the resolved cause and future reporting starts clean.

## Answer

A, C

## Explanation

The overrun's cause is resolved and the rest will run to plan, so the defensible primary forecast adds the remaining work at budgeted cost: EAC = AC + (BAC − EV) (A). Presenting the CPI-persists figure (EAC = BAC / CPI) as a conservative bound alongside it, with the assumption stated, is good practice because it shows the steering group the range and the judgement behind the chosen number (C). (B) is wrong as the primary forecast: applying CPI to all remaining work assumes the resolved, one-off problem will keep recurring, which the evidence contradicts. (D) treats 40% complete data as unusable, but EAC is re-forecast from actual performance, not frozen at BAC. (E) re-baselining to actuals erases the variance signal and the audit trail; the baseline changes only through approved change control, not to tidy a report.
