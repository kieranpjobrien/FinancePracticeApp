# Question

A project manager is finalising the budget. During execution, performance will be measured by earned value against the cost baseline. The sponsor wants a pool of funds held for genuinely unforeseen, out-of-scope events, releasable only with their authorisation. The project manager must decide how this pool sits in the budget so that earned value reporting stays meaningful. How should the pool be structured?

## Options

- **A**: Hold it as management reserve above the cost baseline, so it is excluded from earned value measurement and is added to the baseline only if authorised work draws on it.
- **B**: Include it in the cost baseline as a project-level contingency line, so the full sponsor-approved budget is what performance is measured against.
- **C**: Distribute it proportionally across the control accounts so the reserve is positioned where unforeseen work would most likely arise.
- **D**: Hold it as management reserve but include it in the performance measurement baseline so cost variance reflects the total funds the sponsor has committed.

## Answer

A

## Explanation

Funds for unknown-unknowns are management reserve, and to keep earned value meaningful they must sit outside the cost baseline: EV measures performance against planned work, and the reserve is not planned work, so it is added to the baseline only via change control when authorised. (B) folding it into the baseline as contingency mislabels unknown-unknowns as identified-risk contingency and inflates the measurement baseline, distorting CV and SPI. (C) spreading it across control accounts buries the reserve, removes the sponsor's release control, and contaminates account-level variance with funds that back no defined work. (D) management reserve by definition is outside the performance measurement baseline; including it there is a contradiction that breaks the variance figures it claims to improve.
