# Question

An agile programme integrates a third-party vendor that delivers a key service component each sprint. The vendor's increments pass its own internal tests but repeatedly fail the programme's integration checks, forcing rework at each sprint's end. Investigation shows the vendor genuinely does not know the programme's integration standards, which were never written into the contract or any working agreement. What should the project manager do to address the root cause?

## Options

- **A**: Agree a shared definition of done with the vendor, including the integration standards, and embed it in the working agreement so increments must meet it before hand-off.
- **B**: Give the vendor read access to the programme's integration test suite so it can run the same checks before each hand-off.
- **C**: Add a representative from the vendor to the programme's integration ceremonies so issues surface during the sprint rather than at the end.
- **D**: Establish a continuous-integration pipeline that runs the programme's integration checks automatically on every vendor commit.

## Answer

A

## Explanation

The root cause is that the integration standards were never made a shared, agreed expectation, so the most complete fix is to define a shared definition of done that includes those standards and bind it into the working agreement, making integration-readiness part of what "done" means for the vendor (A). Sharing the integration test suite (B) is a strong enabling step and may well follow, but access to tests without an agreed standard of done leaves it optional and does not establish the obligation that increments meet the bar before hand-off. Embedding a vendor representative in integration ceremonies (C) improves communication and surfaces issues earlier, yet it addresses visibility rather than the missing agreement on what acceptable looks like. A shared CI pipeline (D) automates detection and is excellent practice, but tooling enforces a standard only once one has been agreed; without the shared definition of done it just relocates the same failures earlier. The discriminator is that B, C, and D are mechanisms that operationalise a standard, whereas the actual gap is the absence of the agreed standard itself, which A supplies.
