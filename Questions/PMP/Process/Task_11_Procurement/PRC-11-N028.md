# Question

A project will involve several suppliers under a mix of contract types, and the sponsor is pressing the project manager to get the first solicitation out quickly. The procurement team is eager to start drafting the statement of work for the largest package and to begin shortlisting suppliers it has used before. The project manager wants the procurements to be consistent, defensible, and aligned with how the project is governed. Where should the project manager direct the team's effort first?

## Options

- **A**: Establish the procurement management plan, setting out how contract types, solicitation methods, evaluation criteria, and contract administration will be handled across the project.
- **B**: Draft the statement of work for the largest package, since it sits on the critical path and the detailed requirement will shape every other procurement decision.
- **C**: Run the make-or-buy analysis for each package so the project knows what it is procuring before any approach is chosen.
- **D**: Compile and pre-qualify a bidders list from suppliers the project has worked with before, so a competition can be launched as soon as the requirement is ready.

## Answer

A

## Explanation

The project manager's stated aim is consistency, defensibility, and alignment with project governance across several procurements, and that is precisely what the procurement management plan provides: it frames the approach, methods, evaluation, and administration that every individual package then follows (A). Drafting the statement of work for the biggest package first (B) is real and necessary work, but a single package's requirement does not establish the cross-project approach the PM is asking for, and starting there risks each procurement being run inconsistently. The make-or-buy analysis (C) is genuinely an early step, yet it is one activity carried out within the procurement approach rather than the governing framework itself, and it is informed by the plan's criteria. Pre-qualifying a familiar bidders list (D) is an input used once an approach exists and, done first, both pre-empts the planning and risks bias toward incumbents. The plan is the governing artifact that should be set before the package-level work, which is why it comes first here.
