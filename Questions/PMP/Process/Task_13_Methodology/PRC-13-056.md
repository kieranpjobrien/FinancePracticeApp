# Question

A Scrum team delivers features on time but defect rates have risen each release for a year. The team says refactoring and pair programming feel too expensive. What should the PM and scrum master coach the team on?

## Options

- **A**: Hire a separate QA team to test each release thoroughly so defects are caught before they reach production in future.
- **B**: Add an extra bug-fixing sprint at the end of each release cycle so the team catches up on technical issues periodically.
- **C**: Adopt technical practices such as TDD, refactoring, and pairing, and frame them as clear velocity enablers not overheads.
- **D**: Reduce the sprint commitment by 20% so the team has more time to focus on technical quality during their sprints.

## Answer

C

## Explanation

PMI recognises that sustainable velocity depends on engineering practices, not just process. (C) addresses the root cause. (A) separates quality from development and creates late feedback. (B) accepts a quality-debt cycle. (D) gives the team more time but does not change the practices that create the debt.
