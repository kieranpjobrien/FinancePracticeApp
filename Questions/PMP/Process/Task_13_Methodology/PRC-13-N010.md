# Question

A retailer must launch a loyalty programme before the holiday trading peak, an immovable date eleven weeks away. The full feature set is more than can be built in time, but a viable subset exists. The core data model is uncertain and likely to change as integration with the point-of-sale system is explored. The CIO wants a delivery approach that protects the date while accommodating the technical unknowns. What should the project manager propose?

## Options

- **A**: A predictive plan that fixes scope to the full feature set and compresses the schedule through crashing and fast-tracking.
- **B**: A predictive plan delivering the full scope, with the launch date moved to the first realistic date after the technical unknowns are resolved.
- **C**: A purely iterative approach focused on perfecting the data model before committing to any feature delivery.
- **D**: An agile approach with a fixed date and a prioritised backlog, delivering the highest-value subset first and treating remaining scope as flexible.

## Answer

D

## Explanation

A fixed deadline combined with uncertain requirements and a known viable subset is the classic fit for time-boxed agile with flexible scope: protect the date, vary what ships. (D) lets the team learn about the data model while still guaranteeing a valuable launch. (A) fixes scope against an immovable date despite real unknowns, which almost guarantees the date slips or quality collapses. (B) sacrifices the immovable business date, defeating the primary constraint. (C) perfects an internal artefact while burning the eleven weeks and delivering nothing launchable, inverting the priority of value over polish.
