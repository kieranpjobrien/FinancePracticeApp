# Question

A project manager is combining an adaptive software stream with a predictive infrastructure rollout into a single hybrid delivery. To keep the hybrid coherent rather than two disconnected projects, which actions should the project manager take? (Select TWO.)

## Options

- **A**: Identify the integration points between the streams and agree the criteria and timing for each hand-off.
- **B**: Force the infrastructure rollout into the software stream's iteration cadence so both run identically.
- **C**: Establish unified governance and a combined view of progress and risk across both streams.
- **D**: Run the two streams with no shared milestones so neither ever waits on the other.
- **E**: Use the same detailed up-front estimates for the adaptive software work as for the infrastructure work.

## Answer

A, C

## Explanation

A coherent hybrid explicitly manages the seams between streams and presents one integrated picture to governance. (A) defines the integration points and hand-off criteria so the streams connect deliberately, and (C) gives leadership a single coherent view of progress and risk. (B) imposes an iteration cadence on infrastructure work that does not benefit from it, mixing methods for uniformity rather than fit. (D) pretends the streams have no dependencies, but a hybrid programme almost always has integration milestones one stream relies on. (E) applies predictive up-front estimating to adaptive work, contradicting the very reason that stream is adaptive.
