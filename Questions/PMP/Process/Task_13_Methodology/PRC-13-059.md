# Question

A PM running a data migration project is asked by the PMO to pick "Scrum, Kanban, or waterfall" and stick to it end-to-end. The project has a stable scope but unpredictable volumes of data cleansing. What should the PM recommend?

## Options

- **A**: Use Scrum end-to-end because it is the organisation's default agile method and comes with ready-made ceremonies already.
- **B**: Use waterfall end-to-end because the scope is stable and predictive planning fits a migration with a defined target.
- **C**: Use Kanban end-to-end because unpredictable cleansing volumes suit flow-based work better than rigid timeboxed iterations do here.
- **D**: Tailor the methodology: predictive for migration sequencing and Kanban for cleansing flow, with shared blended programme governance.

## Answer

D

## Explanation

PMI's tailoring principle permits mixing approaches where different work types demand different methods. (D) matches each approach to the nature of its work. (A), (B) and (C) each choose a single lens that fits one part of the project well but the other part poorly.
