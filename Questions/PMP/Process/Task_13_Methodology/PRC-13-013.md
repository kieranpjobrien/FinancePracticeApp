# Question

A single-team software project will be the team's first agile delivery. Two upstream systems have hard quarterly release windows. Which framework is the BEST starting point?

## Options

- **A**: SAFe, so the team's cadence is already aligned to the upstream release windows.
- **B**: Basic Scrum, with a defined approach for managing the cross-team dependencies.
- **C**: Disciplined Agile Delivery (DAD), so the team can pick lifecycle elements as they go.
- **D**: Extreme Programming (XP) with full engineering practices in place from sprint one.

## Answer

B

## Explanation

For a single team new to agile, the starting framework should be the simplest one that lets the team learn. (B) is exactly that, and dependency management is handled with lightweight practices, not by jumping to scaling frameworks. (A) is overkill for one team and adds heavy ceremony. (C) gives too many choices for a team that has not yet built habits. (D) layers technical disciplines on top of unfamiliar process and risks overwhelming the team.
