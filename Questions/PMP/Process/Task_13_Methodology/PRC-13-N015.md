# Question

A medical-device project runs in two phases. In the first, the team is still learning what users need and expects the design to change repeatedly as prototypes are tried out. The second phase submits the device to a certification body whose review steps are set by regulation, run in a defined order, and assess the device against the design recorded at submission; the body re-opens its review if that design is altered afterwards. The project manager is deciding how to structure delivery across both phases. What is the BEST approach?

## Options

- **A**: Run the first phase iteratively, then plan-driven for certification, with a defined design-freeze gate at the hand-off between them.
- **B**: Run the first phase iteratively, then plan-driven for certification, but keep refining the design through user feedback in parallel with the certification submissions.
- **C**: Plan both phases up front against a single fixed design, so the certification body has a stable baseline from the start and the whole schedule is set end to end.
- **D**: Run both phases iteratively, treating certification activities as backlog items the team reprioritises iteration by iteration alongside the design work.

## Answer

A

## Explanation

The two phases behave differently: the first is exploratory with a design that keeps changing, while the second is a regulated, fixed-order review that judges the device against the design as submitted and re-opens if it changes. That combination is what calls for an explicit design-freeze gate at the seam — the decisive point most candidates miss. (A) matches each phase to a fit-for-purpose approach and manages that hand-off. (B) chooses the same two approaches but keeps changing the design during certification, which re-triggers the review and risks invalidating submissions — plausible-sounding but the fatal flaw. (C) locks in one design before the first phase has finished learning, sacrificing the experimentation the early work depends on. (D) keeps reprioritising work iteratively into a review whose steps are set by regulation and cannot be reordered. The choice turns on whether the design must hold still once submitted, not on which phase "feels agile".
