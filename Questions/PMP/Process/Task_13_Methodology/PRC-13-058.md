# Question

A project manager is tailoring practices for a hybrid programme and wants each problem solved by the technique most directly suited to it. Match each delivery problem with the practice that best addresses it.

## Items

- **1**: Demand arrives as an unpredictable, continuous stream and the team is starting too much at once
- **2**: The team keeps repeating avoidable mistakes because improvement is only discussed at project end
- **3**: Estimates for a stable, well-defined construction phase need to be tracked against a committed baseline
- **4**: Stakeholders cannot agree on which features matter most for the next release

## Matches

- **A**: Hold regular retrospectives to inspect and adapt the way of working
- **B**: Use a prioritised backlog with the product owner ranking items by value
- **C**: Apply a Kanban board with work-in-progress limits to manage flow
- **D**: Plan the phase predictively and track earned value against the baseline

## Answer

1-C, 2-A, 3-D, 4-B

## Explanation

Unpredictable continuous demand with too much in progress (1) is managed by a Kanban board with WIP limits (C). Repeated avoidable mistakes from end-only learning (2) are addressed by regular retrospectives that inspect and adapt continuously (A). A stable construction phase needing baseline tracking (3) suits predictive planning with earned-value tracking (D). Disagreement over feature priority (4) is resolved by a prioritised backlog ranked by the product owner (B). Tailoring means selecting the specific practice whose purpose matches each problem, rather than applying one approach uniformly.
