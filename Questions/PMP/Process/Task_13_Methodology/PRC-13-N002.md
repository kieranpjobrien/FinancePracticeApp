# Question

An insurer is replacing its policy administration platform. Five feature teams must deliver against a shared domain model, there are frequent cross-team dependencies, and a single quarterly release is required to satisfy reinsurance reporting. Leadership currently runs each team on independent two-week Scrum cycles and integration problems keep surfacing late. The project manager has been asked how to improve delivery predictability. What is the BEST approach?

## Options

- **A**: Introduce a scaled agile cadence that synchronises the teams' iterations and adds a regular cross-team planning and integration event.
- **B**: Keep independent team Scrum but add a weekly scrum of scrums and a shared definition of done so integration issues surface earlier.
- **C**: Move the programme to a predictive plan with an integrated master schedule, since the fixed quarterly release implies milestone-driven delivery.
- **D**: Merge the five teams into two larger teams to reduce the number of dependency interfaces that have to be coordinated.

## Answer

A

## Explanation

The symptoms — multiple teams, a shared model, dense dependencies, and a fixed integration point — are exactly what synchronised scaling cadences (aligned iterations plus a recurring cross-team planning/integration event) are designed to address. (A) directly targets the integration-late problem with structural alignment. (B) is a useful lightweight step but a scrum of scrums alone rarely fixes systemic late integration at this scale. (C) abandons the agile delivery the teams are already using and does not resolve the dependency coordination, only re-labels it. (D) creates oversized teams, harming flow and communication, and is an organisational change disproportionate to the problem.
