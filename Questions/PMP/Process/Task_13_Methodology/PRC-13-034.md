# Question

What is the defining characteristic that distinguishes Kanban from Scrum?

## Options

- **A**: Kanban requires a dedicated product owner and a scrum master.
- **B**: Kanban uses fixed-length sprints with sprint goals.
- **C**: Kanban is only used for manufacturing and operations work.
- **D**: Kanban uses continuous flow with WIP limits rather than timeboxed iterations.

## Answer

D

## Explanation

Kanban manages the flow of work using WIP limits instead of committing to a fixed scope per sprint. (D) captures that. (A) describes Scrum roles, which Kanban does not require. (B) describes Scrum, the opposite of Kanban's approach. (C) is historically true but practically wrong; Kanban is widely used across software, services, and operations.
