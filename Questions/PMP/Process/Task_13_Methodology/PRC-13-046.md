# Question

A medical device project has a strict regulatory gate at the end but huge UX uncertainty in the user-facing software. The organisation runs Scrum throughout and finds regulatory documentation painful to produce at the end. What should the PM recommend?

## Options

- **A**: Run Scrum for UX discovery and overlay staged predictive milestones that build regulatory artifacts continuously through delivery.
- **B**: Abandon Scrum and use a fully predictive V-model so regulatory documentation is produced in its correct sequential order.
- **C**: Continue pure Scrum and accept that regulatory documentation must be retrofitted near the end of development each release.
- **D**: Keep Scrum but stop sprint reviews during documentation phases so the team can focus on regulatory outputs uninterrupted.

## Answer

A

## Explanation

PMI's tailoring principle combines approaches to fit regulatory and uncertainty drivers together. (A) does both. (B) throws away iterative UX benefits the market demands. (C) accepts documentation pain and regulatory risk that hybrid would prevent. (D) pauses the very feedback loop that Scrum exists to enable.
