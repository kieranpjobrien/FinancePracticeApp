# Question

A utilities company is replacing a high-voltage substation transformer. A new engineering director, who has just delivered a software programme using Scrum, has asked every project to "build in more iteration and feedback so we stop committing to plans too early". On this project the regulator has already approved the substation design, the bespoke transformer is on order against a fixed contract, and the grid operator has issued a single booked outage window. The project manager is preparing the delivery approach to recommend. What should the project manager recommend?

## Options

- **A**: A predictive approach, baselining scope, schedule and cost, while adopting the director's request through regular progress reviews against that baseline.
- **B**: An agile approach with two-week iterations, so the team can re-plan the installation as each stage of site work reveals new information.
- **C**: A hybrid approach, planning procurement predictively but breaking the on-site installation into iterations that can be re-sequenced sprint by sprint.
- **D**: An incremental approach, energising the substation in successive partial releases so the director can review working capability between each.

## Answer

A

## Explanation

The director's "more iteration" message is an organisational preference, not a property of this work; the approach must fit what the work actually is. Here the design is regulator-approved, the equipment is fixed by contract, and the outage is a single booked window, so scope and sequence are settled and the fit is a predictive baseline. (A) holds that baseline and still honours the director's intent through frequent reviews — feedback against the plan, not iteration of the plan. (B) treats the director's preference as the deciding factor and re-plans an installation whose physical sequence and outage window cannot move. (C) sounds balanced but invents re-sequenceable iterations in on-site work that the booked outage and isolation rules do not permit, adding ceremony with no room to use it. (D) proposes partial energisation purely so the director can see progress, which a single bespoke transformer on one outage neither needs nor safely allows. The discriminator is separating an organisational push for agility from the project's real, fixed characteristics — and recognising that "show the director feedback" can be met without changing the lifecycle.
