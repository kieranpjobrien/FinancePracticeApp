# Question

What is the purpose of the backward pass in the Critical Path Method?

## Options

- **A**: To recalculate the schedule after rolling back recent changes that did not produce the expected outcome.
- **B**: To verify completed activities against the original baseline by working backward through the schedule history.
- **C**: To compute Late Start and Late Finish dates by traversing the network backward from the project end date.
- **D**: To identify the path with the fewest activities so that schedule reporting can be simplified.

## Answer

C

## Explanation

(C) is the PMI definition: the backward pass works from the project finish date back to the start, computing Late Start and Late Finish for each activity. Comparing early dates (forward pass) and late dates reveals total float and identifies the critical path. (A) confuses CPM with version control. (B) describes earned value or audit, not CPM. (D) describes a reporting choice, not the backward pass.
