# Question

A project manager finds that team leads consistently pad their activity durations — each privately adds a buffer "to be safe" — inflating the baseline by an estimated 30%. The PM wants more honest, defensible estimates while still protecting the project against genuine duration uncertainty. What is the best approach?

## Options

- **A**: Collect three-point estimates (optimistic, most likely, pessimistic), use the most-likely values in activities, and aggregate the spread into an explicit, managed contingency reserve.
- **B**: Run a Monte Carlo schedule simulation over the leads' duration ranges and set the contingency reserve from the finish-date confidence level the sponsor is willing to accept.
- **C**: Keep duration estimates but ask leads to record the basis of estimate and the assumptions behind each buffer, so the padding becomes visible and reviewable.
- **D**: Adopt reference-class estimating, calibrating activity durations against actuals from comparable past projects to replace individual guesswork.

## Answer

A

## Explanation

None of these is a throwaway — each is a recognised estimating improvement — so the choice turns on what most directly converts hidden padding into transparent, managed contingency for a typical project. (A) does exactly that: three-point estimates expose the range, the most-likely value de-pads the activity, and the uncertainty is pooled into a single visible reserve that the PM controls rather than buffers consumed silently inside every task — the canonical fix. (B) is a more sophisticated extension of the same idea and is excellent, but a full Monte Carlo simulation is heavier than the situation requires and presupposes tooling and quality range data; it is a refinement of (A), not the baseline answer. (C) increases transparency but leaves the padding inside the estimates rather than relocating it to a managed reserve, so it improves visibility without solving the structural problem. (D) is powerful where comparable historical data exists, but it depends on having a suitable reference class and does not itself establish how remaining uncertainty is held as reserve, making it narrower than (A).
