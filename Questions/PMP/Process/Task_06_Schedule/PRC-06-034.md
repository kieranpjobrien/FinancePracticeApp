# Question

A predictive project is transitioning from design to development. Two of seven design deliverables are running a week late, but development resources are scheduled to start on Monday. What should the project manager do?

## Options

- **A**: Start all development work on Monday as originally scheduled and rework anything that conflicts with the late design deliverables once they land.
- **B**: Hold all development work until every design deliverable is signed off so that no rework risk is introduced into the development phase.
- **C**: Identify which development activities depend on the late deliverables, start the independent work Monday, and resequence dependent work.
- **D**: Cancel the development phase start entirely and reconvene the design team for a week of focused completion before resuming work.

## Answer

C

## Explanation

PMI handles phase-overlap situations by analysing dependencies, starting what can safely proceed, and resequencing what cannot. (C) does that. (A) ignores dependency analysis and invites rework on dependent activities. (B) is overly conservative and idles resources that could safely work on independent activities. (D) overreacts by cancelling a planned phase start when only two of seven deliverables are affected.
