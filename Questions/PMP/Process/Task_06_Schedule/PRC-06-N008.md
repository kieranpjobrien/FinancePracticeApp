# Question

A hybrid satellite ground-station project has a firm launch-readiness date 14 months out. The first three months of work are planned to task level. Later integration activities depend on supplier designs that will not be finalised for several months, so the project manager is using rolling-wave planning and has left that integration work as summary-level planning packages. At the steering review the sponsor accepts the approach but says the summary block gives the board no way to judge whether the firm date is actually at risk. What should the project manager do?

## Options

- **A**: Set interim milestones across the summary integration packages, anchored to the supplier design-finalisation dates, and report a milestone-based finish forecast that each wave refines as designs land.
- **B**: Bring the detailed planning of the integration activities forward now, decomposing them against the supplier's latest preliminary designs so the board has task-level dates to review.
- **C**: Run a schedule risk analysis over the summary packages using three-point duration ranges and report the probability of meeting the firm date, refreshing it as each design firms up.
- **D**: Identify the must-finalise-by date for each supplier design that still allows launch readiness, and report integration feasibility as the float remaining against those supplier deadlines.

## Answer

A

## Explanation

The sponsor is not challenging rolling-wave; the gap is that a summary block carries no checkable signal about the firm date, and any fix must add that signal without inventing detail the designs cannot yet support. (A) overlays milestones tied to the real external driver (design finalisation) and forecasts the finish from them, giving the board a feasibility read now that each wave sharpens, which is how rolling-wave stays accountable to a fixed end date. (B) forces task-level detail onto immature designs, abandoning the approach and manufacturing precision that will churn the moment real designs arrive. (C) is genuinely tempting, but a credible probability needs duration ranges grounded in real logic; built on deliberately coarse summary packages it yields a confidence figure with little behind it and reads as more rigorous than it is. (D) is also sound and close to correct, but it reports feasibility of the supplier inputs rather than of the integration work itself, so a comfortable design-date margin can still mask an undeliverable integration sequence once detailed. Milestone-based forecasting against the external dates is the move that makes the uncertain horizon measurable while preserving the planning approach.
