# Question

A predictive biotech project shows four engineers over-allocated in weeks 6-10 on the current schedule. The project end date has three weeks of total float. The sponsor accepts some end-date slip if quality and cost are protected. What technique should the PM apply?

## Options

- **A**: Resource levelling, allowing the end date to move as necessary to keep each engineer within their available capacity.
- **B**: Resource smoothing within existing float so the end date holds, even if some overloads cannot be fully eliminated.
- **C**: Crashing the critical path by adding contract engineers so the overloads disappear and the end date is protected.
- **D**: Fast-tracking the weeks in question to parallelise the overloaded work and compress the over-allocation window.

## Answer

A

## Explanation

Resource levelling can change the end date and is used when capacity constraints are hard and some schedule slip is acceptable. (A) matches the stated conditions. (B) keeps the end date but may leave overloads. (C) adds cost the sponsor's guidance implied should be protected. (D) compresses time but worsens parallelism with already-overloaded resources.
