# Question

A hybrid infrastructure-and-software programme uses predictive planning for the physical build and Scrum for the software layer. Release dates depend on both. The teams plan in different horizons and cadences, and integration milestones keep slipping. What should the PM introduce?

## Options

- **A**: Move both streams to identical Scrum sprints so planning cadences align and integration milestones become sprint events.
- **B**: Move both streams to predictive planning so the software team's cadence follows the physical-build scheduling discipline.
- **C**: Hold integration planning on a shared cadence, align sprint goals with milestones, and sync progress at joint checkpoints.
- **D**: Create a dedicated integration team whose only job is to synchronise deliverables between the software and physical streams.

## Answer

C

## Explanation

Hybrid alignment uses a shared integration cadence that respects each stream's chosen method, not forced uniformity. (C) is the PMI-aligned tailoring answer. (A) forces Scrum onto physical work that may not suit it. (B) forces predictive onto software and kills empirical control. (D) adds headcount and handoff layers instead of fixing the cadence.
