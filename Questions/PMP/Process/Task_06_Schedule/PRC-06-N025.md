# Question

At a status review, a predictive project reports earned value of £420,000 against planned value of £500,000, budget at completion £1,200,000, with the project about 35% through its planned duration. The schedule performance index is therefore 0.84. The sponsor asks the project manager for a revised forecast finish date. What should the project manager do before giving one?

## Options

- **A**: Examine which activities are generating the variance and whether they sit on the critical path, because an aggregate SPI can be inflated by non-critical work that has float to absorb the slip.
- **B**: Extrapolate the finish date by dividing the remaining planned duration by the current SPI of 0.84, treating the index as a steady performance factor.
- **C**: Use the to-complete schedule performance index to work out the efficiency the remaining work must hit to still finish on the baseline date, and report that.
- **D**: Re-run the critical path with the latest actuals to produce a fresh forecast finish date directly from the network logic.

## Answer

A

## Explanation

SPI aggregates all work, so a 0.84 can be driven by lateness on activities that have float and are not delaying the finish at all; the PM must locate the variance before any index-based or network-based forecast is meaningful. (A) diagnoses where the slip sits, which determines whether the finish date is actually threatened. (B) projects the finish purely from the aggregate index, the very error that ignores whether the late work is on the critical path. (C) computes a recovery efficiency target, useful later, but it still assumes the variance bears on the end date rather than checking it. (D) re-running CPM is a sound forecasting step, but doing it before identifying and validating the driving activities and their actual remaining durations risks forecasting from contaminated data.
