# Question

Midway through a predictive project, the project manager's variance analysis shows completed work finishing later than the baseline durations. The slip is not confined to one team or activity type: it appears in broadly similar proportion across most of the activities done so far, all of which were duration-estimated during the same early planning workshop. The activities still to come were estimated the same way. The sponsor has asked for an updated finish-date forecast. What should the project manager do first?

## Options

- **A**: Reforecast the remaining activities by adjusting their planned durations in line with the slip rate the completed work has shown, then give the sponsor the revised finish date.
- **B**: Examine why the completed activities ran long to establish whether the cause is something that will keep recurring on the remaining work or a set of one-off events already behind the project, then forecast on that basis.
- **C**: Recompute the critical path from the latest actual and remaining durations as currently estimated, and report the finish date the network produces.
- **D**: Run a schedule risk analysis over the remaining estimates, sampling each activity's duration range, and report the forecast finish as a confidence range rather than a single date.

## Answer

B

## Explanation

The pattern, a similar-sized slip spread across activities that share one estimating origin, points to a possible systemic cause, but "possible" is the operative word: the forecast is only sound once the project manager has tested whether that cause actually carries forward, so diagnosing it is what must come first. (B) checks whether the slip stems from something ongoing (an optimistic estimating basis that the remaining same-origin estimates also carry) or from one-off disruptions that have passed, and that determination is exactly what decides how the remaining work should be forecast. (A) projects the historical slip rate forward, which is the right move only if the cause is in fact ongoing; applied before that is confirmed, it bakes a possibly-finished disruption into every future activity and overstates the finish. (C) recomputing the network is necessary mechanics, but feeding it the remaining durations untouched assumes the estimating basis was sound, the very thing the uniform slip calls into question, so it forecasts from figures not yet validated. (D) a schedule risk analysis produces an honest range and is a strong technique, but sampling the existing estimate ranges still treats those estimates as the starting point; if their central basis is biased, the whole sampled distribution is shifted, so the cause has to be understood before the ranges mean anything.
