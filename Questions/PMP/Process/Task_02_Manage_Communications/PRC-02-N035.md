# Question

On a hybrid programme, three communication situations have each been handled with a method that is causing a problem. Match each situation (left) to the single change in method that best resolves it (right).

## Items

- **1**: A 60-page technical handbook is emailed to all 40 stakeholders monthly; most never open it, and storage and version confusion are mounting.
- **2**: A contentious budget reallocation has been circulated as a detailed memo, but stakeholders keep replying with conflicting interpretations and no decision is reached.
- **3**: A safety-critical schedule change that three named crews must act on within the hour was posted to the project portal, and one crew has not seen it.

## Matches

- **A**: Move it to interactive communication, a live meeting of the key parties, to resolve it.
- **B**: Move it to pull communication, a self-service repository stakeholders access as needed.
- **C**: Move it to push communication, a direct targeted message to the specific people.

## Answer

1-B, 2-A, 3-C

## Explanation

Each situation is failing because the method does not fit the information's volume, need for dialogue, or urgency. The large reference handbook is being pushed when recipients only need it occasionally, causing inbox bloat and version chaos, so it belongs in a pull repository (1-B). The contentious reallocation is being handled in writing when it needs real-time back-and-forth to converge on a decision, so it calls for interactive communication (2-A). The hour-critical change was placed on a pull channel that a crew did not check in time, so it must be pushed directly to the named crews (3-C). The discrimination is that each item has a tempting wrong fit, for example pushing the handbook harder or posting the urgent change more prominently, and the correct move matches method to the situation's defining constraint.
