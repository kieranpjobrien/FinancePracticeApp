# Question

A PM is running lessons-learned sessions at each sprint review. Feedback is captured but the same issues keep reappearing across sprints. The team has started viewing the sessions as going-through-the-motions exercises. What should the PM change?

## Options

- **A**: Move lessons-learned to end-of-release rather than every sprint so issues accumulate and more significant patterns can emerge over release cycles.
- **B**: Rotate facilitation of lessons-learned among team members so different perspectives surface different aspects of each recurring issue each sprint.
- **C**: Invite stakeholders from outside the team to lessons-learned so fresh observers raise issues the team has stopped noticing over time.
- **D**: Convert each lesson into a tracked improvement action with an owner and due date, reviewed at the start of each next session.

## Answer

D

## Explanation

Lessons-learned without action is a ritual, not a practice; PMI-aligned continuous improvement closes the loop from insight to change. (D) does that explicitly. (A) less frequent sessions don't address the action-closing gap. (B) rotating facilitation changes the voice but not the follow-through. (C) external observers can help but don't solve the core "no action on outputs" problem.
