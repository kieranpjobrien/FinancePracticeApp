# Question

The team uses email, chat, a project tool, a wiki, and three shared drives. Team members report wasting time hunting for the latest version of key documents. What should the PM do?

## Options

- **A**: Designate authoritative sources for each information type (decisions, requirements, schedule, etc.), publish where-things-live guidance, and consolidate redundant repositories.
- **B**: Move all project information into the project management tool and archive the others to remove duplication entirely.
- **C**: Appoint a project administrator to be responsible for filing and retrieving information on behalf of the team.
- **D**: Add a search index across all systems so team members can locate information regardless of where it is stored.

## Answer

A

## Explanation

The cause is unclear ownership of information types, not too many tools or weak search. (A) addresses ownership and is the cheapest, most durable fix. (B) is heavy-handed and usually fails because each tool has legitimate users. (C) treats the symptom by adding a person rather than fixing the system. (D) helps with retrieval but doesn't resolve the fundamental "where should this go" problem.
