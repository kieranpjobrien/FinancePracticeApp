# Question

A hybrid programme runs agile delivery teams beneath a stage-gated governance board that expects a single percent-complete figure each month to support its gate decisions. The teams track progress by working features and story points, and the PM finds that whatever percentage is reported, the number swings or stalls in ways the board reads as evasive, to the point that members have begun questioning whether the teams are being straight with them. The PM has the full backlog, velocity history, and the board's gate criteria to hand. What should the PM do?

## Options

- **A**: Report percent-complete as features accepted divided by total features in the baselined scope, a count the board can verify against the backlog, so the figure is objective and traceable.
- **B**: Replace the single figure with a velocity-based forecast of the completion date carrying a confidence range, giving the board a forward view rather than a contested point-in-time percentage.
- **C**: Keep reporting the percentage but add the burndown and a short note on what moved it this month, so the number arrives with the context that explains its swings.
- **D**: Work with the board to define a reporting view that maps delivered and remaining features to each gate decision they make, replacing the disputed percentage with measures both sides agree show real progress.

## Answer

D

## Explanation

The board's loss of trust is the core problem: a number imposed on them keeps reading as evasive, so the durable fix is to rebuild credibility by jointly defining what "progress" means against the decisions they actually take at each gate, which makes the report something both sides own rather than something done to them. (D) ties the measure to the board's decision needs and repairs the trust gap collaboratively. (A) is a genuine improvement and gives a verifiable number, but a single scope-completion percentage still hides how much value sits in remaining features and can keep stalling in ways the board misreads, so it supplies a cleaner figure without resolving why they distrust it. (B) offers a sound forward-looking metric, yet swapping in a forecast the PM chooses unilaterally still imposes a measure on a board that has not agreed to it, and a confidence range can itself read as hedging to members already primed to suspect evasion. (C) adds useful context but leaves the disputed percentage as the headline, treating a trust-and-fitness problem as if better annotation were enough. The margin between (D) and (A)/(B) is fine: each yields a more defensible number, but only (D) restores trust by agreeing the measure with the board against their own decisions rather than handing them a better figure to take on faith.
