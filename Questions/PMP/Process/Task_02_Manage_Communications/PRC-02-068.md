# Question

At the close of each phase of a multi-phase predictive construction programme, the PM holds a thorough lessons-learned session and writes the findings into a register stored in the project's document area. By the third phase, the team is repeating mistakes that earlier phases had already captured. The register is complete and well organised. What is the most effective improvement?

## Options

- **A**: Schedule the lessons-learned sessions earlier, at the midpoint of each phase, so findings are captured while still fresh.
- **B**: Build the relevant prior lessons into the planning of the next phase as inputs the team actively reviews, rather than leaving them in a register to be consulted.
- **C**: Move the register to a more prominent shared location and remind the team at kick-off that it exists and should be read.
- **D**: Assign an owner to each captured lesson who is accountable for ensuring it is not repeated in later phases.

## Answer

B

## Explanation

Lessons captured but never fed back into work do not change behaviour; the register is a pull repository the team is not pulling from, so the fix is to push the relevant lessons into the next phase's planning as live inputs. (B) closes the loop where decisions are actually made. (C) relies on the same passive consultation that is already failing, more prominence does not make a busy team read a register. (A) improves capture timing but the problem is application, not freshness, the lessons were recorded accurately. (D) creates accountability but a lesson-owner with no mechanism to inject the lesson into planning still depends on memory and goodwill.
