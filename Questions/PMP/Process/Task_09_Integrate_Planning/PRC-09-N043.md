# Question

Midway through a satellite ground-station build, the project manager finds that three teams are each working from a different revision of the interface control document, and recent design changes were applied to two of them without anyone updating the third. No formal record links the document revisions to approved changes. Which action most directly addresses the underlying problem?

## Options

- **A**: Email the latest interface control document to all three teams and instruct them to discard older copies immediately.
- **B**: Raise a change request to reconcile the three document versions and route it through the change control board.
- **C**: Add a standing agenda item to the weekly coordination call to compare document revisions across teams.
- **D**: Establish a configuration management system so document versions and approved changes are controlled and the current baseline is unambiguous to every team.

## Answer

D

## Explanation

The root issue is the absence of configuration management — there is no controlled link between baselined artefacts and approved changes — so installing that system (D) is the structural fix. (A) corrects today's symptom but leaves the next divergence just as likely because nothing controls future versions. (B) addresses one reconciliation through change control but does not create the version-control discipline that prevents recurrence. (C) is a manual workaround that depends on people catching drift, which is exactly what failed.
