# Question

A PM takes over a mid-sized manufacturing project and finds the outgoing PM applied the same heavy predictive plan template used on all company projects, even though this one has significant uncertainty and evolving requirements. What should the PM do?

## Options

- **A**: Keep the existing plan to avoid disruption, since the team has already begun executing against it.
- **B**: Tailor the project management approach to the actual context, revising planning artifacts to match.
- **C**: Replace the plan entirely with a lightweight agile template before the next steering committee review.
- **D**: Raise the mismatch as a risk and continue running with the existing plan documentation unchanged.

## Answer

B

## Explanation

Tailoring is a core PMI principle; plans should fit project context rather than be inherited from a template. (A) continues using a plan that does not fit, accepting known unfit artifacts. (C) overcorrects by switching methodology wholesale instead of adjusting what is needed. (D) records the issue without addressing the underlying fit problem.
