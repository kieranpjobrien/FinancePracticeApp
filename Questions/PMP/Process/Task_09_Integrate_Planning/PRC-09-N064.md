# Question

A newly assigned project manager inherits a project from a manager who left the organisation. The handover folder contains a thorough scope statement, a detailed schedule, and a cost estimate, each produced in its own workshop. The sponsor has told the team to begin executing on Monday. The new project manager has the weekend and wants to spend it on the one action that most reduces the risk of execution going wrong. Where should the project manager focus?

## Options

- **A**: Check the subsidiary plans against each other for conflicts and consolidate them into a single plan that also sets out how changes will be controlled across them.
- **B**: Build a responsibility assignment matrix so every scheduled activity has a named owner before the team starts on Monday.
- **C**: Hold a kickoff with the team and sponsor to confirm shared understanding of the objectives and the way of working before execution begins.
- **D**: Walk the schedule's critical path with the team to validate the durations and logic the previous manager assumed.

## Answer

A

## Explanation

The artefacts exist but were produced separately, so the largest unmanaged risk is that scope, schedule, and cost are mutually inconsistent and that no agreed mechanism exists to control changes once execution starts — the work of integrating the subsidiary plans into the project management plan. (A) both tests the plans for contradictions and supplies the integrating document and change approach, addressing the structural gap directly. (B) assigns ownership but takes the three plans as internally consistent, which is the very assumption that has not been checked. (C) builds alignment and is genuinely valuable, but a kickoff communicates a plan rather than reconciling one, so it can broadcast conflicts the integration step would have caught. (D) validates one plan in isolation; a sound critical path is still unsafe if it contradicts the cost or scope baseline. Integrating and reconciling the plans is the action that most protects execution.
