# Question

A regulated medical device project must produce a defined set of compliance artefacts (design history file, traceability records, validation evidence). The agile delivery team argues these are "agile-incompatible." How should the project manager integrate the requirement?

## Options

- **A**: Honour agile principles and minimise documentation, relying on working software as the primary record of truth.
- **B**: Maximise documentation across the entire project to be comprehensively safe under any regulatory review.
- **C**: Tailor: identify the artefacts the regulator requires, embed production into team flow, keep other docs lean.
- **D**: Run the compliance work as a separate predictive project running in parallel to the agile delivery stream.

## Answer

C

## Explanation

Tailoring is exactly the PMI answer here: regulatory artefacts are non-negotiable, but they can be produced inside an agile cadence with discipline. (A) ignores the regulator. (B) over-corrects and burdens the team. (D) creates parallel governance and breaks the very integration the artefacts are supposed to evidence.
