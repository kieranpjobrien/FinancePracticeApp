# Question

A project manager is integrating planning for a hybrid programme: a predictive data-centre build and an agile analytics platform that will run on it. Which actions genuinely integrate the two streams into one coherent plan? (Select TWO.)

## Options

- **A**: Require the predictive build team to adopt a sprint backlog so both streams use identical artefacts.
- **B**: Have the sponsor receive two separate status reports so each stream is judged on its own terms.
- **C**: Establish shared milestones, such as "infrastructure ready for software deployment," that both streams plan around.
- **D**: Map and document the cross-stream dependencies so each team knows what it needs from the other and when.
- **E**: Freeze the analytics platform's backlog at the start so it cannot change after the build schedule is set.

## Answer

C, D

## Explanation

Shared milestones (C) and explicitly mapped cross-stream dependencies (D) are the connective tissue that makes a hybrid plan integrated while letting each stream keep its own life cycle. (A) forces identical artefacts on fundamentally different work, distorting the predictive stream rather than integrating it. (B) reinforces separation rather than integration. (E) sacrifices the adaptability that justifies running the analytics stream agile, in order to protect a schedule — that is subordination, not integration.
