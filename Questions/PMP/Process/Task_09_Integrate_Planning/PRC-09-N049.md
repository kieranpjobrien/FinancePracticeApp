# Question

An offshore wind project will use predictive planning for foundation installation, which is governed by marine-survey and weather windows, and agile delivery for the SCADA monitoring software. The sponsor asks for one integrated plan that respects both approaches. How should the project manager structure the integrated planning?

## Options

- **A**: Baseline the installation work predictively and plan the software with a backlog and iterations, integrating the two through shared milestones and explicit cross-stream dependencies.
- **B**: Baseline the installation work predictively and require the software stream to commit its full scope and dates to the same integrated baseline, so the plan presents one fixed schedule the sponsor can control.
- **C**: Plan both streams as a single backlog delivered in iterations, treating each survey and weather window as a story the team pulls when conditions allow, so the whole plan stays adaptive.
- **D**: Baseline the installation work predictively now and defer planning the software until the installation schedule is fixed, then attach the software iterations to the dates that remain.

## Answer

A

## Explanation

A genuine hybrid keeps each stream in its appropriate life cycle — a baselined predictive schedule for the physically constrained, weather- and survey-bound installation, and iterative backlog-driven delivery for the software — and stitches them together through shared milestones and named cross-stream dependencies (A). (B) forces the software stream to commit emergent scope to a fixed baseline, stripping the adaptability that is the reason to run it agile. (C) tries to pull immovable marine and weather windows into a backlog as if they were optional stories, when they are hard external constraints the installation must be scheduled around, not iterated through. (D) sequences the planning so the software is shaped only by leftover dates, surrendering its independent value-driven prioritisation and weakening the integration rather than strengthening it.
