# Question

While integrating the plans for two concurrent subprojects under one programme, a project manager finds both subproject schedules assume the same specialist commissioning engineer full-time during the same fortnight. Each subproject plan is sound on its own. What should the project manager do?

## Options

- **A**: Re-sequence the two subprojects' commissioning windows at the programme level so the shared engineer's time no longer overlaps and the combined plan is feasible.
- **B**: Let whichever subproject reaches commissioning first take the engineer, hold the other until released, and record the wait as a managed risk.
- **C**: Baseline both plans as written and apply resource levelling only if both subprojects actually arrive at commissioning in the same fortnight.
- **D**: Escalate the conflict to the programme resource owner to second a second qualified engineer for the fortnight so neither schedule has to move.

## Answer

A

## Explanation

The conflict is a programme-level resource overlap between two otherwise-sound plans, and integrated planning is where it is resolved — here by re-sequencing the commissioning windows so the shared engineer is not double-booked and the combined plan can execute (A). (B) keeps both plans static and converts a known infeasibility into a wait-and-risk, accepting avoidable delay rather than reconciling the schedules. (C) knowingly baselines the over-allocation and defers any action until the clash is already happening, which is reactive. (D) is a legitimate fallback if re-sequencing cannot recover the overlap, but reaching for a second scarce specialist first — when adjusting the windows may resolve it at no cost — escalates before doing the integration work the PM can do.
