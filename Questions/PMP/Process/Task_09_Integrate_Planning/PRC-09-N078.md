# Question

A games studio is integrating planning across five agile squads building a single online title. During a scaled planning event, the PM wants to ensure the squads' separate plans combine into one deliverable product rather than five disconnected efforts. Which integrating practices are appropriate at this event? (Select THREE.)

## Options

- **A**: Surface and sequence cross-squad dependencies so each squad's committed work accounts for what it needs from others.
- **B**: Agree a shared definition of done and common quality bars so all squads' output integrates to the same standard.
- **C**: Align the squads on the common release objectives and the integration milestones the increment must hit.
- **D**: Mandate identical sprint backlogs across all five squads so the plan is uniform and easy to track.
- **E**: Require each squad's lead to commit to their own backlog privately and submit it to the PM for consolidation.

## Answer

A, B, C

## Explanation

Scaled integration depends on managing the seams between teams: sequencing cross-squad dependencies (A), agreeing a shared definition of done and quality bars (B), and aligning on common release objectives and integration milestones (C) are what make five plans combine into one product. (D) imposes uniform backlogs, which destroys each squad's ability to plan its own distinct work and confuses uniformity with integration. (E) collects backlogs in isolation without any cross-squad reconciliation, leaving dependencies and shared objectives unaddressed — assembly, not integration.
