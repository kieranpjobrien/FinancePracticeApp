# Question

After three plan revisions on a hybrid programme, two engineers built features against a superseded version of the integration plan, causing rework. The PM finds team members storing local copies in different folders. What should the PM do to address the root cause?

## Options

- **A**: Send the latest plan as a PDF attachment to the team distribution list whenever it changes.
- **B**: Establish a single authoritative repository for the plan with version control and announce updates through the team's standard channel.
- **C**: Lock the integrated plan after the next revision so team members cannot work from older copies.
- **D**: Add a checkpoint to daily standups where each engineer states which version they are working from.

## Answer

B

## Explanation

Configuration management for project artifacts requires a single source of truth with proper versioning and a defined update notification path. (B) addresses the root cause: people created local copies because there was no clear authoritative location. (A) perpetuates the problem by spawning more local copies that immediately go stale. (C) prevents legitimate updates the plan will need as the programme evolves. (D) treats the symptom and adds standup overhead without fixing the underlying artifact management gap.
