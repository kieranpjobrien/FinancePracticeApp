# Question

An agile team building a mobile telecom self-service app keeps its product backlog, sprint board, release plan, and dependency list in four different tools, and stakeholders complain they cannot see how the pieces relate. The product owner asks the PM to help present a coherent picture of the plan. What is the best first step?

## Options

- **A**: Establish a single integrated view that links the backlog, release plan, and dependencies so stakeholders can see how the planning artifacts connect.
- **B**: Tell stakeholders to consult each tool individually, since each one is the authoritative source for its own data.
- **C**: Replace all four tools with one new platform before the next sprint to eliminate the fragmentation.
- **D**: Produce a static slide deck summarising the current plan for the next stakeholder meeting.

## Answer

A

## Explanation

The problem is that the planning artifacts are not visibly connected, and integration is about making them coherent and transparent. (A) links the artifacts into one view that shows the relationships, directly serving stakeholders' need to see the whole. (B) leaves the fragmentation in place and pushes the integration burden onto stakeholders. (C) is a disruptive tooling overhaul that addresses platforms, not the underlying need for a connected view, and would stall delivery. (D) gives a one-off snapshot that is immediately stale and does not establish an ongoing integrated picture.
