# Question

An agile team building a healthcare scheduling app keeps seeing defects reach release despite a thorough end-of-sprint test pass. The coach wants quality built into the work as the increment is created, not added by a later check. Several improvements are proposed, each defensible on its own. Which three move verification to the point where the work is produced, rather than to a later stage? (Choose three.)

## Options

- **A**: Write the automated acceptance and unit tests for each story as it is developed, so a behaviour is verified the moment it is built.
- **B**: Pair on, or peer-review, code while it is being written so design and logic issues surface during development.
- **C**: Run the automated suite on every check-in through continuous integration, so a breakage is flagged within minutes of the change that caused it.
- **D**: Have an experienced engineer run a structured manual exploratory-testing session on the assembled increment at the end of each sprint.
- **E**: Stand up an automated end-to-end regression suite that runs nightly against the integrated build to catch cross-feature breakages.

## Answer

A, B, C

## Explanation

All five are real quality practices, so the discrimination is not good-versus-bad but where in the flow verification happens. Test-first development as each story is built (A), in-line pairing or review during coding (B), and continuous integration firing on every check-in (C) embed checking into the act of creating the increment, so defects are prevented or caught at the source. (D) is valuable but is still inspection of an already-assembled increment after the work is done — later than the point of creation. (E) is also worth having, but a nightly end-to-end suite detects breakages hours after the change and at the integrated-build level, not as the code is written. The questions to ask of each practice are "how soon after the defect is introduced does this catch it?" and "is the feedback at the developer's hands or downstream?" — A, B, and C are immediate and at-source; D and E are useful but downstream.
