# Question

An agile team building an insurance-quoting engine wants quality to be intrinsic to each sprint rather than concentrated in a test phase at the end. As project manager and servant leader, which practices should you encourage the team to build in? (Select three.)

## Options

- **A**: Maintain a clear, agreed Definition of Done that every backlog item must meet each sprint.
- **B**: Automate tests so they run on each integration and give the team fast feedback on regressions.
- **C**: Refactor continuously so the codebase stays maintainable as features are added.
- **D**: Hold a separate quality-assurance sign-off by a reviewer outside the team before each increment is called done.
- **E**: Set a sprint defect-count target and reward the sprint that logs the fewest defects.

## Answer

A, B, C

## Explanation

Built-in quality relies on a shared Definition of Done applied every sprint (A), automated tests giving continuous regression feedback (B), and continuous refactoring that keeps the code healthy as it grows (C). (D) reintroduces an external, end-of-increment gate outside the team, which pushes quality responsibility off the team and reverts toward inspecting quality in rather than the team building it in. (E) targets and rewards a low logged defect count, which is a vanity metric that perversely discourages people from recording defects and treats a single number as a stand-in for quality rather than improving how the work is done.
