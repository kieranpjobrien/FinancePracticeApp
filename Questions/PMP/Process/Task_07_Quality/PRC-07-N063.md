# Question

A consumer-electronics firmware team is setting its quality budget for the next release. Post-mortems on the previous two releases show that the costly field defects clustered almost entirely in how the team applies one security-coding standard, and that the same handful of mistakes recurred release after release. The budget is fixed, so funding one option means trimming another. Which investment should the project manager prioritise?

## Options

- **A**: Fund an expanded automated test suite that runs against every candidate build before release.
- **B**: Fund a final independent security audit of the release candidate before it ships to customers.
- **C**: Fund coaching on the security-coding standard plus a peer-reviewed checklist applied as the code is written.
- **D**: Fund a faster field-update pipeline so defects found after release can be patched quickly.

## Answer

C

## Explanation

The evidence describes a recurring, concentrated pattern: the same handful of standard-application mistakes reappear release after release, which marks the cause as systematic and preventable rather than random escapes. With a fixed budget, the highest-leverage conformance spend is prevention aimed at that cause. (C) is prevention — it changes how the code is written so the defects are not introduced, attacking the source of the prior failures. (A) and (B) are both appraisal and both genuinely useful, which is what makes them tempting: stronger automated tests and an independent audit would catch more before shipping. But appraisal detects defects that already exist, so against a known recurring cause it pays repeatedly to re-find the same avoidable mistakes, and detection of subtle security flaws is imperfect — defects still escape. (D) is not a conformance cost at all; a faster patch pipeline lowers the cost of an external failure after it reaches customers rather than preventing or detecting it. The discriminator is cost-of-quality category — prevention versus appraisal versus reducing failure cost — and with a constrained budget and a systematic cause, prevention returns more than detecting or recovering from the same defects again.
