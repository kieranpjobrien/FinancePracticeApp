# Question

On a high-volume bottling line, the project manager reviews the fill-weight control chart. No point has ever breached a control limit, but the last seven readings all sit above the mean and form a gentle upward run. Downstream samples still pass specification, and the operator has logged no alarms. What should the project manager do?

## Options

- **A**: Recalculate the control limits from the recent data so they reflect where the process is now running and re-baseline the chart.
- **B**: Keep monitoring and act once a reading actually crosses a control limit or a unit fails specification, since current output still conforms.
- **C**: Start investigating the line for an assignable cause now, treating the run of seven points above the mean as a signal even though no limit was breached.
- **D**: Adjust the line to pull the fill weight back toward the mean, removing the upward drift before it can grow.

## Answer

C

## Explanation

Statistical process control responds to non-random patterns, not only limit breaches. A run of seven consecutive points on one side of the mean is a recognised signal of a probable assignable cause, so the chart is already warning of drift even though everything is within limits and within spec. (C) acts on that signal by investigating the cause while it is still small. (B) waits for a breach or a failure and discards the early warning the pattern is providing — the whole point of the run rule is to act before that. (A) re-baselines the limits to the drifted data, which hides the very shift the chart is detecting and treats abnormal behaviour as the new normal. (D) adjusts the process before the cause is known; tampering with a process in response to an unexplained signal can increase variation, and the correct first step is to find why it is drifting.
