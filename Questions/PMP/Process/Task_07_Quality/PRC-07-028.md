# Question

Code review records show the team is repeatedly catching the same handful of defect categories — null-handling, off-by-one errors, and missing input validation. The PM wants the team to spend less effort fixing these. What should the PM recommend?

## Options

- **A**: Replace the human code-review step entirely with a stronger automated static-analysis tool that catches these defect categories.
- **B**: Build a shared checklist of the recurring defect patterns and have developers self-check against it before submitting code for review.
- **C**: Assign the team's most experienced developer to review every change so the recurring defects are filtered out by deeper expertise.
- **D**: Increase the number of reviewers required per change so more eyes are likely to catch the recurring patterns.

## Answer

B

## Explanation

PMI's continuous-improvement guidance is to convert detection patterns into prevention. (B) shifts the recurring defects upstream by making them visible to the developer before review. (A) trades a complementary control for a single tool and assumes the tool will catch design-level patterns that humans currently catch. (C) creates a single-point bottleneck and prevents the team from learning. (D) scales the existing detection step rather than preventing the issue in the first place.
