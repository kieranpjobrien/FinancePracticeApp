# Question

An agile team building a telehealth scheduling product has strong unit-test coverage and a green build, yet customers keep hitting defects in real booking journeys that the team only learns about through support tickets weeks after release. The product owner asks the project manager to help the team get earlier, more reliable signals about the quality actually reaching users. Which actions would meaningfully strengthen that? (Select all that apply.)

## Options

- **A**: Track the escaped-defect rate — defects found in production versus before release — and review the trend each retrospective.
- **B**: Add end-to-end tests that exercise the real booking journeys customers use, and run them before each release.
- **C**: Instrument the product to capture errors and key booking events so failures surface in near real time rather than via tickets.
- **D**: Raise the unit-test coverage target from 80% to 95% across the codebase to reduce defects overall.
- **E**: Add a manual exploratory-testing pass on the booking journeys each sprint before the release.

## Answer

A, B, C

## Explanation

The defects reaching users are integration- and journey-level and are detected late and indirectly, so the strongest moves give earlier, outcome-based signals about real usage. Tracking escaped defects (A) measures how much is slipping through and whether it is improving; end-to-end tests of real journeys (B) catch the integration failures unit tests miss, before release; product instrumentation (C) turns silent production failures into immediate signals instead of week-late tickets. Raising unit-test coverage (D) deepens testing at a level that already passes green while the failures live above it, so it adds effort without addressing the gap. A manual exploratory pass (E) can find issues and is not worthless, but it is sampled, unrepeatable, and unmeasured — it gives neither the reliable regression coverage of (B) nor the trend signal of (A) and (C), so it is the weakest of the journey-level options and not a dependable earlier signal. The discriminator is signals at the level where defects actually escape, not more of the testing that is already passing.
