# Question

A project manager inherits an organisational quality standard that mandates a heavyweight five-stage inspection for every deliverable. On the current small, low-risk internal-tool project, two of the five stages have never found a defect and consume more effort than building the deliverable. The standard permits tailoring with PMO approval. What should the project manager do?

## Options

- **A**: Apply the tailoring straight away given the project's low risk, and record the rationale so the PMO can review it at the next governance checkpoint.
- **B**: Propose a tailored inspection to the PMO that drops the two non-value-adding stages for this project, with the rationale and risk documented, and apply it once approved.
- **C**: Keep all five stages for this project, and instead raise the inefficiency so the organisation can consider revising the standard for low-risk work generally.
- **D**: Run the three value-adding stages every time but apply the two questioned stages only to the deliverables the project manager judges highest-risk.

## Answer

B

## Explanation

Quality processes should be right-sized to a project's size, risk, and complexity, but tailoring is a governed decision: the standard explicitly requires PMO approval, so the correct path is to propose the change with rationale and risk, then apply it once authorised. (B) does exactly this. (A) reaches the same tailored outcome but acts before approval, treating required governance as an afterthought — tailoring first and informing later breaches the standard's own rule. (C) respects governance but over-corrects: it keeps wasteful stages running on this project while waiting on an organisation-wide change that may never come, when project-level tailoring is available. (D) is unilateral selective application by the project manager without PMO approval and substitutes personal judgement of risk for the authorised process.
