# Question

A hybrid construction-fit-out project is handing over completed floors one at a time. Final inspection on the floors done so far has each time found a wiring fault that then needs rework, pushing out handover. There are several floors still to fit out. The project manager is deciding how to handle the remaining floors and is weighing options that all keep faulty floors from being handed over. Which approach is the best basis for the remaining work?

## Options

- **A**: Have the installation team trace why the fault keeps appearing and adjust the installation method, so the fault stops being introduced on the floors not yet done.
- **B**: Add a second inspection pass partway through each remaining floor's wiring, so the fault is caught and reworked before it reaches the slower final inspection.
- **C**: Build a standard rework allowance into the schedule and cost plan for every remaining floor, treating the fault as a known, accepted cost of this fit-out.
- **D**: Bring final inspection forward on each remaining floor and add inspectors, so any fault is found as early as possible and handover is protected.

## Answer

A

## Explanation

Every option keeps bad floors from being handed over, so the discrimination is whether to remove the fault or to keep managing it. The fault recurs on floor after floor, which points to it being built in by the installation method; correcting that method stops it arising on the floors still to come and eliminates the rework, not just its consequences. (A) is correct and is the only option that reduces the work rather than rearranging when it is absorbed. (B) and (D) both add inspection — earlier or doubled — which detects the same defect sooner but never stops it being created, so rework continues on every remaining floor. (C) normalises avoidable rework into the baseline, accepting a recurring cost that a method fix would remove. The tell of a built-in defect is recurrence; the response is prevention at source, not more or earlier inspection.
