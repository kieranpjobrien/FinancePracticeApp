# Question

A critical defect reached production: customers were briefly able to see other users' order histories. The exposure has already been stopped. The project manager convenes the team to prevent recurrence rather than just patch the symptom. Which actions are consistent with a sound root-cause and corrective-action approach? (Choose three.)

## Options

- **A**: Run a structured root-cause analysis, such as five whys, on how a missing authorisation check passed through to production.
- **B**: Identify the quality control step that should have caught the defect — for example an absent authorisation test — and strengthen it.
- **C**: Implement a preventive check for the identified cause and then verify, with a test that would have failed before, that the corrective action actually works.
- **D**: Add a manual sign-off by the project manager on every future release as a gate, so a person is reviewing each one before it ships.
- **E**: Raise a corrective-action item to add authorisation test cases to every story in the current sprint backlog before any of that work is called done.

## Answer

A, B, C

## Explanation

Sound corrective action investigates the cause, fixes the specific gap, and verifies the fix: root-cause analysis of how the missing check escaped (A), finding and strengthening the quality control step that should have caught it (B), and implementing then verifying a preventive check (C). (D) adds a manual human gate in place of fixing the process — a person-dependent inspection that does not scale, slows every release, and would not reliably catch a missing authorisation check anyway; it substitutes oversight for the automated control the defect actually calls for. (E) is the subtler distractor: it is action-oriented and well-intentioned, but bolting authorisation test cases onto every unrelated story in the current sprint is a broad, unfocused reaction taken before the root-cause analysis has shown where controls genuinely belong — it inflates scope and treats the symptom's category rather than the verified cause. Corrective action should be targeted at the confirmed root cause (A–C), not bolted on as a manual gate (D) or sprayed across the backlog (E).
