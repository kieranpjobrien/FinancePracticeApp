# Question

After a critical bug shipped to production, the team's retrospective traces the root cause to a user story that entered the sprint without clear acceptance criteria. What should the project manager facilitate as the primary improvement?

## Options

- **A**: Add a dedicated regression-test phase after each sprint to catch defects from poorly defined stories before they reach production.
- **B**: Require a senior tester to sign off on every story before it is marked done, regardless of acceptance criteria.
- **C**: Strengthen the team's Definition of Ready so stories cannot enter sprint planning without clear, testable acceptance criteria.
- **D**: Introduce a separate hardening sprint at the end of the release to address residual quality issues before deployment.

## Answer

C

## Explanation

The root cause is upstream — stories entering work without clarity. PMI's prevention-over-detection emphasis says fix the cause. (C) does that by tightening the Definition of Ready. (A) and (D) add expensive downstream inspection that does not stop the cause. (B) creates a bottleneck and substitutes one person's judgement for explicit, testable criteria.
