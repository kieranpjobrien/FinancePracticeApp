# Question

An agile team delivering a brokerage app runs a manual regression checklist that has grown to two days per fortnightly release. Under cadence pressure the team has begun skipping parts of it, and the last two releases shipped regressions to production. A test engineer has mapped which checks cover the highest-risk trading paths. The team debates how to respond. Which approach should the project manager encourage?

## Options

- **A**: Pause new feature work for one sprint and have the whole team automate the full regression suite, then resume delivery with regressions covered.
- **B**: Each sprint, automate the regression checks for the highest-risk paths first while continuing to deliver, expanding coverage as capacity allows.
- **C**: Keep the cadence but make completing the full manual regression a release gate, and add a second tester so it fits the window.
- **D**: Move to monthly releases so the full manual regression fits comfortably before each release.

## Answer

B

## Explanation

The durable fix is automated regression, and the team already knows which paths carry the most risk, so coverage can be grown incrementally without surrendering the delivery cadence that produces value. (B) builds quality in continuously and protects the riskiest paths first, keeping value flowing — consistent with sustainable pace and built-in quality. (A) is a legitimate option and may feel decisive, but a full feature freeze front-loads work that can be spread across sprints and halts value delivery longer than the risk requires. (C) and (D) both preserve a manual gate the team has shown it will skip under pressure: (C) adds cost to defend a non-scaling process, and (D) trades away release frequency rather than improving the process. Incrementally automating, riskiest-first, beats both a big-bang freeze and reinforcing the manual gate.
