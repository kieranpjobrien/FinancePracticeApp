# Question

On a hybrid programme, agile feature teams use a Definition of Done while a waterfall infrastructure stream uses phase-gate quality reviews. Integration testing keeps surfacing interface defects late, and both groups blame each other. What should the project manager do FIRST?

## Options

- **A**: Mandate that the infrastructure team adopt the agile Definition of Done so quality criteria are uniform.
- **B**: Stand up joint interface checkpoints at which both teams jointly verify integration contracts before downstream work proceeds.
- **C**: Add a dedicated integration QA team accountable for catching all interface defects before release.
- **D**: Add interface tests to the agile Definition of Done so every story includes integration coverage.

## Answer

B

## Explanation

The defects sit at the boundary between two delivery models, so the fix has to be at the boundary. Joint checkpoints make ownership explicit while preserving each stream's appropriate methodology. (A) forces one model on work it doesn't suit; infrastructure often genuinely needs phase gates. (C) creates a hand-off team that absorbs the failure rather than removing it, weakening both teams' accountability. (D) only adjusts one side and still leaves the infrastructure team disengaged from interface quality.
