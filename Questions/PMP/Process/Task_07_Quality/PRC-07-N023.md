# Question

On a hybrid programme, the development team's Definition of Done includes unit tests, peer review, and updated documentation. To hit a hard regulatory deadline, the team lead proposes that for the next two sprints the team marks stories done once code is merged and defers the tests and documentation to a backlog item called "DoD catch-up". The sponsor knows the deadline is immovable but has not seen this proposal. The PM has room to act before the next sprint starts. What should the project manager do?

## Options

- **A**: Let the team run the relaxed Definition of Done for the two sprints, and add the deferred testing and documentation to the release plan as scheduled work after the deadline so the catch-up is committed, not just captured.
- **B**: Show the sponsor that marking stories done at merge will make the increment report as complete while testing and documentation remain outstanding, quantify that carried-forward work, and agree the approach with the sponsor before changing the Definition of Done.
- **C**: Keep the full Definition of Done but apply it only to the stories on the regulatory critical path, allowing lower-risk stories to merge with testing and documentation deferred so the deadline is protected where it legally matters.
- **D**: Hold the Definition of Done and instead descope the lowest-priority stories from the two sprints so everything that is delivered is genuinely done, raising the descope with the sponsor as a scope trade.

## Answer

B

## Explanation

Every option here keeps the regulatory deadline in view, so the discriminator is what the PM owes the accountable sponsor *before* the definition of "done" is changed. Redefining done to mean "merged" makes the increment report as finished while tested, reviewed, documented work is silently outstanding — accumulating technical debt and potential regulatory exposure that "done" stories will hide. The PM's first duty is to make that invisible carried-forward work visible and let the accountable sponsor decide with full information (B). (A) commits the catch-up to the plan, which is better than burying it on a backlog, but it still enacts the relaxed definition first and informs the sponsor of the consequences only by implication — it decides, then schedules, rather than surfacing the trade-off for the sponsor's call. (C) is a defensible risk-based application of the DoD, but selectively weakening "done" by story still changes what "done" means and is being chosen without the sponsor seeing the quality trade-off. (D) protecting done by descoping is a legitimate move and is correctly raised with the sponsor — but it presumes descoping is acceptable and acts on the schedule lever before the sponsor has weighed deferring quality versus cutting scope. B is the FIRST move because it equips the sponsor to choose among exactly these trade-offs; the others each pick a trade-off first.
