# Question

Your project's quality objective is "95% user satisfaction" with a new mobile application. The test team's current test plan covers only functional requirements traced to user stories. What is the project manager's BEST next step?

## Options

- **A**: Add automated performance testing to the test plan so the application's response time and throughput can be measured under load before release.
- **B**: Add accessibility testing to the test plan so that the application meets the relevant WCAG criteria before its general-availability release.
- **C**: Add reliability testing to the test plan so crashes and long-run stability can be measured under prolonged use before the release date.
- **D**: Identify which non-functional quality attributes drive user satisfaction for this product and extend the test plan to cover each one.

## Answer

D

## Explanation

User satisfaction is multi-dimensional — functional correctness alone does not deliver it. PMI quality planning is to derive measures from stakeholder-facing quality attributes and design tests for each. (D) does that. (A), (B), and (C) each address one important dimension in isolation; any single one of them on its own would still leave the test plan blind to the others, which is the same gap the question describes.
