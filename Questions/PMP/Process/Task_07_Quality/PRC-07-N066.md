# Question

A bottling line uses a control chart for fill volume. All fifteen recent points sit inside the control limits, but the last eight in a row fall above the centre line. Every fill is still well within the customer's tolerance, and no limit has been breached. The line supervisor wants to keep running and revisit it if a point ever crosses a limit. As project manager, what is the most appropriate response?

## Options

- **A**: Keep running without action, because every fill remains within both the control limits and the customer tolerance.
- **B**: Nudge the fill setpoint down toward the centre line so the points return to straddling it, then continue monitoring.
- **C**: Recalculate the control limits from the recent data so the chart reflects how the process is currently performing.
- **D**: Keep running but open an investigation into the run above the centre line to find and address the assignable cause.

## Answer

D

## Explanation

Eight consecutive points on one side of the mean is a run that signals a non-random shift with an assignable cause, even though no limit was breached and product is still in tolerance — the value of statistical process control is catching the drift before it produces out-of-tolerance fills. (D) keeps production going while investigating the cause, the correct balance. (A) waits for a limit breach and ignores the run rule, missing the early signal the chart exists to provide. (B) adjusts the setpoint in reaction to a shift whose cause is unknown — tampering, which can increase variation if the cause is transient. (C) widens or re-centres the limits to fit drifted data, masking the very signal that should be investigated. All are real SPC responses; the discriminator is acting on the run rule by finding the cause rather than ignoring, tampering, or hiding the signal.
