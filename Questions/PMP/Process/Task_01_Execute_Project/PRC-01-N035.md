# Question

A biotech lab is building a sample-tracking system. The team and product owner need to define a first releasable increment that delivers real value quickly under a tight grant-reporting deadline. Which principles should guide how they slice the first increment? (Select three.)

## Options

- **A**: The increment should let a lab technician complete a genuine end-to-end task, such as logging and retrieving a sample.
- **B**: The increment should include every reporting field auditors might eventually request, to avoid rework later.
- **C**: The increment should be prioritised by the value it delivers to users and the grant deadline, not by which components are easiest to build.
- **D**: The increment should be releasable and demonstrable, not a horizontal layer like a database schema with no usable function.
- **E**: The increment should be deferred until the full data model for all future modules is finalised.

## Answer

A, C, D

## Explanation

A valuable first increment is a thin, vertical, releasable slice that lets a real user complete a real task and is chosen by value, not ease. (A), (C), and (D) capture end-to-end usefulness, value-based prioritisation, and releasability. (B) is gold-plating that loads the first slice with speculative fields and delays value. (E) blocks all delivery on finalising a complete data model, the opposite of slicing for early value under a deadline.
