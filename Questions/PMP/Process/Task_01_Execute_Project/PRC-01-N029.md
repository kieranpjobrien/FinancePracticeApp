# Question

A media-streaming team is building a personalised recommendations feature. The highest-value story, "recommend based on viewing history", depends on an analytics data pipeline that another team will not finish for three sprints. Several lower-value but independent stories, such as "trending now" and "because you watched", can be delivered immediately. The product owner wants value flowing this sprint. What should the project manager recommend?

## Options

- **A**: Wait until the analytics pipeline is ready so the team can deliver the highest-value story first.
- **B**: Deliver the independent lower-value stories now to keep value flowing, while pressing the other team to firm up the pipeline date for the dependent story.
- **C**: Build a temporary throwaway version of the analytics pipeline so the highest-value story can ship this sprint.
- **D**: Reassign the team to help the other team finish the pipeline faster, pausing the recommendations work.

## Answer

B

## Explanation

When the top-value item is externally blocked, urgency is served by delivering the next-best independent value now rather than idling. (B) keeps a stream of value flowing and actively manages the dependency for the blocked story. (A) stalls all delivery for three sprints waiting on one item. (C) invests in throwaway infrastructure for a single feature, likely costing more than it returns. (D) abandons the team's own valuable independent work to subsidise another team, with no guarantee of net speed-up.
