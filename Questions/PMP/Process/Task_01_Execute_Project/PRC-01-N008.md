# Question

A government agency's project manager is coaching a new product owner on ordering a backlog so value reaches citizens as early as possible. The product owner lists several factors they are tempted to sort by. Which THREE are sound primary bases for sequencing the backlog to maximise early value delivery? (Select THREE.)

## Options

- **A**: The mission value each item is expected to deliver to citizens relative to its size, so high-value, low-effort items surface first.
- **B**: Whether an item unblocks or enables several other high-value items that cannot start until it is done.
- **C**: The cost of delay of each item, prioritising those whose value erodes fastest if deferred.
- **D**: How well-understood and ready-to-start each item is, doing the clearest, lowest-uncertainty items first so the team keeps moving without getting stuck.
- **E**: The development effort each item requires, building the lowest-effort items first to maximise the number of items shipped.

## Answer

A, B, C

## Explanation

Early value delivery is driven by benefit relative to effort, enablement, and the rate at which value decays. (A) ranks by value-to-effort, surfacing the strongest return first. (B) recognises enablers unlock downstream value and can be worth doing early despite modest standalone benefit. (C) uses cost of delay to prioritise items whose value is perishable. (D) is the subtler trap: sequencing by readiness keeps the team busy and feels efficient, but "clearest first" optimises for low uncertainty, not value — it can park a high-value but fuzzier item behind several easy, well-understood low-value ones, and readiness is properly handled through refinement, not by letting it drive the value order. (E) optimises for *count* of items shipped, not value delivered; cheapest-first maximises throughput of low-impact work and can defer the items citizens actually need. Both D and E reward something real (flow, throughput) that is not the same as early value, which is the distinction a PM should teach a new product owner.
