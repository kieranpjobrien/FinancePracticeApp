# Question

A regulatory compliance project has a fixed scope (mandatory controls) and a fixed deadline (statutory). The team prefers agile practices. How should the PM tailor the approach?

## Options

- **A**: Convert to a strict waterfall plan since both scope and the deadline are fixed; agile won't help in this context.
- **B**: Run an agile cadence to deliver and validate controls in priority order, with a must-have/should-have split to flex on.
- **C**: Commit to delivering all the controls and rely on overtime in the final month if execution starts to fall behind schedule.
- **D**: Sequence controls by technical complexity so the team builds momentum on the simpler items first, then tackles the harder ones.

## Answer

B

## Explanation

When scope and time are both fixed, the only real lever is the order in which work is done — and agile cadence with prioritisation gives the earliest signal of trouble. (B) keeps agile's strengths (early delivery, fast feedback, clear must-have backstop) inside the regulatory constraint. (A) discards useful practice based on a misunderstanding — agile and predictive can coexist. (C) plans to use heroics as a recovery strategy. (D) sequences by ease, which prioritises team comfort over compliance risk.
