# Question

A Scrum team has a release candidate meeting the Definition of Done for all committed stories. Marketing wants two extra "nice-to-have" features bundled before the release goes public. The PO agrees, but engineering estimates another four sprints. What should the PM advocate?

## Options

- **A**: Release the current build now and plan the additional features as a clearly scoped follow-on release candidate.
- **B**: Delay release by four sprints to bundle marketing's features so the launch lands with maximum promotional impact.
- **C**: Split the team into two streams so marketing work runs in parallel without delaying the current release scope.
- **D**: Ask marketing to lower their ambition and accept a faster, smaller launch without the extra feature bundle.

## Answer

A

## Explanation

Value is realised when working software reaches users; delay burns optionality. (A) banks realised value now and keeps the follow-on clean and prioritised. (B) delays realised value for unvalidated marketing assumptions. (C) splitting a Scrum team mid-flight breaks cohesion and velocity without evidence it pays back. (D) frames marketing as the problem rather than facilitating a prioritisation conversation.
