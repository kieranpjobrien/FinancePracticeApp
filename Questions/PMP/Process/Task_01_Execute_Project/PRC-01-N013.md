# Question

After releasing the first increment of a customer self-service portal for an energy retailer — built to cut call-centre load and let customers resolve account tasks themselves — the project manager wants measures that confirm the increment is *realising that value*, not merely showing activity. Which TWO measures best evidence the intended value being realised? (Select TWO.)

## Options

- **A**: The reduction in call-centre contacts for the tasks the portal now handles, compared with before the release.
- **B**: The number of self-service tasks customers start on the portal each week since launch.
- **C**: The proportion of customers who start a self-service task and successfully complete it without falling back to the call centre.
- **D**: The average satisfaction rating customers leave after using the portal.
- **E**: The retailer's overall Net Promoter Score measured across all channels this quarter.

## Answer

A, C

## Explanation

The intended benefit is deflected call-centre load and customers self-resolving, so value is evidenced by outcomes tied to *that* goal. (A) measures the benefit directly — fewer contacts for the migrated tasks means real deflection and cost saving. (C) measures successful self-service completion without fallback, the precise usage outcome the portal exists to produce. (B) counts tasks *started*, which captures intent and traffic but not value: a customer can start, fail, and still phone in, so attempts alone do not show self-resolution. (D) satisfaction is a worthwhile signal but measures sentiment, not the deflection benefit — customers can rate the portal pleasant yet still complete their task by calling, and only those who choose to rate are counted. (E) is a broad, lagging organisation-wide measure influenced by pricing, outages and every other channel; it cannot isolate this increment's contribution, so it is unsuitable for confirming *this* release realised its value.
