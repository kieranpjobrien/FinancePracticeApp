# Question

A sprint's highest-value item — a payments feature — is blocked because another department has not provisioned a third-party sandbox credential. The project manager raised it with that department's team lead two days ago and chased again yesterday; the lead keeps agreeing it matters but it stays undone, apparently stuck behind their own queue. The sprint has six days left. A sandbox can be self-provisioned in a test tenant within a day, letting development proceed, though the credential would still be needed before release. What should the project manager do?

## Options

- **A**: Stand up the self-provisioned test sandbox now so development continues immediately, and in parallel escalate the stalled credential to the department's manager so it is in place before release.
- **B**: Escalate the credential to the department's manager and the sponsor and wait for it, since the official credential is what the feature ultimately needs and a workaround risks divergent test conditions.
- **C**: Keep working the team lead daily and offer to help clear whatever is blocking them, preserving the working relationship rather than going over their head.
- **D**: Self-provision the test sandbox and treat that as the fix for the sprint, raising the official credential later since development can proceed for now.

## Answer

A

## Explanation

The impediment must be removed, but politely re-chasing the same lead has twice failed and the clock is running, so the best move unblocks the work *now* and fixes the root cause *in parallel*. (A) restores progress immediately via the self-provisioned sandbox and simultaneously escalates the stuck credential up the chain so it lands before release — protecting both flow and the eventual release gate. (B) escalates correctly but then *waits*, idling the team for a day or more when a safe interim path exists; it over-indexes on the official credential when development could already be moving. (C) maintains harmony but repeats the approach that has not worked, tolerating continued stall on the top item — relationship preservation at the cost of the sprint's value. (D) takes the interim unblock but quietly drops the escalation, so the release-blocking credential is still unprovisioned and the same delay simply resurfaces at release time.
