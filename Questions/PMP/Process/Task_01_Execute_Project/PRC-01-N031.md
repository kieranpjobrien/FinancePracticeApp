# Question

An online-learning platform team has just completed a working, tested course-progress tracker that learners have been requesting. The next planned feature is unrelated and several weeks away. The release process is low-cost and the feature can ship independently. Marketing notes that a larger combined announcement next month, bundling the tracker with upcoming features, would draw more attention than a standalone release. What should the project manager advise?

## Options

- **A**: Release the completed tracker now so learners get the requested value immediately, and let marketing still feature it in the larger announcement next month.
- **B**: Hold the tracker and release it with the bundle next month, so a frequently requested feature anchors a higher-impact announcement.
- **C**: Release the tracker now to a subset of learners as a soft launch, holding the wider rollout to coincide with the bundled announcement.
- **D**: Release the tracker now but delay publicising it at all until the combined announcement, so the marketing moment is preserved.

## Answer

A

## Explanation

The feature is finished, requested, and cheap to ship on its own, so the value belongs in learners' hands now, and the marketing upside does not require withholding it. (A) delivers the benefit immediately and still lets marketing reference the tracker in next month's announcement, capturing both. (B) is the clearest competing case but trades a month of real learner value for announcement optics on a feature that ships independently. (C) gives some learners the value sooner yet deliberately withholds it from most for a marketing date, slowing benefit for rollout-control the situation does not demand. (D) ships the feature but suppresses any communication, which understates a wanted improvement and risks learners not discovering value that is already live.
