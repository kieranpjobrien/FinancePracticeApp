# Question

On a hybrid telecom project modernising a billing platform, the sponsor is frustrated that working billing changes are reaching production slowly and wants that fixed before the next quarterly review. The team's board shows many items started, few finishing; cycle times have been climbing for three iterations. The sponsor proposes funding two more developers to push the work through faster. The project manager has flow metrics for the period. How should the project manager respond?

## Options

- **A**: Accept the additional developers, since more capacity is the most direct way to convert the sponsor's funding into faster delivery of billing changes.
- **B**: Walk the sponsor through the flow data, propose capping items in progress and finishing started work first, and revisit adding people only if throughput stays constrained afterward.
- **C**: Keep capacity unchanged but lengthen the iteration so the many in-flight items have more time to reach completion before the review.
- **D**: Reprioritise so the billing changes most visible at the quarterly review are finished first, leaving overall flow as it is.

## Answer

B

## Explanation

The data pattern — many items started, few finishing, rising cycle time — points to excess work in progress as the constraint, so the disciplined response is to show the sponsor the flow evidence, cap WIP, and finish started work before adding anything else (B). (A) is the strongest distractor because the sponsor is offering real capacity and more developers feels like obvious progress, but adding people to a flow that is already drowning in started work raises coordination overhead and usually slows delivery further; the PM owes the sponsor the diagnosis, not just compliance with the funding offer. (C) lengthening the iteration gives stuck items more time without addressing why they stall, masking the WIP problem and delaying feedback. (D) reprioritising the review-visible changes can produce a short-term demo but leaves the underlying flow constraint untouched, so delivery stays slow once the review passes; it treats the symptom the sponsor will see rather than the cause.
