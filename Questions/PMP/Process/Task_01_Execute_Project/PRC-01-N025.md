# Question

A hybrid government-services portal is mid-execution with a legislated launch date in seven weeks. A newly appointed director, who carries weight with the programme board, asks to add a digital-identity verification module that was never in scope, arguing it would greatly improve citizen experience. The team's first read is that fitting it in seven weeks is doubtful but not obviously impossible. What should the project manager do first?

## Options

- **A**: Have the team scope the module's effort and its effect on the legislated date, then take the assessed options to the director and the change board before committing either way.
- **B**: Commit to delivering the module in this release and replan toward it, treating a senior director's citizen-experience priority as a clear value signal worth reorganising around.
- **C**: Tell the director the legislated date is fixed and the module will be logged for the next release, protecting the committed launch without spending effort assessing an out-of-scope ask.
- **D**: Offer to include a lightweight version of identity verification now and defer the full module, as a middle path that gives the director something in this release.

## Answer

A

## Explanation

The director's request is a real change against a legally fixed date, and whether it fits is genuinely uncertain, so the disciplined first move is to size the impact and let the board decide with evidence rather than the PM pre-empting it. (A) produces an assessed set of options and routes the call through change control, honouring both the stakeholder and the deadline. (B) commits to the date-threatening scope before anyone knows if it fits, betting a statutory launch on an optimistic first read. (C) reaches a plausible end-state but gets there by refusing to assess a senior stakeholder's idea, which damages engagement and may be wrong if the module actually does fit. (D) invents a scope compromise on the spot without analysis, and a half-built identity-verification feature in a government portal carries security and compliance exposure of its own.
