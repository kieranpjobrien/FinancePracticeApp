# Question

A project manager runs a hybrid programme with two work streams sharing one delivery team this fortnight. Stream X has a feature that, if shipped now, lets the sales team close a large deal already in late-stage negotiation. Stream Y has a compliance fix with a regulatory deadline three weeks away. The team can fully complete only one stream this fortnight; the other will slip by two weeks but both would still meet their respective deadlines. How should the project manager prioritise?

## Options

- **A**: Deliver Stream X's feature now, because doing so unlocks time-sensitive revenue while Stream Y still has slack to meet its regulatory deadline.
- **B**: Deliver Stream Y's compliance fix first, because regulatory work always takes precedence over revenue features.
- **C**: Split the team across both streams to make partial progress on each, reducing the slip on both.
- **D**: Escalate to the sponsor to decide, since the project manager should not choose between revenue and compliance.

## Answer

A

## Explanation

Both deadlines can still be met under either order, so the tie-breaker is value timing: delivering the revenue-unlocking feature now captures a time-sensitive deal while the compliance fix retains enough slack to land safely (A). (B) applies a blanket rule that ignores the slack analysis and needlessly delays revenue. (C) splits focus, increasing work in progress and likely finishing neither cleanly this fortnight. (D) escalates a decision the PM is equipped to make, since the data already shows both deadlines are safe.
