# Question

A SaaS team has been delivering features fast for two quarters and velocity is now dropping sharply as defects and rework accumulate. The product owner still pushes for new features each sprint. What should the PM raise in the next sprint planning?

## Options

- **A**: Propose a dedicated hardening sprint where the team pauses feature work and clears the entire backlog of accumulated technical debt.
- **B**: Ask the PO to slow new feature intake temporarily so the team can absorb the current defect load before adding more.
- **C**: Suggest allocating a fixed percentage of each sprint to technical-debt items, making the trade-off visible to the PO on every cycle.
- **D**: Accept the pace and raise velocity decline as a risk so leadership understands the long-term cost of current choices.

## Answer

C

## Explanation

Technical debt is a continuous cost; PMI-aligned agile practice is to surface the trade-off to the PO every sprint so prioritisation stays transparent. (C) makes debt a first-class backlog concern. (A) a pure hardening sprint is a big-bang fix that rarely survives contact with deadlines. (B) slowing intake without structure leaves the trade-off ambiguous. (D) raising risk without proposing a response abdicates the PM's action role.
