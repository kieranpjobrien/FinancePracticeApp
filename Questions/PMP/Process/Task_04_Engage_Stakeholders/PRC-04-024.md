# Question

A PM inherits a project mid-execution. The stakeholder register hasn't been updated in four months and several new stakeholders have emerged. What should the PM do FIRST?

## Options

- **A**: Continue using the existing register since it was approved previously.
- **B**: Discard the old register and build one from scratch.
- **C**: Conduct a stakeholder re-identification exercise to add new stakeholders, retire obsolete entries, and update classifications.
- **D**: Delegate register maintenance to a team member going forward without reviewing current state.

## Answer

C

## Explanation

Stakeholder identification is iterative — changes in scope, phase, and environment produce new stakeholders. (C) refreshes the register in a structured way. (A) stale information guides poor engagement decisions. (B) throwing away institutional knowledge in the existing register is wasteful. (D) delegating without first understanding the current state means the PM can't direct the work meaningfully.
