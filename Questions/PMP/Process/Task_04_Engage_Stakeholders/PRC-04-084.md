# Question

A project manager has identified and analysed the stakeholders for a new university student-portal project and understands each group's interest and influence. Several stakeholders have very different information needs and preferred channels. What should the project manager do next to keep them effectively engaged?

## Options

- **A**: Develop a stakeholder engagement plan defining the approach, channels, and frequency tailored to each group's needs.
- **B**: Schedule a single project-wide briefing so all stakeholders receive the same baseline information at the same time.
- **C**: Begin sending regular status reports to every stakeholder so no group feels uninformed during delivery.
- **D**: Wait for stakeholders to request information so engagement effort matches each group's actual demand.

## Answer

A

## Explanation

Once stakeholders are identified and analysed, the planned, deliberate step is to define how each will be engaged through a stakeholder engagement plan that tailors approach, channel, and frequency. (A) is that step and accounts for the differing needs already identified. (B) a single uniform briefing ignores the analysis showing varied needs and channels. (C) blanket status reports are untailored and risk over- or under-informing different groups. (D) waiting for requests is reactive and lets disengagement set in, the opposite of proactive engagement.
