# Question

Stakeholders of an agile marketing-automation project keep emailing the PM individually asking for status, progress against features, and what is coming next. The PM spends hours each week answering near-identical questions, and some stakeholders complain the answers they get differ from one another. What is the best way to improve stakeholder engagement here?

## Options

- **A**: Make current progress continuously visible to all stakeholders through a shared board and release burndown they can check whenever they want.
- **B**: Compile one detailed status email each week from a single source and send it to the full stakeholder list.
- **C**: Set up a shared inbox or channel for status questions so any team member can answer and the load is spread.
- **D**: Add a standing stakeholder Q&A to the sprint review so the recurring questions get answered together each iteration.

## Answer

A

## Explanation

The pattern — many individual asks and answers that disagree — points to information that is not openly and continuously available, so the fix is to radiate live progress that any stakeholder can self-serve at any time, giving everyone the same current picture (A). (B) is the strongest alternative and does fix consistency by writing once from one source, but a weekly push goes stale between sends and still leaves the ad-hoc "what's the status today" questions that drive the load. (C) spreads the work but does not reduce it and, with multiple people answering, can reintroduce the very inconsistency being complained about. (D) batches the questions usefully but makes stakeholders wait for a meeting for information that should be available on demand. The margin is between continuous self-serve transparency (A) and periodic or distributed pushes that still leave gaps (B, C, D).
