# Question

A logistics project manager receives a written change request from a customer asking for an additional reporting dashboard, and logs it in the change log. The change control board meets in two weeks. To prepare the request for that meeting, what should the project manager do next?

## Options

- **A**: Work with the team to assess the change's effect on scope, schedule, cost, quality, and risk, and document the findings for the board.
- **B**: Brief the sponsor and key stakeholders that the request exists so they are aware of the potential change before the board meets.
- **C**: Have the technical lead prepare a design for the dashboard so a worked solution is ready to present alongside the request.
- **D**: Ask the customer to clarify and confirm the detailed requirements for the dashboard so the request is unambiguous.

## Answer

A

## Explanation

Once a change is logged, the procedural next step is to analyse its impact across the project dimensions so the board can approve, defer, or reject it on evidence; this assessment is what a change control board decision depends on (A). Briefing stakeholders that the request exists (B) is reasonable communication but premature and incomplete before the impact is even understood, and it is not what prepares the board to decide. Preparing a full design (C) sinks solution effort into an unapproved change and presumes the answer; design work belongs after approval. Clarifying requirements with the customer (D) can be a genuine input, but it is a sub-step folded into scoping the impact assessment, not the overarching next action that readies the request for the board. The discriminator is that a complete impact analysis is the immediate deliverable the board needs, whereas the others are partial, premature, or subordinate to it.
