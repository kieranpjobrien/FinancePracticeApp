# Question

A new piece of regulation just passed changes data retention requirements across an in-flight financial services programme. Compliance is mandatory within ninety days. The PM has not yet raised a change request because implementation feels unavoidable anyway. What should the PM do?

## Options

- **A**: Implement compliance without a change request, since regulatory mandates do not require CCB approval formally.
- **B**: Raise the change after implementation to document the regulatory response retrospectively for audit trail purposes.
- **C**: Raise the change request now so impact is assessed, baselines updated, and governance visibility maintained.
- **D**: Reduce programme scope elsewhere to absorb the compliance work without raising a formal change request now.

## Answer

C

## Explanation

Regulatory changes still go through integrated change control so impact on cost, schedule, and scope is visible and baselines stay current. (A) bypasses governance because the outcome seems certain. (B) reverses the proper sequence and produces retrospective records that are less rigorous. (D) absorbs impact silently and obscures the true programme picture.
