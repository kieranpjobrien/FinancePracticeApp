# Question

The cost and schedule baselines were set three months ago. Since then, fifteen approved changes have modified scope and EVM is now showing CPI 0.74 and SPI 0.81. A stakeholder argues the baseline is no longer meaningful and the project should be rebaselined. What should the PM do FIRST?

## Options

- **A**: Submit a change request to rebaseline immediately so EVM reflects the approved scope after all the changes.
- **B**: Continue measuring against the original baseline because baselines are immutable once formally approved by CCB.
- **C**: Report performance against the original baseline alongside an informal current forecast each reporting cycle.
- **D**: Evaluate whether cumulative approved change warrants rebaseline and route a recommendation through change control.

## Answer

D

## Explanation

Rebaselining is itself a change requiring evaluation and CCB approval; the PM should assess whether the original baseline still gives meaningful insight before recommending action. (A) jumps to rebaselining without analysis, which can mask underlying performance problems. (B) clings to a baseline that may no longer be useful. (C) creates parallel reporting without resolving the underlying baseline question and risks confusion.
