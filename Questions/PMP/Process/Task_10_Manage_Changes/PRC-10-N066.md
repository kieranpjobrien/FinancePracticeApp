# Question

A project manager is coaching a new coordinator on where different change-related entries belong. For each situation on the left, match it to the artifact or register where it should primarily be recorded or actioned (right). Each option is used once.

## Items

- **1**: A stakeholder asks the project manager to add a new interface to the agreed scope and wants it formally considered.
- **2**: After a change is approved, the team needs to issue the affected design drawing as a new controlled revision and keep its prior versions traceable.
- **3**: The coordinator wants to see, at a glance, every change that has been raised and how each was dispositioned.
- **4**: A supplier has missed a milestone and the slippage now needs owners and actions to resolve it.

## Matches

- **A**: Change log
- **B**: Configuration management system
- **C**: Change request (raised into integrated change control)
- **D**: Issue log

## Answer

1-C, 2-B, 3-A, 4-D

## Explanation

A request to alter agreed scope is initiated as a change request entering integrated change control, where it is assessed and dispositioned (1-C). Issuing a controlled drawing as a new traceable revision is configuration management, which governs the version and state of controlled items (2-B). A consolidated view of every change raised and its disposition is the change log, the register of change requests and their outcomes (3-A). A missed supplier milestone that has already occurred is a current problem needing owners and actions, which is the issue log (4-D). The traps are pairing the change request that starts the process with the change log that records its disposition (1 versus 3), and pairing a controlled-item revision with the change log rather than the configuration management system (2 versus 3); the discriminator is initiation versus disposition record, item-version control versus change record, and change versus issue.
