# Question

A PM's approved change to a shared data model will ripple into a sister project that depends on the current model. The sister project's PM has not been consulted. The change has already been approved by the CCB. What should the PM do?

## Options

- **A**: Notify the sister project after implementation since they can adapt once the new model is actually in place.
- **B**: Assume the programme manager has already notified the sister project as part of cross-project coordination.
- **C**: Reverse the CCB approval until the sister project has been consulted, even though CCB approval is formal.
- **D**: Inform the sister project's PM immediately, review cross-project impact, and adjust implementation if needed.

## Answer

D

## Explanation

Cross-project dependencies require proactive notification and joint impact review even after CCB approval, so the change lands without breaking a neighbouring project. (A) prioritises speed over coordination and damages the sister project. (B) assumes coordination that has not been confirmed. (C) reverses a formal decision rather than coordinating around it sensibly.
