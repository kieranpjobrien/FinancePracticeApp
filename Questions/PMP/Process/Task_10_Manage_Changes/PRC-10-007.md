# Question

A project is running behind schedule and the PM adds a second tester to recover the variance. In PMI terminology, this is best classified as which of the following?

## Options

- **A**: A preventive action to avoid future variance.
- **B**: A defect repair to fix a deliverable.
- **C**: A corrective action to bring performance back to plan.
- **D**: A scope change to the project baseline.

## Answer

C

## Explanation

A corrective action realigns future project performance with the project management plan when actual performance has diverged from it. (A) preventive actions address risks that have not yet caused variance. (B) defect repair fixes a non-conforming deliverable, not a schedule variance. (D) a scope change modifies the baseline, whereas adding a tester is execution work to recover against the existing baseline.
