# Question

A project manager is reviewing recent events on a predictive rail-signalling project to decide which warrant raising a change request. Which three of the following should normally result in a change request? (Choose three.)

## Options

- **A**: A new safety regulation requires an additional redundancy circuit that was not in the approved scope.
- **B**: An activity finishes two days late but inside its free float, so the project end date is unaffected and the baseline still holds.
- **C**: An approved risk response is triggered that will draw on the contingency reserve and shift the schedule baseline.
- **D**: A deliverable fails verification against its existing specification and must be reworked to conform.
- **E**: The project manager re-sequences two non-critical activities within the existing baseline to smooth resource use, with no change to dates, cost or scope.

## Answer

A, C, D

## Explanation

A regulatory scope addition (A), enacting a risk response that consumes reserve and moves a baseline (C), and reworking a deliverable that failed verification, which is defect repair (D), all change or restore baselined work and so are processed through change requests. An activity slipping but staying within its free float (B) leaves the baseline intact and is a variance managed within tolerance, not a change. Re-sequencing non-critical activities inside the baseline with no effect on dates, cost or scope (E) is a routine optimisation the project manager makes within their existing authority, not a baseline change. The trigger for a change request is an impact on, or restoration of, an approved baseline, not variation that the baseline and tolerances already absorb.
