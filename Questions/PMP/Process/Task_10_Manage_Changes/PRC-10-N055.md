# Question

On a predictive bridge-strengthening project, weekly performance data shows a non-critical activity trending two weeks behind, though the milestone is still forecast on time. The project manager can recover it by resequencing crews within the existing plan, which leaves the cost and schedule baselines unchanged but means a subcontractor's crew works a different week than the published lookahead showed. How should the project manager handle this?

## Options

- **A**: Implement the resequencing as corrective action within the approved plan, update the lookahead and notify the subcontractor, without raising a change request.
- **B**: Raise a change request before resequencing, because the subcontractor's published working week is changing.
- **C**: Raise a change request because an activity is running behind the schedule, which is variance from the baseline that change control must record.
- **D**: Rebaseline the schedule to absorb the two-week slippage so the plan matches actual performance going forward.

## Answer

A

## Explanation

Resequencing crews to recover slippage while the cost and schedule baselines stay intact is corrective action taken within the approved plan; it is communicated and the lookahead refreshed, but it does not require a change request. (A) correctly separates managing to the plan from changing the plan, and still handles the stakeholder communication. (B) treats a routine, within-baseline resequencing as a formal change because a short-term lookahead shifted, which would burden delivery with change requests for ordinary adjustments. (C) confuses performance variance with a baseline change: variance is analysed and recovered, not change-controlled, when the baseline itself is unchanged. (D) rebaselines to make a recoverable slippage permanent, discarding the original baseline the project should still be measured against.
