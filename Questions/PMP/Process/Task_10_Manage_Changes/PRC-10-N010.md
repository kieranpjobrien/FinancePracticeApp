# Question

On a manufacturing-line project, a near-miss during installation reveals an unguarded pinch point. No unit has yet been built incorrectly, but the project manager judges that without a guard and a revised procedure a defect is likely to arise in later units. Adding the guard and updating the procedure will draw on time and budget not in the current plan. How should the project manager proceed?

## Options

- **A**: Raise the guard and procedure update as a preventive action through change control, since it changes planned work and consumes unplanned time and budget.
- **B**: Implement the guard and procedure change immediately under the project's health-and-safety obligation, then record what was done.
- **C**: Hold the guard as a contingency and only act if a unit is actually built with the defect, to avoid spending on a problem that has not occurred.
- **D**: Log the near-miss as a defect repair and process the rework of the affected work through change control.

## Answer

A

## Explanation

Reducing the probability of a future problem that has not yet occurred is a preventive action, and when it alters planned work and consumes unplanned time and cost it is an approved change action processed through integrated change control. (A) handles it correctly. (B) is tempting because safety is urgent, but acting first and recording later bypasses the assessment and authorisation that the unplanned effort needs; genuine emergencies have their own provision, and a near-miss with no unit yet affected is a preventive matter, not an emergency fix. (C) waits for the defect to materialise, which contradicts the purpose of preventive action and accepts avoidable risk. (D) calls it defect repair, but nothing nonconforming has been produced yet, so there is no defect to repair. The correct classification and route is a preventive action through change control.
