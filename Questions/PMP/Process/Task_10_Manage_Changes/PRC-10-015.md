# Question

Eight CCB-approved changes are queued for implementation but the team has been pulled to firefight other work. Three of the queued changes block tasks on the critical path. What should the PM do?

## Options

- **A**: Implement the queued changes in the order they were approved by the CCB to be fair to requesters.
- **B**: Defer all queued changes until dedicated resources can be assigned to the change backlog.
- **C**: Reprioritise the change implementation queue based on critical-path dependencies and business impact, and replan resource allocation accordingly.
- **D**: Return the three blocking changes to the CCB so they can be re-approved with explicit priority.

## Answer

C

## Explanation

Approved changes are work the team must plan and execute like any other deliverable; sequencing should follow dependencies and value rather than approval order. (A) ignores critical-path impact and lets blockers persist. (B) compounds the schedule risk by deferring blocking changes. (D) wastes CCB time and conflates approval with prioritisation, which is the PM's responsibility once changes are approved.
