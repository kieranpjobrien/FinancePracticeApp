# Question

On a predictive payroll-system upgrade, a business analyst mentions she has been quietly making small adjustments that end users requested informally during testing, none of them logged. The project manager is concerned the accumulation is eroding the agreed scope. What should the project manager do to bring this under control?

## Options

- **A**: Have the analyst write up the adjustments already made and route each, with any future requests, through the change-control process to be assessed and dispositioned.
- **B**: Set a threshold so the team can implement minor adjustments directly and only escalate larger requests to change control, keeping a log of the minor ones.
- **C**: Trace the adjustments back to their cause by reviewing whether the requirements baseline captured the users' needs, then revise the baseline if it was incomplete.
- **D**: Brief the users and the analyst on the change-control procedure and the impact of unlogged changes, so requests come through the proper channel from now on.

## Answer

A

## Explanation

The adjustments are unassessed changes to a baselined scope, and control is restored by bringing both the changes already made and all future requests into the change-control process, where each is logged, assessed for impact, and dispositioned (A). A minor-change threshold (B) is a real governance option, but inventing one mid-project to legitimise direct, unapproved implementation institutionalises the very creep at issue and still bypasses impact assessment; a logged tweak is not an assessed one. Investigating whether the requirements baseline was incomplete (C) is a worthwhile root-cause inquiry, yet it diagnoses why requests arise without governing the unassessed changes already flowing, so the creep continues while the analysis proceeds. Briefing everyone on the procedure (D) is prevention that helps going forward but, on its own, leaves the adjustments already made outside assessment and disposition. The discriminator is that regaining control requires channelling the existing and future changes through assessment and approval now, not setting a bypass, deferring to root-cause work, or relying on communication alone.
