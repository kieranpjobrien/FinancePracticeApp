# Question

A project is in detailed planning. The scope statement is drafted but the performance measurement baseline has not yet been approved. A key stakeholder requests a significant addition to scope. A junior project manager proposes raising a formal change request to the change control board. What should the experienced project manager advise?

## Options

- **A**: Work the addition into the evolving plan through normal planning and progressive elaboration, since there is no approved baseline yet for a change request to act against, after confirming the addition with the affected stakeholders.
- **B**: Raise the change request to the change control board so the significant addition is assessed and approved before it is built into the plan.
- **C**: Record the request in the change log now and analyse its impact, then fold the result into the plan once planning resumes.
- **D**: Ask the sponsor to approve the addition before incorporating it, since a significant scope increase needs sign-off whatever the project's phase.

## Answer

A

## Explanation

Formal change control governs changes to approved baselines; before a baseline exists, evolving requirements are handled through planning and progressive elaboration, with normal stakeholder agreement, not through change requests. (A) reflects that the project has nothing baselined to change yet. (B) applies the change control board to a pre-baseline situation, imposing ceremony when there is no baseline to protect. (C) opens a change-log entry and impact analysis as though a controlled change were under way, which is premature for requirements still being shaped in planning. (D) seeks sponsor sign-off, which sounds prudent, but during planning a significant addition is incorporated and agreed through the planning process and its normal approvals, not handled as a controlled change against a baseline that does not yet exist. Knowing when the baseline is set distinguishes planning from controlled change.
