# Question

On a predictive project, the project manager discovers that a senior engineer, acting in good faith to save time, has already modified a baselined interface specification and begun building to the new version. The modification looks technically sound and the engineer says reverting it would waste two days of completed work. What should the project manager do first?

## Options

- **A**: Direct the engineer to revert to the baselined specification immediately, then raise a change request if the modification is still wanted.
- **B**: Halt further work on the modified specification and raise a change request so the deviation is assessed and formally dispositioned before any more is built on it.
- **C**: Raise a change request for the modification and, given it looks sound, let the engineer keep building while the request is assessed so the two days are not lost.
- **D**: Document the deviation in the change log and present the already-completed work to the change control board as a fait accompli for ratification.

## Answer

B

## Explanation

Unauthorised work against a baseline has to be brought back under control, and the controlled response is to stop adding to the deviation and route it through change control so its full impact is assessed before any further commitment, while not destroying sound work prematurely. (B) does exactly that: it freezes the divergence at its current extent and lets the process decide, which is the disciplined first action. (C) is the sharpest trap because it correctly raises a change request, yet it simultaneously lets construction continue on an unapproved baseline change, deepening the divergence the project manager is supposed to be containing and pre-committing the project to an outcome the board has not approved. (A) over-reacts by ordering an immediate revert before the change is even assessed, potentially discarding work the board might well approve and inverting the order of analyse-then-decide. (D) treats the completed work as leverage to force ratification, which corrupts the board's independence and rewards working around the baseline. Stopping the spread and assessing through change control, without either bulldozing or rubber-stamping, is the correct first move.
