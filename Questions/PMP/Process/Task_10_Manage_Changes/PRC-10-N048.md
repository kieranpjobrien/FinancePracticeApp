# Question

A project manager is setting up change handling for a hybrid project: an agile delivery team feeding into a regulated predictive release gate. Which of the following correctly describe how changes should be handled in this arrangement? (Choose three.)

## Options

- **A**: The product owner orders and reprioritises the product backlog within the approved scope without raising change requests for routine reordering.
- **B**: Changes that alter the committed, funded scope or breach the regulated release criteria go through formal integrated change control.
- **C**: Emerging requirements are welcomed into the backlog rather than suppressed, then prioritised against existing work.
- **D**: Because the release is regulated, the change control board should pre-approve each sprint's selected backlog items before the team commits to them in sprint planning.
- **E**: New requirements may be added to the backlog freely, but any that would change the committed release scope can be absorbed by the team without change control as long as they fit within the team's velocity.

## Answer

A, B, C

## Explanation

A workable hybrid model lets the product owner reorder the backlog freely within approved scope (A), routes genuine scope or compliance-affecting changes through formal change control (B), and treats emerging requirements as welcome backlog candidates to be prioritised (C). (D) is plausible because the release is regulated, but funnelling each sprint's backlog selection through the board for pre-approval collapses the team's ordinary planning into governance overhead; the board controls changes to committed scope and release criteria, not which in-scope items the team sequences into a sprint. (E) correctly allows new requirements into the backlog but then smuggles in the error that fitting within velocity makes a change to committed release scope exempt from change control; capacity to build something is not authority to change what was committed. The art of hybrid change control is drawing the line between what the team owns (ordering and refining within approved scope) and what governance must approve (changes to the committed scope and the regulated release criteria).
