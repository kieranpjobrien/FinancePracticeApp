# Question

A Scrum product owner on a media platform has added seven items to the release backlog in four weeks, each individually reasonable. The release goal is at risk and the team is nervous. What should the PM raise with the product owner?

## Options

- **A**: Revisit release goals in light of the added items and decide what to drop, defer, or re-scope explicitly.
- **B**: Commit to a longer release to accommodate the additions and communicate the new date to stakeholders now.
- **C**: Tell the team to improve velocity to absorb the additions without compromising the original release goal.
- **D**: Freeze the release backlog for the next two weeks so the team can focus on completing existing items.

## Answer

A

## Explanation

Release goals protect value delivery; changes to the backlog must be reconciled against the goal, not absorbed silently. (B) extends without explicitly discussing value trade-offs. (C) pushes the team to hide the problem through effort. (D) freezes without reconciling value and postpones the conversation that needs to happen.
