# Question

A team member on a school construction project raises a minor query about the wording of a signage label. The project manager judges it trivial, with no impact on cost, time, quality or scope, and easily resolved within the work package. How should this be handled under PRINCE2?

## Options

- **A**: Open a formal issue report and add it to the issue register for an audit trail
- **B**: Escalate it to the change authority as a request for change
- **C**: Ignore it entirely, since trivial matters need no record at all
- **D**: Manage it informally and record it in the daily log

## Answer

D

## Explanation

PRINCE2 permits trivial issues with no material impact to be handled informally, with the project manager noting them in the daily log rather than invoking the full formal procedure. (A) imposes unnecessary formality and overhead for a trivial matter. (B) misclassifies a wording query as a baseline change and over-escalates. (C) is the near-miss — informal handling still calls for a lightweight record in the daily log, not no record at all.
