# Question

A project manager joining a delayed airport signage project finds that products are being built and tested, but there is no documented basis for what "good" looks like and testers are applying their own judgement. Which corrective action addresses the root cause?

## Options

- **A**: Add more reviewers to each quality check so testers' judgements are cross-validated
- **B**: Define quality criteria, tolerances and methods in each product description so quality control has an objective basis, then capture activities in the quality register
- **C**: Escalate an exception to the project board because testing is unreliable
- **D**: Replace appraisal methods with in-process methods so testing is no longer required

## Answer

B

## Explanation

The underlying problem is missing quality planning: without defined quality criteria, tolerances and methods in the product descriptions, quality control has nothing objective to test against, so the fix is to establish that basis and then record activities in the quality register. (A) more reviewers still lack an agreed standard to judge against; (C) escalation does not supply the missing definitions; (D) in-process methods complement rather than remove the need for appraisal, and still require defined criteria.
