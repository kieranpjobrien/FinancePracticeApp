# Question

On a banking app project, an urgent regulatory request for change arrives that the project manager assesses as high priority but low cost, well inside the delegated change budget. The change authority is available. What should the project manager do?

## Options

- **A**: Implement the change at once, then record it retrospectively because it is urgent
- **B**: Escalate it to the project board because regulation makes it strategically important
- **C**: Add the new requirement directly to the product baseline, since it is mandatory
- **D**: Capture and examine the request, then pass it to the change authority to decide

## Answer

D

## Explanation

Even an urgent, mandatory change follows the procedure: capture, examine, then let the appropriate authority decide; here it is in budget, so the change authority decides. (A) implements before the issue is recorded and authorised, breaching change control. (B) is the near-miss — urgency and importance affect priority but the decision still sits within the change authority's delegated budget, so escalation to the board is unnecessary. (C) alters a baseline without any authorisation, which change control exists to prevent.
