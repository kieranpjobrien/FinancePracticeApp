# Question

A project manager is initiating a new visitor-centre construction project. The sponsor keeps describing what "good" would look like but nothing has been written down about how the finished centre will be judged acceptable. What should the project manager do?

## Options

- **A**: Proceed to the product breakdown structure, capturing acceptance details later when convenient
- **B**: Work with the sponsor to define and document the project product description, including acceptance criteria
- **C**: Ask each team manager to set their own acceptance criteria for the products they build
- **D**: Record the sponsor's comments as risks in the risk register

## Answer

B

## Explanation

The project product description captures the customer's quality expectations and acceptance criteria for the project's overall output and should be agreed early; the project manager should formalise these with the sponsor. (A) deferring acceptance criteria undermines the basis for the whole plan. (C) acceptance criteria for the final product are agreed with the customer, not set independently by team managers. (D) these are quality requirements to be defined, not risks.
