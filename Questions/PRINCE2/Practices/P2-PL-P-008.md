# Question

A project manager on a museum-exhibition project has a complete product breakdown structure and product descriptions, but the team keeps starting products in the wrong order, causing rework. Which planning product would BEST address this?

## Options

- **A**: A revised project product description with tighter acceptance criteria
- **B**: A larger product breakdown structure capturing more sub-products
- **C**: A product flow diagram showing the sequence in which products must be created and their dependencies
- **D**: An exception plan to recover the time lost to rework

## Answer

C

## Explanation

The product flow diagram arranges the identified products into the order in which they must be produced and shows the dependencies between them, which is exactly what is missing here. (A) acceptance criteria define quality, not sequence. (B) more hierarchy detail does not establish the order of creation. (D) an exception plan addresses tolerance breaches, not the underlying lack of a defined product sequence; it is a tempting but incorrect response to the rework symptom.
