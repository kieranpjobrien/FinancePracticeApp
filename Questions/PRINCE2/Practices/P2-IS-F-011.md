# Question

In PRINCE2 issue management, how do severity and priority differ?

## Options

- **A**: Severity determines who can decide the issue, while priority indicates how important and urgent it is
- **B**: Severity and priority are different names for the same attribute
- **C**: Severity ranks issues by cost, while priority ranks them by the date they were raised
- **D**: Severity applies only to off-specifications, while priority applies only to requests for change

## Answer

A

## Explanation

In PRINCE2 7, severity guides the level of management at which an issue can be decided (for example team, project manager or project board), whereas priority reflects how important and urgent the issue is and helps sequence the response. (B) is wrong because they answer different questions. (C) invents fixed cost-and-date definitions PRINCE2 does not use. (D) wrongly restricts each attribute to a single issue type; both apply across all issues.
