# Question

While preparing the business case for a warehouse-automation project, the Project Manager realises the new robots will displace twelve long-serving staff, an outcome several stakeholders see as negative. How should this be reflected in the business case?

## Options

- **A**: Omit it, so the case looks stronger to the project board
- **B**: Record it as a project risk, since it is only a possibility
- **C**: Record it as a dis-benefit so the project board can weigh the net value of the project
- **D**: Record it as an additional benefit because automation reduces headcount cost

## Answer

C

## Explanation

A negative outcome that stakeholders regard as a disadvantage is a dis-benefit and must be stated so the board can judge the project's net value honestly. (A) omitting it undermines the integrity of the justification; (B) it is a near-certain consequence of the chosen solution, not merely a risk; (D) presenting staff displacement purely as a saving ignores its perceived harm. Recording dis-benefits keeps the business case balanced and credible.
