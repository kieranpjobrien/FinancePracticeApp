# Question

During testing on a railway signalling upgrade, a team member discovers that a delivered component fails to meet one of its agreed acceptance criteria. The shortfall is significant and cannot be ignored. What should the project manager do first?

## Options

- **A**: Immediately raise an exception report to the project board
- **B**: Instruct the team to rework the component before recording anything
- **C**: Note it informally in the daily log and continue with the stage
- **D**: Capture the off-specification as a formal issue and record it in the issue register

## Answer

D

## Explanation

A significant failure to meet an agreed acceptance criterion is an off-specification, and the first step of the issue and change control procedure is to capture it formally in the issue register with an issue report. (A) is premature; an exception report is only needed if tolerances are forecast to be exceeded, which has not yet been assessed. (B) acts before the issue is captured and examined, skipping the procedure. (C) treats a significant issue as informal, which is inappropriate.
