# Question

A stakeholder asks for an approved, baselined product to be modified to add functionality that was never part of its specification. How should this be classified in PRINCE2?

## Options

- **A**: An off-specification, because the product currently lacks the functionality
- **B**: A problem/concern, because it requires the project manager's attention
- **C**: A risk, because the change might affect the project's objectives
- **D**: A request for change, because it proposes a change to an agreed baseline

## Answer

D

## Explanation

A request for change is a proposal to alter an agreed baseline, which is exactly what adding new functionality to an approved product is. (A) is wrong because an off-specification is a failure to meet the agreed specification, but here the specification never included the functionality. (B) is the residual category used only when an issue is neither of the other two. (C) confuses an issue that has happened (the request) with an uncertain future event.
