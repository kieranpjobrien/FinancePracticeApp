# Question

In PRINCE2, what is a baseline in the context of change control?

## Options

- **A**: The minimum acceptable level of quality that a product must reach to be approved
- **B**: A reference level against which an entity is monitored and controlled
- **C**: The first version of the project plan before any stages have started
- **D**: The list of tolerances delegated from the project board to the project manager

## Answer

B

## Explanation

A baseline is a reference level against which an entity such as a product or plan is monitored and controlled; once baselined, changes must pass through change control. (A) describes a quality criterion or acceptance criterion, not a baseline. (C) is too narrow — any plan version can be a baseline, not only the first. (D) describes tolerance, a separate control concept.
