# Question

In PRINCE2 7, how is a dis-benefit best described?

## Options

- **A**: A measurable decline resulting from an outcome perceived as negative by one or more stakeholders
- **B**: A benefit that has been delayed beyond the planned realisation date
- **C**: An expected cost overrun against the project budget
- **D**: A risk that has occurred and become an issue

## Answer

A

## Explanation

A dis-benefit is an outcome perceived as negative by one or more stakeholders, an actual consequence of an activity, distinct from a risk which is only a possibility. (B) describes a late benefit, not a dis-benefit; (C) is a budget variance; (D) confuses a dis-benefit with an issue. Dis-benefits are deliberately identified so the net value of the project can be judged honestly.
