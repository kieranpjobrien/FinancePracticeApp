# Question

In PRINCE2 7, what is the BEST definition of a risk?

## Options

- **A**: An event that has already occurred and harmed the project
- **B**: An uncertain event or set of events that, should it occur, will affect the achievement of objectives
- **C**: A problem that the project manager must escalate to the project board immediately
- **D**: A change to the scope baseline requested by a stakeholder

## Answer

B

## Explanation

PRINCE2 7 defines a risk as an uncertain event or set of events that, should it occur, will affect the achievement of objectives — it can be a threat (negative) or an opportunity (positive). (A) describes an issue that has already happened, not an uncertainty. (C) confuses risk with escalation, and (D) describes a request for change, which is a type of issue rather than a risk.
