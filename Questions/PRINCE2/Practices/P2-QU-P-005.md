# Question

A senior user on a public library digitisation project wants independent confidence that the user's quality interests are being properly addressed throughout the project. Which PRINCE2 mechanism is the appropriate route for this?

## Options

- **A**: Quality assurance provided by the corporate centre, since project assurance is not relevant to the senior user
- **B**: A change authority, which exists to confirm that quality interests are protected
- **C**: Project assurance, with the senior user appointing or carrying out the user-side assurance role on the project board's behalf
- **D**: The project manager performing assurance, as the project manager is accountable to the board for quality

## Answer

C

## Explanation

Project assurance is the project board's responsibility for monitoring the project independently of the project manager, and each board role, including the senior user, may carry out or delegate its own assurance perspective. (A) quality assurance is independent of the project entirely and is not the senior user's board-level mechanism; (B) a change authority handles change decisions, not assurance of user quality interests; (D) the project manager cannot provide the board's independent assurance over their own work.
