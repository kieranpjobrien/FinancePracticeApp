# Question

When is the project product description first created, and what key information does it contain?

## Options

- **A**: During the initiation stage; it contains the detailed designs for each specialist product
- **B**: At the end of each management stage; it records the quality activities completed in that stage
- **C**: During pre-project (starting up a project); it contains the customer's quality expectations, acceptance criteria and project-level quality tolerances
- **D**: During project closure; it confirms which products the customer has accepted

## Answer

C

## Explanation

The project product description is created during the starting up a project process and defines the overall project output, capturing the customer's quality expectations, the acceptance criteria, acceptance methods and project-level quality tolerances. (A) is wrong on both timing and content; (B) describes the quality register; (D) is too late, although the project product description is used at closure to confirm acceptance.
