# Question

Midway through a retail loyalty-app project, the project board is being interrupted weekly to decide on small, low-cost change requests, slowing the project down. What is the BEST response?

## Options

- **A**: Establish a Change Authority with a defined change budget and delegated limits so minor changes are handled without the full board
- **B**: Allow the Project Manager to approve all changes to keep momentum
- **C**: Freeze all change requests until the next stage boundary
- **D**: Ask Project Support to decide on the changes administratively

## Answer

A

## Explanation

When the board is burdened by frequent minor changes, PRINCE2 recommends delegating decisions to a Change Authority operating within limits and, often, a change budget set by the board. (B) removes board oversight entirely and exceeds the Project Manager's authority; (C) blocks legitimate change and harms the product; (D) misassigns a decision-making responsibility to an administrative support function.
