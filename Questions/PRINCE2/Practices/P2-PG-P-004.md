# Question

On a manufacturing line upgrade, a team manager believes a work package can be delivered faster by adding scope that was not agreed. The change would keep the work package within its time and cost tolerances. What should the team manager do?

## Options

- **A**: Add the extra scope, since the work package stays within its time and cost tolerances
- **B**: Raise the suggestion with the project manager, who owns the work package and its scope, before acting
- **C**: Escalate the idea directly to the project board as a request for change
- **D**: Record the idea in the lessons log and take no further action

## Answer

B

## Explanation

Scope is one of the seven performance targets, and the work package's scope is defined and owned by the project manager; the team manager must refer the change upward rather than alter scope unilaterally. (A) staying within time and cost does not authorise a scope change; (C) bypasses the project manager in the line of authority; (D) merely logging it ignores a live decision the project manager needs to make.
