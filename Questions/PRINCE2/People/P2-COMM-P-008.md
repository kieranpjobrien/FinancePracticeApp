# Question

On a pharmaceutical compliance project, the senior user repeatedly approves stage reports without comment, yet user representatives later raise serious objections that should have been caught earlier. The project manager suspects feedback is not flowing up through the senior user. What should the project manager do?

## Options

- **A**: Bypass the senior user and gather feedback directly from individual users in future
- **B**: Work with the senior user to strengthen the feedback loop so that user community concerns are genuinely gathered and represented before approval
- **C**: Accept the senior user's sign-offs at face value, as they hold the user assurance responsibility
- **D**: Ask the project board to replace the senior user immediately

## Answer

B

## Explanation

The senior user is accountable for representing the user community, so the right move is to reinforce the feedback loop with them so user concerns are captured and reflected before approvals are given. (A) bypassing the senior user undermines the defined role and the board's structure. (C) accepting silent sign-offs ignores clear evidence that representation is failing. (D) removing the senior user is a board decision and a disproportionate first step before improving the feedback mechanism.
