# Question

When is the communication management approach normally created in the PRINCE2 7 lifecycle?

## Options

- **A**: During the Closing a Project process, ready for the next project
- **B**: Only when the first issue is escalated to the project board
- **C**: At the end of each management stage, as part of the stage boundary
- **D**: During the Initiating a Project process, as part of the project initiation documentation

## Answer

D

## Explanation

The communication management approach is produced during Initiating a Project and forms part of the project initiation documentation, so engagement can be planned from the outset. (A) closing is far too late to begin planning communication. (B) communication should be planned proactively, not triggered by an issue. (C) the approach is established once at initiation and reviewed, not freshly created each stage boundary.
