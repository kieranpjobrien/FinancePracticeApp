# Question

A hospital has gone live with a new e-rostering tool delivered by the project. Adoption is patchy: many ward managers quietly revert to spreadsheets, putting the forecast efficiency benefits at risk. The project is still open. What is the BEST course of action?

## Options

- **A**: Close the project on schedule, since the output was delivered and accepted
- **B**: Personally mandate that all wards stop using spreadsheets and switch immediately
- **C**: Lead targeted engagement with the ward managers to understand and remove the barriers to adoption, working with the senior user who is accountable for realising the benefits
- **D**: Log a request for change to add more tool features in the hope that uptake improves

## Answer

C

## Explanation

Adoption is the bridge from output to benefit, and the senior user is accountable for the benefits, so the project manager should lead engagement to remove adoption barriers in partnership with them. (A) is the tempting near-miss: the output was accepted, but closing while adoption is failing abandons the benefits the project exists to enable. (B) a blunt mandate from the project manager neither addresses the underlying barriers nor respects the senior user's accountability. (D) adding features assumes a product gap without evidence and sidesteps the behavioural change actually needed.
