# Question

What is the primary purpose of a role description in a PRINCE2 project?

## Options

- **A**: To set out the responsibilities, accountabilities and skills associated with a role so expectations are clear
- **B**: To record the personal performance appraisal of each team member
- **C**: To replace the need for a project initiation documentation
- **D**: To list the salary and contractual terms of each appointee

## Answer

A

## Explanation

A role description defines what a role entails — its responsibilities, accountabilities and the skills or competencies needed — so that the people filling it understand what is expected. (B) confuses a role description with an individual appraisal; (C) is wrong because role descriptions support, not replace, the project initiation documentation; (D) concerns employment and contract terms, which are outside the scope of a PRINCE2 role description.
