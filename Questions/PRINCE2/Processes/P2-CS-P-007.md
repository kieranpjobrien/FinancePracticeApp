# Question

Near the end of a stage on a medical-device project, testing shows a delivered component does not meet one of its agreed quality criteria. Fixing it would breach the stage's cost tolerance, but the senior user indicates the product may still be usable as-is. What should the project manager do?

## Options

- **A**: Authorise the team to grant a concession and accept the product, since the senior user is content
- **B**: Record it as an off-specification, assess the impact, and escalate to the project board because correcting it would breach tolerance, noting that a concession is the board's decision
- **C**: Quietly close the work package as complete and move on to the stage boundary
- **D**: Reduce the quality criteria in the project product description so the component passes

## Answer

B

## Explanation

The failure is an off-specification; the project manager assesses it, and because remedying it would breach stage tolerance the matter is escalated to the project board, which alone can decide whether to grant a concession (accept the product despite the off-specification). (A) The project manager and team cannot grant a concession; that authority rests with the board. (C) Closing it as complete conceals a known defect. (D) Rewriting baselined quality criteria to mask a failure corrupts the change control and quality records, even though it superficially resolves the issue.
