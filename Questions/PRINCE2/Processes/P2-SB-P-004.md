# Question

As a software platform project nears the end of its second stage, the project manager learns that a key third-party API will be deprecated during the next stage, creating a significant new threat. What should the project manager do as part of Managing a Stage Boundary?

## Options

- **A**: Wait until the threat materialises before recording anything
- **B**: Record and assess the threat in the risk register so the board can weigh it in the authorisation decision
- **C**: Quietly add contingency time to the next stage plan without telling the board
- **D**: Escalate the project to closure because of the new threat

## Answer

B

## Explanation

Managing a Stage Boundary includes updating the risk register so the board's go/no-go decision reflects the project's current exposure, including newly identified threats. (A) withholds material information; (C) hides the risk and its response from the board, breaching transparency; (D) is a disproportionate over-reaction to a single manageable threat.
