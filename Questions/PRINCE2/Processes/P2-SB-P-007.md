# Question

On a city-wide broadband rollout, the project manager forecasts that the current stage will breach its time tolerance. They have not yet contacted the project board. What is the correct first step, and in which process does it sit?

## Options

- **A**: Produce an exception plan straight away within Managing a Stage Boundary
- **B**: Raise an exception report to the project board within Controlling a Stage, then await direction
- **C**: Absorb the slippage by reducing the next stage's scope within Managing a Stage Boundary
- **D**: Replan the whole project within Initiating a Project

## Answer

B

## Explanation

A forecast tolerance breach is first escalated by raising an exception report to the board within Controlling a Stage; only if the board then requests an exception plan does Managing a Stage Boundary produce one. (A) jumps ahead of the board's direction; (C) silently alters scope without authority and out of sequence; (D) wrongly returns to initiation for an in-flight stage issue.
