# Question

Three weeks into a six-week stage on a banking compliance project, the project manager realises the project board has received no highlight reports because no reporting frequency was ever set. The stage is currently progressing within tolerance. What is the BEST action?

## Options

- **A**: Wait until the stage boundary and cover everything in the end stage report
- **B**: Raise an exception report to apologise for the missing reports
- **C**: Agree a highlight report frequency with the project board and begin issuing reports at that cadence
- **D**: Send daily checkpoint reports to the project board to make up for lost time

## Answer

C

## Explanation

Highlight reporting is a time-driven control whose frequency the project board sets, usually in the project initiation documentation; the right fix is to agree that frequency and start reporting, restoring the board's visibility while the stage is still in tolerance. (A) Waiting for the stage boundary leaves the board uninformed for too long, defeating the purpose of regular reporting. (B) No tolerance is threatened, so an exception report is the wrong product. (D) Checkpoint reports run from team manager to project manager, not project manager to board, so this misuses the management product.
