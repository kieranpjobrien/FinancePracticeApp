# Question

A bank's mobile-app project is complete and the project manager has finished all Closing a Project activities. Wanting to free the team quickly, the project manager issues a project closure notification to stakeholders and releases the resources, intending to brief the project board afterwards. What should the project manager have done?

## Options

- **A**: Nothing different; once the work is done the project manager may close the project and inform the board later
- **B**: Recommended closure to the project board and waited for the board to authorise closure before the closure notification is issued
- **C**: Asked corporate or programme management to authorise closure instead of the board
- **D**: Released the resources first, then sought board authorisation to confirm the decision

## Answer

B

## Explanation

The project board, not the project manager, authorises closure through "authorise project closure" in Directing a Project, and only then is the closure notification distributed and resources released. (A) The project manager cannot close the project unilaterally and inform the board afterwards. (C) Closure is authorised by the project board, although it then notifies corporate or programme management. (D) Releasing resources before authorisation pre-empts a decision that is not the project manager's to make — a plausible but incorrect ordering.
