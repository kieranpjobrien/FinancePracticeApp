# Question

During Controlling a Stage, when must the project manager escalate to the project board?

## Options

- **A**: Whenever any issue or risk is raised by a team
- **B**: When a stage is forecast to exceed, or has exceeded, its agreed tolerances
- **C**: At the end of every checkpoint reporting cycle
- **D**: Whenever a work package is completed and accepted

## Answer

B

## Explanation

The manage by exception principle means the project manager works within delegated tolerances and only escalates to the project board when a tolerance is forecast to be exceeded (or has been), normally via an exception report. (A) Routine issues and risks are handled by the project manager and logged, not escalated. (C) and (D) are normal stage events reported through highlight reports, not triggers for escalation.
