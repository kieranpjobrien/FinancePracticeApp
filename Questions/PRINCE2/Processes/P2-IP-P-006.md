# Question

While initiating a defence-logistics software project, the project manager expects frequent change requests during delivery and wants to avoid escalating every one to the project board. Within the change management approach, what is the BEST way to handle this?

## Options

- **A**: Define a change authority and a change budget so that changes within agreed limits can be decided without troubling the board
- **B**: Grant the project manager open authority to approve all changes regardless of impact
- **C**: State in the approach that all change requests will be approved automatically to keep delivery moving
- **D**: Record that changes will be assessed only at each stage boundary

## Answer

A

## Explanation

The change management approach can delegate decision-making by defining a change authority and a change budget, allowing changes within set limits to be decided without escalating each one to the board. (B) Unlimited authority removes proper control and ignores impact on tolerances. (C) Automatic approval abandons impact assessment entirely. (D) Deferring all change decisions to stage boundaries is not the mechanism PRINCE2 uses; a delegated change authority within defined limits is the correct, tempting-but-distinct answer.
