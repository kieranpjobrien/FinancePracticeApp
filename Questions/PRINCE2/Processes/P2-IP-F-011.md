# Question

Once the Project Initiation Documentation is approved by the project board, how is it used for the remainder of the project?

## Options

- **A**: It is archived unchanged and referred to only at project closure
- **B**: It is rewritten in full at each stage boundary to reflect current circumstances
- **C**: It is baselined and used as the benchmark against which the project's progress and ongoing viability are assessed
- **D**: It is handed to the delivery teams as their work package instructions

## Answer

C

## Explanation

The approved PID is baselined, providing the reference point against which actual progress and the project's continued viability are measured throughout delivery. (A) It is actively used, not merely archived. (B) The PID may be reviewed and refined at stage boundaries, but it is not rewritten in full each time; the baseline is preserved for comparison. (D) Work packages are separate products derived during Controlling a Stage, not the PID itself.
