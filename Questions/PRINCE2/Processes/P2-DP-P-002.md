# Question

Starting Up a Project has just completed for a regional bridge-strengthening programme, and the project brief with its outline business case has been submitted. The executive is keen to "get moving" and asks the project manager to begin writing the project initiation documentation straight away. What is the BEST response?

## Options

- **A**: Begin initiation work immediately because the executive has informally given approval
- **B**: Wait for the project board to authorise initiation before committing resources to the initiation stage
- **C**: Ask the team manager to start delivering the first specialist products in parallel
- **D**: Skip initiation and proceed directly to authorising the project to save time

## Answer

B

## Explanation

After Starting Up a Project, the board must formally "authorise initiation" before the initiation stage begins; the executive's verbal eagerness is not a board decision and resources should not be committed until it is given. (A) treats informal comment as authorisation. (C) starts delivery before any plan or authorisation exists. (D) is wrong because the project cannot be authorised before initiation has produced the documentation the board needs.
