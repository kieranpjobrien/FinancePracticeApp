# Question

On a hospital IT rollout, a team manager receives a work package to configure a new patient-records module. Reviewing it, she believes the timescale is unrealistic given her team's other commitments. What should she do?

## Options

- **A**: Accept the work package as written and hope the team can catch up later
- **B**: Discuss her concerns with the project manager and only confirm acceptance once achievable tolerances and constraints have been agreed
- **C**: Reduce the scope of the module herself so the work fits the timescale
- **D**: Decline the work package and ask the project board to reassign it

## Answer

B

## Explanation

Accept a work package requires the team manager to agree that the work is achievable within the stated tolerances and constraints; if it is not, she negotiates with the project manager before committing. (A) accepting work she cannot deliver breaches the principle of agreed acceptance. (C) altering scope unilaterally bypasses change control. (D) the project board is not involved at work-package level; the dialogue is with the project manager.
