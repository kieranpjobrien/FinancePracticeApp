# Question

Which statement about the Project Brief is correct?

## Options

- **A**: It is maintained and updated throughout the project as a living document
- **B**: It replaces the Project Initiation Documentation once the project is authorised
- **C**: It is assembled during Starting Up a Project to provide a firm and full basis for initiating the project
- **D**: It is produced by the project board to instruct the Project Manager

## Answer

C

## Explanation

The Project Brief is assembled at the end of Starting Up a Project to give a firm, full basis for the decision to initiate, drawing together the project definition, outline Business Case, project approach and team structure. (A) The Project Brief is not maintained beyond initiation; the Project Initiation Documentation supersedes it as the baseline. (B) It is the other way around — the PID is built up from the Project Brief, not replaced by it. (D) The Project Brief is assembled by the Project Manager during start-up, not issued by the project board.
