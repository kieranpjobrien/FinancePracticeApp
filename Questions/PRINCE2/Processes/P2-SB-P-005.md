# Question

A rail-signalling upgrade has just completed a stage that ran 10% under budget and slightly ahead of schedule. The project board will soon review the stage boundary. What should the project manager do with the project plan?

## Options

- **A**: Leave the project plan unchanged because the variance was favourable
- **B**: Update the project plan with the stage's actuals and revised forecasts before the board reviews it
- **C**: Delete the project plan now that the stage figures are known
- **D**: Hand the project plan to the team manager to maintain from now on

## Answer

B

## Explanation

During Managing a Stage Boundary the project plan is updated with actuals and the latest forecasts — favourable variances matter just as much as adverse ones for an accurate whole-project view. (A) leaves the board working from stale data; (C) discards a live management product; (D) misassigns ownership of the project plan, which remains with the project manager.
