# Question

When a project is closed prematurely, what additional consideration applies that would not normally arise in a planned closure?

## Options

- **A**: The project board must approve the next stage plan before closing
- **B**: There may be products that were created but not yet approved, and the project may need to recover value or make safe any partially completed work
- **C**: The end project report is no longer required
- **D**: Acceptance of the project product is automatically granted by the project board

## Answer

B

## Explanation

In a premature closure the project has not finished, so the project manager must check what has been produced, identify any gaps, and consider salvaging or making safe work in progress as instructed by the board. (A) No further stage is being authorised when the project is being stopped. (C) An end project report is still produced, adapted to reflect the early closure. (D) Acceptance is never automatic; products are assessed against what was actually delivered.
