# Question

Why must the project plan be created before the project controls are finalised during Initiating a Project?

## Options

- **A**: Because the project plan identifies the management stages and key decision points around which controls are set
- **B**: Because the project plan replaces the need for stage plans throughout delivery
- **C**: Because controls cannot be delegated until the project board has approved the plan
- **D**: Because the project plan is the only product the board reviews when authorising the project

## Answer

A

## Explanation

The project plan defines how and when the project's objectives are achieved, including the management stages and major control points, so the project controls — such as tolerances, reporting and stage boundaries — are designed to align with it. (B) Stage plans are still produced for each delivery stage. (C) Controls are established during initiation, not gated on a separate plan approval. (D) The board reviews the whole PID, of which the project plan is only one part.
