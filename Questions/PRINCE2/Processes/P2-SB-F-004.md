# Question

Why does the project manager produce a detailed plan for the next management stage during Managing a Stage Boundary?

## Options

- **A**: So the team manager can authorise the work themselves without board involvement
- **B**: So the project board has a sufficiently detailed basis on which to authorise the next stage
- **C**: To replace the project plan, which is no longer needed after stage one
- **D**: To record the actual effort already spent in the current stage

## Answer

B

## Explanation

A management stage is planned in detail only as it approaches, giving the project board a credible, near-term plan against which to make its authorisation decision. (A) is wrong because authorising a stage is a project board decision, not the team manager's; (C) is wrong because the project plan is updated, not replaced; (D) describes performance data, not the forward plan.
