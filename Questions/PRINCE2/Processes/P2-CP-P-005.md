# Question

As a university finishes a project to deliver a new online enrolment portal, the project manager has confirmed acceptance and handover but has not yet produced any closure documentation. The board expects a recommendation to close. What should the project manager do?

## Options

- **A**: Recommend closure now and write the documentation only if the board asks for it
- **B**: Produce the end project report and lessons report to evaluate the project before recommending closure to the board
- **C**: Ask each team member to keep their own informal notes instead of formal reports
- **D**: Defer all evaluation to the post-project benefits review

## Answer

B

## Explanation

Closing a Project includes evaluating the project, which is done through the end project report and the lessons report, and these inform the project manager's recommendation to close. (A) Recommending closure without the evaluation products skips a required part of the process. (C) Informal personal notes do not satisfy the need for a structured project evaluation and shared lessons. (D) The post-project benefits review measures benefits later; it does not replace evaluating how the project itself performed.
