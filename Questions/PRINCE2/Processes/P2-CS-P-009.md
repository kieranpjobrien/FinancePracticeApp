# Question

A project manager on an airport baggage-system upgrade has submitted an exception report forecasting a breach of stage tolerance. While awaiting the project board's response, a team manager asks whether to keep working on the affected work packages. What should the project manager do?

## Options

- **A**: Tell the team to stop all work immediately, since the stage is in exception
- **B**: Instruct the team to continue exactly as planned and ignore the exception
- **C**: Keep the work packages running as authorised within their tolerances while awaiting the board's direction, then act on the board's decision
- **D**: Produce an exception plan now and start executing it without waiting for the board

## Answer

C

## Explanation

Raising an exception report does not by itself halt or change the work; the project manager continues controlling the authorised work packages within their tolerances and waits for the project board's direction, which may be to request an exception plan, before changing anything. (A) Stopping everything pre-empts the board and may waste valid progress. (B) Ignoring the exception abandons control of the situation. (D) Producing and executing an exception plan is only done after the board directs it; acting first usurps the board's decision-making authority.
