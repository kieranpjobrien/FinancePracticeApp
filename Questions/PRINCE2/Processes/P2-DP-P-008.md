# Question

A clinical-trials data platform has delivered all its products and the project manager has prepared the end project report recommending closure. The executive wants to keep the project "open" informally so the team remains on hand for possible future tweaks. What is the BEST response?

## Options

- **A**: Agree to leave the project open indefinitely so resources stay available
- **B**: Recommend the board authorise closure now, with any future work handled through follow-on action recommendations or a new project
- **C**: Move the remaining team into a new, unauthorised maintenance stage of the same project
- **D**: Close the project but continue reporting against it for several more months

## Answer

B

## Explanation

Once products are delivered and justification for further project work has ended, the board should perform "authorise closure" for a controlled end, with any future needs captured as follow-on action recommendations or addressed by a separate initiative. (A) keeps the project running without justification, contrary to a controlled close. (C) is the near-miss: bolting an undefined maintenance stage onto a finished project bypasses proper authorisation. (D) closes on paper but leaves the project effectively running, defeating the purpose of authorising closure.
