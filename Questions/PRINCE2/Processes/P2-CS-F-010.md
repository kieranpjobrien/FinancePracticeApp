# Question

What distinguishes a request for change from an off-specification when capturing issues in Controlling a Stage?

## Options

- **A**: A request for change proposes a modification to an agreed baseline, whereas an off-specification is a product missing or not meeting its specification
- **B**: A request for change is always raised by the project board, an off-specification by the team
- **C**: A request for change is a type of risk, an off-specification is a type of issue
- **D**: There is no difference; the two terms are interchangeable

## Answer

A

## Explanation

Both are types of issue, but a request for change is a proposal to change something already agreed (a baseline), while an off-specification is a product that is missing or does not meet its agreed specification. (B) Either type can be raised by anyone. (C) Both are issues, not risks. (D) They are distinct and trigger different handling under the Issues practice and change control approach.
