# Question

When evaluating the project at closure, the end project report compares actual results against several baselines. Which of the following does it typically NOT cover?

## Options

- **A**: Performance against the project's time, cost, quality, scope, benefits, risk and sustainability targets
- **B**: A summary of the project's achievements against the approved PID
- **C**: The detailed day-to-day allocation of work packages for the next management stage
- **D**: An assessment of how well the project's management approaches and controls performed

## Answer

C

## Explanation

The end project report evaluates the completed project against its PID baseline and performance targets and reviews how the project was managed; it does not plan future work. (C) Allocating work packages for a future stage is delivery planning and has no place in a closure evaluation, especially as the project is ending. (A), (B) and (D) are all legitimate components of evaluating project performance at closure.
