# Question

A government grants-processing project is closing out a stage in which the team discovered a far more efficient data-validation technique worth sharing across the wider portfolio. The project manager is assembling the stage boundary products. What is the BEST way to handle this finding?

## Options

- **A**: Keep it informal, as lessons are only recorded at the very end of the project
- **B**: Summarise the stage's performance in the end stage report and record the reusable technique so it can be passed on as a lesson
- **C**: Add the technique to the next stage plan only, with no wider record
- **D**: Hold the finding back until the project board specifically asks for lessons

## Answer

B

## Explanation

Managing a Stage Boundary produces the end stage report summarising performance and is the natural point to capture lessons learned in the stage so they can benefit other initiatives, consistent with the principle of learning from experience. (A) wrongly defers all lessons to closure; (C) confines a portfolio-wide lesson to one stage plan; (D) makes proactive learning contingent on a board request rather than the project manager's duty to record it.
