# Question

Midway through a retail website build, a senior user emails the project manager asking to add a new live-chat feature that was not in the agreed scope. What should the project manager do first?

## Options

- **A**: Add the feature to the next work package, since a senior user has requested it
- **B**: Capture it as a request for change in the issue register and assess its impact before any decision
- **C**: Reject the request because it was not in the original scope
- **D**: Raise an exception report immediately to the project board

## Answer

B

## Explanation

A proposal to alter the agreed scope is a request for change, which must be captured as an issue and assessed for impact under the change control approach before any action is taken. (A) Acting on it directly bypasses change control and could breach tolerance. (C) Outright rejection pre-empts proper assessment. (D) An exception report is only warranted once assessment shows a forecast tolerance breach, not as an automatic first step.
