# Question

The senior user on a retail loyalty-app project has begun emailing the development team daily, reprioritising their tasks and asking for progress updates directly. The project manager notices delivery is being disrupted. What should the project manager do?

## Options

- **A**: Accept the senior user's involvement because board members may direct the team whenever they wish
- **B**: Remind the board, through the appropriate channel, that day-to-day management is delegated to the project manager and direction should flow through agreed reporting
- **C**: Escalate an exception report immediately, as any board contact with the team breaches tolerance
- **D**: Stop reporting to the board until the senior user stops contacting the team

## Answer

B

## Explanation

Under manage by exception the board directs but delegates day-to-day management to the project manager; the senior user reaching into the team undermines this, so the project manager should reassert the agreed lines of control. (A) misstates the board's role, which is to direct, not to run the team. (C) is disproportionate; board interference is not in itself a tolerance breach requiring an exception report. (D) withholding reporting damages the board's legitimate oversight and resolves nothing.
