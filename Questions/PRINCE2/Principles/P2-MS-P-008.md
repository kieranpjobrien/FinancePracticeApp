# Question

Midway through the current stage of a banking-platform migration, the project board asks the project manager to bring forward the next stage plan and seek authorisation now, so resources can be locked in early. The project manager believes too much remains uncertain to plan the next stage reliably. What is the BEST response?

## Options

- **A**: Comply at once and present a detailed next stage plan built on current assumptions, however uncertain
- **B**: Skip the next stage plan and ask the board to authorise the rest of the project from the high-level project plan
- **C**: Explain that detailed next-stage planning is best done near the stage boundary when information is reliable, and propose deferring authorisation to that point
- **D**: Refuse the request and stop reporting to the board until the boundary

## Answer

C

## Explanation

PRINCE2 deliberately produces the detailed stage plan close to the boundary because planning distant work too early yields unreliable plans; the project manager should explain this and propose authorising at the boundary. (A) Building a detailed plan on shaky assumptions undermines reliable control. (B) The high-level project plan lacks the detail the board needs to authorise an individual stage. (D) Withholding reporting breaches the board's right to oversight.
