# Question

Why does the "define roles, responsibilities and relationships" principle insist on clearly defined accountabilities?

## Options

- **A**: So that blame can be assigned quickly when something goes wrong
- **B**: So that everyone knows what they are answerable for, avoiding gaps and overlaps in decision-making
- **C**: So that the project manager can take all key decisions alone
- **D**: So that the number of people involved in the project can be minimised

## Answer

B

## Explanation

Clearly defined accountabilities ensure each person understands what they are answerable for, which prevents gaps and overlaps and supports effective, timely decisions. (A) The aim is clear ownership for direction and control, not apportioning blame. (C) Key decisions remain with the project board, not the project manager alone. (D) The principle is about clarity of roles, not reducing headcount.
