# Question

On a bridge construction project, the design "technical stage" will overlap the build "technical stage" because early build can start while later design continues. The project board still wants a formal review before committing the large build budget. How should the project manager structure the management stages?

## Options

- **A**: Set a management stage boundary at the point the board needs its go/no-go decision, regardless of where the technical stages overlap
- **B**: Align each management stage exactly to one technical stage so design and build never overlap
- **C**: Abandon management stages and rely only on the technical stages
- **D**: Place the boundary only after both technical stages are completely finished

## Answer

A

## Explanation

Management stages are units of control defined by the board's decision needs and need not coincide with technical stages, so the boundary is placed where the board wants its go/no-go review before releasing the build budget. (B) Forcing alignment would remove the beneficial overlap of design and build. (C) Technical stages alone provide no governance control points. (D) Waiting until both technical stages finish defeats the purpose of a pre-build commitment review.
