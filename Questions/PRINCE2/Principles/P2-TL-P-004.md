# Question

A new project manager joins a charity that has its own PRINCE2-based standards, mandatory templates and tailored terminology used across all its projects. How should the project manager treat these organisational standards when tailoring their own project?

## Options

- **A**: Ignore the organisational standards and tailor purely from the official PRINCE2 manual
- **B**: Use the organisation's embedded standards as the starting point, then tailor further to suit this specific project
- **C**: Adopt the organisational standards exactly with no project-level tailoring at all
- **D**: Create a completely new method unique to this one project

## Answer

B

## Explanation

The charity has embedded PRINCE2 into its way of working, so the project manager should start from those embedded standards and then tailor further for the specific project's needs. (A) wrongly discards relevant organisational context; (C) confuses embedding with tailoring and skips adapting to the individual project; (D) needlessly reinvents an approach and ignores both the organisation's standards and PRINCE2.
