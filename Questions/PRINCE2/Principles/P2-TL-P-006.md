# Question

A project sits within a wider transformation programme that already provides risk management standards and a quality register for all its projects. The project manager is about to create separate, project-specific versions of these from scratch. What is the BEST action?

## Options

- **A**: Tailor the project's approach to use and reference the programme's existing standards rather than duplicating them
- **B**: Build fully independent project versions so the project is self-contained
- **C**: Adopt the programme standards verbatim and skip producing any project-level risk approach
- **D**: Omit risk management on the project because the programme already covers it

## Answer

A

## Explanation

When a project operates within a programme, tailoring should make use of the programme's existing management approaches and registers rather than wastefully recreating them. (B) is the tempting near-miss: duplicating standards is unnecessary effort and works against tailoring, even though self-containment sounds tidy; (C) skips the project's own application of how risks will actually be managed; (D) breaches the spirit of the risk practice by leaving the project with no risk management at all.
