# Question

A charity is running its very first PRINCE2 project, a new donor-management platform, and therefore has no internal lessons log from past projects to draw on. During start-up, how should the project manager apply 'learn from experience'?

## Options

- **A**: Skip seeking lessons at the start, since the charity has no previous projects of its own
- **B**: Defer all learning until the project has generated lessons of its own
- **C**: Rely solely on the project manager's personal memory and record nothing formally
- **D**: Seek lessons from external sources and similar projects elsewhere, recording relevant ones in a new lessons log

## Answer

D

## Explanation

The principle expects lessons from previous projects, this project AND external sources; with no internal history, the project manager should actively draw on external and similar projects and record them in a newly created lessons log. (A) the absence of internal projects is not a reason to skip the start-up search. (B) deferring all learning ignores readily available external lessons. (C) relying on memory without recording fails the record element of the principle.
