# Question

An organisation runs PRINCE2 projects but uses its own job titles instead of the standard PRINCE2 role names. How should this be treated?

## Options

- **A**: This is forbidden, because PRINCE2 role names must always be used unchanged
- **B**: This is acceptable tailoring of terminology, provided the underlying roles and accountabilities are still fulfilled
- **C**: This means the projects are no longer PRINCE2 projects
- **D**: This is only allowed if the project board approves it for each individual

## Answer

B

## Explanation

Tailoring terminology, including mapping PRINCE2 role names to an organisation's own job titles, is acceptable as long as the responsibilities and accountabilities of those roles are still genuinely carried out. (A) is too rigid, as PRINCE2 explicitly allows terminology to be tailored; (C) is wrong because the defined roles principle is met even under different names; (D) invents an approval rule and confuses tailoring of terminology with per-person sign-off.
