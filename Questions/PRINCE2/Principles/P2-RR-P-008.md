# Question

On a university campus Wi-Fi rollout, the project board is frustrated by the volume of minor change requests it must review. The senior user proposes that the project manager simply approve any change they judge to be "small". What should the project manager recommend instead?

## Options

- **A**: Agree, and start approving small changes using personal judgement to ease the board's workload
- **B**: Insist that every change, however minor, continues to go to the full board to keep control
- **C**: Recommend the board formally delegate authority to a defined change authority within an agreed change budget and tolerance
- **D**: Refer all change requests to corporate management to decide on the board's behalf

## Answer

C

## Explanation

PRINCE2 allows the board to delegate change decisions to a defined change authority operating within an agreed change budget and tolerance, keeping accountabilities clear while reducing board load. (A) is the tempting near-miss: informally approving "small" changes by personal judgement lacks defined authority, budget and limits, so accountability becomes unclear. (B) Routing every minor change to the full board ignores the available delegation mechanism. (D) Change decisions belong to the project board, not corporate management above it.
