# Question

How does managing by stages support the way authority is delegated in PRINCE2?

## Options

- **A**: It lets the project board delegate authority to the project manager one stage at a time, within agreed tolerances
- **B**: It removes the need for the project board once initiation is complete
- **C**: It transfers all decision-making to the team managers for the whole project
- **D**: It requires the project board to approve every individual work package

## Answer

A

## Explanation

Managing by stages allows the project board to commit resources and delegate day-to-day authority to the project manager for one stage at a time, each governed by stage tolerances, retaining overall control at the boundaries. (B) The project board remains accountable throughout. (C) Team managers receive only delegated work packages, not authority over the whole project. (D) Routine work package authorisation is the project manager's job, not the board's.
