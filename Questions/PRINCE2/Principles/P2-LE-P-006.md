# Question

On a software migration project, a developer realises the team's chosen deployment tool is poorly suited to the organisation's environment, a problem that recurs across the organisation's projects. The current project can still cope, but future projects should avoid this tool. What is the BEST response?

## Options

- **A**: Raise it only as an issue, since it affects the current project's work
- **B**: Do nothing, because the current project can manage despite the problem
- **C**: Record it as a lesson for future projects and also act on it within the current project where helpful
- **D**: Wait for the project board to ask for lessons before recording anything

## Answer

C

## Explanation

This is a lesson with organisation-wide relevance, so it should be captured for future projects, while still being acted upon now where it can help, in line with seek-record-act. (A) treating it purely as a current-project issue loses the broader learning for future projects. (B) coping with a problem does not remove the duty to record the lesson. (D) everyone records lessons proactively; this does not wait on a board request.
