# Question

Midway through a retail loyalty-app stage, a senior user asks the project manager to add a substantial new feature. Delivering it stays within the stage's time and cost tolerances, but the project manager judges it would erode a benefit the project board explicitly protected with a benefits tolerance. What should the project manager do?

## Options

- **A**: Add the feature, since time and cost tolerances are not threatened
- **B**: Implement it and record it as a minor change within delegated authority
- **C**: Raise the matter to the project board, because a forecast breach of benefits tolerance is an exception even though time and cost are unaffected
- **D**: Reject the senior user's request outright without referring it anywhere

## Answer

C

## Explanation

A forecast breach of any performance tolerance — here benefits — is an exception, regardless of whether time and cost remain within tolerance, so it goes to the project board. (A) is the tempting near-miss: staying within time and cost does not license a forecast breach of a different protected tolerance. (B) The project manager cannot absorb a board-protected benefits impact as a minor change. (D) Unilateral rejection bypasses the board, whose business case and benefits are affected.
