# Question

Why does PRINCE2 require the business justification to be documented rather than simply held as a shared understanding?

## Options

- **A**: So it can be objectively reviewed and confirmed at decision points throughout the project
- **B**: Because documentation is needed only to satisfy external auditors
- **C**: Because an undocumented justification automatically invalidates the project
- **D**: So that the project manager, rather than the executive, owns the justification

## Answer

A

## Explanation

A documented justification gives a stable, agreed reference that can be objectively reassessed at stage boundaries and other decision points, which a vague shared understanding cannot. (B) understates the purpose — documentation supports decision-making, not just audit. (C) overstates the consequence; the issue is that an undocumented justification cannot be reliably reviewed. (D) misassigns ownership, which sits with the executive.
