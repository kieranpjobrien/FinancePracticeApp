# Question

The distinction between utility tokens and security tokens is important because:

## Options

- **A**: They are treated identically by regulators
- **B**: Security tokens may be subject to securities laws while utility tokens may not
- **C**: Only utility tokens have any value

## Answer

B

## Explanation

Token classification:

**Security tokens:**
- Represent ownership/investment
- Subject to securities laws
- Registration requirements
- Investor protections apply

**Utility tokens:**
- Access to platform/service
- Consumable value
- May not be securities
- Usage-based

**Howey Test (US):**
- Investment of money
- Common enterprise
- Expectation of profits
- From efforts of others
- If met = security

**Regulatory implications:**
- Securities: Registration, disclosure
- Utility: May avoid securities laws
- Grey areas exist
- Enforcement ongoing

**Examples:**
- Security: Tokenized equity
- Utility: Platform access token
- Hybrid: Some features of both

**Common mistakes:**
- Option A: Different regulatory treatment
- Option C: Both types can have value
