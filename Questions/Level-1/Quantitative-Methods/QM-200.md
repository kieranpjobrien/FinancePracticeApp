# Question

Out-of-sample testing in backtesting:

## Options

- **A**: Uses the same data to develop and test the trading strategy
- **B**: Reserves a portion of historical data not used in strategy development for testing
- **C**: Requires live trading with real capital

## Answer

B

## Explanation

Out-of-sample testing holds back a portion of historical data during strategy development. The strategy is developed using only in-sample data, then tested on the reserved out-of-sample data to evaluate performance on data the model has never "seen." This helps detect overfitting and provides a more realistic estimate of future performance. Walk-forward analysis extends this concept.

**Common mistakes:**
- Option A describes in-sample testing, which overstates expected performance
- Option C describes live or paper trading, which is different from out-of-sample backtesting
