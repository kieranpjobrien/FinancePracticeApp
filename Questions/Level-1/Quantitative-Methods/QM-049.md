# Question

K-fold cross-validation is used primarily to:

## Options

- **A**: Increase the amount of training data available
- **B**: Estimate how well a model will perform on unseen data
- **C**: Reduce the computational time required for model training

## Answer

B

## Explanation

K-fold cross-validation divides the data into K equal parts (folds). The model is trained on K-1 folds and tested on the remaining fold, rotating through all K combinations. The average performance across all folds provides a more robust estimate of how the model will perform on new, unseen data compared to a single train-test split. This helps in model selection and detecting overfitting.

**Common mistakes:**
- Option A is incorrect; cross-validation doesn't increase data but uses existing data more efficiently for validation
- Option C is incorrect; cross-validation actually increases computational time by requiring multiple model trainings
